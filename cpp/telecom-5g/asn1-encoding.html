ASN.1 Encoding

ASN.1 (Abstract Syntax Notation One) is a standard interface definition language used to define data structures that telecommunications protocols serialize on the wire. 3GPP protocols -- RRC, NGAP, S1AP, X2AP, F1AP, E1AP -- all use ASN.1 to define their message formats.

BER vs PER Encoding

AspectBER (Basic Encoding Rules)PER (Packed Encoding Rules)
Size overheadHigh (TLV with full length)Low (bit-aligned, no padding)
Typical size2-3x of PERCompact (close to struct)
Decode speedSlower (tag-length-value parse)Faster (offset-based)
Used inCSS, LDAP, SNMP3GPP RRC, NGAP, S1AP, X2AP
AlignmentByte-alignedBit-aligned

3GPP choice: All 3GPP radio and NG interface protocols use Aligned PER (Packed Encoding Rules, aligned variant). PER encodes data at the bit level with optional byte alignment for efficiency. This is critical for air interface signaling where every byte of bandwidth matters.

TLV Structure (BER)

  BER TLV (Tag-Length-Value):
  +--------+---------+----------+
  | Tag    | Length  | Value    |
  | 1 byte | 1+ bytes| N bytes  |
  +--------+---------+----------+

  Tag:   Class (2 bits) | P/C (1 bit) | Tag Number (5 bits)
  Length: Short form (< 128): 1 byte
          Long form (>= 128): first byte = 0x80 | N, then N bytes

  Example (INTEGER value 42):
  02 01 2A
  |  |  |
  Tag=L |  Value=42 (0x2A)
       Length=1

PER Bit-Level Encoding

  PER encodes values at the bit level:
  - BOOLEAN: 1 bit (0 or 1)
  - INTEGER (constrained): min bits for range
  - ENUMERATED: log2(max_value) bits
  - BIT STRING: length + bits
  - OCTET STRING: length + bytes
  - SEQUENCE: fields concatenated at bit level
  - CHOICE: index + chosen alternative

  Example: ENUMERATED {idle(0), connected(1), inactive(2)}
  PER encoding: 2 bits (enough for 3 values)

  Example: INTEGER (0..4095) -> 12 bits
  Value 1000 = 0011 1110 1000 in 12 bits

C++ Manual PER Encoding Example

#include <cstdint>
#include <cstring>
#include <vector>
#include <stdexcept>

// Bit-level encoder for PER encoding
class PerEncoder {
public:
    explicit PerEncoder(size_t max_bytes = 1024)
        : buffer_(max_bytes, 0), bit_pos_(0) {}

    // Encode a boolean (1 bit)
    void encodeBool(bool value) {
        if (value) setBit(bit_pos_);
        bit_pos_++;
    }

    // Encode constrained integer
    // value must be in [min_val, max_val]
    void encodeConstrainedInt(int32_t value,
                                int32_t min_val,
                                int32_t max_val) {
        uint32_t range = max_val - min_val;
        int n_bits = 0;
        uint32_t temp = range;
        while (temp > 0) { n_bits++; temp >>= 1; }

        uint32_t offset = value - min_val;
        for (int i = n_bits - 1; i >= 0; --i) {
            if ((offset >> i) & 1) setBit(bit_pos_);
            bit_pos_++;
        }
    }

    // Encode BIT STRING
    void encodeBitString(const uint8_t* bits, size_t n_bits) {
        // Length determinant (if not fixed)
        // For simplicity, assume fixed length known from ASN.1
        for (size_t i = 0; i < n_bits; ++i) {
            if ((bits[i / 8] >> (7 - (i % 8))) & 1)
                setBit(bit_pos_);
            bit_pos_++;
        }
    }

    // Encode OCTET STRING (byte-aligned)
    void encodeOctetString(const uint8_t* data, size_t len) {
        alignToByte();
        // Length determinant for variable-length
        encodeLength(len);
        for (size_t i = 0; i < len; ++i) {
            encodeByte(data[i]);
        }
    }

    // Align to byte boundary (required by Aligned PER)
    void alignToByte() {
        while (bit_pos_ % 8 != 0) bit_pos_++;
    }

    // Get encoded data and size
    const uint8_t* data() const { return buffer_.data(); }
    size_t byteSize() const { return (bit_pos_ + 7) / 8; }

private:
    std::vector<uint8_t> buffer_;
    size_t bit_pos_;

    void setBit(size_t pos) {
        buffer_[pos / 8] |= (1 << (7 - (pos % 8)));
    }

    void encodeByte(uint8_t byte) {
        for (int i = 7; i >= 0; --i) {
            if ((byte >> i) & 1) setBit(bit_pos_);
            bit_pos_++;
        }
    }

    void encodeLength(size_t len) {
        if (len < 128) {
            encodeByte(static_cast<uint8_t>(len));
        } else if (len < 16384) {
            encodeByte(static_cast<uint8_t>(0x80 | (len >> 8)));
            encodeByte(static_cast<uint8_t>(len & 0xFF));
        }
    }
};

// Example: encode a simple RRC-like message
void encodeRrcSetupRequest(PerEncoder& enc) {
    // Message type (ENUMERATED, 6 bits)
    enc.encodeConstrainedInt(0, 0, 63);  // RRCSetupRequest

    // ng-5G-S-TMSI-Value (BIT STRING, 48 bits)
    uint8_t guti[6] = {0x12, 0x34, 0x56, 0x78, 0x9A, 0xBC};
    enc.encodeBitString(guti, 48);

    // establishmentCause (ENUMERATED, 0..7, 3 bits)
    enc.encodeConstrainedInt(4, 0, 7);  // mt-Access
}

Open-Source ASN.1 Compilers

Production tools: In production telecom systems, you rarely hand-code PER encoding. Use asn1c (open-source, most common in telecom), or commercial tools like OSS ASN.1 Tools or Objective Systems. These compile .asn files into C/C++ structs with encode/decode functions.

  # Using asn1c to compile RRC ASN.1 definition:
  asn1c -pdu=all -fper -fnative-types RRC_NR.asn

  Generated files:
  - RRC_NR.h    : C structs matching ASN.1 definitions
  - RRC_NR.c    : encode/decode functions
  - converter-sample.c : usage example

  Compile generated code with your application:
  gcc -o my_app my_app.c RRC_NR.c -I.
