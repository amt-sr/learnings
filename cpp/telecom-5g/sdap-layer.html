SDAP Layer

SDAP (Service Data Adaptation Protocol) is a new protocol introduced in 5G NR. It sits between the application/transport layer (IP) and PDCP. Its primary purpose is to map QoS flows to Data Radio Bearers (DRBs), enabling fine-grained QoS differentiation within a single PDU session.

SDAP in the 5G Protocol Stack

  +------------------+
  | Application / IP |
  +--------+---------+
           |  QoS Flows (QFI)
  +--------+---------+
  | SDAP             |  Maps QoS Flow -> DRB
  +--------+---------+
           |  SDAP SDU = PDCP SDU
  +--------+---------+
  | PDCP             |  Ciphering, integrity, header compression
  +--------+---------+
  | RLC              |  Segmentation, ARQ
  +--------+---------+
  | MAC / PHY        |
  +------------------+

  Note: SDAP is 5G-only. LTE does not have SDAP.
  In LTE, QoS is mapped at the eNodeB to EPS bearers directly.

QoS Flow to DRB Mapping

5G introduces QoS Flows identified by a QoS Flow Identifier (QFI). Each QoS flow has a 5QI (5G QoS Identifier) that determines delay, loss rate, and throughput characteristics. SDAP maps these QoS flows to DRBs.

QoS Flow5QICharacteristicsDRB Mapping
Conversational voice1100 ms delay, 1e-2 lossDefault DRB (QFI=5)
Video (buffered)2150 ms delay, 1e-3 lossNon-GBR DRB
Signaling5Default QoSDefault DRB
AR/VR8010 ms delay, 1e-5 lossGBR DRB (dedicated)
URLLC control195 ms delay, 1e-5 lossGBR DRB

Key insight: Multiple QoS flows can be mapped to a single DRB (reflective mapping), or a single QoS flow can have a dedicated DRB (explicit mapping). Reflective QoS mapping is the default: the UE learns the mapping by observing which DRB carries which QFI.

SDAP Header Format

  SDAP PDU (Downlink/Uplink):
  +-----+-----+-----------+
  | D/C | RDI | QoS Flow  |
  | 1b  | 1b  | ID (QFI)  |
  |     |     | 6 bits    |
  +-----+-----+-----------+
  (2 bytes total)

  D/C  = Data (0) / Control (1) PDU
  RDI  = Reflective QoS mapping Delayed Indication
          0 = QFI is end-marked (no mapping instruction)
          1 = Reflective mapping applies
  QFI  = QoS Flow Identifier (6 bits, values 1-63)

  SDAP Control PDU (End Marker):
  +-----+-----+------+---------+
  | D/C | RDI | E.Flg| QFI     |
  |  1  |  0  |  1   | 6 bits  |
  +-----+-----+------+---------+
  Used to indicate end of a QoS flow on a DRB.

C++ SDAP Entity

#include <vector>
#include <unordered_map>
#include <cstdint>
#include <optional>

// QoS Flow Identifier (6 bits: 1-63)
using Qfi = uint8_t;

// QoS characteristics per 5QI
struct QosCharacteristics {
    uint8_t  fiveqi;          // 5QI value
    uint32_t pdbr_ul;         // Packet Delay Budget UL (microseconds)
    uint32_t pdbr_dl;         // Packet Delay Budget DL (microseconds)
    double   per;             // Packet Error Rate (e.g., 1e-2)
    bool     is_gbr;          // Guaranteed Bit Rate
    uint32_t gbr_ul;          // GBR UL (bps)
    uint32_t gbr_dl;          // GBR DL (bps)
};

// QoS Flow configuration
struct QosFlowConfig {
    Qfi                qfi;
    QosCharacteristics qos_chars;
    uint8_t            drb_id;       // Associated DRB
    bool               reflective_mapping;  // Enable reflective QoS
};

// SDAP header (2 bytes)
struct SdapHeader {
    bool   dc;            // Data/Control
    bool   rdi;           // Reflective mapping delayed indication
    uint8_t qfi;          // QoS Flow ID (6 bits)
} __attribute__((packed));

// SDAP entity (per PDU session)
class SdapEntity {
public:
    explicit SdapEntity(uint8_t pdu_session_id)
        : pdu_session_id_(pdu_session_id) {}

    // Configure QoS flow -> DRB mapping
    void addQosFlow(const QosFlowConfig& cfg) {
        qos_flows_[cfg.qfi] = cfg;
        // Add reverse mapping: DRB -> list of QFIs
        drb_to_qfis_[cfg.drb_id].push_back(cfg.qfi);
    }

    // TX: receive IP packet with QFI, map to DRB
    void handleDownlinkData(Qfi qfi, const uint8_t* data, size_t len) {
        auto it = qos_flows_.find(qfi);
        if (it == qos_flows_.end()) return;

        uint8_t drb_id = it->second.drb_id;

        // Build SDAP header (2 bytes)
        uint8_t pdu[8192];
        pdu[0] = (0 << 7)  // D/C = 0 (Data)
               | (it->second.reflective_mapping ? 0 : (1 << 6))  // RDI
               | ((qfi & 0x3F) << 0);  // QFI
        pdu[1] = 0;  // Reserved / padding

        // Copy payload
        memcpy(pdu + 2, data, len);

        // Send to PDCP
        sendToPdcp(drb_id, pdu, len + 2);
    }

    // RX: receive SDAP PDU from PDCP, extract QFI, deliver
    void handleUplinkPdu(uint8_t drb_id, const uint8_t* pdu, size_t len) {
        if (len < 1) return;

        // Parse SDAP header
        bool dc  = (pdu[0] & 0x80) != 0;
        bool rdi = (pdu[0] & 0x40) != 0;
        Qfi qfi  = pdu[0] & 0x3F;

        // Reflective QoS mapping: learn QFI -> DRB association
        if (!rdi && dc == 0) {
            updateReflectiveMapping(qfi, drb_id);
        }

        // Deliver payload (skip SDAP header)
        deliverToUpper(qfi, pdu + 1, len - 1);
    }

private:
    uint8_t pdu_session_id_;
    std::unordered_map<Qfi, QosFlowConfig> qos_flows_;
    std::unordered_map<uint8_t, std::vector<Qfi>> drb_to_qfis_;

    void updateReflectiveMapping(Qfi qfi, uint8_t drb_id) {
        auto it = qos_flows_.find(qfi);
        if (it != qos_flows_.end()) {
            it->second.drb_id = drb_id;
        }
    }

    void sendToPdcp(uint8_t drb_id, const uint8_t* data, size_t len);
    void deliverToUpper(Qfi qfi, const uint8_t* data, size_t len);
};

Reflective vs Explicit Mapping

AspectReflective MappingExplicit Mapping
Who decidesUE (observes DL packets)gNB (RRC config)
SignalingNone neededRRC reconfiguration
LatencyFast (automatic)Slower (config change)
Use caseDefault for best-effort trafficGBR and critical QoS flows

4G vs 5G: LTE does not have SDAP. In LTE, the eNodeB maps S1 bearers to EPS bearers (DRBs) directly. The addition of SDAP in 5G provides more flexible QoS handling, especially for network slicing where multiple slices share the same DRB.
