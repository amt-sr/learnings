Circular Buffers

Circular buffers (ring buffers) are the backbone of telecom data-path communication. They provide fixed-size, O(1) FIFO queues for passing packets between processing stages, threads, and layers. When combined with power-of-2 sizing, index wrapping becomes a simple bitwise AND, eliminating modulo division.

Why Power-of-2 Size + Masking?

  // Modulo (any size):
  index = (index + 1) % N;       // Division: ~30 cycles on x86

  // Bitwise AND (power-of-2 size):
  index = (index + 1) & (N - 1); // AND: 1 cycle on x86

  Example with N = 1024 (power of 2):
  mask = 1023 = 0x3FF

  write_idx = 1023; write_idx + 1 = 1024
  1024 & 0x3FF = 0       // Wraps to beginning (1 cycle!)

  This is why DPDK rte_ring, all telecom ring buffers,
  and lock-free queues use power-of-2 sizes.

Design choice: Always size circular buffers to a power of 2 (64, 128, 256, 512, 1024, 2048, 4096, 8192). The single-cycle wrap-around is critical in data-path code that runs millions of times per second.

Read/Write Pointer Management

  SPSC (Single Producer, Single Consumer):
  +---+---+---+---+---+---+---+---+
  |   | D | D | D |   |   |   |   |  Capacity = 8
  +---+---+---+---+---+---+---+---+
        ^               ^
        |               |
     read_ptr       write_ptr

  Empty:     read_ptr == write_ptr
  Full:      (write_ptr + 1) % N == read_ptr
  Count:     (write_ptr - read_ptr) % N
  Available: N - 1 (one slot wasted for empty/full distinction)

C++ Template Circular Buffer

#include <array>
#include <cstddef>
#include <atomic>
#include <optional>
#include <stdexcept>

// Lock-free SPSC circular buffer
// N must be power of 2
template<typename T, size_t N>
class CircularBuffer {
    static_assert((N & (N - 1)) == 0, "N must be power of 2");

public:
    CircularBuffer() : buffer_{} {}

    // Push item (producer only, single-threaded)
    bool push(const T& item) {
        size_t write = write_idx_.load(std::memory_order_relaxed);
        size_t next = (write + 1) & MASK_;

        if (next == read_idx_.load(std::memory_order_acquire)) {
            return false;  // Buffer full
        }

        buffer_[write] = item;
        write_idx_.store(next, std::memory_order_release);
        return true;
    }

    // Pop item (consumer only, single-threaded)
    std::optional<T> pop() {
        size_t read = read_idx_.load(std::memory_order_relaxed);

        if (read == write_idx_.load(std::memory_order_acquire)) {
            return std::nullopt;  // Buffer empty
        }

        T item = buffer_[read];
        read_idx_.store((read + 1) & MASK_,
                         std::memory_order_release);
        return item;
    }

    // Peek without removing
    std::optional<T> peek() const {
        size_t read = read_idx_.load(std::memory_order_acquire);
        if (read == write_idx_.load(std::memory_order_acquire)) {
            return std::nullopt;
        }
        return buffer_[read];
    }

    [[nodiscard]] size_t size() const {
        size_t w = write_idx_.load(std::memory_order_acquire);
        size_t r = read_idx_.load(std::memory_order_acquire);
        return (w - r) & MASK_;
    }

    [[nodiscard]] bool empty() const {
        return size() == 0;
    }

    [[nodiscard]] bool full() const {
        return size() == (N - 1);
    }

    [[nodiscard]] size_t capacity() const { return N - 1; }

    // Batch push (for burst processing)
    size_t pushBatch(const T* items, size_t count) {
        size_t written = 0;
        for (size_t i = 0; i < count; ++i) {
            if (!push(items[i])) break;
            written++;
        }
        return written;
    }

    // Batch pop
    size_t popBatch(T* items, size_t max_count) {
        size_t read = 0;
        for (size_t i = 0; i < max_count; ++i) {
            auto item = pop();
            if (!item.has_value()) break;
            items[i] = std::move(*item);
            read++;
        }
        return read;
    }

private:
    static constexpr size_t MASK_ = N - 1;

    alignas(64) std::array<T, N> buffer_;
    alignas(64) std::atomic<size_t> write_idx_{0};
    alignas(64) std::atomic<size_t> read_idx_{0};
};

// Usage: inter-layer communication
// PHY -> MAC ring, MAC -> RLC ring, etc.
void example() {
    CircularBuffer<uint8_t*, 4096> mac_to_rlc;

    // MAC thread (producer):
    uint8_t* pdu = nullptr;  // from MAC processing
    mac_to_rlc.push(pdu);

    // RLC thread (consumer):
    auto item = mac_to_rlc.pop();
    if (item.has_value()) {
        // Process RLC PDU
    }
}

Overrun and Underrun

ConditionCauseHandling
Overrun (buffer full)Producer faster than consumerDrop oldest packets, backpressure, or grow buffer
Underrun (buffer empty)Consumer faster than producerReturn empty/padding, sleep briefly

Telecom consideration: In a RAN data path, overrun means packet loss and underrun means wasted CPU cycles. Size the buffer to handle burst scenarios. A common approach: 2x the maximum burst size expected between producer and consumer. For 5G, MAC-to-PHY buffer is sized based on the TTI processing budget.
