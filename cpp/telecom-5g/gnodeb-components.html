gNB Components

A gNB (Next-Generation NodeB) in 5G is logically decomposed into four main functional units: the Central Unit Control Plane (CU-CP), Central Unit User Plane (CU-UP), Distributed Unit (DU), and Radio Unit (RU). Each unit handles specific protocol layers and has distinct real-time requirements.

Complete gNB Architecture

Assembling the Full gNB

class GnbSystem {
public:
    GnbSystem(const GnbId& id)
        : cu_cp_(std::make_unique<CuControlPlane>("10.0.0.1")),
          cu_up_(std::make_unique<CuUserPlane>()),
          du_(std::make_unique<DistributedUnit>()),
          ru_(std::make_unique<RadioUnit>()),
          gnb_id_(id) {}

    bool start() {
        // Initialize in order: CU-CP, CU-UP, DU, RU
        if (!cu_cp_->initialize() || !cu_cp_->start())
            return false;
        if (!cu_up_->initialize() || !cu_up_->start())
            return false;
        if (!du_->initialize() || !du_->start())
            return false;
        if (!ru_->initialize() || !ru_->start())
            return false;

        // Establish E1 between CU-CP and CU-UP
        // Establish F1 between CU and DU
        return true;
    }

    void stop() {
        ru_->stop();
        du_->stop();
        cu_up_->stop();
        cu_cp_->stop();
    }

private:
    std::unique_ptr<CuControlPlane> cu_cp_;
    std::unique_ptr<CuUserPlane>    cu_up_;
    std::unique_ptr<DistributedUnit> du_;
    std::unique_ptr<RadioUnit>      ru_;
    GnbId gnb_id_;
};
