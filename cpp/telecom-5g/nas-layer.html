NAS Layer

NAS (Non-Access Stratum) is the highest protocol layer in the 5G/LTE signaling stack, managing signaling between the UE and the Core Network (AMF/MME) that passes transparently through the RAN (gNB/eNodeB). NAS messages handle registration, authentication, security mode setup, and session management.

NAS vs Access Stratum (AS)

AspectNAS (Non-Access Stratum)AS (Access Stratum)
EndpointsUE <-> Core Network (AMF/MME)UE <-> RAN (gNB/eNodeB)
ProtocolsNAS (MM, SM), SIPRRC, PDCP, RLC, MAC, PHY
TransportCarried inside RRC (AS)Radio bearers (SRB/NR)
ScopeRegistration, authentication, PDU sessionsRadio connection, handover, measurements
Survives handover?Yes (core network context persists)No (re-established at target)

Key concept: NAS messages are opaque to the gNB. The RAN transports them as payload without inspecting contents (except for initial NAS during RRC Setup). This is why NAS is called "Non-Access Stratum" -- it bypasses access layer processing.

NAS Message Protocol

  NAS Message Structure:
  +------+------------+-------+---------+
  | EPD  | Protocol   | Security Header / Plain
  | 0.5B | Disc. (1B) | Message Type (1B)
  +------+------------+-------+---------+

  Extended Protocol Discriminator (EPD):
  0x7E = Mobility Management (5G MM / 5GS)
  0x7A = Session Management (5G SM / 5GS)
  0x7B = Mobility Management (4G EMM / EPS)
  0x2E = Session Management (4G ESM / EPS)

  MM Message Types (5GS):
  0x41 = Registration Request
  0x42 = Registration Accept
  0x43 = Registration Complete
  0x44 = Registration Reject
  0x56 = Authentication Request
  0x57 = Authentication Response
  0x5E = Security Mode Command
  0x5F = Security Mode Complete
  0x47 = Deregistration Request
  0x4B = Deregistration Accept

  SM Message Types (5GS):
  0xC1 = PDU Session Establishment Request
  0xC2 = PDU Session Establishment Accept
  0xC3 = PDU Session Establishment Reject
  0xC6 = PDU Session Release Request
  0xD0 = PDU Session Modification Command

C++ NAS Message Parser

#include <cstdint>
#include <cstring>
#include <string>
#include <optional>
#include <array>

// NAS protocol discriminator
enum class NasProtocol : uint8_t {
    MM_5GS     = 0x7E,
    SM_5GS     = 0x7A,
    MM_EPS     = 0x7B,
    SM_EPS     = 0x2E
};

// NAS MM message types (5G)
enum class NasMmType : uint8_t {
    REG_REQUEST           = 0x41,
    REG_ACCEPT            = 0x42,
    REG_COMPLETE          = 0x43,
    REG_REJECT            = 0x44,
    DEREG_REQUEST         = 0x47,
    AUTH_REQUEST          = 0x56,
    AUTH_RESPONSE         = 0x57,
    AUTH_REJECT           = 0x58,
    AUTH_FAILURE           = 0x5D,
    SECURITY_MODE_CMD     = 0x5E,
    SECURITY_MODE_COMPLETE = 0x5F,
    IDENTITY_REQUEST       = 0x5B,
    IDENTITY_RESPONSE      = 0x5C
};

// Parsed NAS message header
struct NasMessageHeader {
    bool               security_header;  // Has security header?
    uint8_t            epd;              // Extended protocol discriminator
    uint8_t            security_type;    // Security header type (if present)
    uint8_t            message_type;
    NasProtocol        protocol;
};

// NAS message parser
class NasMessageParser {
public:
    // Parse NAS message header from buffer
    std::optional<NasMessageHeader> parseHeader(
            const uint8_t* data, size_t len) {
        if (len < 2) return std::nullopt;

        NasMessageHeader hdr{};
        hdr.epd = data[0];

        // Check if security header is present
        // Plain NAS: EPD is the protocol discriminator directly
        // Security-protected NAS: first byte is security header type
        if (isSecurityProtected(data[0])) {
            hdr.security_header = true;
            hdr.security_type = data[0];
            hdr.epd = data[1];
            hdr.message_type = data[2];
            if (len < 3) return std::nullopt;
        } else {
            hdr.security_header = false;
            hdr.message_type = data[1];
        }

        hdr.protocol = static_cast<NasProtocol>(hdr.epd);
        return hdr;
    }

    // Extract 5G-GUTI from Registration Request
    bool extract5gGuti(const uint8_t* msg, size_t len,
                        uint8_t (&guti)[12]) {
        // Skip header, find Mobile Identity IE (type 0x77)
        size_t offset = 2;  // Skip EPD + message type
        while (offset < len) {
            uint8_t ie_type = msg[offset];
            if (ie_type == 0x77) {  // 5G-GUTI type
                // TLV: type(1) + length(1) + value(N)
                uint8_t ie_len = msg[offset + 1];
                if (ie_len >= 12 && offset + 2 + 12 <= len) {
                    memcpy(guti, msg + offset + 3, 12);
                    return true;
                }
            }
            uint8_t ie_len = msg[offset + 1];
            offset += 2 + ie_len;
        }
        return false;
    }

    // Get message type name for logging
    static const char* mmTypeName(uint8_t type) {
        switch (static_cast<NasMmType>(type)) {
            case NasMmType::REG_REQUEST:
                return "Registration Request";
            case NasMmType::REG_ACCEPT:
                return "Registration Accept";
            case NasMmType::AUTH_REQUEST:
                return "Authentication Request";
            case NasMmType::AUTH_RESPONSE:
                return "Authentication Response";
            case NasMmType::SECURITY_MODE_CMD:
                return "Security Mode Command";
            case NasMmType::SECURITY_MODE_COMPLETE:
                return "Security Mode Complete";
            default:
                return "Unknown";
        }
    }

private:
    bool isSecurityProtected(uint8_t first_byte) {
        return (first_byte >= 0x00 && first_byte <= 0x0F);
    }
};

Security Considerations

Security: Before security mode is activated, NAS messages are sent in plain text. After security mode command/complete exchange, all subsequent NAS messages are ciphered and integrity-protected. The keys (KNASenc, KNASint) are derived from KAMF using the selected algorithm (NEA/NIA). This is why the registration procedure is critical -- it establishes the security context.
