S1/X2 Interfaces

S1 and X2 are the key interfaces in the LTE architecture. S1 connects the eNodeB to the EPC (MME and S-GW), while X2 connects neighboring eNodeBs for handover coordination and inter-cell interference management.

S1 Interface (S1-MME + S1-U)

  S1 Interface splits into control and user planes:

  eNodeB ----[S1-MME]---- MME     (Control plane, S1AP protocol)
  eNodeB ----[S1-U]------ S-GW    (User plane, GTP-U protocol)

  S1-MME (Control Plane):
  - UE-related: Initial UE Message, UE Context Release,
    Handover Preparation/Request
  - eNB-related: S1 Setup, eNB Configuration Update,
    S1 Reset

  S1-U (User Plane):
  - GTP-U tunnels carry user data packets
  - Each EPS bearer has a dedicated GTP-U tunnel
  - TEID (Tunnel Endpoint ID) identifies each tunnel

X2 Interface

  eNodeB1 ----[X2]---- eNodeB2
  - Control plane: X2AP protocol
  - User plane: GTP-U tunnels

  X2AP Procedures:
  - Handover Preparation (Source -> Target)
  - Handover Request/Acknowledge
  - SN Status Transfer (PDCP sequence numbers)
  - UE Context Release
  - Load Indication
  - eNB Configuration Update
  - Resource Status Reporting

Interface Protocol Stacks

InterfacePlaneProtocolTransport

4G to 5G Interface Evolution

LTE5G EquivalentChange
S1-MMENG (N2)gNB to AMF, same SCTP transport
S1-UNG-U (N3)gNB to UPF, GTP-U still used
X2-CPXn-CgNB to gNB, XnAP protocol
X2-UPXn-UGTP-U still used for data forwarding

Key difference: In LTE, the eNodeB connects to a single MME and single S-GW. In 5G, a gNB can connect to multiple AMFs (AMF set) and multiple UPFs, enabling better load balancing and redundancy. The NG interface also supports network slicing via S-NSSAI.

C++ S1AP Handler

#include <cstdint>
#include <unordered_map>
#include <vector>

// S1AP procedure codes
enum class S1apProcedure : uint8_t {
    INITIAL_UE_MESSAGE        = 9,
    INITIAL_CONTEXT_SETUP_REQ = 10,
    INITIAL_CONTEXT_SETUP_RSP  = 11,
    UE_CONTEXT_RELEASE_CMD   = 21,
    UE_CONTEXT_RELEASE_COMP  = 22,
    HANDOVER_REQUEST          = 13,
    HANDOVER_REQUEST_ACK      = 14,
    PATH_SWITCH_REQUEST       = 24,
    E_RAB_SETUP_REQ           = 15,
    E_RAB_SETUP_RSP           = 16,
    S1_SETUP_REQUEST          = 5,
    S1_SETUP_RESPONSE         = 6
};

// S1AP E-RAB to be setup item
struct ErabToBeSetup {
    uint8_t  erab_id;       // EPS radio bearer ID
    uint32_t teid_dl;        // S1-U DL TEID (SGW)
    uint32_t teid_ul;        // S1-U UL TEID (eNB)
    uint32_t sgw_addr;      // S-GW IP address
    uint8_t  qci;           // QoS Class Identifier
    uint32_t gbr_dl;        // Guaranteed bit rate DL
    uint32_t gbr_ul;        // Guaranteed bit rate UL
};

// S1AP handler at eNodeB side
class S1apHandler {
public:
    void handleInitialContextSetup(
            uint32_t mme_ue_s1ap_id, uint32_t enb_ue_s1ap_id,
            const std::vector<ErabToBeSetup>& erabs,
            const uint8_t* nas_pdu, size_t nas_len) {

        // Map MME UE ID to local context
        uint32_t ue_id = getUeId(enb_ue_s1ap_id);

        // Setup each E-RAB (DRB via RRC)
        for (const auto& erab : erabs) {
            setupDrb(ue_id, erab);
            // Store S1-U TEID for user plane
            s1u_teids_[ue_id][erab.erab_id] = {
                erab.teid_dl, erab.teid_ul, erab.sgw_addr
            };
        }

        // Send RRC Reconfiguration with NAS to UE
        sendRrcReconfig(ue_id, nas_pdu, nas_len);

        // Send S1AP Initial Context Setup Response to MME
        sendInitialContextSetupResponse(
            mme_ue_s1ap_id, enb_ue_s1ap_id);
    }

    void handleErabSetupRequest(
            uint32_t mme_ue_id, uint32_t enb_ue_id,
            const std::vector<ErabToBeSetup>& erabs) {
        for (const auto& erab : erabs) {
            setupDrb(getUeId(enb_ue_id), erab);
        }
    }

private:
    struct S1uTeidInfo {
        uint32_t dl_teid;
        uint32_t ul_teid;
        uint32_t sgw_ip;
    };

    std::unordered_map<uint32_t,
        std::unordered_map<uint8_t, S1uTeidInfo>> s1u_teids_;

    uint32_t getUeId(uint32_t enb_ue_id) { return enb_ue_id; }
    void setupDrb(uint32_t ue_id, const ErabToBeSetup& erab) {}
    void sendRrcReconfig(uint32_t, const uint8_t*, size_t) {}
    void sendInitialContextSetupResponse(uint32_t, uint32_t) {}
};

SCTP in Telecom

SCTP advantages over TCP: All control plane interfaces (S1AP, X2AP, NGAP, XnAP, F1AP) use SCTP. SCTP provides multi-homing (survives link failure), multi-streaming (avoid head-of-line blocking), and message-oriented delivery. Each UE association typically gets a dedicated SCTP stream.
