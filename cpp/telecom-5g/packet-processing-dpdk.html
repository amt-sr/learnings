Packet Processing with DPDK

DPDK (Data Plane Development Kit) is an open-source set of libraries and drivers for fast packet processing. It enables user-space applications to directly access NIC hardware, bypassing the OS kernel. DPDK is the foundation for vRAN, 5G UPF, and high-throughput networking appliances.

DPDK Architecture Overview

  Traditional Kernel Networking:
  NIC -> IRQ -> Kernel (sk_buff) -> Syscall -> User Space

  DPDK Networking:
  NIC -> Poll Mode Driver (PMD) -> Hugepages -> User Space
         (no interrupts, no syscalls, no context switches)

  Key Components:
  +------------------+
  | rte_mbuf         |  Packet buffer descriptor
  +------------------+
  | rte_mempool      |  Pre-allocated mbuf pool
  +------------------+
  | rte_ring         |  Lock-free ring buffer (SPSC/MPSC)
  +------------------+
  | PMD (Poll Mode)  |  NIC driver, no IRQ
  +------------------+
  | Hugepages        |  2 MB / 1 GB pages, TLB-friendly
  +------------------+

Hugepages Configuration

  # Allocate 1024 x 2MB hugepages
  echo 1024 > /sys/kernel/mm/hugepages/hugepages-2048kB/nr_hugepages

  # Mount hugetlbfs
  mkdir -p /dev/hugepages
  mount -t hugetlbfs nodev /dev/hugepages

  # Verify
  cat /proc/meminfo | grep HugePages

  Why hugepages?
  - Standard 4 KB pages -> TLB misses at 10 Mpps
  - 2 MB pages -> 512x fewer TLB entries needed
  - 1 GB pages -> even fewer, best for massive memory pools
  - Reduces page walk latency from ~100 cycles to ~10 cycles

NUMA Awareness

NUMA penalty: Accessing memory on a remote NUMA node adds ~40-80ns latency per access. In DPDK, always allocate memory (mempools, rings) on the same NUMA node as the core that uses them. Cross-NUMA access kills throughput in packet processing.

// NUMA-aware allocation
struct rte_mempool* createMempool(int socket_id) {
    return rte_mempool_create("pkt_pool", 8192,
        2048 + sizeof(struct rte_mbuf),  // size + mbuf overhead
        256,       // cache size per core
        128,       // private data size
        socket_id, // NUMA socket ID
        NULL, NULL, NULL, NULL,
        NULL, NULL);  // MP init, obj init
}

// Pin thread to NUMA-local core
void pinToNumaCore(int core_id) {
    cpu_set_t cpuset;
    CPU_ZERO(&cpuset);
    CPU_SET(core_id, &cpuset);
    pthread_setaffinity_np(pthread_self(),
                            sizeof(cpuset), &cpuset);
}

rte_ring: Lock-Free Ring Buffer

  rte_ring modes:
  - SPSC: Single Producer, Single Consumer (fastest)
  - MPSC: Multi Producer, Single Consumer
  - SPMC: Single Producer, Multi Consumer
  - MPMC: Multi Producer, Multi Consumer

  Operations (lock-free with CAS):
  - rte_ring_sp_enqueue(): Single-producer enqueue (burst)
  - rte_ring_mp_enqueue(): Multi-producer enqueue (burst)
  - rte_ring_sc_dequeue(): Single-consumer dequeue (burst)
  - rte_ring_mc_dequeue(): Multi-consumer dequeue (burst)

  Burst API (process N packets at once):
  - rte_ring_sp_enqueue_burst(): enqueue up to N items
  - rte_ring_sc_dequeue_burst(): dequeue up to N items

C++ DPDK Receive Loop

#include <rte_eal.h>
#include <rte_ethdev.h>
#include <rte_mbuf.h>
#include <rte_mempool.h>
#include <rte_ring.h>
#include <cstdint>
#include <cstdio>

constexpr uint16_t RX_BURST_SIZE = 32;

// Initialize DPDK EAL (Environment Abstraction Layer)
int initDpdk(int argc, char* argv[]) {
    return rte_eal_init(argc, argv);
}

// Create packet memory pool
struct rte_mempool* createPacketPool(int socket_id) {
    return rte_pktmbuf_pool_create("pkt_pool",
        8192 * 16,     // 16K mbufs
        256,           // cache per lcore
        0,             // private size
        2048,          // data room size
        socket_id);
}

// Main packet receive loop (runs on dedicated core)
void rxLoop(uint16_t port_id, struct rte_mempool* mbuf_pool,
            struct rte_ring* rx_ring) {
    while (true) {
        struct rte_mbuf* mbufs[RX_BURST_SIZE];
        uint16_t nb_rx = rte_eth_rx_burst(port_id, 0,
                            mbufs, RX_BURST_SIZE);

        if (nb_rx == 0) continue;  // Poll, no yield

        // Process burst of packets
        for (uint16_t i = 0; i < nb_rx; ++i) {
            struct rte_mbuf* m = mbufs[i];

            // Process Ethernet/IP/GTP header
            processPacket(m);

            // Enqueue to application ring
            rte_ring_sp_enqueue_burst(rx_ring,
                (void**)mbufs, nb_rx, NULL);
        }

        // Free processed mbufs (if not forwarded)
        for (uint16_t i = 0; i < nb_rx; ++i) {
            rte_pktmbuf_free(mbufs[i]);
        }
    }
}

// Process individual packet (L2/L3/L4)
void processPacket(struct rte_mbuf* m) {
    uint8_t* pkt_data = rte_pktmbuf_mtod(m, uint8_t*);
    uint16_t pkt_len = rte_pktmbuf_pkt_len(m);

    // Ethernet header (14 bytes)
    // IP header (20 bytes)
    // GTP-U header (8 bytes minimum)
    // Inner Ethernet + IP + UDP/TCP

    // Extract GTP-U TEID (bytes 8-11 of GTP header)
    if (pkt_len > 38) {
        uint32_t teid = (pkt_data[34] << 24)
                      | (pkt_data[35] << 16)
                      | (pkt_data[36] << 8)
                      |  pkt_data[37];

        // Route based on TEID to correct bearer/Ue
        routePacket(teid, m);
    }
}

void routePacket(uint32_t teid, struct rte_mbuf* m) {}

// Entry point
int main(int argc, char* argv[]) {
    initDpdk(argc, argv);

    int socket_id = rte_socket_id();
    auto* pool = createPacketPool(socket_id);

    // Configure NIC port
    uint16_t port = 0;
    rte_eth_dev_configure(port, 1, 1, nullptr);
    rte_eth_rx_queue_setup(port, 0, 1024, socket_id, nullptr, pool);
    rte_eth_tx_queue_setup(port, 0, 1024, socket_id, nullptr);
    rte_eth_dev_start(port);

    auto* ring = rte_ring_create("rx_ring", 8192, socket_id, 0);
    rxLoop(port, pool, ring);
    return 0;
}

Burst Processing Benefits

ApproachPackets/SyscallCache EfficiencyThroughput
Per-packet processing1Poor (cold cache each time)5-10 Mpps
Burst of 3232Good (prefetch within burst)30-50 Mpps
Burst of 6464Best (L1/L2 cache hot)50-80 Mpps

Key DPDK principle: Batch processing is everything. Processing packets one at a time defeats the purpose of DPDK. Always use burst APIs, prefetch next packets, and keep working data in L1/L2 cache.
