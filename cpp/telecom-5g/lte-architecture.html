LTE Architecture

LTE (Long-Term Evolution) is the 4G cellular architecture defined by 3GPP. It consists of the Evolved UTRAN (E-UTRAN) for radio access and the Evolved Packet Core (EPC) for core networking. Understanding LTE architecture is essential for appreciating 5G design changes.

Architecture Diagram

  LTE Architecture (EPC + E-UTRAN):

  UE ---[Uu]--- eNodeB ---[S1-MME]--- MME
                  |                       |
               [S1-U]                [S6a] | [S11]
                  |                       |    |
              S-GW ----[S5/S8]---- P-GW ---[SGi]--- Internet
                                    |
                               PCRF (Rx+)
                               HSS  (S6a)

  Key Nodes:
  eNodeB  = Base station (radio + control + user plane)
  MME     = Mobility Management Entity (control plane)
  S-GW    = Serving Gateway (user plane anchor, mobility)
  P-GW    = PDN Gateway (UE IP allocation, packet filtering)
  HSS     = Home Subscriber Server (subscription data)
  PCRF    = Policy and Charging Rules Function (QoS)

C++ Architecture Model

#include <string>
#include <vector>
#include <unordered_map>
#include <memory>
#include <cstdint>

// Cell configuration for an eNodeB
struct ENodeBConfig {
    uint32_t    cell_id;           // ECI (28-bit)
    uint16_t    tac;               // Tracking Area Code
    uint16_t    plmn_id;           // PLMN (MCC+MNC)
    uint32_t    earfcn;            // ARFCN
    uint8_t     bandwidth;         // 1.4, 3, 5, 10, 15, 20 MHz
    uint8_t     pci;               // Physical Cell ID (0-503)
    uint16_t    max_ues;           // Max UEs supported
};

// EPS Bearer context
struct EpsBearerContext {
    uint8_t  bearer_id;           // EPS bearer ID (5-15)
    uint8_t  qci;                 // QoS Class Identifier (1-9)
    uint32_t arp;                 // Allocation and Retention Priority
    uint32_t ambr_ul;             // Aggregate MBR UL (bps)
    uint32_t ambr_dl;             // Aggregate MBR DL (bps)
    uint32_t gbr_ul;              // Guaranteed BR UL
    uint32_t gbr_dl;              // Guaranteed BR DL
};

// Simplified EPC node abstraction
class EpcCore {
public:
    struct UeContext {
        uint32_t                ue_id;
        std::string             imsi;
        uint32_t                enb_ue_id;
        uint16_t                cell_id;
        uint16_t                tac;
        std::vector<EpsBearerContext> bearers;
        uint8_t                 mme_ue_s1ap_id;
    };

    // MME function: handle UE attach
    void handleAttachRequest(uint32_t enb_id, uint16_t cell_id,
                              const std::string& imsi) {
        uint32_t ue_id = allocateUeId();
        UeContext ctx;
        ctx.ue_id = ue_id;
        ctx.imsi = imsi;
        ctx.enb_ue_id = enb_id;
        ctx.cell_id = cell_id;
        ctx.tac = getTacForCell(cell_id);

        // Query HSS for subscription
        auto sub_info = queryHss(imsi);
        if (!sub_info.allowed) {
            sendAttachReject(ue_id);
            return;
        }

        // Create default bearer
        EpsBearerContext default_bearer{};
        default_bearer.bearer_id = 5;
        default_bearer.qci = sub_info.default_qci;
        default_bearer.ambr_ul = sub_info.subscribed_ambr_ul;
        default_bearer.ambr_dl = sub_info.subscribed_ambr_dl;
        ctx.bearers.push_back(default_bearer);

        ue_contexts_[ue_id] = ctx;
        sendAttachAccept(ue_id, default_bearer);
    }

private:
    std::unordered_map<uint32_t, UeContext> ue_contexts_;

    uint32_t allocateUeId() { return next_ue_id_++; }
    uint16_t getTacForCell(uint16_t cell_id) { return 0x01A0; }
    void sendAttachAccept(uint32_t, const EpsBearerContext&) {}
    void sendAttachReject(uint32_t) {}

    struct SubscriptionInfo {
        bool     allowed;
        uint8_t  default_qci;
        uint32_t subscribed_ambr_ul;
        uint32_t subscribed_ambr_dl;
    };
    SubscriptionInfo queryHss(const std::string& imsi);

    uint32_t next_ue_id_{1};
};

4G LTE vs 5G NR Comparison

Aspect4G LTE (EPC)5G NR (5GC)
CoreEPC (MME + S-GW + P-GW)5GC (AMF + SMF + UPF)
RAN nodeeNodeB (integrated CU+DU)gNB (CU-DU split)
Control planeMMEAMF (access) + SMF (session)
User planeS-GW + P-GWUPF
PagingMME -> eNodeBAMF -> gNB
QoS granularityEPS bearer (QCI)QoS flow (5QI) via SDAP
Network slicingNot supportedNSSAI-based slicing
State modelIDLE, CONNECTEDIDLE, INACTIVE, CONNECTED

5G evolution: The biggest architectural change is the separation of control and user plane. In 5G, AMF handles mobility while SMF manages sessions. UPF is a pure user-plane function. This CUPS (Control and User Plane Separation) enables flexible network deployment and edge computing.

S1-AP Message Flow

Here's a C++ simulation of the S1-AP attach procedure showing how messages flow between UE, eNodeB, and EPC nodes:

#include <iostream>
#include <string>
#include <functional>

// Simplified S1-AP message types
enum class S1apMsgType : uint8_t {
    INITIAL_UE_MESSAGE,        // eNodeB -> MME (NAS attach request)
    INITIAL_CONTEXT_SETUP_REQ, // MME -> eNodeB (security + bearers)
    INITIAL_CONTEXT_SETUP_RSP, // eNodeB -> MME (setup complete)
    ATTACH_ACCEPT,              // MME -> eNodeB -> UE (with EPS bearer)
};

struct S1apMessage {
    S1apMsgType type;
    uint32_t    mme_ue_id;
    uint32_t    enb_ue_id;
    uint32_t    cell_id;
    std::string nas_payload;
};

// S1 interface simulation
class S1Interface {
public:
    void sendToMme(const S1apMessage& msg) {
        std::cout << "[S1] eNodeB -> MME: "
                  << msgName(msg.type) << "\n";
    }

    void sendToEnb(const S1apMessage& msg) {
        std::cout << "[S1] MME -> eNodeB: "
                  << msgName(msg.type) << "\n";
    }

private:
    const char* msgName(S1apMsgType t) {
        static const char* names[] = {
            "InitialUEMessage", "InitCtxSetupReq",
            "InitCtxSetupRsp", "AttachAccept"
        };
        return names[static_cast<int>(t)];
    }
};

void simulateAttach(S1Interface& s1) {
    uint32_t enb_ue_id = 1, mme_ue_id = 100;

    // Step 1: eNodeB forwards NAS Attach Request to MME
    S1apMessage msg1{S1apMsgType::INITIAL_UE_MESSAGE, mme_ue_id,
                     enb_ue_id, 0x01A0, "NAS_ATTACH_REQUEST"};
    s1.sendToMme(msg1);

    // Step 2: MME sends Initial Context Setup (after auth + HSS query)
    S1apMessage msg2{S1apMsgType::INITIAL_CONTEXT_SETUP_REQ, mme_ue_id,
                     enb_ue_id, 0x01A0, "SECURITY+BEARER_CTX"};
    s1.sendToEnb(msg2);

    // Step 3: eNodeB acknowledges context setup
    S1apMessage msg3{S1apMsgType::INITIAL_CONTEXT_SETUP_RSP, mme_ue_id,
                     enb_ue_id, 0x01A0, "CONTEXT_ACTIVE"};
    s1.sendToMme(msg3);

    // Step 4: MME sends Attach Accept (via eNodeB to UE)
    S1apMessage msg4{S1apMsgType::ATTACH_ACCEPT, mme_ue_id,
                     enb_ue_id, 0x01A0, "NAS_ATTACH_ACCEPT"};
    s1.sendToEnb(msg4);
}

int main() {
    S1Interface s1;
    simulateAttach(s1);
    return 0;
}
