Real-Time Processing

Telecom systems operate under strict real-time constraints. An OFDM symbol in 5G NR must be processed within microseconds, and MAC scheduling decisions must be made within one TTI (Transmission Time Interval). Missing deadlines causes packet drops, increased latency, or radio link failure.

Deadline Scheduling (SCHED_DEADLINE)

Linux SCHED_DEADLINE: The most appropriate scheduler for real-time RAN workloads. It provides guaranteed CPU time using the sporadic task model (C, D, P) where C = computation time, D = deadline, P = period. The kernel ensures each task gets at least C microseconds every P microseconds.

// Set SCHED_DEADLINE via sched_setattr()
#define _GNU_SOURCE
#include <sched.h>
#include <unistd.h>
#include <cstring>

void setDeadlineScheduling(int cpu) {
    cpu_set_t cpuset;
    CPU_ZERO(&cpuset);
    CPU_SET(cpu, &cpuset);

    // Pin to specific CPU core
    sched_setaffinity(0, sizeof(cpuset), &cpuset);

    // SCHED_DEADLINE parameters
    struct sched_attr attr{};
    attr.size = sizeof(attr);
    attr.sched_policy = SCHED_DEADLINE;
    attr.sched_runtime = 50000;    // 50 us computation budget
    attr.sched_deadline = 125000; // 125 us deadline (1 OFDM symbol)
    attr.sched_period = 125000;  // 125 us period (matching SCS)

    if (sched_setattr(0, &attr, 0) == -1) {
        // Fallback to SCHED_FIFO if DEADLINE unavailable
        attr.sched_policy = SCHED_FIFO;
        attr.sched_priority = 99;
        sched_setscheduler(0, SCHED_FIFO, &attr);
    }
}

Priority Inversion and Solutions

  Priority Inversion Scenario:

  High-priority (PHY)       Low-priority (logging)
       |                         |
       |  needs mutex M          |
       |  blocked...             |
       |                    locks M
       |                    preempted by...
                                  Medium-priority (monitor)
                                  runs indefinitely
       |  DEADLOCK (effectively)

  Solutions:
  1. Priority Inheritance (PI) mutex: Low-priority thread
     temporarily inherits high priority while holding the lock.
     Linux: pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_INHERIT)

  2. Priority Ceiling: Mutex assigned a ceiling priority.
     Any thread locking it runs at ceiling priority.
     Linux: pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_PROTECT)

  3. Lock-free data structures: Eliminate locks entirely
     using atomics and ring buffers (preferred in data path).

CPU Affinity and NO_HZ

#include <sched.h>
#include <unistd.h>

// Thread-per-core pinning for RAN
void pinThreadToCore(int core_id) {
    cpu_set_t cpuset;
    CPU_ZERO(&cpuset);
    CPU_SET(core_id, &cpuset);
    pthread_setaffinity_np(pthread_self(),
                            sizeof(cpuset), &cpuset);
}

// Kernel boot parameters for NO_HZ (tickless):
// nohz_full=2-7 rcu_nocbs=2-7
// Cores 2-7 run without timer interrupts (dedicated to RAN)
// Core 0-1 handle system/OS tasks

// Isolate cores via kernel command line:
// isolcpus=2-7
// Prevents scheduler from placing other tasks on RAN cores

Critical: In production RAN systems, timer interrupts, softirqs, and workqueues on data-path cores cause jitter. NO_HZ full, rcu_nocbs, and isolcpus are essential kernel parameters. Without them, latency spikes of 10-100 us are common, which can violate URLLC deadlines.

DPDK Overview for Telecom

DPDK (Data Plane Development Kit) bypasses the Linux kernel to achieve line-rate packet processing. Instead of system calls and interrupt-driven networking, DPDK uses poll-mode drivers, hugepages for zero-copy, and user-space memory for packet buffers.

C++ Real-Time Timer

#include <chrono>
#include <atomic>
#include <iostream>

class HighResolutionTimer {
public:
    void start() {
        start_ = std::chrono::high_resolution_clock::now();
    }

    uint64_t elapsedMicroseconds() const {
        auto now = std::chrono::high_resolution_clock::now();
        return std::chrono::duration_cast<
            std::chrono::microseconds>(now - start_).count();
    }

    bool checkDeadline(uint64_t deadline_us) const {
        return elapsedMicroseconds() <= deadline_us;
    }

private:
    std::chrono::high_resolution_clock::time_point start_;
};

// Usage in data path
void processSlot() {
    HighResolutionTimer timer;
    timer.start();

    // Process PHY symbol
    processPhySymbol();

    if (!timer.checkDeadline(125)) {
        // Deadline miss -- log and potentially trigger
        // scheduler adjustment or alarm
        std::atomic<uint64_t>::add_fetch(&deadline_misses_, 1);
    }
}

static std::atomic<uint64_t> deadline_misses_{0};
void processPhySymbol() {}
