Signal Processing Basics

Digital signal processing (DSP) is at the heart of the PHY layer in both LTE and 5G NR. Understanding sampling, FFT, and OFDM is essential for anyone working on telecom PHY or even optimizing upper-layer processing that interacts with the radio.

Sampling Theorem (Nyquist-Shannon)

A continuous signal can be perfectly reconstructed from its samples if the sampling rate is at least twice the highest frequency component. For a signal with bandwidth B, the minimum sampling rate is f_s >= 2 * B.

FFT Basics

The Fast Fourier Transform converts time-domain samples to frequency-domain (and IFFT does the reverse). In OFDM, IFFT creates orthogonal subcarriers from frequency-domain data symbols.

  DFT formula:
  X[k] = sum(x[n] * e^(-j*2*pi*k*n/N)) for n = 0 to N-1

  FFT complexity: O(N * log2(N))
  DFT complexity: O(N^2)

  For N = 4096:
  FFT: ~4096 * 12 = 49,152 operations
  DFT: ~4096 * 4096 = 16,777,216 operations

  In 5G NR, one OFDM symbol requires one FFT/IFFT per TTI.
  At 120 kHz SCS: one FFT per ~8.3 us.

OFDM Basics

  OFDM (Orthogonal Frequency Division Multiplexing):
  - Data transmitted on multiple parallel subcarriers
  - Subcarriers are orthogonal (integers of base frequency)
  - IFFT at TX creates time-domain OFDM symbol
  - FFT at RX recovers frequency-domain symbols
  - Cyclic Prefix (CP) copies tail of symbol to front to
    handle multi-path delay spread

  5G NR Subcarrier Spacing (SCS):
  +--------+-------+----------+-----------+
  | SCS    | Slots | Symbol   | CP length |
  | (kHz)  | /ms   | duration | (us)      |
  +--------+-------+----------+-----------+
  | 15     | 1     | 66.7 us  | 4.7 us    |
  | 30     | 2     | 33.3 us  | 2.3 us    |
  | 60     | 4     | 16.7 us  | 1.2 us    |
  | 120    | 8     | 8.3 us   | 0.59 us   |
  | 240    | 16    | 4.2 us   | 0.29 us   |
  +--------+-------+----------+-----------+

  Higher SCS = shorter symbol = lower latency (for URLLC)
  Lower SCS = longer symbol = better coverage (for mMTC)

Cyclic Prefix (CP)

Purpose of CP: The cyclic prefix is a copy of the last part of the OFDM symbol prepended to its beginning. It converts a linear convolution (from multi-path channel) into a circular convolution, which the FFT can handle efficiently. The CP length must be longer than the maximum expected delay spread. Normal CP: ~4.7 us (covers ~1.4 km extra path).

C++ FFT Pseudocode

#include <vector>
#include <complex>
#include <cmath>

using Complex = std::complex<double>;
const double PI = 3.14159265358979323846;

// DFT (naive, O(N^2)) -- for understanding only
std::vector<Complex> dft(const std::vector<Complex>& x) {
    size_t N = x.size();
    std::vector<Complex> X(N);
    for (size_t k = 0; k < N; ++k) {
        X[k] = 0;
        for (size_t n = 0; n < N; ++n) {
            double angle = -2.0 * PI * k * n / N;
            Complex W(std::cos(angle), std::sin(angle));
            X[k] += x[n] * W;
        }
    }
    return X;
}

// Radix-2 Cooley-Tukey FFT (O(N log N))
void fftInPlace(std::vector<Complex>& x) {
    size_t N = x.size();
    if (N == 0 || (N & (N - 1)) != 0) return; // Must be power of 2

    // Bit-reversal permutation
    size_t j = 0;
    for (size_t i = 1; i < N; ++i) {
        size_t bit = N >> 1;
        while (j & bit) {
            j ^= bit;
            bit >>= 1;
        }
        j ^= bit;
        if (i < j) std::swap(x[i], x[j]);
    }

    // Butterfly stages
    for (size_t len = 2; len <= N; len <<= 1) {
        double angle = -2.0 * PI / len;
        Complex Wlen(std::cos(angle), std::sin(angle));
        for (size_t i = 0; i < N; i += len) {
            Complex w(1.0);
            for (size_t k = 0; k < len / 2; ++k) {
                Complex u = x[i + k];
                Complex v = x[i + k + len/2] * w;
                x[i + k] = u + v;
                x[i + k + len/2] = u - v;
                w *= Wlen;
            }
        }
    }
}

// IFFT = FFT with conjugated twiddles, then divide by N
void ifftInPlace(std::vector<Complex>& x) {
    // Conjugate input
    for (auto& c : x) c = std::conj(c);
    // Forward FFT
    fftInPlace(x);
    // Conjugate and scale
    for (auto& c : x) c = std::conj(c) / static_cast<double>(x.size());
}

// Generate OFDM symbol from subcarrier data
std::vector<Complex> generateOfdmSymbol(
        const std::vector<Complex>& subcarriers,
        size_t fft_size) {
    // Map subcarriers to FFT bins (centered around DC)
    std::vector<Complex> freq_domain(fft_size, 0.0);
    size_t n_sc = subcarriers.size();
    size_t offset = fft_size / 2 - n_sc / 2;
    for (size_t i = 0; i < n_sc; ++i) {
        freq_domain[offset + i] = subcarriers[i];
    }

    // IFFT to get time-domain OFDM symbol
    ifftInPlace(freq_domain);
    return freq_domain;
}

// Add cyclic prefix
std::vector<Complex> addCyclicPrefix(
        const std::vector<Complex>& symbol,
        size_t cp_length) {
    size_t N = symbol.size();
    std::vector<Complex> cp_symbol(N + cp_length);
    // Copy CP from end of symbol
    for (size_t i = 0; i < cp_length; ++i) {
        cp_symbol[i] = symbol[N - cp_length + i];
    }
    // Copy full symbol after CP
    for (size_t i = 0; i < N; ++i) {
        cp_symbol[cp_length + i] = symbol[i];
    }
    return cp_symbol;
}

Windowing

Window functions are applied to OFDM symbols to reduce out-of-band emissions. In 5G NR, time-domain windowing is applied before transmission to smooth symbol transitions at the CP boundary.

Production FFT: Real PHY implementations use FFTW (Fastest Fourier Transform in the West), ARM CMSIS FFT, or custom SIMD-optimized FFT kernels. The C++ code above is for educational purposes. Production FFT achieves sub-microsecond latency using AVX-512 or NEON vectorization.
