enum class

enum class (scoped enumeration, introduced in C++11) provides type-safe enumerations that avoid the pitfalls of traditional C-style enums. They prevent implicit conversions, namespace pollution, and comparison between unrelated enumerators.

C-Style enum Problems

#include <iostream>

// C-style enum — many problems
enum Color { RED, GREEN, BLUE };
enum TrafficLight { RED, YELLOW, GREEN };  // ERROR: redefines RED and GREEN

// Even if it compiled:
void c_enum_problems() {
    Color c = RED;
    int x = c;             // implicit conversion to int — not type-safe
    c = 42;                // assign int to enum — no error!
    if (c == GREEN) {}     // works, but RED and GREEN are in global scope

    // Both REDs would be ambiguous
    // Color r = RED;  // Which RED? Color::RED or TrafficLight::RED?
}

Traditional enum issues: Enumerators leak into the enclosing scope, implicitly convert to and from integers, and cannot specify the underlying type portably. These issues led to many subtle bugs in large codebases.

enum class — Scoped Enumerations

#include <iostream>
#include <string>

// Scoped enumerations — each is its own type
enum class Color { Red, Green, Blue };
enum class TrafficLight { Red, Yellow, Green };
enum class Fruit { Apple, Orange, Banana };

void enum_class_basics() {
    Color c = Color::Red;
    // Color c2 = Red;           // ERROR: must qualify with Color::
    // int x = c;                // ERROR: no implicit conversion
    // if (c == Fruit::Apple) {} // ERROR: cannot compare different enum classes

    // Explicit conversion to underlying type
    int val = static_cast<int>(c);  // 0
    std::cout << "Color::Red as int: " << val << "\n";  // 0

    // No ambiguity between Color::Red and TrafficLight::Red
    TrafficLight t = TrafficLight::Red;  // OK: different type
}

Underlying Type

You can specify the underlying type of an enum class to control its size and representation.

#include <iostream>
#include <cstdint>

// Specify underlying type for memory layout control
enum class Status : uint8_t { Ok = 0, Error = 1, Pending = 2, Timeout = 3 };
enum class Priority : int8_t { Low = -1, Normal = 0, High = 1, Critical = 2 };
enum class Flags : uint32_t { None = 0, Read = 1, Write = 2, Execute = 4 };

void underlying_type_demo() {
    std::cout << "sizeof(Status) = " << sizeof(Status) << "\n";      // 1
    std::cout << "sizeof(Priority) = " << sizeof(Priority) << "\n";  // 1
    std::cout << "sizeof(Flags) = " << sizeof(Flags) << "\n";       // 4

    // Using flags with bitwise operators
    Flags perms = Flags::Read | Flags::Write;
    bool can_read = (static_cast<uint32_t>(perms) & static_cast<uint32_t>(Flags::Read)) != 0;
    std::cout << "Can read: " << can_read << "\n";  // true
}

Switch with enum class

#include <iostream>
#include <string>

enum class HttpStatusCode {
    OK = 200,
    NotFound = 404,
    ServerError = 500,
    ServiceUnavailable = 503
};

std::string status_message(HttpStatusCode code) {
    switch (code) {
        case HttpStatusCode::OK:           return "Everything is fine";
        case HttpStatusCode::NotFound:     return "Resource not found";
        case HttpStatusCode::ServerError:  return "Internal server error";
        case HttpStatusCode::ServiceUnavailable: return "Service unavailable";
    }
    return "Unknown status code";  // warning in some compilers if not all cases covered
}

void switch_demo() {
    std::cout << status_message(HttpStatusCode::OK) << "\n";
    std::cout << status_message(HttpStatusCode::NotFound) << "\n";
}

enum class with Bit Flags

For bitmask operations, define overloaded operators to keep enum class ergonomic.

#include <iostream>

enum class Permission : uint32_t {
    None    = 0,
    Read    = 1u << 0,
    Write   = 1u << 1,
    Execute = 1u << 2,
    All     = Read | Write | Execute
};

// Enable bitwise operators for enum class
constexpr Permission operator|(Permission a, Permission b) {
    return static_cast<Permission>(static_cast<uint32_t>(a) | static_cast<uint32_t>(b));
}

constexpr Permission operator&(Permission a, Permission b) {
    return static_cast<Permission>(static_cast<uint32_t>(a) & static_cast<uint32_t>(b));
}

constexpr Permission operator~(Permission a) {
    return static_cast<Permission>(~static_cast<uint32_t>(a));
}

constexpr bool has_permission(Permission perms, Permission flag) {
    return (perms & flag) != Permission::None;
}

void bit_flags_demo() {
    Permission user_perms = Permission::Read | Permission::Execute;

    std::cout << std::boolalpha;
    std::cout << "Can read: "    << has_permission(user_perms, Permission::Read) << "\n";    // true
    std::cout << "Can write: "   << has_permission(user_perms, Permission::Write) << "\n";   // false
    std::cout << "Can execute: " << has_permission(user_perms, Permission::Execute) << "\n"; // true

    // Add write permission
    user_perms = user_perms | Permission::Write;
    std::cout << "Can write now: " << has_permission(user_perms, Permission::Write) << "\n"; // true
}

enum class vs enum — Comparison

Featureenum classTraditional enum
ScopeScoped (must qualify)Unscoped (leaks into enclosing scope)
Implicit int conversionNoYes
Specify underlying typeDefault int, configurableImplementation-defined
Forward declarationYes (always)Only with specified underlying type
Type safetyHighLow

Guideline: Always prefer enum class over traditional enum. The slight extra verbosity of Color::Red is worth the type safety, namespace isolation, and prevention of implicit conversions.

Practical Example: State Machine

#include <iostream>
#include <string>

enum class ConnectionState { Disconnected, Connecting, Connected, Disconnecting };

class Connection {
    ConnectionState state_ = ConnectionState::Disconnected;
public:
    void connect() {
        if (state_ == ConnectionState::Disconnected) {
            state_ = ConnectionState::Connecting;
            std::cout << "Connecting...\n";
            state_ = ConnectionState::Connected;
            std::cout << "Connected.\n";
        }
    }

    void disconnect() {
        if (state_ == ConnectionState::Connected) {
            state_ = ConnectionState::Disconnecting;
            std::cout << "Disconnecting...\n";
            state_ = ConnectionState::Disconnected;
            std::cout << "Disconnected.\n";
        }
    }

    ConnectionState state() const { return state_; }
};
