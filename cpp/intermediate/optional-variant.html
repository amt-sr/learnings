Optional / Variant

C++17 introduced std::optional and std::variant as type-safe alternatives to "magic values" (like -1 or nullptr) and raw unions. Together with std::any, they form the modern C++ vocabulary for representing values that may or may not exist, or values that hold one of several types.

std::optional (C++17)

std::optional<T> wraps a value that may or may not be present. It is the type-safe replacement for returning sentinel values or using output parameters.

#include <optional>
#include <string>
#include <iostream>
#include <fstream>

// Instead of returning -1 or throwing on "not found"
std::optional<std::string> find_user(int id) {
    if (id == 1) return "Alice";
    if (id == 2) return "Bob";
    return std::nullopt;  // "no value"
}

void optional_demo() {
    auto user = find_user(1);
    if (user.has_value()) {
        std::cout << "Found: " << *user << "\n";
    }

    // value_or: provide default if empty
    auto name = find_user(99).value_or("Unknown");
    std::cout << name << "\n";  // Unknown

    // Using the arrow operator
    std::optional<std::string> config = read_config("path");
    if (config) {  // implicit bool conversion
        std::cout << config->length() << "\n";
    }
}

Never dereference empty optional: Calling *opt or opt.value() on a disengaged optional is undefined behavior (for *) or throws std::bad_optional_access (for .value()). Always check with has_value(), operator bool(), or use value_or().

Common optional Patterns

#include <optional>
#include <string_view>
#include <charconv>

// Parse with error handling (no exceptions)
std::optional<int> parse_int(std::string_view str) {
    int value = 0;
    auto [ptr, ec] = std::from_chars(str.begin(), str.end(), value);
    if (ec == std::errc{} && ptr == str.end())
        return value;
    return std::nullopt;
}

// Chaining optionals
std::optional<double> safe_divide(int a, int b) {
    if (b == 0) return std::nullopt;
    return static_cast<double>(a) / b;
}

void optional_patterns() {
    if (auto val = parse_int("42")) {
        std::cout << "Parsed: " << *val << "\n";
    }

}

std::variant (C++17)

std::variant<Types...> holds exactly one value from a set of alternative types at any time. It is a type-safe replacement for C-style unions.

#include <variant>
#include <string>
#include <iostream>
#include <vector>

// A value that can be int, double, or string
using Value = std::variant<int, double, std::string>;

void variant_basics() {
    Value v1 = 42;               // holds int
    Value v2 = 3.14;             // holds double
    Value v3 = std::string("hi"); // holds string (need explicit type)

    // Check which type is held
    std::cout << "v1 index: " << v1.index() << "\n";  // 0 (int)
    std::cout << std::holds_alternative<int>(v1) << "\n";  // true

    // Access with std::get (throws if wrong type)
    std::cout << std::get<int>(v1) << "\n";    // 42
    // std::get<double>(v1);  // THROWS std::bad_variant_access

    // Safe access with std::get_if (returns pointer or nullptr)
    if (auto* p = std::get_if<std::string>(&v3)) {
        std::cout << "String value: " << *p << "\n";
    }
}

The Visit Pattern

std::visit applies a callable to the active alternative. This is the primary way to process variant values.

#include <variant>
#include <iostream>
#include <string>

using Value = std::variant<int, double, std::string>

// Visitor as overloaded function object
struct PrintVisitor {
    void operator()(int v) const { std::cout << "int: " << v << "\n"; }
    void operator()(double v) const { std::cout << "double: " << v << "\n"; }
    void operator()(const std::string& v) const { std::cout << "string: " << v << "\n"; }
};

// C++17: lambda-based overloaded (helper)
template<class... Fs>
struct overloaded : Fs... { using Fs::operator()...; };
template<class... Fs>
overloaded(Fs...) -> overloaded<Fs...>;

void visit_demo() {
    Value v = 42;

    // Using a visitor struct
    std::visit(PrintVisitor{}, v);

    // Using lambdas with overloaded (C++17)
    std::visit(overloaded{
        [](int v) { std::cout << "INT: " << v << "\n"; },
        [](double v) { std::cout << "DBL: " << v << "\n"; },
        [](const std::string& v) { std::cout << "STR: " << v << "\n"; }
    }, v);
}

std::any

std::any can hold any copyable type. It is less type-safe than variant but more flexible when the set of possible types is unknown at compile time.

#include <any>
#include <iostream>
#include <string>

void any_demo() {
    std::any data = 42;
    data = std::string("hello");

    // Check type
    if (data.type() == typeid(std::string)) {
        std::cout << "Holds string\n";
    }

    // Retrieve with any_cast (throws if wrong type)
    try {
        auto& s = std::any_cast<std::string&>(data);
        std::cout << s << "\n";
    } catch (const std::bad_any_cast& e) {
        std::cout << "Wrong type: " << e.what() << "\n";
    }

    // Pointer version (returns nullptr on mismatch)
    if (auto* p = std::any_cast<int>(&data)) {
        std::cout << *p << "\n";  // not reached
    }
}

Best practice: Prefer optional for nullable values (instead of pointers or sentinel values). Prefer variant when a value can be one of several known types (instead of unions or inheritance hierarchies). Avoid any unless you truly need to store arbitrary types.

Optional and variant work well with tuples and structured bindings. For error handling via exceptions, see exception handling.
