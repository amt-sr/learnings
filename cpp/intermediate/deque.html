Deque

std::deque (double-ended queue) is a sequence container that supports efficient insertion and removal at both ends. Unlike std::vector, which uses a contiguous array, std::deque uses a collection of fixed-size arrays (chunks) managed by a central map of pointers, giving it O(1) push/pop at both front and back.

Deque Basics

#include <deque>
#include <iostream>

void deque_basics() {
    std::deque<int> dq;

    // Insert at both ends — O(1) amortized
    dq.push_back(10);   // [10]
    dq.push_front(5);   // [5, 10]
    dq.push_back(20);   // [5, 10, 20]
    dq.push_front(1);   // [1, 5, 10, 20]

    // Random access — O(1)
    std::cout << "Element at index 2: " << dq[2] << "\n";  // 10
    std::cout << "Front: " << dq.front() << "\n";       // 1
    std::cout << "Back: " << dq.back() << "\n";         // 20
    std::cout << "Size: " << dq.size() << "\n";           // 4

    // Remove from both ends
    dq.pop_front();  // [5, 10, 20]
    dq.pop_back();   // [5, 10]

    // Iterate
    for (const auto& val : dq) {
        std::cout << val << " ";
    }
    // Output: 5 10
}

Memory Layout

A std::deque internally maintains a map (array of pointers) to fixed-size blocks. Elements are stored sequentially within each block. This provides fast front/back operations while maintaining random access capability.

Memory allocation: A deque allocates memory in chunks (typically 512 bytes per chunk on 64-bit systems). The central map is a dynamically allocated array of pointers to these chunks. When the map runs out of space, it is reallocated -- but element blocks are not moved.

Practical: Sliding Window / Queue

#include <deque>
#include <vector>
#include <iostream>

// Sliding window maximum
std::vector<int> sliding_window_max(const std::vector<int>& nums, int k) {
    std::deque<int> dq;  // stores indices
    std::vector<int> result;

    for (int i = 0; i < (int)nums.size(); ++i) {
        // Remove indices outside the window
        while (!dq.empty() && dq.front() <= i - k) {
            dq.pop_front();
        }
        // Remove indices whose values are less than current
        while (!dq.empty() && nums[dq.back()] < nums[i]) {
            dq.pop_back();
        }
        dq.push_back(i);

        if (i >= k - 1) {
            result.push_back(nums[dq.front()]);
        }
    }
    return result;
}

void sliding_window_demo() {
    std::vector<int> nums = {1, 3, -1, -3, 5, 3, 6, 7};
    auto result = sliding_window_max(nums, 3);
    for (int v : result) std::cout << v << " ";
    // Output: 3 3 5 5 6 7
}

Practical: Undo/Redo Buffer

#include <deque>
#include <string>
#include <iostream>

class CommandHistory {
    std::deque<std::string> undo_stack;
    std::deque<std::string> redo_stack;
    static constexpr size_t MAX_HISTORY = 50;
public:
    void execute(const std::string& cmd) {
        undo_stack.push_back(cmd);
        if (undo_stack.size() > MAX_HISTORY) undo_stack.pop_front();
        redo_stack.clear();  // new command invalidates redo
    }

    std::string undo() {
        if (undo_stack.empty()) return "Nothing to undo";
        std::string cmd = undo_stack.back();
        undo_stack.pop_back();
        redo_stack.push_back(cmd);
        return "Undo: " + cmd;
    }

    std::string redo() {
        if (redo_stack.empty()) return "Nothing to redo";
        std::string cmd = redo_stack.back();
        redo_stack.pop_back();
        undo_stack.push_back(cmd);
        return "Redo: " + cmd;
    }
};

void undo_redo_demo() {
    CommandHistory history;
    history.execute("insert A");
    history.execute("insert B");
    history.execute("delete A");
    std::cout << history.undo() << "\n";  // Undo: delete A
    std::cout << history.redo() << "\n";  // Redo: delete A
}

Use deque when: You need fast insertion/removal at both ends (queue, sliding window, undo buffer). Use vector when you primarily need random access and back insertion with contiguous memory.

Cache performance: Vectors have better cache locality due to contiguous memory. If you do heavy random access (no front insertion), vector is faster. Deque chunk boundaries cause occasional cache misses during iteration.

Deque complements vector for general-purpose storage and list/forward_list for frequent insertion in the middle. See STL overview for the full container hierarchy.
