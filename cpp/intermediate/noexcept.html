noexcept

The noexcept specifier tells the compiler that a function is guaranteed not to throw exceptions. This enables compiler optimizations (no exception cleanup code), allows containers to make better choices (e.g., std::vector uses move instead of copy during reallocation if the move constructor is noexcept), and serves as documentation for API contracts.

noexcept Specifier

#include <iostream>
#include <vector>

// Unconditionally noexcept
void safe_function() noexcept {
    std::cout << "This will never throw\n";
}

// noexcept(false) — explicitly may throw (default for most functions)
void risky_function() noexcept(false) {
    // throw std::runtime_error("oops");  // OK
}

// Conditional noexcept — depends on whether T's move is noexcept
template<typename T>
void swap_values(T& a, T& b) noexcept(noexcept(T(std::move(a)))) {
    T tmp = std::move(a);
    a = std::move(b);
    b = std::move(tmp);
}

void noexcept_demo() {
    std::cout << std::boolalpha;
    std::cout << "safe_function noexcept? "
              << noexcept(safe_function()) << "\n";  // true
    std::cout << "risky_function noexcept? "
              << noexcept(risky_function()) << "\n";  // false

    // Conditional: depends on int's move constructor
    std::cout << "swap_values<int> noexcept? "
              << noexcept(swap_values<int>(std::declval<int&>(),
                                            std::declval<int&>()))
              << "\n";  // true (int move is noexcept)
}

noexcept Operator

The noexcept(expr) operator is a compile-time check that returns true if the compiler can determine that expr will not throw. It evaluates the expression at compile time without actually executing it.

#include <iostream>
#include <vector>
#include <string>

void noexcept_operator_demo() {
    std::cout << std::boolalpha;

    // Built-in operations are noexcept
    std::cout << "int + int noexcept? "
              << noexcept(std::declval<int>() + std::declval<int>()) << "\n";  // true

    // vector::size() is noexcept
    std::cout << "vector::size noexcept? "
              << noexcept(std::declval<std::vector<int>>().size()) << "\n";  // true

    // vector::at() is NOT noexcept (throws out_of_range)
    std::cout << "vector::at noexcept? "
              << noexcept(std::declval<std::vector<int>>().at(0)) << "\n";  // false
}

Move Constructors and noexcept

Marking move constructors and move assignment operators as noexcept has a significant practical impact: std::vector (and other containers) will only use move operations during reallocation if they are noexcept. Otherwise, it falls back to copy to maintain the strong exception guarantee.

#include <iostream>
#include <vector>

class NoThrowMovable {
    int* data_;
public:
    explicit NoThrowMovable(int val) : data_(new int(val)) {}
    ~NoThrowMovable() { delete data_; }

    // Move constructor — noexcept allows vector to move during realloc
    NoThrowMovable(NoThrowMovable&& other) noexcept
        : data_(other.data_) {
        other.data_ = nullptr;
    }

    NoThrowMovable& operator=(NoThrowMovable&& other) noexcept {
        if (this != &other) {
            delete data_;
            data_ = other.data_;
            other.data_ = nullptr;
        }
        return *this;
    }

    // Copy operations
    NoThrowMovable(const NoThrowMovable& other)
        : data_(new int(*other.data_)) {}
    NoThrowMovable& operator=(const NoThrowMovable& other) {
        *data_ = *other.data_;
        return *this;
    }
};

void move_noexcept_demo() {
    std::vector<NoThrowMovable> v;
    v.reserve(2);
    v.emplace_back(1);
    v.emplace_back(2);

    // When vector grows, it MOVES elements (fast, no copies)
    // because the move constructor is noexcept
    v.emplace_back(3);
    std::cout << "Vector moved elements during growth\n";
}

If a noexcept function throws: std::terminate() is called immediately. The program aborts without stack unwinding. This is why you must be certain that noexcept functions truly cannot throw -- or catch any potential exceptions internally.

What Should Be noexcept

Should be noexceptShould NOT be noexcept
Destructors (implicitly noexcept in C++11+)Functions that allocate memory (new)
Move constructors/assignment (when safe)Functions that call potentially-throwing code
swap functionsFunctions with complex logic (file I/O, parsing)
getters and simple accessorsVirtual functions (subclass may throw)
hash functionsFunctions returning by value of complex types
#include <utility>

class Point {
    double x_, y_;
public:
    Point(double x, double y) : x_(x), y_(y) {}

    // Getters: simple, no throwing
    double x() const noexcept { return x_; }
    double y() const noexcept { return y_; }

    // Swap: should always be noexcept
    friend void swap(Point& a, Point& b) noexcept {
        using std::swap;
        swap(a.x_, b.x_);
        swap(a.y_, b.y_);
    }
};

Rule of thumb: Make functions noexcept whenever they cannot throw. It enables better optimization and is part of a correct API contract. Destructors are implicitly noexcept since C++11 -- do not add throw to destructors.

noexcept interacts with exception handling and move semantics. For smart pointer usage, see smart pointers.
