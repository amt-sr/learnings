Intermediate Practice: Build a Task Scheduler

Topics Covered: Smart Pointers, Move Semantics, Lambda Expressions, auto & decltype, constexpr, Enum Class, Structured Bindings, Virtual Functions / vtable, Abstract Classes, RAII, STL Algorithms, Exception Handling, std::thread, Mutex & Locks, Condition Variable, async & Futures, Thread-Safe Data Structures, std::function & std::bind

Problem Statement

Build a thread-safe task scheduler with the following capabilities:

  • Priority queue: Tasks have priority levels (CRITICAL, HIGH, NORMAL, LOW). Higher-priority tasks are executed first.
  • Thread pool: A fixed-size pool of worker threads that process tasks concurrently.
  • Futures: Scheduling a task returns a std::future so the caller can retrieve the result or detect exceptions.
  • Timers: Support delayed/timed tasks that execute at a specified future time.
  • Cancellation: Tasks can be cancelled before they begin execution.
  • Result aggregation: A when_all-style mechanism to wait for multiple tasks and collect their results.

This exercise combines many intermediate C++ features: smart pointers, move semantics, lambda expressions, enum class, RAII, STL algorithms, exception handling, multithreading primitives, and std::function.

Design Considerations:

  • Use enum class for priority levels to avoid implicit conversions and namespace pollution.
  • Use std::function for type-erased task payloads so callers can use lambdas, function pointers, or bind expressions.
  • Use std::packaged_task / std::promise to bridge tasks with std::future for result retrieval.
  • The priority queue comparator must order by priority first, then by submission time (FIFO within the same priority).
  • The scheduler should own the thread pool (RAII) and shut down gracefully in its destructor.
  • Timed tasks require a separate timer thread that uses condition_variable::wait_until and promotes tasks to the ready queue when their deadline arrives.

// ---- Type aliases and enum ----

using TaskId = std::uint64_t;
using Clock = std::chrono::steady_clock;
using TimePoint = Clock::time_point;

enum class Priority {
    CRITICAL = 0,   // Highest
    HIGH     = 1,
    NORMAL   = 2,
    LOW      = 3    // Lowest
};

// Enumerate priority names for logging
inline const char* priorityName(Priority p) {
    switch (p) {
        case Priority::CRITICAL: return "CRITICAL";
        case Priority::HIGH:     return "HIGH";
        case Priority::NORMAL:   return "NORMAL";
        case Priority::LOW:      return "LOW";
    }
    return "UNKNOWN";
}

// ---- TaskResult: wraps a future and metadata ----

template<typename T>
class TaskResult {
public:
    TaskResult() = default;
    explicit TaskResult(std::future<T>&& fut, TaskId id, Priority prio)
        : future_(std::move(fut)), id_(id), priority_(prio) {}

    // Non-copyable, movable
    TaskResult(const TaskResult&) = delete;
    TaskResult& operator=(const TaskResult&) = delete;
    TaskResult(TaskResult&&) = default;
    TaskResult& operator=(TaskResult&&) = default;

    T get() { return future_.get(); }

    std::future_status wait_for(const std::chrono::milliseconds& dur) const {
        return future_.wait_for(dur);
    }

    bool valid() const { return future_.valid(); }
    TaskId id() const { return id_; }
    Priority priority() const { return priority_; }

private:
    std::future<T> future_;
    TaskId id_{0};
    Priority priority_{Priority::NORMAL};
};

// ---- Internal task wrapper ----

struct Task {
    std::function<void()> execute;   // Type-erased callable
    Priority priority;
    TaskId id;
    TimePoint submitTime;              // For FIFO ordering within same priority
    TimePoint readyTime;               // For delayed tasks (defaults to now)

    // Comparator for priority queue: lower Priority value = higher priority
    // Within same priority, earlier submitTime goes first
    bool operator>(const Task& other) const {
        if (static_cast<int>(priority) != static_cast<int>(other.priority))
            return static_cast<int>(priority) > static_cast<int>(other.priority);
        return submitTime > other.submitTime;
    }
};

int main() {
    // Demonstrate type usage
    std::promise<int> prom;
    TaskResult<int> result(prom.get_future(), 42, Priority::HIGH);
    std::cout << "TaskResult valid: " << result.valid() << "\n";
    std::cout << "Task ID: " << result.id() << "\n";
    std::cout << "Priority: " << priorityName(result.priority()) << "\n";
    return 0;
}

Explanation:

  • enum class Priority: Scoped enum prevents implicit conversion to int. Lower numeric values represent higher urgency.
  • TaskId: Strong typedef via using for clarity. Every submitted task gets a unique ID.
  • TaskResult<T>: Wraps a std::future<T> with metadata (ID, priority). Move-only because std::future is move-only. The caller holds this to retrieve the result later.
  • Task struct: Internal representation. std::function<void()> type-erases the callable so we can store any callable. The operator> makes it compatible with std::priority_queue (which is a max-heap by default).
  • readyTime: For delayed tasks. Defaults to submission time (immediate execution). Will be used by the timer thread in Step 5.

Step 2: Implement the Priority Queue

#include <iostream>
#include <queue>
#include <mutex>
#include <condition_variable>
#include <chrono>
#include <atomic>
#include <vector>
#include <string>
#include <functional>
#include <optional>

using TaskId = std::uint64_t;
using Clock = std::chrono::steady_clock;
using TimePoint = Clock::time_point;

enum class Priority { CRITICAL = 0, HIGH = 1, NORMAL = 2, LOW = 3 };

struct Task {
    std::function<void()> execute;
    Priority priority;
    TaskId id;
    TimePoint submitTime;
    TimePoint readyTime;

    bool operator>(const Task& other) const {
        if (static_cast<int>(priority) != static_cast<int>(other.priority))
            return static_cast<int>(priority) > static_cast<int>(other.priority);
        return submitTime > other.submitTime;
    }
};

// ---- Thread-safe priority queue ----

class TaskPriorityQueue {
public:
    TaskPriorityQueue() = default;

    // Non-copyable
    TaskPriorityQueue(const TaskPriorityQueue&) = delete;
    TaskPriorityQueue& operator=(const TaskPriorityQueue&) = delete;

    // Push a task into the queue and notify one waiter
    void push(Task task) {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (stopped_) return;
            queue_.push(std::move(task));
        }
        cond_.notify_one();
    }

    // Pop the highest-priority task (blocks until available or stopped)
    std::optional<Task> pop() {
        std::unique_lock<std::mutex> lock(mutex_);
        cond_.wait(lock, [this] {
            return !queue_.empty() || stopped_;
        });
        if (queue_.empty()) return std::nullopt;

        Task task = std::move(const_cast<Task&>(queue_.top()));
        queue_.pop();
        return task;
    }

    // Pop with timeout
    std::optional<Task> pop_for(const std::chrono::milliseconds& timeout) {
        std::unique_lock<std::mutex> lock(mutex_);
        if (!cond_.wait_for(lock, timeout, [this] {
            return !queue_.empty() || stopped_;
        })) {
            return std::nullopt;  // Timeout
        }
        if (queue_.empty()) return std::nullopt;

        Task task = std::move(const_cast<Task&>(queue_.top()));
        queue_.pop();
        return task;
    }

    // Non-blocking pop
    std::optional<Task> try_pop() {
        std::lock_guard<std::mutex> lock(mutex_);
        if (queue_.empty()) return std::nullopt;

        Task task = std::move(const_cast<Task&>(queue_.top()));
        queue_.pop();
        return task;
    }

    std::size_t size() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return queue_.size();
    }

    bool empty() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return queue_.empty();
    }

    // Stop the queue and wake all waiters
    void stop() {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            stopped_ = true;
        }
        cond_.notify_all();
    }

private:
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    // Max-heap with std::greater comparator so CRITICAL (0) is at top
    std::priority_queue<Task, std::vector<Task>, std::greater<Task>> queue_;
    bool stopped_ = false;
};

int main() {
    TaskPriorityQueue pq;
    auto now = Clock::now();

    // Push tasks with different priorities
    pq.push({[](){ std::cout << "LOW task\n"; }, Priority::LOW, 3, now, now});
    pq.push({[](){ std::cout << "CRITICAL task\n"; }, Priority::CRITICAL, 1, now, now});
    pq.push({[](){ std::cout << "NORMAL task\n"; }, Priority::NORMAL, 4, now, now});
    pq.push({[](){ std::cout << "HIGH task\n"; }, Priority::HIGH, 2, now, now});

    // Pop all -- they should come out in priority order
    while (auto task = pq.try_pop()) {
        std::cout << "Processing task " << task->id << "\n";
        task->execute();
    }

    pq.stop();
    std::cout << "Queue stopped. Size: " << pq.size() << "\n";
    return 0;
}

Explanation:

  • std::priority_queue with std::greater: By default, std::priority_queue is a max-heap. Using std::greater as the comparator inverts the ordering so that the smallest priority value (CRITICAL = 0) is at the top.
  • Thread safety: Every public method acquires mutex_ before accessing the queue. The condition_variable allows workers to block efficiently when the queue is empty.
  • Move semantics: Tasks are moved out of the queue via std::move. The internal storage uses const_cast to move from top() since priority_queue::top() returns a const&.
  • std::optional: Used for pop return types to cleanly signal "no result" without exceptions or sentinel values.
  • Graceful stop: The stop() method sets the flag and wakes all waiting threads, allowing them to exit cleanly.

Step 3: Build the Thread Pool

#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>
#include <vector>
#include <functional>
#include <atomic>
#include <chrono>
#include <sstream>

// Simple thread pool with graceful shutdown

class ThreadPool {
public:
    explicit ThreadPool(std::size_t numThreads)
        : stopped_(false) {
        for (std::size_t i = 0; i < numThreads; ++i) {
            workers_.emplace_back([this, i]() { workerLoop(i); });
        }
        std::cout << "ThreadPool: started " << numThreads << " workers\n";
    }

    ~ThreadPool() {
        shutdown();
    }

    // Non-copyable, non-movable
    ThreadPool(const ThreadPool&) = delete;
    ThreadPool& operator=(const ThreadPool&) = delete;

    // Submit a task (void return for now -- we add futures in Step 4)
    void submit(std::function<void()> task) {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (stopped_) return;
            tasks_.push(std::move(task));
        }
        cond_.notify_one();
    }

    // Graceful shutdown: finish queued work, then join all threads
    void shutdown() {
        if (stopped_.exchange(true)) return;  // Already stopped
        cond_.notify_all();

        for (auto& w : workers_) {
            if (w.joinable()) w.join();
        }
        std::cout << "ThreadPool: all " << workers_.size()
                  << " workers stopped\n";
    }

    std::size_t pendingTasks() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return tasks_.size();
    }

private:
    void workerLoop(std::size_t id) {
        while (true) {
            std::function<void()> task;
            {
                std::unique_lock<std::mutex> lock(mutex_);
                cond_.wait(lock, [this] {
                    return !tasks_.empty() || stopped_;
                });

                if (stopped_ && tasks_.empty()) return;

                task = std::move(tasks_.front());
                tasks_.pop();
            }
            // Execute outside the lock
            task();
        }
    }

    std::vector<std::thread> workers_;
    std::queue<std::function<void()>> tasks_;
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    std::atomic<bool> stopped_;
};

// ---- Demo ----

int main() {
    {
        ThreadPool pool(4);

        // Submit 10 tasks
        for (int i = 0; i < 10; ++i) {
            pool.submit([i]() {
                std::ostringstream oss;
                oss << "Task " << i << " running on thread "
                    << std::this_thread::get_id() << "\n";
                std::cout << oss.str();
                std::this_thread::sleep_for(std::chrono::milliseconds(50));
            });
        }

        std::cout << "Submitted 10 tasks. Pending: "
                  << pool.pendingTasks() << "\n";

        // RAII: pool destructor calls shutdown()
        std::this_thread::sleep_for(std::chrono::milliseconds(500));
    }  // pool destroyed here
    std::cout << "Pool destroyed. All tasks completed.\n";
    return 0;
}

Explanation:

  • RAII: The destructor calls shutdown(), ensuring threads are joined even if an exception is thrown. This is the Resource Acquisition Is Initialization pattern in action.
  • std::atomic<bool> stopped_: Thread-safe flag. exchange(true) atomically sets it and returns the old value, providing a test-and-set that prevents double-shutdown.
  • condition_variable::wait with predicate: Workers sleep until there is work or the pool is stopped. The predicate handles spurious wakeups correctly.
  • Task execution outside the lock: The lock is released before calling task(), maximizing concurrency. This is critical for performance -- holding the lock during execution would serialize all tasks.
  • std::function<void()>: Type-erases the callable so the pool can store lambdas with different captures, member functions, or any callable object.

Step 4: Add Task Scheduling with Futures

#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>
#include <vector>
#include <functional>
#include <future>
#include <atomic>
#include <chrono>
#include <memory>
#include <cstdint>

using TaskId = std::uint64_t;
using Clock = std::chrono::steady_clock;
using TimePoint = Clock::time_point;

enum class Priority { CRITICAL = 0, HIGH = 1, NORMAL = 2, LOW = 3 };

struct Task {
    std::function<void()> execute;
    Priority priority;
    TaskId id;
    TimePoint submitTime;

    bool operator>(const Task& other) const {
        if (static_cast<int>(priority) != static_cast<int>(other.priority))
            return static_cast<int>(priority) > static_cast<int>(other.priority);
        return submitTime > other.submitTime;
    }
};

// ---- Thread pool with priority scheduling and futures ----

class TaskScheduler {
public:
    explicit TaskScheduler(std::size_t numThreads = 4)
        : stopped_(false), nextId_(1) {
        for (std::size_t i = 0; i < numThreads; ++i) {
            workers_.emplace_back([this, i]() { workerLoop(i); });
        }
        std::cout << "TaskScheduler: " << numThreads << " workers started\n";
    }

    ~TaskScheduler() { shutdown(); }

    TaskScheduler(const TaskScheduler&) = delete;
    TaskScheduler& operator=(const TaskScheduler&) = delete;

    // Schedule a task with a return value, returns a future
    template<typename F>
    auto schedule(F&& f, Priority prio = Priority::NORMAL)
        -> std::future<typename std::invoke_result_t<F>>
    {
        using ReturnType = typename std::invoke_result_t<F>;

        // Wrap in a shared_ptr so the packaged_task outlives the lambda capture
        auto packaged = std::make_shared<std::packaged_task<ReturnType()>>(
            std::forward<F>(f)
        );
        auto future = packaged->get_future();

        TaskId id = nextId_++;
        TimePoint now = Clock::now();

        Task task{
            [packaged]() { (*packaged)(); },  // Execute the packaged_task
            prio,
            id,
            now
        };

        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (!stopped_) {
                queue_.push(std::move(task));
            }
        }
        cond_.notify_one();

        return future;
    }

    // Schedule a void task (simplified overload)
    template<typename F>
    std::future<void> scheduleVoid(F&& f, Priority prio = Priority::NORMAL) {
        return schedule(std::forward<F>(f), prio);
    }

    void shutdown() {
        if (stopped_.exchange(true)) return;
        cond_.notify_all();
        for (auto& w : workers_) {
            if (w.joinable()) w.join();
        }
        std::cout << "TaskScheduler: shutdown complete\n";
    }

private:
    void workerLoop(std::size_t workerId) {
        while (true) {
            Task task;
            {
                std::unique_lock<std::mutex> lock(mutex_);
                cond_.wait(lock, [this] {
                    return !queue_.empty() || stopped_;
                });

                if (stopped_ && queue_.empty()) return;

                task = std::move(const_cast<Task&>(queue_.top()));
                queue_.pop();
            }
            // Execute outside the lock -- exceptions propagate through future
            task.execute();
        }
    }

    std::vector<std::thread> workers_;
    std::priority_queue<Task, std::vector<Task>, std::greater<Task>> queue_;
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    std::atomic<bool> stopped_;
    std::atomic<TaskId> nextId_;
};

// ---- Demo ----

int main() {
    TaskScheduler scheduler(4);

    // Schedule tasks with different priorities
    auto f1 = scheduler.schedule([]() { return 42; }, Priority::CRITICAL);
    auto f2 = scheduler.schedule([val = 7]() { return val * val; }, Priority::HIGH);
    auto f3 = scheduler([]() {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        return std::string("hello");
    }, Priority::NORMAL);
    auto f4 = scheduler([]() { return 3.14; }, Priority::LOW);

    // Retrieve results (blocks until ready)
    std::cout << "f1 (CRITICAL): " << f1.get() << "\n";
    std::cout << "f2 (HIGH): " << f2.get() << "\n";
    std::cout << "f3 (NORMAL): " << f3.get() << "\n";
    std::cout << "f4 (LOW): " << f4.get() << "\n";

    scheduler.shutdown();
    return 0;
}

Explanation:

  • std::packaged_task: Wraps any callable and creates a std::future via get_future(). When the packaged_task is invoked, the result (or exception) is stored in the shared state, making it available through the future.
  • std::shared_ptr capture: The packaged_task is captured by shared_ptr in the lambda. This ensures the task object lives until the lambda executes, even if the original future is moved or destroyed.
  • Exception propagation: If the callable throws, std::packaged_task catches the exception and stores it in the shared state. Calling future.get() will re-throw it.
  • Priority ordering: CRITICAL tasks are dequeued first, ensuring urgent work is not blocked behind lower-priority tasks.
  • std::invoke_result_t: C++17 trait that deduces the return type of a callable. This makes the schedule() function work with any callable, returning the correct std::future<T> type.

Step 5: Implement Delayed/Timed Tasks

#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>
#include <vector>
#include <functional>
#include <future>
#include <atomic>
#include <chrono>
#include <memory>
#include <cstdint>

using TaskId = std::uint64_t;
using Clock = std::chrono::steady_clock;
using TimePoint = Clock::time_point;

enum class Priority { CRITICAL = 0, HIGH = 1, NORMAL = 2, LOW = 3 };

struct Task {
    std::function<void()> execute;
    Priority priority;
    TaskId id;
    TimePoint submitTime;
    TimePoint readyTime;       // When the task becomes eligible for execution

    bool operator>(const Task& other) const {
        if (static_cast<int>(priority) != static_cast<int>(other.priority))
            return static_cast<int>(priority) > static_cast<int>(other.priority);
        return submitTime > other.submitTime;
    }
};

// Comparator for the timer queue: orders by readyTime (earliest first)
struct TimerCompare {
    bool operator()(const Task& a, const Task& b) const {
        return a.readyTime > b.readyTime;
    }
};

// ---- Scheduler with timer thread ----

class TimedTaskScheduler {
public:
    explicit TimedTaskScheduler(std::size_t numWorkers = 4)
        : stopped_(false), nextId_(1) {
        // Start worker threads
        for (std::size_t i = 0; i < numWorkers; ++i) {
            workers_.emplace_back([this, i]() { workerLoop(i); });
        }
        // Start timer thread
        timerThread_ = std::thread([this]() { timerLoop(); });
        std::cout << "TimedScheduler: " << numWorkers
                  << " workers + 1 timer started\n";
    }

    ~TimedTaskScheduler() { shutdown(); }

    TimedTaskScheduler(const TimedTaskScheduler&) = delete;
    TimedTaskScheduler& operator=(const TimedTaskScheduler&) = delete;

    // Schedule a task for immediate execution
    template<typename F>
    auto schedule(F&& f, Priority prio = Priority::NORMAL)
        -> std::future<typename std::invoke_result_t<F>>
    {
        return scheduleAt(std::chrono::milliseconds(0),
                          std::forward<F>(f), prio);
    }

    // Schedule a task after a delay
    template<typename F>
    auto scheduleAfter(const std::chrono::milliseconds& delay,
                        F&& f, Priority prio = Priority::NORMAL)
        -> std::future<typename std::invoke_result_t<F>>
    {
        return scheduleAt(delay, std::forward<F>(f), prio);
    }

    // Schedule a task at a specific time point
    template<typename F>
    auto scheduleAt(TimePoint when, F&& f, Priority prio = Priority::NORMAL)
        -> std::future<typename std::invoke_result_t<F>>
    {
        using ReturnType = typename std::invoke_result_t<F>;

        auto packaged = std::make_shared<std::packaged_task<ReturnType()>>(
            std::forward<F>(f)
        );
        auto future = packaged->get_future();

        TaskId id = nextId_++;
        TimePoint now = Clock::now();

        Task task{
            [packaged]() { (*packaged)(); },
            prio,
            id,
            now,
            when
        };

        if (when <= now) {
            // Already ready: push directly to ready queue
            {
                std::lock_guard<std::mutex> lock(readyMutex_);
                readyQueue_.push(std::move(task));
            }
            readyCond_.notify_one();
        } else {
            // Future execution: push to timer queue
            {
                std::lock_guard<std::mutex> lock(timerMutex_);
                timerQueue_.push(std::move(task));
            }
            timerCond_.notify_one();
        }

        return future;
    }

    void shutdown() {
        if (stopped_.exchange(true)) return;
        readyCond_.notify_all();
        timerCond_.notify_all();
        for (auto& w : workers_) {
            if (w.joinable()) w.join();
        }
        if (timerThread_.joinable()) timerThread_.join();
        std::cout << "TimedScheduler: shutdown complete\n";
    }

private:
    // Timer loop: moves tasks to ready queue when their time arrives
    void timerLoop() {
        while (true) {
            std::unique_lock<std::mutex> lock(timerMutex_);
            timerCond_.wait(lock, [this] {
                return !timerQueue_.empty() || stopped_;
            });

            if (stopped_) {
                // Drain remaining timed tasks
                while (!timerQueue_.empty()) {
                    Task task = std::move(
                        const_cast<Task&>(timerQueue_.top()));
                    timerQueue_.pop();
                    // Move to ready queue
                    std::lock_guard<std::mutex> rlock(readyMutex_);
                    readyQueue_.push(std::move(task));
                }
                readyCond_.notify_all();
                return;
            }

            // Check if the earliest task is ready
            TimePoint now = Clock::now();
            while (!timerQueue_.empty() &&
                   timerQueue_.top().readyTime <= now) {
                Task task = std::move(
                    const_cast<Task&>(timerQueue_.top()));
                timerQueue_.pop();
                readyCond_.notify_one();
                lock.unlock();  // Release timer lock before acquiring ready lock
                {
                    std::lock_guard<std::mutex> rlock(readyMutex_);
                    readyQueue_.push(std::move(task));
                }
                lock.lock();  // Re-acquire for next iteration
            }

            // If timer queue not empty, wait until the next task's ready time
            if (!timerQueue_.empty()) {
                timerCond_.wait_until(lock, timerQueue_.top().readyTime);
            }
        }
    }

    // Worker loop: processes tasks from the ready queue
    void workerLoop(std::size_t workerId) {
        while (true) {
            Task task;
            {
                std::unique_lock<std::mutex> lock(readyMutex_);
                readyCond_.wait(lock, [this] {
                    return !readyQueue_.empty() || stopped_;
                });
                if (stopped_ && readyQueue_.empty()) return;
                task = std::move(const_cast<Task&>(readyQueue_.top()));
                readyQueue_.pop();
            }
            task.execute();
        }
    }

    // Ready queue (priority-based)
    std::priority_queue<Task, std::vector<Task>, std::greater<Task>> readyQueue_;
    mutable std::mutex readyMutex_;
    std::condition_variable readyCond_;

    // Timer queue (time-ordered)
    std::priority_queue<Task, std::vector<Task>, TimerCompare> timerQueue_;
    mutable std::mutex timerMutex_;
    std::condition_variable timerCond_;

    std::vector<std::thread> workers_;
    std::thread timerThread_;
    std::atomic<bool> stopped_;
    std::atomic<TaskId> nextId_;
};

// ---- Demo ----

int main() {
    TimedTaskScheduler scheduler(2);
    auto start = Clock::now();

    // Immediate task
    auto f1 = scheduler.schedule([]() {
        return std::string("immediate");
    }, Priority::HIGH);
    std::cout << "Immediate: " << f1.get() << "\n";

    // Delayed task (500ms from now)
    auto f2 = scheduler.scheduleAfter(std::chrono::milliseconds(500), []() {
        return std::string("delayed 500ms");
    }, Priority::NORMAL);

    // Another delayed task (200ms from now) -- should complete first
    auto f3 = scheduler.scheduleAfter(std::chrono::milliseconds(200), []() {
        return std::string("delayed 200ms");
    }, Priority::CRITICAL);

    std::cout << "200ms task: " << f3.get() << "\n";
    std::cout << "500ms task: " << f2.get() << "\n";

    auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>(
        Clock::now() - start).count();
    std::cout << "Total elapsed: " << elapsed << "ms\n";

    scheduler.shutdown();
    return 0;
}

Explanation:

  • Dual-queue design: A timer queue (ordered by readyTime) and a ready queue (ordered by priority). The timer thread promotes tasks from the timer queue to the ready queue when their deadline arrives.
  • condition_variable::wait_until: The timer thread sleeps efficiently until the earliest timed task's readyTime. This avoids busy-waiting and uses zero CPU while idle.
  • Lock ordering: The timer loop carefully releases the timer lock before acquiring the ready lock to avoid deadlock. This is a subtle but critical detail in multi-lock designs.
  • std::chrono: Uses steady_clock (monotonic) for timing, which is immune to system clock adjustments. system_clock could jump backward/forward due to NTP or DST changes.
  • Graceful shutdown: On shutdown, the timer thread drains all remaining timed tasks into the ready queue so they get a chance to execute.

#include <iostream>

using TaskId = std::uint64_t;
using Clock = std::chrono::steady_clock;
using TimePoint = Clock::time_point;

enum class Priority { CRITICAL = 0, HIGH = 1, NORMAL = 2, LOW = 3 };

struct Task {
    std::function<void()> execute;
    Priority priority;
    TaskId id;
    TimePoint submitTime;

    bool operator>(const Task& other) const {
        if (static_cast<int>(priority) != static_cast<int>(other.priority))
            return static_cast<int>(priority) > static_cast<int>(other.priority);
        return submitTime > other.submitTime;
    }
};

// ---- Scheduler with exception propagation and cancellation ----

class CancelableScheduler {
public:
    explicit CancelableScheduler(std::size_t numThreads = 4)
        : stopped_(false), nextId_(1) {
        for (std::size_t i = 0; i < numThreads; ++i) {
            workers_.emplace_back([this, i]() { workerLoop(i); });
        }
    }

    ~CancelableScheduler() { shutdown(); }

    CancelableScheduler(const CancelableScheduler&) = delete;
    CancelableScheduler& operator=(const CancelableScheduler&) = delete;

    // Schedule and return a future + task ID for cancellation
    template<typename F>
    auto schedule(F&& f, Priority prio = Priority::NORMAL)
        -> std::pair<std::future<typename std::invoke_result_t<F>>, TaskId>
    {
        using ReturnType = typename std::invoke_result_t<F>;

        auto packaged = std::make_shared<std::packaged_task<ReturnType()>>(
            std::forward<F>(f)
        );
        auto future = packaged->get_future();
        TaskId id = nextId_++;

        Task task{
            [this, packaged, id]() {
                // Check if cancelled before executing
                if (isCancelled(id)) {
                    throw std::runtime_error("Task cancelled");
                }
                (*packaged)();
            },
            prio,
            id,
            Clock::now()
        };

        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (stopped_) {
                throw std::runtime_error("Scheduler is stopped");
            }
            activeIds_.insert(id);
            queue_.push(std::move(task));
        }
        cond_.notify_one();

        return {std::move(future), id};
    }

    // Cancel a task by ID (only works if not yet executing)
    bool cancel(TaskId id) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = activeIds_.find(id);
        if (it == activeIds_.end()) return false;  // Already completed/running
        activeIds_.erase(it);

        // Rebuild the queue without the cancelled task
        std::priority_queue<Task, std::vector<Task>, std::greater<Task>> newQueue;
        bool found = false;
        while (!queue_.empty()) {
            Task t = std::move(const_cast<Task&>(queue_.top()));
            queue_.pop();
            if (t.id == id && !found) {
                found = true;  // Skip this task
            } else {
                newQueue.push(std::move(t));
            }
        }
        queue_ = std::move(newQueue);
        return found;
    }

    bool isCancelled(TaskId id) const {
        std::lock_guard<std::mutex> lock(mutex_);
        return activeIds_.find(id) == activeIds_.end();
    }

    void shutdown() {
        if (stopped_.exchange(true)) return;
        cond_.notify_all();
        for (auto& w : workers_) {
            if (w.joinable()) w.join();
        }
    }

private:
    void workerLoop(std::size_t workerId) {
        while (true) {
            Task task;
            {
                std::unique_lock<std::mutex> lock(mutex_);
                cond_.wait(lock, [this] {
                    return !queue_.empty() || stopped_;
                });
                if (stopped_ && queue_.empty()) return;
                task = std::move(const_cast<Task&>(queue_.top()));
                queue_.pop();
            }
            // Execute -- exceptions propagate through packaged_task
            try {
                task.execute();
            } catch (...) {
                // packaged_task already captured the exception
                // Clean up active ID
                std::lock_guard<std::mutex> lock(mutex_);
                activeIds_.erase(task.id);
                throw;  // Re-throw so packaged_task can store it
            }
            // Mark as completed
            {
                std::lock_guard<std::mutex> lock(mutex_);
                activeIds_.erase(task.id);
            }
        }
    }

    std::vector<std::thread> workers_;
    std::priority_queue<Task, std::vector<Task>, std::greater<Task>> queue_;
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    std::atomic<bool> stopped_;
    std::atomic<TaskId> nextId_;
    std::unordered_set<TaskId> activeIds_;
};

// ---- Demo ----

int main() {
    CancelableScheduler scheduler(2);

    // Schedule a normal task
    auto [f1, id1] = scheduler.schedule([]() {
        std::cout << "Task 1 executed\n";
        return 100;
    }, Priority::NORMAL);

    // Schedule a task that throws
    auto [f2, id2] = scheduler.schedule([]() -> int {
        std::cout << "Task 2 throwing...\n";
        throw std::runtime_error("Intentional error in task 2");
    }, Priority::HIGH);

    // Schedule a task and cancel it
    auto [f3, id3] = scheduler.schedule([]() {
        std::cout << "Task 3 executed (should not appear)\n";
        return 300;
    }, Priority::LOW);

    bool cancelled = scheduler.cancel(id3);
    std::cout << "Task 3 cancelled: " << std::boolalpha << cancelled << "\n";

    // Get results
    std::cout << "Task 1 result: " << f1.get() << "\n";

    // Task 2 threw -- get() re-throws the exception
    try {
        f2.get();
    } catch (const std::exception& e) {
        std::cout << "Task 2 exception caught: " << e.what() << "\n";
    }

    // Task 3 was cancelled -- get() re-throws the cancellation exception
    try {
        f3.get();
    } catch (const std::exception& e) {
        std::cout << "Task 3 exception caught: " << e.what() << "\n";
    }

    scheduler.shutdown();
    return 0;
}

Explanation:

  • std::packaged_task exception capture: When a task throws, std::packaged_task automatically stores the exception in the shared state. Calling future.get() re-throws it in the calling thread.
  • Cancellation via cancel(): Removes the task from the queue by rebuilding it without the target task. The cancelled task's lambda throws "Task cancelled" if it somehow executes (race condition guard).
  • activeIds_ set: Tracks which tasks are still pending or running. Used for both cancellation checks and cleanup. The unordered_set provides O(1) lookup.
  • std::pair return: schedule() returns both the future and the task ID, giving the caller control over cancellation.
  • Structured bindings: auto [f1, id1] = ... (C++17) decomposes the returned pair for clean code.

#include <iostream>

// ---- when_all result aggregator ----

// A result that holds either a value or an exception
template<typename T>
class Try {
public:
    // Success case
    Try(T value) : data_(std::move(value)) {}

    // Failure case
    Try(std::exception_ptr ex) : data_(ex) {}

    bool hasValue() const {
        return std::holds_alternative<T>(data_);
    }

    T& get() {
        if (auto* val = std::get_if<T>(&data_)) return *val;
        std::rethrow_exception(std::get<std::exception_ptr>(data_));
    }

    const T& get() const {
        if (auto* val = std::get_if<T>(&data_)) return *val;
        std::rethrow_exception(std::get<std::exception_ptr>(data_));
    }

private:
    std::variant<T, std::exception_ptr> data_;
};

// Aggregated result from multiple tasks
template<typename T>
class WhenAllResult {
public:
    WhenAllResult() = default;

    // Wait for all futures and collect results
    explicit WhenAllResult(std::vector<std::future<T>>&& futures)
        : results_(futures.size()) {
        // Launch a thread to wait on all futures concurrently
        std::vector<std::thread> waiters;
        waiters.reserve(futures.size());

        for (std::size_t i = 0; i < futures.size(); ++i) {
            waiters.emplace_back([this, i](std::future<T> fut) {
                try {
                    T value = fut.get();
                    results_[i] = Try<T>(std::move(value));
                } catch (...) {
                    results_[i] = Try<T>(std::current_exception());
                }
                // Signal completion
                {
                    std::lock_guard<std::mutex> lock(mutex_);
                    completedCount_++;
                }
                cond_.notify_one();
            }, std::move(futures[i]));
        }

        // Wait for all waiter threads
        for (auto& t : waiters) {
            if (t.joinable()) t.join();
        }
    }

    std::size_t size() const { return results_.size(); }

    // Get individual result (re-throws if that task failed)
    const T& get(std::size_t index) const {
        return results_.at(index).get();
    }

    // Check if a specific task succeeded
    bool hasValue(std::size_t index) const {
        return results_.at(index).hasValue();
    }

    // Get the Try object for detailed inspection
    const Try<T>& tryGet(std::size_t index) const {
        return results_.at(index);
    }

    // Count of successful tasks
    std::size_t successCount() const {
        return std::count_if(results_.begin(), results_.end(),
            [](const Try<T>& t) { return t.hasValue(); });
    }

    // Count of failed tasks
    std::size_t failureCount() const {
        return results_.size() - successCount();
    }

    // Collect all successful values into a vector
    std::vector<T> values() const {
        std::vector<T> vals;
        for (const auto& r : results_) {
            if (r.hasValue()) vals.push_back(r.get());
        }
        return vals;
    }

private:
    std::vector<Try<T>> results_;
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    std::size_t completedCount_{0};
};

// ---- Convenience: when_all that waits for all futures ----

template<typename T>
WhenAllResult<T> when_all(std::vector<std::future<T>> futures) {
    return WhenAllResult<T>(std::move(futures));
}

// ---- Demo ----

int main() {
    // Create 6 tasks: some succeed, some throw
    std::vector<std::future<int>> futures;
    auto launch = [&futures](int delayMs, int value, bool shouldThrow) {
        futures.push_back(std::async(std::launch::async, [=]() -> int {
            std::this_thread::sleep_for(std::chrono::milliseconds(delayMs));
            if (shouldThrow) {
                throw std::runtime_error("Task " + std::to_string(value) + " failed");
            }
            return value;
        }));
    };

    launch(100, 1, false);   // succeeds
    launch(50,  2, true);    // throws
    launch(200, 3, false);   // succeeds
    launch(150, 4, false);   // succeeds
    launch(80,  5, true);    // throws
    launch(300, 6, false);   // succeeds

    // Aggregate all results
    WhenAllResult<int> all = when_all(std::move(futures));

    std::cout << "Total tasks: " << all.size() << "\n";
    std::cout << "Succeeded:   " << all.successCount() << "\n";
    std::cout << "Failed:      " << all.failureCount() << "\n";

    // Print each result
    for (std::size_t i = 0; i < all.size(); ++i) {
        if (all.hasValue(i)) {
            std::cout << "Task " << i << ": " << all.get(i) << "\n";
        } else {
            try {
                all.get(i);
            } catch (const std::exception& e) {
                std::cout << "Task " << i << ": EXCEPTION - "
                          << e.what() << "\n";
            }
        }
    }

    // Collect all successful values
    auto vals = all.values();
    std::cout << "\nAll successful values: ";
    for (auto v : vals) std::cout << v << " ";
    std::cout << "\n";

    return 0;
}

Explanation:

  • Try<T>: A result wrapper that stores either a value or an exception. Uses std::variant (C++17) to hold either case. Similar to Rust's Result type or Haskell's Either.
  • WhenAllResult<T>: Collects results from multiple futures. Each future is waited on in a separate thread, so all tasks run concurrently. Results are indexed by their original position.
  • std::current_exception(): Captures the current exception as an std::exception_ptr, allowing it to be stored and re-thrown later in a different thread.
  • std::async(std::launch::async): Forces asynchronous launch (the default may defer execution). Each task runs in its own thread for true parallelism in this demo.
  • std::count_if with lambda: STL algorithm to count successful results. Demonstrates functional-style programming with lambdas.

Follow-up Questions

Variations to Try

  1. Work stealing: Extend the thread pool with per-thread local deques. When a worker's local queue is empty, it steals from other workers' queues (steal from front, pop from back). This reduces contention on the global queue.
  2. Task dependencies (DAG scheduler): Support dependent tasks where task B can only start after task A completes. Implement a directed acyclic graph with in-degree tracking. When a task completes, decrement the in-degree of its dependents and schedule any that become ready.
  3. Dynamic priority adjustment: Allow task priority to change after submission (e.g., elevate a task from NORMAL to CRITICAL based on external events). Use a std::unordered_map<TaskId, Priority> for O(1) lookup and rebuild the queue when priorities change.
  4. Rate limiting: Add per-priority rate limits to prevent low-priority tasks from starving under heavy CRITICAL load. Implement token bucket or leaky bucket algorithms.

Key Takeaways:

  • RAII is essential for thread safety: The scheduler destructor joins all threads, preventing dangling threads. Smart pointers manage task lifetime automatically.
  • std::packaged_task bridges callables and futures: It type-erases the callable and provides a future for result retrieval. Exceptions in the task are automatically propagated.
  • condition_variable enables efficient waiting: Workers sleep until work is available, using zero CPU while idle. The predicate overload handles spurious wakeups.
  • enum class provides type safety: Priority levels cannot accidentally convert to int, preventing subtle bugs.
  • Move semantics are critical: Tasks, futures, and packaged_tasks are all move-only types. Understanding std::move and perfect forwarding (std::forward) is essential for building efficient concurrent systems.
  • std::variant enables safe sum types: The Try<T> wrapper stores either a value or an exception without dynamic allocation, combining the safety of tagged unions with modern C++ ergonomics.
  • Lock ordering prevents deadlock: When using multiple mutexes (timer queue + ready queue), always acquire them in the same order or release one before acquiring another.
  • std::atomic for simple shared state: Flags like stopped_ and counters like nextId_ can use atomics instead of mutexes for lock-free access.

Complexity Analysis:

Common Pitfalls:

  • Priority inversion: If a low-priority task holds a lock needed by a high-priority task, priorities become meaningless. Consider using std::shared_mutex for read-heavy paths to reduce contention.
  • Starvation: Under continuous CRITICAL load, LOW-priority tasks may never execute. Use aging (gradually increase priority over time) to prevent this.
  • Future lifetime: If a std::future is destroyed before calling get(), the result is discarded. If the task threw, the exception is silently swallowed -- this can hide bugs.
  • Timer queue drift: Under heavy load, the timer thread may not wake exactly at the deadline. For hard real-time requirements, consider dedicated timer hardware or OS-level timers.
  • Shared_ptr overhead: Capturing std::shared_ptr<packaged_task> in the task lambda incurs reference counting overhead. For extremely high-frequency tasks, consider custom allocators or intrusive ref counting.
