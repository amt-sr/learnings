Operator Overloading

Operator overloading allows user-defined types to work with C++ operators (+, ==, <<, [], etc.) in a natural way. When done correctly, overloaded operators make custom types behave like built-in types, enabling intuitive APIs.

Rules and Guidelines

Rules:

  • You cannot invent new operators (no **, %%, etc.)
  • You cannot change the arity (number of operands) of an operator
  • ?:, ::, ., .*, and sizeof cannot be overloaded
  • The precedence and associativity of operators cannot be changed
  • At least one operand must be a user-defined type

Arithmetic Operators

#include <iostream>

class Money {
    long dollars_;
    int cents_;
public:
    Money(long d = 0, int c = 0) : dollars_(d), cents_(c) {
        normalize();
    }

    // Binary + operator (member function)
    Money operator+(const Money& other) const {
        return Money(dollars_ + other.dollars_, cents_ + other.cents_);
    }

    // Compound assignment +=
    Money& operator+=(const Money& other) {
        dollars_ += other.dollars_;
        cents_ += other.cents_;
        normalize();
        return *this;
    }

    // Unary - operator
    Money operator-() const {
        return Money(-dollars_, -cents_);
    }

    // Binary - operator
    Money operator-(const Money& other) const {
        return *this + (-other);
    }

    // Comparison operators
    bool operator==(const Money& other) const {
        return dollars_ == other.dollars_ && cents_ == other.cents_;
    }

    bool operator!=(const Money& other) const {
        return !(*this == other);
    }

    bool operator<(const Money& other) const {
        return (dollars_ < other.dollars_) ||
               (dollars_ == other.dollars_ && cents_ < other.cents_);
    }

    friend std::ostream& operator<<(std::ostream& os, const Money& m);

private:
    void normalize() {
        if (cents_ >= 100) { dollars_ += cents_ / 100; cents_ %= 100; }
        else if (cents_ < 0) { dollars_ -= 1; cents_ += 100; }
    }
};

void arithmetic_demo() {
    Money a(10, 50);   // $10.50
    Money b(3, 75);     // $3.75

    Money sum = a + b;       // $14.25
    Money diff = a - b;      // $6.75

    a += b;                  // a is now $14.25
    std::cout << "a: " << a << "\n";   // $14.25
    std::cout << "sum: " << sum << "\n";  // $14.25
    std::cout << (a == sum) << "\n";     // 1 (true)
}

Stream Operators (<< and >>)

Stream operators must be overloaded as non-member (free) functions because the left operand is std::ostream/std::istream, not your type.

// Must be a non-member function (left operand is ostream)
std::ostream& operator<<(std::ostream& os, const Money& m) {
    os << "$" << m.dollars_ << "."
       << (m.cents_ < 10 ? "0" : "") << m.cents_;
    return os;
}

// Input operator (left operand is istream)
std::istream& operator>>(std::istream& is, Money& m) {
    long d;
    int c;
    char dot;
    is >> d >> dot >> c;
    m = Money(d, c);
    return is;
}

Convention: Return a reference to the stream from operator<< and operator>> so they can be chained: std::cout << a << " + " << b << "\n".

Subscript Operator

#include <vector>
#include <stdexcept>
#include <iostream>

class Matrix {
    std::vector<std::vector<double>> data_;
    size_t rows_, cols_;
public:
    Matrix(size_t rows, size_t cols, double init = 0.0)
        : rows_(rows), cols_(cols), data_(rows, std::vector<double>(cols, init)) {}

    // const access
    const double& operator()(size_t row, size_t col) const {
        return data_.at(row).at(col);
    }

    // mutable access
    double& operator()(size_t row, size_t col) {
        return data_.at(row).at(col);
    }

    // Single subscript — returns row (useful for expression templates, etc.)
    const std::vector<double>& operator[](size_t row) const {
        return data_.at(row);
    }

    std::vector<double>& operator[](size_t row) {
        return data_.at(row);
    }
};

void matrix_demo() {
    Matrix m(3, 3, 0.0);
    m(0, 0) = 1.0;
    m(1, 1) = 2.0;
    m(2, 2) = 3.0;

    std::cout << m(0, 0) << " " << m(1, 1) << " " << m(2, 2) << "\n";  // 1 2 3

    const Matrix cm = m;
    // cm(0, 0) = 5.0;  // ERROR: const version returns const reference
}

Comparison Operators

// C++20 provides spaceship operator <=> for defaulting all comparisons.
// Before C++20, you need to define each operator or use a helper base class.

#include <iostream>

class Version {
    int major_, minor_, patch_;
public:
    Version(int maj, int min, int pat) : major_(maj), minor_(min), patch_(pat) {}

    // Implement all six comparison operators
    bool operator==(const Version& o) const {
        return std::tie(major_, minor_, patch_) == std::tie(o.major_, o.minor_, o.patch_);
    }
    bool operator!=(const Version& o) const { return !(*this == o); }
    bool operator<(const Version& o) const {
        return std::tie(major_, minor_, patch_) < std::tie(o.major_, o.minor_, o.patch_);
    }
    bool operator<=(const Version& o) const { return !(o < *this); }
    bool operator>(const Version& o) const { return o < *this; }
    bool operator>=(const Version& o) const { return !(*this < o); }
};

void comparison_demo() {
    Version v1(1, 0, 0);
    Version v2(2, 0, 0);
    Version v3(1, 0, 0);

    std::cout << std::boolalpha;
    std::cout << (v1 == v3) << "\n";  // true
    std::cout << (v1 < v2) << "\n";   // true
    std::cout << (v2 > v1) << "\n";   // true
}

Member vs Non-Member

OperatorMust be MemberShould be Non-Member
=, [], (), ->, ->*Yes—
+=, -=, *=, etc.Yes (modifies left operand)—
==, !=, <, >—Yes (symmetric operands)
+, -, *, /—Yes (symmetric; allows implicit conversions on left)
<<, >>—Yes (left operand is stream)

Tip: Define binary operators that don't modify the left operand as non-member friends. This allows implicit conversions on both sides. For +, implement += as a member and define + in terms of it as a non-member.

// Canonical pattern: implement compound assignment as member, binary as non-member
class Vec2 {
    double x_, y_;
public:
    Vec2(double x, double y) : x_(x), y_(y) {}

    Vec2& operator+=(const Vec2& rhs) {
        x_ += rhs.x_;
        y_ += rhs.y_;
        return *this;
    }
};

// Non-member operator+ implemented in terms of operator+=
Vec2 operator+(Vec2 lhs, const Vec2& rhs) {  // pass lhs by value
    lhs += rhs;
    return lhs;
}

Operator overloading works closely with friend functions for non-member operators that need access to private members. It also relates to multiple inheritance where ambiguous operators between base classes can cause issues.
