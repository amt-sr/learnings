Chrono & Time

The <chrono> library (C++11) provides a type-safe, precision-independent system for dealing with time. It defines clocks, time points, and durations as distinct types, preventing the common bugs of mixing seconds with milliseconds or using raw integers for time arithmetic.

Clocks

A clock is a source of time points. The standard provides three clocks:

ClockMonotonicUse Case
std::chrono::system_clockNo (adjusts for wall-clock changes)Timestamps, logging, dates
std::chrono::steady_clockYes (never goes backward)Measuring elapsed time
std::chrono::high_resolution_clockImplementation-definedShortest tick period available

Always use steady_clock for measuring elapsed time. system_clock can jump backward if the system clock is adjusted (NTP sync, DST, manual change), leading to negative or wildly incorrect durations.

time_point and duration

#include <chrono>
#include <iostream>

void time_point_basics() {
    using namespace std::chrono;

    // Get the current time point from a clock
    auto now = system_clock::now();
    auto steady_now = steady_clock::now();

    // Duration types
    auto d_sec    = 5s;                   // 5 seconds
    auto d_ms     = 250ms;               // 250 milliseconds
    auto d_us     = 100us;               // 100 microseconds
    auto d_ns     = 50ns;                // 50 nanoseconds
    auto d_min    = 2min;                // 2 minutes
    auto d_hours  = 1h;                  // 1 hour

    // Arithmetic on durations
    auto total = d_sec + d_ms + d_us;    // 5.250100 seconds
    auto doubled = d_sec * 2;            // 10 seconds

    // Arithmetic on time_points
    auto future = now + 1h;             // 1 hour from now
    auto past   = now - 30min;          // 30 minutes ago

    // Difference between time_points = duration
    auto elapsed = steady_now - steady_now;
}

Measuring Elapsed Time

#include <chrono>
#include <iostream>

void measure_elapsed() {
    using namespace std::chrono;

    auto start = steady_clock::now();

    // ... code to measure ...
    volatile double sum = 0;
    for (int i = 0; i < 1'000'000; ++i) {
        sum += i * 0.001;
    }

    auto end = steady_clock::now();

    // Convert duration to desired unit
    auto ns   = duration_cast<nanoseconds>(end - start);
    auto us   = duration_cast<microseconds>(end - start);
    auto ms   = duration_cast<milliseconds>(end - start);
    auto sec  = duration_cast<seconds>(end - start);

    std::cout << "Elapsed: " << ns.count() << " ns\n";
    std::cout << "Elapsed: " << us.count() << " us\n";
    std::cout << "Elapsed: " << ms.count() << " ms\n";
}

Best practice: Always use duration_cast to convert between duration units. It truncates toward zero. If you need floating-point precision, use duration<double> instead.

Floating-Point Durations

#include <chrono>
#include <iostream>

void floating_duration() {
    using namespace std::chrono;

    auto start = steady_clock::now();
    // ... work ...
    auto end = steady_clock::now();

    // Get duration as a double in seconds (preserves sub-second precision)
    double seconds = duration<double>(end - start).count();
    std::cout << "Took " << seconds << " seconds\n";

    // Or in milliseconds as double
    double ms = duration<double, std::milli>(end - start).count();
    std::cout << "Took " << ms << " ms\n";
}

Sleeping

#include <chrono>
#include <thread>
#include <iostream>

void sleeping_demo() {
    using namespace std::chrono_literals;

    // Sleep for a duration
    std::this_thread::sleep_for(500ms);       // 500 milliseconds
    std::this_thread::sleep_for(2s);         // 2 seconds
    std::this_thread::sleep_for(std::chrono::microseconds(100));

    // Sleep until a specific time point
    auto deadline = std::chrono::steady_clock::now() + 1s;
    std::this_thread::sleep_until(deadline);
}

Note: sleep_for and sleep_until require #include <thread>. They may sleep longer than requested due to OS scheduling granularity. For real-time guarantees, platform-specific APIs are needed. See std::thread for threading basics.

Example: Benchmarking Utility

#include <chrono>
#include <iostream>
#include <functional>
#include <string>

// Reusable benchmark timer
class Timer {
    std::chrono::steady_clock::time_point start_;
    std::string label_;
public:
    explicit Timer(std::string label = "")
        : start_(std::chrono::steady_clock::now())
        , label_(std::move(label)) {}

    ~Timer() {
        auto end = std::chrono::steady_clock::now();
        double ms = std::chrono::duration<double, std::milli>(
            end - start_).count();
        std::cout << "[" << label_ << "] "
                  << ms << " ms\n";
    }
};

void benchmark_demo() {
    {
        Timer t("vector sort");
        // std::sort(vector.begin(), vector.end());
    }
    {
        Timer t("hash map insert");
        // for (...) map.insert({key, value});
    }
    // Output:
    //   [vector sort] 12.3 ms
    //   [hash map insert] 4.7 ms
}

Date/Time Formatting (C++20)

C++20 introduced the <format> library and std::chrono::format for calendar-based date/time formatting, replacing the need for std::put_time or strftime.

#include <chrono>
#include <iostream>

// C++20 calendar types
void calendar_demo() {
    using namespace std::chrono;

    // Create specific dates
    year_month_day date = year{2025} / July / day{6};

    // Get today
    auto now = system_clock::now();
    auto today = year_month_day{
        floor<days>(now)
    };

    std::cout << "Year:  " << today.year() << "\n";
    std::cout << "Month: " << today.month() << "\n";
    std::cout << "Day:   " << today.day() << "\n";

    // C++20: Format a time_point (if std::format is available)
    // std::cout << std::format("{:%Y-%m-%d %H:%M:%S}", now) << "\n";

    // Pre-C++20 alternative with put_time
    auto time_t_now = system_clock::to_time_t(now);
    std::cout << std::put_time(std::localtime(&time_t_now), "%F %T") << "\n";
}

Compiler support: C++20 chrono calendar types (year_month_day, hh_mm_ss) require GCC 12+, Clang 14+, or MSVC 19.29. std::chrono::format support is still spotty across compilers as of 2025.
