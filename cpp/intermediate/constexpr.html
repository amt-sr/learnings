constexpr

constexpr (constant expression) enables compile-time evaluation of variables, functions, and even objects. Introduced in C++11 and significantly enhanced in C++14 and C++17, constexpr bridges the gap between runtime and compile-time programming, enabling zero-cost abstractions.

constexpr Variables

A constexpr variable must be initialized by a constant expression and its value is computed at compile time.

#include <iostream>

void constexpr_variables() {
    constexpr int max_size = 1024;          // compile-time constant
    constexpr double pi = 3.14159265358979;

    // Used where compile-time values are required:
    int buffer[max_size];                    // array size must be constant
    static_assert(max_size > 0, "must be positive");  // see static_assert

    // ERROR: cannot modify constexpr
    // max_size = 2048;

    // ERROR: not a constant expression
    // int x = 10;
    // constexpr int y = x;

    std::cout << "max_size = " << max_size << "\n";
    std::cout << "pi = " << pi << "\n";
}

constexpr Functions (C++11)

In C++11, constexpr functions are heavily restricted: a single return statement, no loops, no local variables, no if statements.

// C++11 constexpr — very restricted
constexpr int factorial_11(int n) {
    return (n <= 1) ? 1 : n * factorial_11(n - 1);
}

static_assert(factorial_11(5) == 120, "5! should be 120");

// C++14 constexpr — much more flexible
constexpr int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

static_assert(factorial(6) == 720, "6! should be 720");

// Can be called at runtime too
void runtime_usage() {
    int n = 7;
    std::cout << "7! = " << factorial(n) << "\n";  // evaluated at runtime
}

Key insight: A constexpr function can be evaluated at compile time, but it is not required to. If called with non-constant arguments, it executes at runtime like any normal function. The compiler evaluates at compile time only when the result is used in a constant expression context.

constexpr with Loops and Variables (C++14+)

#include <array>
#include <algorithm>
#include <iostream>

// Compile-time string length
constexpr size_t str_len(const char* s) {
    size_t len = 0;
    while (s[len] != '\0') ++len;
    return len;
}

// Compile-time Fibonacci
constexpr int fibonacci(int n) {
    if (n <= 1) return n;
    int prev = 0, curr = 1;
    for (int i = 2; i <= n; ++i) {
        int next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}

static_assert(fibonacci(10) == 55, "fib(10) should be 55");

// constexpr function returning a complex type
constexpr int max_of_three(int a, int b, int c) {
    int m = a;
    if (b > m) m = b;
    if (c > m) m = c;
    return m;
}

void advanced_constexpr() {
    constexpr int fib20 = fibonacci(20);
    std::cout << "Fibonacci(20) = " << fib20 << "\n";  // 6765
    std::cout << "max(3, 7, 5) = " << max_of_three(3, 7, 5) << "\n";  // 7
}

if constexpr (C++17)

if constexpr enables compile-time branching. The discarded branch is not even compiled, which means it does not need to be valid code. This is extremely useful for template programming.

#include <type_traits>
#include <iostream>
#include <string>

// Generic toString that works differently based on type
template<typename T>
std::string to_string_impl(const T& value) {
    if constexpr (std::is_arithmetic_v<T>) {
        return std::to_string(value);
    } else if constexpr (std::is_same_v<T, std::string>) {
        return value;
    } else {
        // This branch is only compiled for other types
        return "[unknown type]";
    }
}

void if_constexpr_demo() {
    std::cout << to_string_impl(42) << "\n";           // "42"
    std::cout << to_string_impl(3.14) << "\n";          // "3.14"
    std::cout << to_string_impl(std::string("hi")) << "\n";  // "hi"
}

// Replace enable_if/SFINAE with if constexpr
template<typename T>
auto process_value(T val) {
    if constexpr (std::is_integral_v<T>) {
        return val * 2;       // only compiled for integers
    } else if constexpr (std::is_floating_point_v<T>) {
        return val + 0.5;     // only compiled for floats
    } else {
        return val;            // fallback
    }
}

if constexpr vs regular if: A regular if evaluates the condition at runtime and both branches must be valid code. With if constexpr, the condition is evaluated at compile time and the discarded branch is not instantiated. See SFINAE for the older approach to similar problems.

constexpr Objects

#include <array>
#include <iostream>

// constexpr struct — usable at compile time
struct Point {
    double x;
    double y;

    constexpr double distance_to_origin() const {
        // C++14 allows this in constexpr
        double sum = x * x + y * y;
        // sqrt is constexpr in C++26; for now use at runtime
        return sum;
    }

    constexpr Point translate(double dx, double dy) const {
        return Point{x + dx, y + dy};
    }
};

// Compile-time point
constexpr Point origin{0.0, 0.0};
constexpr Point p{3.0, 4.0};

// Compile-time computation
constexpr double dist_sq = p.distance_to_origin();  // 25.0

// Compile-time array of constexpr points
constexpr std::array<Point, 3> triangle = {
    Point{0, 0}, Point{4, 0}, Point{2, 3}
};

void constexpr_objects_demo() {
    std::cout << "Distance squared: " << dist_sq << "\n";  // 25.0
    constexpr auto moved = p.translate(1.0, 1.0);
    std::cout << "Moved point: (" << moved.x << ", " << moved.y << ")\n";  // (4, 5)
}

constexpr and Standard Library

#include <array>
#include <algorithm>
#include <iostream>

// Many std algorithms are constexpr since C++20
constexpr int find_max_in_array() {
    std::array<int, 5> arr = {3, 7, 1, 9, 4};
    return *std::max_element(arr.begin(), arr.end());  // 9
}

static_assert(find_max_in_array() == 9);

// std::string_view is constexpr friendly
#include <string_view>

constexpr std::string_view greeting() {
    return "Hello, constexpr world!";
}
