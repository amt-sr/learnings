Type Traits

Type traits are a compile-time mechanism for inspecting, modifying, and transforming types. Defined in the <type_traits> header (C++11), they form the foundation of generic programming and template metaprogramming. They produce no runtime overhead -- all work happens at compile time.

Type Inspection Traits

These traits query properties of types and return a boolean or a type as a static constexpr member.

#include <type_traits>
#include <iostream>
#include <string>
#include <vector>

void inspection_demo() {
    // Primary type categories
    static_assert(std::is_integral_v<int>);
    static_assert(std::is_floating_point_v<double>);
    static_assert(std::is_pointer_v<int*>);
    static_assert(std::is_reference_v<int&>);
    static_assert(std::is_array_v<int[5]>);
    static_assert(std::is_class_v<std::string>);

    // Composite type categories
    static_assert(std::is_arithmetic_v<int>);       // integral or floating point
    static_assert(std::is_fundamental_v<char>);       // arithmetic, void, or nullptr_t
    static_assert(std::is_compound_v<std::string>);  // everything else

    // Type relationships
    static_assert(std::is_same_v<int, int>);
    static_assert(!std::is_same_v<int, unsigned int>);
    static_assert(std::is_base_of_v<std::ios_base, std::iostream>);
    static_assert(std::is_convertible_v<int, double>);

    // Property queries
    static_assert(std::is_const_v<const int>);
    static_assert(std::is_volatile_v<volatile int>);
    static_assert(std::is_signed_v<int>);
    static_assert(std::is_unsigned_v<unsigned int>);

    std::cout << "All type trait assertions passed!\n";
}

Type Modification Traits

Modification traits produce a new type based on the input type. They are accessed via the ::type member or the _t alias template (C++14).

#include <type_traits>
#include <iostream>
#include <string>

void modification_demo() {
    // Remove/add const
    using PlainInt = std::remove_const_t<const int>;       // int
    using ConstInt = std::add_const_t<int>;                // const int

    // Remove/add pointer
    using NoPtr = std::remove_pointer_t<int*>;             // int
    using WithPtr = std::add_pointer_t<int>;              // int*

    // Remove/add reference
    using NoRef = std::remove_reference_t<int&>;          // int
    using WithRef = std::add_lvalue_reference_t<int>;      // int&

    // Decay: removes reference, converts array to pointer, function to pointer
    using Decayed = std::decay_t<const int&>;              // int
    using ArrayDecayed = std::decay_t<int[10]>;            // int*
    using FuncDecayed = std::decay_t<void(int)>;           // void(*)(int)

    // Common type: the type all arguments can convert to
    using Common = std::common_type_t<int, double, float>; // double

    std::cout << "Modification traits compile successfully\n";
}

std::conditional

std::conditional is the compile-time equivalent of the ternary operator for types.

#include <type_traits>

// Select unsigned type large enough to hold values up to N bits
template<size_t N>
using UintForBits = std::conditional_t<
    (N <= 8),  uint8_t,
    std::conditional_t<
        (N <= 16), uint16_t,
        std::conditional_t<
            (N <= 32), uint32_t,
            uint64_t
        >
    >
>;

void conditional_demo() {
    static_assert(std::is_same_v<UintForBits<7>,  uint8_t>);
    static_assert(std::is_same_v<UintForBits<10>, uint16_t>);
    static_assert(std::is_same_v<UintForBits<24>, uint32_t>);
    static_assert(std::is_same_v<UintForBits<48>, uint64_t>);
}

static_assert with Type Traits

static_assert combined with type traits provides compile-time validation of template constraints.

#include <type_traits>
#include <string>
#include <vector>

template<typename T>
class NumericBuffer {
    static_assert(std::is_arithmetic_v<T>,
        "NumericBuffer requires an arithmetic type (int, float, double, etc.)");
    static_assert(std::is_trivially_copyable_v<T>,
        "NumericBuffer requires a trivially copyable type");

    T* data_;
    size_t size_;
public:
    explicit NumericBuffer(size_t n) : data_(new T[n]), size_(n) {}
    ~NumericBuffer() { delete[] data_; }
};

// NumericBuffer<int> ok1(100);       // OK
// NumericBuffer<std::string> err("hi");  // ERROR: string is not arithmetic

// Container type validation
template<typename Container>
void sort_container(Container& c) {
    static_assert(std::is_same_v<typename Container::value_type, int>,
        "sort_container only works with int containers");
    // sorting logic...
}

enable_if Revisited

std::enable_if is itself a type trait that conditionally defines a type member.

#include <type_traits>

// enable_if<true, int>::type is int
// enable_if<false, int> has no ::type member (SFINAE!)
template<bool Cond, typename T = void>
struct enable_if {};

template<typename T>
struct enable_if<true, T> { using type = T; };

// Convenience alias (C++14)
template<bool Cond, typename T = void>
using enable_if_t = typename enable_if<Cond, T>::type;

Practical: Type-Preserving Serializer

#include <type_traits>
#include <string>
#include <iostream>

struct Serializer {
    // Overload for integral types
    template<typename T>
    std::enable_if_t<std::is_integral_v<T>, std::string>
    serialize(T value) const {
        return "INT:" + std::to_string(value);
    }

    // Overload for floating-point types
    template<typename T>
    std::enable_if_t<std::is_floating_point_v<T>, std::string>
    serialize(T value) const {
        return "FLOAT:" + std::to_string(value);
    }

    // Overload for std::string
    std::string serialize(const std::string& value) const {
        return "STR:" + value;
    }
};

void serializer_demo() {
    Serializer s;
    std::cout << s.serialize(42) << "\n";      // INT:42
    std::cout << s.serialize(3.14) << "\n";     // FLOAT:3.140000
    std::cout << s.serialize("hello") << "\n";  // STR:hello
}

Performance: All type traits are compile-time operations. They generate zero runtime code. The compiler evaluates them during template instantiation and may even optimize away branches enabled/disabled by type traits.

Type traits are essential for SFINAE and template metaprogramming. In C++20, concepts build on type traits to provide cleaner syntax. See also auto and decltype for type deduction and template specialization for type-specific implementations.
