RAII

RAII (Resource Acquisition Is Initialization) is the most important idiom in C++. The principle is simple: acquire a resource in a constructor and release it in a destructor. Since destructors are called automatically when an object goes out of scope (even when exceptions are thrown), resources are always properly cleaned up.

The Core Principle

#include <iostream>
#include <fstream>

// RAII class wrapping a file handle
class FileGuard {
    std::ofstream file_;
    std::string filename_;
public:
    explicit FileGuard(const std::string& filename)
        : filename_(filename) {
        file_.open(filename_);
        if (!file_.is_open()) {
            throw std::runtime_error("Cannot open file: " + filename_);
        }
        std::cout << "Opened: " << filename_ << "\n";
    }

    ~FileGuard() {
        if (file_.is_open()) {
            file_.close();
            std::cout << "Closed: " << filename_ << "\n";
        }
    }

    // Prevent copying (exclusive resource ownership)
    FileGuard(const FileGuard&) = delete;
    FileGuard& operator=(const FileGuard&) = delete;

    // Allow moving
    FileGuard(FileGuard&& other) noexcept
        : file_(std::move(other.file_)), filename_(std::move(other.filename_)) {}

    std::ofstream& stream() { return file_; }
};

void raii_demo() {
    FileGuard file("output.txt");
    file.stream() << "Hello, RAII!\n";

    // Even if an exception is thrown here, the file is closed
    // throw std::runtime_error("oops");
}
// Destructor called automatically — file is closed

Scope-Based Resource Management

RAII turns scope-based object lifetime into resource lifetime management. This is the foundation for smart pointers, lock guards, and many other C++ facilities.

#include <iostream>

class Timer {
    std::string label_;
    // Use a high-resolution clock for timing
public:
    explicit Timer(std::string label) : label_(std::move(label)) {
        std::cout << "[" << label_ << "] START\n";
    }

    ~Timer() {
        std::cout << "[" << label_ << "] END\n";
    }

    Timer(const Timer&) = delete;
    Timer& operator=(const Timer&) = delete;
};

void scope_management() {
    std::cout << "Entering scope_management\n";

    {
        Timer t("Inner block");
        // Do some work here...
    }
    // Timer destructor called here: "[Inner block] END"

    std::cout << "Leaving scope_management\n";
}

// Exception safety example
void exception_safe_operation() {
    Timer t("Database Query");

    // Step 1
    // Step 2
    // If any step throws, Timer's destructor still runs
    // throw std::runtime_error("Connection failed");
}

Standard Library RAII Examples

#include <memory>     // unique_ptr, shared_ptr — memory RAII
#include <mutex>      // lock_guard, unique_lock — mutex RAII
#include <fstream>    // fstream — file handle RAII
#include <vector>     // vector — dynamic array RAII
#include <string>     // string — character buffer RAII

// All of these clean up automatically:
void standard_raii() {
    // Memory
    auto ptr = std::make_unique<int>(42);

    // File
    std::ofstream file("data.txt");
    file << "data\n";
    // file closed automatically when it goes out of scope

    // Mutex (see mutex and locks)
    std::mutex mtx;
    {
        std::lock_guard<std::mutex> lock(mtx);
        // Critical section — mutex released when lock goes out of scope
    }
}

Building a Custom RAII Wrapper

#include <iostream>

// Generic RAII wrapper for any callable (cleanup function)
template<typename Cleanup>
class ScopeGuard {
    Cleanup cleanup_;
    bool active_ = true;
public:
    explicit ScopeGuard(Cleanup cleanup) : cleanup_(std::move(cleanup)) {}

    ~ScopeGuard() {
        if (active_) cleanup_();
    }

    // Dismiss — cancel the cleanup
    void dismiss() { active_ = false; }

    // Prevent copying
    ScopeGuard(const ScopeGuard&) = delete;
    ScopeGuard& operator=(const ScopeGuard&) = delete;

    // Allow moving
    ScopeGuard(ScopeGuard&& other) noexcept
        : cleanup_(std::move(other.cleanup_)), active_(other.active_) {
        other.active_ = false;
    }
};

// Helper function for type deduction
template<typename Cleanup>
ScopeGuard<Cleanup> make_scope_guard(Cleanup cleanup) {
    return ScopeGuard<Cleanup>(std::move(cleanup));
}

void scope_guard_demo() {
    std::cout << "Before work\n";
    auto guard = make_scope_guard([]() {
        std::cout << "Cleanup: releasing resources\n";
    });

    // Do work...
    std::cout << "During work\n";

    // If we succeed and don't need cleanup:
    // guard.dismiss();

    // Otherwise, cleanup runs automatically when guard goes out of scope
}
// Output:
//   Before work
//   During work
//   Cleanup: releasing resources

RAII and Exception Safety

RAII is the foundation of exception safety in C++. With RAII, you never need to remember to release resources — the destructor handles it, whether the scope exits normally or via an exception.

#include <iostream>

class DatabaseConnection {
public:
    DatabaseConnection() { std::cout << "DB connected\n"; }
    ~DatabaseConnection() { std::cout << "DB disconnected\n"; }

    void execute(const std::string& query) {
        std::cout << "Executing: " << query << "\n";
    }
};

void transactional_operation() {
    DatabaseConnection db;  // Connected

    db.execute("BEGIN TRANSACTION");
    db.execute("INSERT INTO users VALUES ('Alice')");
    db.execute("UPDATE inventory SET count = count - 1");

    // If exception thrown here, db destructor disconnects properly
    if (false) throw std::runtime_error("Network error");

    db.execute("COMMIT");
}
// db.~DatabaseConnection() called — connection closed even if exception thrown

RAII Guarantee: Resources are released deterministically when the owning object goes out of scope. This applies whether the scope exits normally, via return, via break, via continue, or via an exception. This is why C++ does not need a finally block — destructors serve that purpose.
