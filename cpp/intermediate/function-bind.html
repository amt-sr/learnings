Function / Bind

std::function is a general-purpose polymorphic function wrapper that can store, copy, and invoke any callable target -- free functions, member functions, lambda expressions, and function objects. std::bind creates partial function applications by binding arguments to a callable. Together they form the foundation of the callback and event patterns in C++.

std::function

#include <functional>
#include <iostream>
#include <string>

// Free function
int add(int a, int b) { return a + b; }

// Functor (function object)
struct Multiplier {
    int factor;
    explicit Multiplier(int f) : factor(f) {}
    int operator()(int x) const { return x * factor; }
};

void function_basics() {
    // Wrap different callable types in std::function
    std::function<int(int, int)> f1 = add;          // free function
    std::function<int(int)> f2 = Multiplier{5};       // functor
    std::function<int(int)> f3 = [](int x) {           // lambda
        return x * x;
    };

    std::cout << f1(3, 4) << "\n";   // 7
    std::cout << f2(10) << "\n";   // 50
    std::cout << f3(7) << "\n";    // 49

    // Can be empty (default-constructed)
    std::function<void()> f4;
    if (!f4) {
        std::cout << "f4 is empty\n";
    }
    f4 = []() { std::cout << "Now assigned\n"; };
    f4();  // "Now assigned"
}

Callback Pattern

#include <functional>
#include <vector>
#include <string>
#include <iostream>

class EventSystem {
    std::vector<std::function<void(const std::string&)>> listeners_;
public:
    using Callback = std::function<void(const std::string&)>;

    void subscribe(Callback cb) {
        listeners_.push_back(std::move(cb));
    }

    void notify(const std::string& event) {
        for (const auto& cb : listeners_) {
            cb(event);
        }
    }
};

void callback_demo() {
    EventSystem es;

    // Register lambdas as callbacks
    es.subscribe([](const std::string& e) {
        std::cout << "[Logger] " << e << "\n";
    });

    int count = 0;
    es.subscribe([&count](const std::string& e) {
        ++count;
        std::cout << "[Counter] event #" << count << ": " << e << "\n";
    });

    es.notify("user_login");
    // [Logger] user_login
    // [Counter] event #1: user_login
}

std::bind and Placeholders

std::bind creates a callable by fixing some arguments of another callable. std::placeholders mark positions where arguments will be passed when the bound function is called.

#include <functional>
#include <iostream>
#include <cmath>

using namespace std::placeholders;  // _1, _2, _3...

int pow_int(int base, int exp) {
    return static_cast<int>(std::pow(base, exp));
}

void bind_demo() {
    // Bind second argument to 2: square(int) from pow_int(int, int)
    auto square = std::bind(pow_int, _1, 2);
    std::cout << square(5) << "\n";   // 25

    // Bind first argument to 3: cube(int)
    auto cube = std::bind(pow_int, 3, _1);
    std::cout << cube(2) << "\n";    // 9

    // Swap argument order
    auto subtract = std::bind(std::minus<int>{}, _2, _1);
    std::cout << subtract(3, 10) << "\n";  // 7 (10 - 3)

    // Bind member function
    struct Printer {
        void print(const std::string& msg) const {
            std::cout << msg << "\n";
        }
    };

    Printer p;
    auto bound_print = std::bind(&Printer::print, &p, _1);
    bound_print("Hello via bind");
}

Prefer lambdas over std::bind: In modern C++, lambdas are usually clearer and more efficient than std::bind. Lambdas avoid the complexity of placeholders and are easier to read. Use std::bind primarily when interfacing with APIs that explicitly require it or when using member function pointers with object instances.

std::function with Member Functions

#include <functional>
#include <iostream>

class Sensor {
    double value_ = 0.0;
public:
    void update(double delta) { value_ += delta; }
    double read() const { return value_; }
};

void member_function_callback() {
    Sensor s;
    // Wrap member function with object pointer
    std::function<void(double)> updater =
        [&s](double delta) { s.update(delta); };

    // Alternatively using mem_fn (C++11)
    auto reader = std::mem_fn(&Sensor::read);

    updater(1.5);
    updater(2.5);
    std::cout << "Sensor value: " << reader(s) << "\n";  // 4.0
}

Performance Considerations

#include <functional>
#include <iostream>

// Lambda: no overhead — compiler can inline
void with_lambda() {
    auto f = [](int x) { return x * 2; };
    int result = f(5);
}

// std::function: may prevent inlining, possible heap allocation
void with_function() {
    std::function<int(int)> f = [](int x) { return x * 2; };
    int result = f(5);
}

// For maximum performance in critical paths, use templates:
template<typename F>
void apply_and_print(F f, int x) {
    int result = f(x);
    std::cout << result << "\n";
}

Best practice: Use lambdas for local callbacks and templates for generic interfaces. Reserve std::function for cases where type erasure is needed: storing heterogeneous callables in a container, returning a function from another function, or defining callback APIs. The overhead is small but measurable in tight loops.

std::function complements lambda expressions and relates to operator() overloading (which creates functors). See observer pattern for a full callback system design.
