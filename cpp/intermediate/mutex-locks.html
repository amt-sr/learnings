Mutex / Locks

Mutexes (mutual exclusion locks) protect shared data from concurrent access by multiple threads. When a thread locks a mutex, other threads attempting to lock the same mutex block until it is unlocked. C++11 provides several RAII lock classes that manage the lifetime of mutex locks automatically.

std::mutex

#include <mutex>
#include <iostream>

class ThreadSafeCounter {
    int count_ = 0;
    std::mutex mutex_;
public:
    void increment() {
        std::lock_guard<std::mutex> lock(mutex_);
        ++count_;
    }

    void decrement() {
        std::lock_guard<std::mutex> lock(mutex_);
        --count_;
    }

    int get() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return count_;
    }
};

void mutex_demo() {
    ThreadSafeCounter counter;

    std::thread t1([&]() {
        for (int i = 0; i < 10000; ++i) counter.increment();
    });
    std::thread t2([&]() {
        for (int i = 0; i < 10000; ++i) counter.decrement();
    });

    t1.join();
    t2.join();
    std::cout << "Counter: " << counter.get() << "\n";  // 0
}

Lock Classes Comparison

Lock TypeBehaviorUse Case
std::lock_guardLocks in constructor, unlocks in destructor. Non-copyable.Simple scoped locks. Default choice.
std::unique_lockLike lock_guard but can defer, unlock early, and transfer ownership.Condition variables, deferred locking, manual unlock.
std::scoped_lock (C++17)Locks multiple mutexes deadlock-free using a deadlock avoidance algorithm.Multiple mutexes simultaneously.

std::unique_lock

#include <mutex>
#include <iostream>

void unique_lock_demo() {
    std::mutex mtx;

    // Defer locking
    std::unique_lock<std::mutex> lock(mtx, std::defer_lock);

    // ... do some work without holding the lock ...

    lock.lock();  // explicitly lock
    std::cout << "Protected section\n";
    lock.unlock();  // explicitly unlock early

    // ... do more work without holding the lock ...

    // Lock automatically released when 'lock' goes out of scope
    // Even if we didn't call unlock(), destructor handles it
}

std::scoped_lock (C++17)

std::scoped_lock can lock multiple mutexes simultaneously without deadlock. It uses the same algorithm as std::lock() to avoid deadlock when acquiring multiple locks.

#include <mutex>
#include <iostream>

class BankAccount {
    int balance_;
    std::mutex mtx_;
public:
    explicit BankAccount(int balance) : balance_(balance) {}

    int balance() { return balance_; }

    friend void transfer(BankAccount& from, BankAccount& to, int amount) {
        // Lock both mutexes deadlock-free
        std::scoped_lock lock(from.mtx_, to.mtx_);
        from.balance_ -= amount;
        to.balance_ += amount;
    }
};

void scoped_lock_demo() {
    BankAccount a(1000);
    BankAccount b(500);

    transfer(a, b, 200);
    std::cout << "A: " << a.balance() << ", B: " << b.balance() << "\n";
    // A: 800, B: 700
}

Deadlocks

A deadlock occurs when two or more threads each hold a lock and wait for a lock held by another thread. No thread can proceed.

Deadlock conditions (Coffman conditions): Mutual exclusion, hold and wait, no preemption, circular wait. All four must hold for a deadlock. Break any one to prevent deadlocks.

// DEADLOCK EXAMPLE (do not copy this!)
void deadlock_example() {
    std::mutex mtx_a, mtx_b;

    std::thread t1([&]() {
        std::lock_guard<std::mutex> lock1(mtx_a);
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
        std::lock_guard<std::mutex> lock2(mtx_b);  // DEADLOCK
    });

    std::thread t2([&]() {
        std::lock_guard<std::mutex> lock1(mtx_b);
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
        std::lock_guard<std::mutex> lock2(mtx_a);  // DEADLOCK
    });

    t1.join();
    t2.join();
}

// SOLUTION 1: Always lock in the same order
std::lock(mtx_a, mtx_b);  // locks both without deadlock

// SOLUTION 2: Use std::scoped_lock (C++17)
std::scoped_lock lock(mtx_a, mtx_b);  // deadlock-free

Deadlock Prevention Strategies

  1. Lock ordering: Always acquire locks in a fixed global order.
  2. std::lock / scoped_lock: Use the standard library's deadlock-free multi-lock.
  3. Minimal lock scope: Hold locks for the shortest possible time.
  4. Avoid nested locks: Minimize the number of locks held simultaneously.
  5. std::try_lock: Use try_lock with backoff for non-blocking acquisition.

Thread-Safe Counter (Complete Example)

#include <mutex>
#include <thread>
#include <vector>
#include <iostream>

class AtomicCounter {
    std::mutex mtx_;
    int value_ = 0;
public:
    void increment() {
        std::lock_guard<std::mutex> lock(mtx_);
        ++value_;
    }

    void add(int n) {
        std::lock_guard<std::mutex> lock(mtx_);
        value_ += n;
    }

    int get() const {
        std::lock_guard<std::mutex> lock(mtx_);
        return value_;
    }
};

void thread_safe_counter_demo() {
    AtomicCounter counter;
    std::vector<std::thread> threads;

    for (int i = 0; i < 10; ++i) {
        threads.emplace_back([&counter]() {
            for (int j = 0; j < 1000; ++j) {
                counter.increment();
            }
        });
    }

    for (auto& t : threads) t.join();
    std::cout << "Counter: " << counter.get() << "\n";  // 10000
}

Mutex locks work with std::thread and are used by condition variables for waiting/signaling. For lock-free approaches, see lock-free programming. Practice with thread-safe queue.
