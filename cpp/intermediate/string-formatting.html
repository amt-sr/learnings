String Formatting

C++ has evolved through several approaches to string formatting: from C-style printf through std::stringstream to the modern C++20 std::format. Each approach has tradeoffs in safety, performance, and ergonomics. This topic covers them all, building on std::string fundamentals and data types.

C-style printf/sprintf (Legacy)

Inherited from C, printf and sprintf use format specifiers but offer no type safety.

#include <cstdio>
#include <string>

void printf_demo() {
    int age = 30;
    double score = 95.5;
    const char* name = "Alice";

    // Print to stdout
    printf("Name: %s, Age: %d, Score: %.1f\n", name, age, score);

    // Write to buffer (DANGEROUS — buffer overflow risk)
    char buf[128];
    snprintf(buf, sizeof(buf), "Name: %s, Age: %d", name, age);
    // Always use snprintf, never sprintf
}

Why avoid printf: Format specifiers must match argument types exactly. Mismatched types cause undefined behavior at runtime. The compiler cannot check these, making bugs easy to introduce and hard to find. Prefer C++ alternatives.

std::stringstream

Type-safe and flexible, std::stringstream uses operator<< for formatting. The tradeoff is verbosity and runtime overhead.

#include <sstream>
#include <string>
#include <iomanip>

void stringstream_demo() {
    std::ostringstream oss;

    int id = 42;
    double value = 3.14159;
    std::string name = "sensor";

    oss << "[" << id << "] " << name
        << " = " << std::fixed << std::setprecision(2) << value;

    std::string result = oss.str();
    // "[42] sensor = 3.14"

    // Parsing with stringstream (input direction)
    std::istringstream iss("42 3.14 hello");
    int a; double b; std::string c;
    iss >> a >> b >> c;
    // a=42, b=3.14, c="hello"
}

Manipulators: std::setw(n) sets field width, std::setprecision(n) sets decimal precision, std::hex / std::oct change the base, and std::boolalpha prints booleans as text. Defined in <iomanip>.

std::to_string

For simple numeric-to-string conversions, std::to_string is concise and efficient.

#include <string>

void to_string_demo() {
    int i = 42;
    double d = 3.14;
    float f = 2.718f;
    long l = 9999999L;
    unsigned long ul = 42UL;

    std::string s1 = std::to_string(i);    // "42"
    std::string s2 = std::to_string(d);    // "3.140000"
    std::string s3 = std::to_string(f);    // "2.718000"
    std::string s4 = std::to_string(l);    // "9999999"
    std::string s5 = std::to_string(ul);   // "42"
}

Limitation: std::to_string uses fixed precision (6 decimal places for floating-point). For custom precision, use std::format (C++20) or std::ostringstream.

C++20 std::format

std::format (C++20, <format>) provides Python-style format strings with compile-time type checking. This is the recommended approach for all new code.

#include <format>
#include <string>
#include <iostream>

void format_demo() {
    std::string name = "Alice";
    int age = 30;
    double score = 95.675;

    // Basic formatting
    std::string s1 = std::format("Name: {}, Age: {}", name, age);
    // "Name: Alice, Age: 30"

    // Ordered arguments (positional)
    std::string s2 = std::format("{0} is {1} years old. {0} scored {2:.1f}.",
                                  name, age, score);
    // "Alice is 30 years old. Alice scored 95.7."

    // Width and alignment
    std::string s3 = std::format("{:>10}", name);   // "     Alice"
    std::string s4 = std::format("{:<10}", name);   // "Alice     "
    std::string s5 = std::format("{:^10}", name);    // "  Alice   "

    // Numeric formatting
    std::string s6 = std::format("{:.2f}", score);   // "95.68"
    std::string s7 = std::format("{:x}", 255);       // "ff" (hex)
    std::string s8 = std::format("{:#x}", 255);      // "0xff"
    std::string s9 = std::format("{:+d}", 42);       // "+42"
    std::string s10 = std::format("{:08d}", 42);     // "00000042"

    std::cout << s1 << "\n";
}

Format Specifier Syntax

// Full format specifier grammar:
// { [arg_index] [: [fill] align] [sign] [#] [0] [width] [.precision] [type] }

std::string s = std::format(
    "{1:<+020.2f}",   // arg 1, left-align, sign, zero-pad, width 20, 2 decimals
    "unused", 42.5
);
// "+000000000000042.50      "  (left-aligned within width 20)

Recommendation: Use std::format for all new code (C++20+). For codebases stuck on C++17, std::to_string handles simple cases and std::ostringstream handles complex formatting. Avoid printf/sprintf in modern C++.

Custom Type Formatting with std::format

#include <format>
#include <string>
#include <string_view>

struct Point3D {
    double x, y, z;
};

// Specialize std::formatter for custom types
template<>
struct std::formatter<Point3D> : std::formatter<std::string> {
    auto format(const Point3D& p, format_context& ctx) const {
        return std::format_to(ctx.out(), "({:.2f}, {:.2f}, {:.2f})",
                              p.x, p.y, p.z);
    }
};

void custom_format_demo() {
    Point3D p{1.0, 2.5, -3.75};
    std::string s = std::format("Point: {}", p);
    std::cout << s << "\n";  // "Point: (1.00, 2.50, -3.75)"
}

Compiler support: std::format requires GCC 13+, Clang 14+, or MSVC 19.29. For older compilers, use the fmt library (the basis for std::format) as a drop-in replacement: #include <fmt/core.h> and fmt::format(...).

Common Formatting Patterns

#include <format>

void common_patterns() {
    // Padding and alignment table
    std::string header = std::format("{:<15} {:>8} {:>10}", "Name", "Age", "Score");
    std::string row    = std::format("{:<15} {:>8} {:>10.1f}", "Alice", 30, 95.5);
    std::string sep    = std::format("{:->35}", "");

    // Thousands separator (C++23: std::format("{:Ld}", 1000000))
    // Pre-C++23 workaround: manual insertion

    // Percentage formatting
    double ratio = 0.8567;
    std::string pct = std::format("{:.1f}%", ratio * 100);  // "85.7%"

    // Hex with prefix
    std::string hex = std::format("0x{:04X}", 255);  // "0x00FF"

    std::cout << header << "\n" << sep << "\n" << row << "\n";
}

String formatting pairs naturally with std::string operations and regular expressions for parsing. For type deduction of format arguments, auto & decltype is relevant background. See also chrono & time for time-specific formatting.
