Variadic Templates

Variadic templates allow templates to accept an arbitrary number of arguments of any type. Introduced in C++11, they are the foundation for many modern C++ features including std::tuple, std::variant, and type-safe formatting. With C++17, fold expressions simplified variadic template code significantly.

Parameter Packs

A parameter pack is a template parameter that accepts zero or more arguments. There are two kinds: type parameter packs (typename... Ts) and non-type parameter packs (int... Is).

#include <iostream>

// Type parameter pack
template<typename... Ts>
void print_types() {
    std::cout << sizeof...(Ts) << " types\n";  // sizeof... gives pack size
}

// Function parameter pack
template<typename... Ts>
void print_all(const Ts&... args) {
    // pack expansion: args expands to arg1, arg2, ...
    // but we need a way to iterate over them...
}

Pack Expansion

Pack expansion uses the ... operator to expand a pattern over each element in the pack. The pattern is evaluated once for each element.

#include <iostream>

// Expand with a function call
template<typename... Ts>
void print_all(const Ts&... args) {
    // C++17 fold expression (see below for details)
    (std::cout << ... << args) << "\n";
}

// Expand into a container initializer
template<typename... Ts>
auto make_vector(const Ts&... args) {
    return std::vector<std::common_type_t<Ts...>>{args...};
}

Expansion contexts: Pack expansion works inside: function arguments, initializer lists, template argument lists, base class lists, lambda captures, and more. The ... must follow the pattern containing the pack.

Recursive Variadic Templates

Before C++17 fold expressions, recursion was the primary way to process parameter packs. The pattern: a base case (zero args) plus a recursive case (peel one arg, recurse on the rest).

#include <iostream>

// Base case: no arguments
void print() {
    std::cout << "\n";
}

// Recursive case: print first arg, recurse on rest
template<typename T, typename... Ts>
void print(const T& first, const Ts&... rest) {
    std::cout << first;
    if constexpr (sizeof...(rest) > 0) {
        std::cout << ", ";
    }
    print(rest...);  // expand pack into recursive call
}

void recursive_demo() {
    print(1, "hello", 3.14, 'X');
    // Output: 1, hello, 3.14, X
}

C++17 Fold Expressions

Fold expressions eliminate the need for recursive template boilerplate when you need to apply a binary operator across all pack elements.

FormSyntaxExpansion
Unary right fold(pack op ...)((a1 op a2) op a3) op ...
Unary left fold(... op pack)a1 op (a2 op (a3 op ...))
Binary right fold(pack op ... op init)((a1 op a2) op ...) op init
Binary left fold(init op ... op pack)init op (a1 op (a2 op ...))
#include <iostream>
#include <string>

// Type-safe printf using fold expressions
template<typename... Args>
void safe_printf(const char* fmt, const Args&... args) {
    std::cout << fmt << ": ";
    // Right fold: (cout << arg1) << arg2 << arg3 ...
    (std::cout << ... << args) << "\n";
}

void fold_demo() {
    safe_printf("Result", 42, " items", 3.14);
    // Output: Result: 42 items3.14

    // Sum all arguments with a left fold
    auto sum = [](auto... vals) {
        return (... + vals);
    };
    std::cout << "Sum: " << sum(1, 2, 3, 4, 5) << "\n";  // 15

    // Comma fold to execute actions in order
    auto actions = [](auto&... args) {
        ((std::cout << args << " "), ...);
    };
    actions(std::cout, "hello", "world");
    // Output: hello world
}

// All elements satisfy a condition (unary left fold with &&)
template<typename... Args>
bool all_positive(const Args&... args) {
    return (... && (args > 0));
}

// Any element satisfies a condition (unary right fold with ||)
template<typename... Args>
bool any_negative(const Args&... args) {
    return (... || (args < 0));
}

Best practice: Prefer C++17 fold expressions over manual recursion. They are simpler, more readable, and often generate better code. The recursive approach is still useful for complex cases where fold expressions do not apply.

Type-Safe printf

A practical variadic template: a type-safe printf replacement that avoids format string vulnerabilities inherent in C's printf.

#include <iostream>
#include <string>
#include <sstream>

// Helper: convert any type to string
template<typename T>
std::string to_string_arg(const T& val) {
    if constexpr (std::is_same_v<T, std::string>) {
        return val;
    } else if constexpr (std::is_arithmetic_v<T>) {
        return std::to_string(val);
    } else {
        std::ostringstream oss;
        oss << val;
        return oss.str();
    }
}

// Base case
void format_impl(std::ostringstream& oss) {}

// Recursive case with placeholders
template<typename T, typename... Ts>
void format_impl(std::ostringstream& oss, const T& first, const Ts&&... rest) {
    oss << to_string_arg(first);
    if constexpr (sizeof...(rest) > 0) {
        oss << ", ";
    }
    format_impl(oss, rest...);
}

template<typename... Ts>
std::string format(const Ts&&... args) {
    std::ostringstream oss;
    format_impl(oss, args...);
    return oss.str();
}

void printf_demo() {
    std::cout << format("user:", 42, 3.14, "active") << "\n";
    // Output: user:, 42, 3.14, active
}

Variadic Class Templates

#include <iostream>
#include <typeinfo>

// Variadic class template
template<typename... Types>
class TypeList {};

// Specialization to get the head type
template<typename Head, typename... Tail>
class TypeList<Head, Tail...> {
public:
    using head = Head;
    using tail = TypeList<Tail...>;
    static constexpr size_t size = 1 + sizeof...(Tail);
};

void class_demo() {
    using MyTypes = TypeList<int, double, std::string>;
    std::cout << "Size: " << MyTypes::size << "\n";  // 3
    std::cout << "Head is int: "
              << std::is_same_v<MyTypes::head, int> << "\n";  // 1
}

Comparison: Recursion vs Fold Expressions

AspectRecursiveFold Expression (C++17)
VerbosityBase case + recursive caseSingle expression
Operator supportAny operationBinary operators only
ComplexityDeep recursion possibleFlat, no recursion depth
Best forComplex per-element logicSimple reductions (sum, print, etc.)

Compilation cost: Variadic templates instantiate a separate function for each recursive step (in the recursive approach). With many parameters, this increases compile time and binary size. Fold expressions are more efficient in this regard.

Variadic templates are the backbone of advanced template metaprogramming. They interact closely with template specialization and SFINAE for conditional compilation. See type traits for compile-time type inspection used within variadic templates.
