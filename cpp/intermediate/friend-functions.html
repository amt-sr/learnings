Friend Functions

The friend keyword grants a non-member function or another class access to the private and protected members of the class that declares the friendship. Friend declarations are a controlled exception to encapsulation — they allow specific external functions to access internals without making those members public.

Basic friend Function

#include <iostream>
#include <string>

class BankAccount {
    double balance_;
    std::string owner_;
public:
    BankAccount(std::string owner, double initial)
        : owner_(std::move(owner)), balance_(initial) {}

    void deposit(double amount) { balance_ += amount; }
    void withdraw(double amount) { if (amount <= balance_) balance_ -= amount; }

    // Grant friend access — audit function can see balance_
    friend void audit(const BankAccount& account);
    friend bool transfer(BankAccount& from, BankAccount& to, double amount);

    // Friend class — all methods of TransactionLog can access private members
    friend class TransactionLog;
};

// Non-member function with access to private members
void audit(const BankAccount& account) {
    std::cout << "Audit: " << account.owner_
              << " has balance $" << account.balance_ << "\n";
}

// Friend function operating on two objects
bool transfer(BankAccount& from, BankAccount& to, double amount) {
    if (amount > from.balance_ || amount <= 0) return false;
    from.balance_ -= amount;
    to.balance_ += amount;
    std::cout << "Transferred $" << amount << " from "
              << from.owner_ << " to " << to.owner_ << "\n";
    return true;
}

void friend_demo() {
    BankAccount alice("Alice", 1000.0);
    BankAccount bob("Bob", 500.0);

    audit(alice);  // "Audit: Alice has balance $1000"

    transfer(alice, bob, 200.0);  // Transferred $200

    audit(alice);  // "Audit: Alice has balance $800"
    audit(bob);    // "Audit: Bob has balance $700"
}

Friend for Operator Overloading

The most common legitimate use of friend is for overloading << and >> stream operators.

#include <iostream>
#include <string>

class Vector3D {
    double x_, y_, z_;
public:
    Vector3D(double x, double y, double z) : x_(x), y_(y), z_(z) {}

    // Friend declarations for stream operators
    friend std::ostream& operator<<(std::ostream& os, const Vector3D& v);
    friend std::istream& operator>>(std::istream& is, Vector3D& v);

    // Friend for symmetric operators
    friend Vector3D operator*(double scalar, const Vector3D& v);

    double dot(const Vector3D& other) const {
        return x_ * other.x_ + y_ * other.y_ + z_ * other.z_;
    }
};

std::ostream& operator<<(std::ostream& os, const Vector3D& v) {
    os << "(" << v.x_ << ", " << v.y_ << ", " << v.z_ << ")";
    return os;
}

std::istream& operator>>(std::istream& is, Vector3D& v) {
    is >> v.x_ >> v.y_ >> v.z_;
    return is;
}

// Friend allows scalar * vector (not possible as member because
// the left operand is double, not Vector3D)
Vector3D operator*(double scalar, const Vector3D& v) {
    return Vector3D(v.x_ * scalar, v.y_ * scalar, v.z_ * scalar);
}

void stream_operator_demo() {
    Vector3D v(1.0, 2.0, 3.0);
    std::cout << "v = " << v << "\n";  // (1, 2, 3)

    Vector3D scaled = 2.0 * v;  // (2, 4, 6) — left operand is double
    std::cout << "2.0 * v = " << scaled << "\n";

    // v * 2.0 would need a member operator* — different from friend
}

Friend Class

class TransactionLog {
    struct Entry {
        std::string from;
        std::string to;
        double amount;
        std::string timestamp;
    };
    std::vector<Entry> entries_;
public:
    void log_transfer(const BankAccount& from, const BankAccount& to, double amount) {
        // Can access from.balance_, to.owner_ because TransactionLog is a friend of BankAccount
        entries_.push_back({from.owner_, to.owner_, amount, "now"});
    }

    void print_log() const {
        for (const auto& e : entries_) {
            std::cout << e.from << " -> " << e.to
                      << ": $" << e.amount << " at " << e.timestamp << "\n";
        }
    }
};

Friend with Templates

template<typename T>
class Stack {
    std::vector<T> data_;
public:
    void push(const T& val) { data_.push_back(val); }
    void pop() { data_.pop_back(); }
    const T& top() const { return data_.back(); }
    bool empty() const { return data_.empty(); }

    // Friend a specific template instantiation
    friend bool operator==<>(const Stack<T>& a, const Stack<T>& b);

    // Friend all template instantiations
    template<typename U>
    friend void print_stack(const Stack<U>& s);
};

template<typename T>
bool operator==(const Stack<T>& a, const Stack<T>& b) {
    return a.data_ == b.data_;  // Can access private data_
}

template<typename U>
void print_stack(const Stack<U>& s) {
    for (const auto& item : s.data_)  // Can access private data_
        std::cout << item << " ";
    std::cout << "\n";
}

Pros and Cons

Legitimate uses:

  • Overloading << / >> stream operators
  • Symmetric binary operators where the left operand is not your class
  • Factory functions or utility functions that need deep access
  • Tightly coupled classes (e.g., Iterator and Container)

Avoid when:

  • Public getters/setters would work just as well
  • Too many friends — indicates poor encapsulation design
  • Breaking the class invariant (friend functions bypass invariants)

Friendship is not inherited: If class B is a friend of A, a class C derived from B is not automatically a friend of A. Friendship is also not transitive: if B is a friend of A and C is a friend of B, C is not a friend of A.

Friend functions are commonly used with operator overloading for stream operators and symmetric operations. They complement the RAII pattern for resource management. Next, we explore RAII.
