Set / Unordered Set

std::set and std::unordered_set are associative containers that store unique elements. std::set keeps elements sorted using a balanced binary tree (typically red-black tree), while std::unordered_set uses a hash table for O(1) average-case lookups.

std::set (Ordered)

std::set stores elements in sorted order. It provides bidirectional iterators and guarantees logarithmic-time insert, erase, and lookup operations.

#include <set>
#include <iostream>
#include <string>

void set_basics() {
    std::set<int> s = {30, 10, 20, 10, 40};  // duplicates are ignored
    // s = {10, 20, 30, 40}  (sorted)

    s.insert(25);           // s = {10, 20, 25, 30, 40}
    s.insert(10);           // no effect — already present

    std::cout << "Size: " << s.size() << "\n";       // 5
    std::cout << "Count 10: " << s.count(10) << "\n";  // 1
    std::cout << "Count 99: " << s.count(99) << "\n";  // 0

    // Find and erase
    if (auto it = s.find(25); it != s.end()) {
        std::cout << "Found: " << *it << "\n";
        s.erase(it);  // erase by iterator — O(1) amortized
    }

    // Range-based for (always sorted order)
    for (const auto& val : s) {
        std::cout << val << " ";
    }
    // Output: 10 20 30 40
}

std::unordered_set (Hash Table)

std::unordered_set provides average O(1) insert, erase, and lookup but elements have no defined order. It requires elements to be hashable via std::hash.

#include <unordered_set>
#include <iostream>
#include <string>

void unordered_set_basics() {
    std::unordered_set<std::string> names;
    names.insert("Alice");
    names.insert("Bob");
    names.insert("Charlie");
    names.insert("Alice");  // duplicate — ignored

    std::cout << "Size: " << names.size() << "\n";  // 3
    std::cout << "Bucket count: " << names.bucket_count() << "\n";
    std::cout << "Load factor: " << names.load_factor() << "\n";

    if (names.contains("Bob")) {  // C++20
        std::cout << "Bob is in the set\n";
    }

    // Custom bucket count for performance tuning
    std::unordered_set<int> nums;
    nums.reserve(1000);  // avoid rehashing when inserting ~1000 elements
    for (int i = 0; i < 1000; ++i) nums.insert(i);
}

Custom Comparator and Hasher

#include <set>
#include <unordered_set>
#include <iostream>
#include <string>

// Custom comparator: case-insensitive set
struct CaseInsensitiveLess {
    bool operator()(const std::string& a, const std::string& b) const {
        return std::lexicographical_compare(
            a.begin(), a.end(), b.begin(), b.end(),
            [](char c1, char c2) { return tolower(c1) < tolower(c2); });
    }
};

void custom_comparator_demo() {
    std::set<std::string, CaseInsensitiveLess> words;
    words.insert("Hello");
    words.insert("hello");   // considered duplicate
    words.insert("WORLD");

    std::cout << "Size: " << words.size() << "\n";  // 2
}

// Custom hasher for unordered_set
struct StringHash {
    size_t operator()(const std::string& s) const {
        size_t h = 0;
        for (char c : s) h = h * 31 + c;
        return h;
    }
};

void custom_hasher_demo() {
    std::unordered_set<std::string, StringHash> custom;
    custom.insert("test");
    std::cout << "Contains 'test': " << custom.count("test") << "\n";
}

Practical: Removing Duplicates

#include <set>
#include <vector>
#include <iostream>

std::vector<int> remove_duplicates(const std::vector<int>& input) {
    std::set<int> unique(input.begin(), input.end());
    return std::vector<int>(unique.begin(), unique.end());  // sorted
}

void dedup_demo() {
    std::vector<int> data = {5, 3, 5, 1, 3, 7, 1, 9};
    auto unique = remove_duplicates(data);
    for (int v : unique) std::cout << v << " ";
    // Output: 1 3 5 7 9
}

When to Use Which

Use std::set when: You need ordered iteration, range queries (lower_bound/upper_bound), or when you need guaranteed O(log n) worst case.

Use std::unordered_set when: You primarily need fast existence checks and do not care about ordering. This is the most common case for deduplication and membership testing.

Performance tip: Call reserve() on unordered_set before inserting many elements to avoid costly rehashing. The load factor should stay below 1.0 for optimal performance.

Sets complement the map containers (which store key-value pairs) and are used extensively with STL algorithms for set operations like intersection and union.
