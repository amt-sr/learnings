auto & decltype

C++11 introduced auto for type deduction and decltype for querying the type of an expression. C++14 added decltype(auto) and relaxed auto rules. Together, they reduce verbosity, improve readability, and enable generic programming patterns.

auto Type Deduction

auto tells the compiler to deduce the type from the initializer. The deduction rules follow the same pattern as template argument deduction.

#include <iostream>
#include <vector>
#include <map>
#include <type_traits>

void auto_basics() {
    // Basic usage
    auto x = 42;           // int
    auto y = 3.14;         // double
    auto z = "hello";      // const char*
    auto s = std::string("world");  // std::string

    // With references and const
    int a = 10;
    int& ref = a;
    const int& cref = a;

    auto b = a;     // int (not int&, not const)
    auto c = ref;   // int (reference is stripped)
    auto d = cref;  // int (both const and reference are stripped)

    // To keep const/reference, use explicitly
    auto& e = a;      // int&
    const auto f = a;  // const int
    auto&& g = 42;     // int&& (forwarding reference)
}

auto with Iterators

#include <map>
#include <vector>

void auto_iterators() {
    // Before auto — verbose iterator types
    std::map<std::string, std::vector<int>> data;
    for (std::map<std::string, std::vector<int>>::iterator it = data.begin();
         it != data.end(); ++it) { /* ... */ }

    // With auto — clean and readable
    for (auto it = data.begin(); it != data.end(); ++it) { /* ... */ }

    // Even better: range-based for (see range-based for)
    for (const auto& [key, value] : data) { /* ... */ }
}

Best practice: Use auto when the type is obvious from context or overly complex to write out. Do not use auto when the explicit type adds clarity, especially for arithmetic conversions that may hide bugs.

auto and Function Return Types

// C++14: auto return type deduction
auto add(int a, int b) {
    return a + b;  // deduced as int
}

auto get_name() {
    return std::string("Alice");  // deduced as std::string
}

// Can be tricky — what does this return?
auto compute(int x) {
    if (x > 0)
        return x;       // int
    else
        return 3.14;     // double — ERROR: inconsistent return types
}

// All return statements must deduce to the same type.
// Use decltype(auto) or an explicit return type when mixing types.

decltype

decltype yields the declared type of an expression without evaluating it. Unlike auto, it preserves references and const qualifiers.

#include <iostream>
#include <typeinfo>

void decltype_basics() {
    int x = 10;
    int& ref = x;
    const int c = 20;

    decltype(x) a = 30;        // int
    decltype(ref) b = x;       // int& — preserves reference!
    decltype(c) d = 40;         // const int — preserves const!
    decltype(x + 1) e = 50;    // int — x+1 is an rvalue (prvalue)

    // decltype with parenthesized variable names
    decltype((x)) f = x;        // int& — parenthesized lvalue is reference!
    decltype(x) g = x;          // int   — plain name is the declared type

    std::cout << "a: " << a << "\n";
    std::cout << "b: " << b << "\n";
    std::cout << "d: " << d << "\n";
}

decltype((variable)) vs decltype(variable): When the expression inside decltype is a parenthesized variable name, the result is a reference to that variable's type. decltype(x) gives int, but decltype((x)) gives int&. This is a subtle but important difference.

decltype(auto)

decltype(auto) (C++14) uses decltype deduction rules instead of auto rules. It preserves references and const, which auto normally strips.

#include <vector>
#include <iostream>

struct Container {
    std::vector<int> data = {1, 2, 3, 4, 5};

    // auto return — strips reference, returns a copy
    auto get_at_copy(size_t idx) {
        return data[idx];     // returns int (copy)
    }

    // decltype(auto) return — preserves reference
    decltype(auto) get_at_ref(size_t idx) {
        return data[idx];     // returns int& (reference)
    }

    // decltype(auto) with auto — return forwarding reference
    template<typename T>
    decltype(auto) forward_example(T&& arg) {
        return std::forward<T>(arg);
    }
};

void decl_auto_demo() {
    Container c;
    c.get_at_copy(0) = 99;  // modifies a temporary — no effect
    c.get_at_ref(0) = 99;   // modifies actual element
    std::cout << "data[0] = " << c.data[0] << "\n";  // 99
}

Trailing Return Types

C++11 introduced trailing return types using ->. While largely superseded by auto return type deduction (C++14), trailing return types are still needed in some template scenarios where the return type depends on template parameters.

// Trailing return type with decltype
template<typename T, typename U>
auto multiply(T a, U b) -> decltype(a * b) {
    return a * b;
}

// In C++14+, this is cleaner:
template<typename T, typename U>
auto multiply2(T a, U b) {
    return a * b;
}

// Trailing return type with decltype still useful in complex cases:
template<typename Container>
auto begin(Container& c) -> decltype(c.begin()) {
    return c.begin();
}

#include <vector>
#include <iostream>

// Practical example: generic function returning element reference
template<typename C>
decltype(auto) front_element(C& container) {
    return container.front();
}

void practical_example() {
    std::vector<int> v = {10, 20, 30};

    // decltype(auto) preserves the reference returned by front()
    decltype(auto) ref = front_element(v);
    ref = 99;  // modifies v[0]
    std::cout << "v[0] = " << v[0] << "\n";  // 99
}
