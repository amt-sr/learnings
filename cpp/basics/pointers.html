1.1.7 Pointers

A pointer is a variable that stores the memory address of another variable. Pointers are one of the most powerful and most dangerous features of C++. They enable dynamic memory allocation, efficient pass-by-reference semantics, and direct hardware access, but incorrect pointer usage is a leading source of crashes and security vulnerabilities.

Declaration Syntax

To declare a pointer, place an asterisk * after the type. The pointer type must match the type of the variable it points to:

int x = 42;
int* ptr = &x;    // ptr stores the address of x
double* dptr;      // pointer to a double (uninitialized)
char* cptr = nullptr;  // null pointer (safe default)

Style tip: The asterisk can go next to the type (int* ptr) or next to the variable (int *ptr). The first style emphasizes that int* is a "pointer-to-int" type. However, int* a, b; declares a as a pointer but b as a plain int, which surprises many beginners. Declare one pointer per line to avoid confusion.

Address-of and Dereference Operators

OperatorSymbolPurpose
Address-of&Get the memory address of a variable
Dereference*Access the value at the address a pointer holds
#include <iostream>

int main() {
    int value = 100;
    int* ptr = &value;    // ptr now holds the address of value

    std::cout << "Value:  " << value << std::endl;   // 100
    std::cout << "Address: " << ptr << std::endl;     // e.g. 0x7ffd...
    std::cout << "Deref:  " << *ptr << std::endl;      // 100

    // Modify value through the pointer
    *ptr = 200;
    std::cout << "Value after modification: " << value << std::endl; // 200

    return 0;
}

Pointer Arithmetic

Pointer arithmetic allows you to navigate through contiguous memory, which is especially useful with arrays and strings. When you add an integer n to a pointer of type T*, the address advances by n * sizeof(T) bytes:

#include <iostream>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int* ptr = arr;  // points to arr[0]

    std::cout << *ptr << std::endl;       // 10
    std::cout << *(ptr + 2) << std::endl;   // 30
    std::cout << ptr[3] << std::endl;        // 40 (ptr[i] is *(ptr + i))

    ++ptr;  // now points to arr[1]
    std::cout << *ptr << std::endl;         // 20

    // Pointer difference: number of elements between two pointers
    int* end = arr + 5;
    std::cout << (end - ptr) << std::endl;  // 4 elements apart

    return 0;
}

Warning: Pointer arithmetic is only well-defined within a single array (or one past the last element). Dereferencing a pointer that has been moved outside the bounds of its array results in undefined behavior.

Null Pointers

A null pointer explicitly points to no valid memory location. In modern C++, always use nullptr (introduced in C++11) instead of NULL or 0:

#include <iostream>

int main() {
    int* ptr = nullptr;  // C++11 recommended

    // Always check before dereferencing
    if (ptr != nullptr) {
        std::cout << *ptr << std::endl;
    } else {
        std::cout << "Pointer is null" << std::endl;
    }

    // Prefer this concise form
    if (ptr) {
        std::cout << *ptr << std::endl;
    }

    return 0;
}

Danger: Dereferencing a null pointer causes undefined behavior, which on most systems results in a segmentation fault and program crash. Always check that a pointer is not null before dereferencing it.

Pointers and const

Combining const with pointers creates several distinct semantics:

DeclarationMeaning
const int* pPointer to const int — cannot change the value via *p
int* const pConst pointer to int — cannot change p itself
const int* const pConst pointer to const int — neither can change
#include <iostream>

int main() {
    int x = 10, y = 20;

    const int* p1 = &x;   // can change p1, but not *p1
    p1 = &y;               // OK
    // *p1 = 30;             // ERROR: cannot modify const value

    int* const p2 = &x;     // can change *p2, but not p2
    *p2 = 30;               // OK
    // p2 = &y;             // ERROR: cannot reassign const pointer

    const int* const p3 = &x; // cannot change p3 or *p3

    std::cout << "x = " << x << ", y = " << y << std::endl;
    return 0;
}

Common Pitfalls

Dangling Pointers

A dangling pointer points to memory that has been freed or has gone out of scope. Accessing it is undefined behavior:

#include <iostream>

int* dangling_example() {
    int local = 42;
    return &local;  // WARNING: returning address of a local variable
} // local is destroyed here, ptr returned is dangling

int main() {
    int* ptr = dangling_example();
    // *ptr is undefined behavior — the variable no longer exists
    // The memory may have been reused for something else
    std::cout << *ptr << std::endl;  // DANGEROUS
    return 0;
}

Wild Pointers

A wild pointer is an uninitialized pointer that holds a garbage address. Dereferencing it can corrupt arbitrary memory:

int* wild;          // uninitialized — points to a random address
// *wild = 100;     // DANGEROUS: writes to unknown memory location

Rule: Always initialize pointers. If you do not have a valid address yet, set the pointer to nullptr. Uninitialized pointers are the root cause of countless bugs, memory corruption, and security vulnerabilities. See Memory Layout for how memory is organized.

Pointers with Dynamic Memory

Pointers are the mechanism for accessing dynamically allocated memory on the heap:

#include <iostream>

int main() {
    // Allocate a single int on the heap
    int* p = new int(42);
    std::cout << *p << std::endl;  // 42
    delete p;  // must free when done
    p = nullptr;  // prevent dangling pointer

    // Allocate an array on the heap
    int* arr = new int[10];
    for (int i = 0; i < 10; ++i) {
        arr[i] = i * i;
    }
    delete[] arr;  // use delete[] for arrays
    arr = nullptr;

    return 0;
}

Best practice: In modern C++, prefer smart pointers (std::unique_ptr, std::shared_ptr) over raw new/delete. Smart pointers automatically manage lifetime and prevent memory leaks. See new & delete for the full details on dynamic allocation.
