1.5.5 Iterators Basics

Iterators are the glue that connects STL containers to algorithms. An iterator is a generalization of a pointer: it provides a way to access, traverse, and modify elements within a container. Every STL container provides its own iterator type, and all STL algorithms operate on iterator ranges.

What Iterators Provide

At its core, an iterator supports these operations:

  • Dereference: *it accesses the element the iterator points to.
  • Increment: ++it moves to the next element.
  • Compare: it == end checks if we have reached the end of the range.
  • Arrow: it->member accesses a member of the pointed-to object.
#include <iostream>
#include <vector>

int main() {
    std::vector<int> nums = {10, 20, 30, 40, 50};

    // Obtain iterators to the beginning and past-the-end
    auto it = nums.begin();   // Points to nums[0] (10)
    auto end = nums.end();     // Points past nums[4] (one past last)

    // Traverse using iterators
    while (it != end) {
        std::cout << *it << " ";
        ++it;
    }
    std::cout << std::endl;
    // Output: 10 20 30 40 50

    return 0;
}

Iterator Categories

Not all iterators support the same operations. The STL defines five categories, from weakest to strongest. More powerful iterators support all operations of weaker ones.

CategoryOperationsContainers
Input IteratorRead (*it), increment (++it), compare (==)std::istream_iterator
Output IteratorWrite (*it = val), increment (++it)std::ostream_iterator
Forward IteratorInput + multi-pass (can revisit)std::forward_list, std::unordered_map
Bidirectional IteratorForward + decrement (--it)std::list, std::map, std::set
Random Access IteratorBidirectional + arithmetic (it + n, it - n, it[n])std::vector, std::deque, std::array
#include <iostream>
#include <vector>
#include <list>

int main() {
    // Random access iterator: supports +, -, [], <, >
    std::vector<int> v = {10, 20, 30, 40, 50};
    auto vit = v.begin();
    vit += 3;                   // Jump 3 positions
    std::cout << *vit << std::endl;   // 40
    std::cout << vit[1] << std::endl;  // 50 (offset from current position)

    // Bidirectional iterator: supports -- but not +, -, []
    std::list<int> lst = {10, 20, 30, 40, 50};
    auto lit = lst.end();
    --lit;                      // Move back one position
    std::cout << *lit << std::endl;    // 50
    // lit + 2;  // ERROR: list iterator does not support +

    return 0;
}

Note: The iterator category determines which algorithms you can use with a given container. For example, std::sort requires random access iterators, so it works with std::vector but not with std::list. Use lst.sort() (member function) for lists instead.

begin() / end() and rbegin() / rend()

Every container provides begin()/end() for forward iteration and rbegin()/rend() for reverse iteration.

#include <iostream>
#include <vector>

int main() {
    std::vector<int> nums = {1, 2, 3, 4, 5};

    // Forward: begin() to end()
    std::cout << "Forward:  ";
    for (auto it = nums.begin(); it != nums.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << std::endl;  // Forward:  1 2 3 4 5

    // Reverse: rbegin() to rend()
    std::cout << "Reverse:  ";
    for (auto rit = nums.rbegin(); rit != nums.rend(); ++rit) {
        std::cout << *rit << " ";
    }
    std::cout << std::endl;  // Reverse:  5 4 3 2 1

    // Const iterators (cbegin/cend): read-only access
    std::cout << "Const:    ";
    for (auto cit = nums.cbegin(); cit != nums.cend(); ++cit) {
        std::cout << *cit << " ";
        // *cit = 99;  // ERROR: cannot modify through const_iterator
    }
    std::cout << std::endl;

    return 0;
}

Tip: Use cbegin()/cend() (C++11) when you only need to read elements. This prevents accidental modification and makes your intent clear. Use auto to let the compiler deduce the correct iterator type.

Using Iterators with Algorithms

STL algorithms work on half-open ranges [begin, end) — the range includes begin but excludes end. This convention is used everywhere in the STL.

#include <iostream>
#include <vector>
#include <algorithm>
#include <numeric>

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9, 3};

    // Sort: random access iterators required
    std::sort(nums.begin(), nums.end());
    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;  // 1 2 3 5 8 9

    // Reverse: bidirectional iterators sufficient
    std::reverse(nums.begin(), nums.end());
    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;  // 9 8 5 3 2 1

    // Find: input iterators sufficient
    auto it = std::find(nums.begin(), nums.end(), 5);
    if (it != nums.end()) {
        std::cout << "Found 5 at index: " << (it - nums.begin()) << std::endl;
    }

    // Count_if with lambda
    int big = std::count_if(nums.begin(), nums.end(), [](int x) {
        return x > 5;
    });
    std::cout << "Elements > 5: " << big << std::endl;  // 3

    // Accumulate
    int sum = std::accumulate(nums.begin(), nums.end(), 0);
    std::cout << "Sum: " << sum << std::endl;  // 28

    return 0;
}

Iterator Types

Containers provide both mutable and const iterator types. The auto keyword is your friend here.

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // Mutable iterator (can modify elements)
    std::vector<int>::iterator mut_it = v.begin();
    *mut_it = 100;  // OK: v[0] is now 100

    // Const iterator (read-only)
    std::vector<int>::const_iterator const_it = v.cbegin();
    std::cout << *const_it << std::endl;  // 100
    // *const_it = 99;  // ERROR: cannot modify

    // Using auto (preferred in modern C++)
    for (auto it = v.begin(); it != v.end(); ++it) {
        *it += 10;  // OK: begin() returns mutable iterator
    }

    for (auto it = v.cbegin(); it != v.cend(); ++it) {
        std::cout << *it << " ";  // Read-only
    }
    std::cout << std::endl;  // 110 12 13 14 15

    return 0;
}

Range-Based For (Under the Hood)

The range-based for loop (C++11) is syntactic sugar over iterators. These two loops are equivalent:

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {10, 20, 30};

    // Range-based for (modern, preferred)
    for (const auto& elem : v) {
        std::cout << elem << " ";
    }
    std::cout << std::endl;

    // Equivalent using iterators (what the compiler generates)
    for (auto it = v.begin(); it != v.end(); ++it) {
        const auto& elem = *it;
        std::cout << elem << " ";
    }
    std::cout << std::endl;

    return 0;
}

See also: Range-Based For for more details on range-based loops, Ranges (C++20) for composable range views, and STL Algorithms for the full set of algorithms that operate on iterator ranges.

Common Iterator Pitfalls

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5, 6};

    // PITFALL: Modifying container while iterating
    // This can invalidate iterators!
    // BAD:
    // for (auto it = v.begin(); it != v.end(); ++it) {
    //     if (*it % 2 == 0) v.erase(it);  // Undefined behavior!
    // }

    // CORRECT: erase returns iterator to next element
    for (auto it = v.begin(); it != v.end(); ) {
        if (*it % 2 == 0) {
            it = v.erase(it);  // erase returns next valid iterator
        } else {
            ++it;
        }
    }
    for (int n : v) std::cout << n << " ";
    std::cout << std::endl;  // 1 3 5

    return 0;
}

Warning: Inserting into or erasing from a std::vector can invalidate all iterators at or after the modification point (because the vector may reallocate). Always use the return value of insert() or erase() to get a valid iterator after modification. See std::vector for vector-specific iterator invalidation rules.
