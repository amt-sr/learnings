1.2.4 Polymorphism

Polymorphism means "many forms." In C++, it is the ability to use a single interface to represent different underlying behaviors. There are two main types: compile-time polymorphism (resolved at compile time) and runtime polymorphism (resolved at runtime). Both enable flexible and extensible inheritance-based designs.

Compile-Time Polymorphism

Compile-time polymorphism is resolved by the compiler before the program runs. The two primary forms are function overloading and operator overloading.

Function Overloading

Multiple functions can share the same name if they differ in their parameter types or count. The compiler selects the correct version based on the arguments provided:

#include <iostream>

class Printer {
public:
    void print(int value) const {
        std::cout << "int: " << value << std::endl;
    }

    void print(double value) const {
        std::cout << "double: " << value << std::endl;
    }

    void print(const std::string& value) const {
        std::cout << "string: " << value << std::endl;
    }

    void print(int a, int b) const {
        std::cout << "two ints: " << a << ", " << b << std::endl;
    }
};

int main() {
    Printer p;
    p.print(42);             // calls print(int)
    p.print(3.14);           // calls print(double)
    p.print("hello");        // calls print(const string&)
    p.print(10, 20);         // calls print(int, int)
    return 0;
}

Operator Overloading (Overview)

C++ allows you to redefine how operators work for user-defined types. This is covered in depth in Operator Overloading. Here is a brief example:

#include <iostream>

class Complex {
private:
    double real, imag;

public:
    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    // Overload + operator
    Complex operator+(const Complex& other) const {
        return Complex{real + other.real, imag + other.imag};
    }

    void print() const {
        std::cout << real << " + " << imag << "i" << std::endl;
    }
};

int main() {
    Complex a{3.0, 4.0};
    Complex b{1.0, 2.0};
    Complex c = a + b;  // uses overloaded operator+
    c.print();          // 4 + 6i
    return 0;
}

Runtime Polymorphism

Runtime polymorphism uses virtual functions to determine which function to call at runtime based on the actual type of the object. This is the foundation of dynamic dispatch in C++:

#include <iostream>
#include <vector>
#include <memory>

class Shape {
public:
    virtual double area() const = 0;           // pure virtual
    virtual void describe() const {
        std::cout << "A shape" << std::endl;
    }
    virtual ~Shape() = default;              // virtual destructor
};

class Rectangle : public Shape {
    double w, h;
public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
    void describe() const override {
        std::cout << "Rectangle " << w << "x" << h
                  << " (area=" << area() << ")" << std::endl;
    }
};

class Circle : public Shape {
    double r;
public:
    Circle(double r) : r(r) {}
    double area() const override { return 3.14159265 * r * r; }
    void describe() const override {
        std::cout << "Circle r=" << r
                  << " (area=" << area() << ")" << std::endl;
    }
};

int main() {
    // Base class pointers/references can point to derived objects
    Shape* s1 = new Rectangle{4.0, 5.0};
    Shape* s2 = new Circle{3.0};

    s1->describe();   // calls Rectangle::describe()
    s2->describe();   // calls Circle::describe()

    delete s1;
    delete s2;
    return 0;
}

When to Use virtual

Use virtualDo not use virtual
Base class with derived classes that need different behaviorFinal (leaf) classes with no derived types
Functions called through base class pointers or referencesInternal helper functions never called polymorphically
Interface / abstract base classesSmall, performance-critical inner loops
Destructors in base classes (always)Functions called only from within the class

Critical rule: If a class has any virtual functions, its destructor must also be virtual. Without a virtual destructor, deleting a derived object through a base pointer results in undefined behavior (the derived destructor is never called).

// WRONG: Base has virtual functions but non-virtual destructor
class Base {
public:
    virtual void foo() {}
    ~Base() {}           // NOT virtual!
};

class Derived : public Base {
public:
    ~Derived() { /* cleanup derived resources */ }
};

// DANGER: Derived destructor NOT called
Base* p = new Derived{};
delete p;  // only Base::~Base() runs; ~Derived() is skipped

vtable Basics

When a class has virtual functions, the compiler creates a vtable (virtual function table) for that class. Each object of the class contains a hidden pointer (vptr) to its class's vtable. At runtime, a virtual function call looks up the correct function address in the vtable:

ConceptDescription
vtableA static table (per class) containing function pointers for all virtual functions
vptrA hidden pointer (per object) pointing to the class's vtable
CostOne pointer per object + one vtable per polymorphic class + indirect call per virtual invocation

Deep dive: For a detailed explanation of how vtables are laid out, how multiple inheritance affects them, and performance implications, see Virtual Functions & vtable.

Combining Both Types

A well-designed C++ class hierarchy often uses both compile-time and runtime polymorphism together:

#include <iostream>
#include <string>

class Sensor {
public:
    // Runtime: derived classes override this
    virtual std::string read() const = 0;
    virtual ~Sensor() = default;

    // Compile-time: overloaded for different parameter types
    void calibrate(double offset) const {
        std::cout << "Calibrating with offset: " << offset << std::endl;
    }
    void calibrate(double offset, double scale) const {
        std::cout << "Calibrating offset: " << offset
                  << ", scale: " << scale << std::endl;
    }
};

class TemperatureSensor : public Sensor {
public:
    std::string read() const override {
        return "25.3 C";
    }
};

int main() {
    TemperatureSensor ts;
    ts.calibrate(0.5);                    // compile-time overload
    ts.calibrate(0.1, 1.2);               // compile-time overload

    Sensor* s = &ts;
    std::cout << s->read() << std::endl;  // runtime dispatch

    return 0;
}
