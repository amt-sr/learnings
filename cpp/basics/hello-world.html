1.1.1 Hello World & Compilation

Every C++ journey begins with the classic Hello World program. Understanding how to write, compile, and run a C++ program is the essential first step before diving into language features.

Basic Program Structure

A minimal C++ program consists of preprocessor directives, a main function, and statements. Here is the simplest valid C++ program:

#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

Let us break down every line:

  • #include <iostream> — A preprocessor directive that includes the Standard I/O library. This gives us access to std::cout for output and std::cin for input.
  • int main() — The entry point of every C++ program. It returns an int to the operating system: 0 means success, any non-zero value indicates an error.
  • std::cout << "Hello, World!" << std::endl; — Sends the string to standard output using the stream insertion operator (<<). std::endl flushes the buffer and inserts a newline.
  • return 0; — Signals successful program termination.

Tip: In C++ you can omit return 0; in main(). The compiler implicitly returns 0 when control reaches the end of main. This is unique to main() — all other functions that declare a return type must return a value explicitly.

Header Files

Header files provide declarations for functions, classes, and other entities. Common standard library headers include:

HeaderPurpose
<iostream>Input/output streams (cin, cout)
<string>std::string class
<vector>std::vector dynamic array
<cmath>Math functions (sqrt, pow, sin)
<cstdio>C-style I/O (printf, scanf)
<cstdlib>General utilities (malloc, exit)

Note that C++ standard headers do not use a .h suffix. Headers like <stdio.h> are C compatibility headers and place their contents in the global namespace, whereas <cstdio> places them in the std namespace. See Namespaces for more detail.

Compiling with g++

The most common compiler on Linux and macOS is GCC's g++. The basic compilation command is:

# Compile a single source file
g++ -o hello hello.cpp

# Run the compiled program
./hello

Essential Compiler Flags

FlagPurpose
-WallEnable all common warnings
-WextraEnable extra warnings beyond -Wall
-WpedanticWarn about non-standard C++
-std=c++17Use the C++17 standard
-std=c++20Use the C++20 standard
-O2Optimization level 2 (recommended for builds)
-gInclude debug symbols
-o <name>Specify the output binary name

Recommended compile command:

g++ -Wall -Wextra -std=c++17 -o hello hello.cpp

Warning: Always compile with at least -Wall. Compiler warnings catch bugs early. Ignoring warnings is a common source of undefined behavior and hard-to-find bugs in production code.

Using Namespaces

You will frequently see using namespace std; in tutorials and older codebases. This brings all names from the std namespace into the global scope:

#include <iostream>
using namespace std;  // Avoid this in large projects

int main() {
    cout << "Hello without std:: prefix" << endl;
    return 0;
}

Avoid in header files: Never put using namespace std; in a header file. It pollutes the namespace of every file that includes it, potentially causing name collisions. In implementation files ( .cpp ), it is acceptable but still discouraged in larger projects. Prefer using the std:: prefix explicitly.

Multi-file Programs

Real projects span multiple source files. A typical project structure:

project/
  main.cpp
  math_utils.h    // declarations
  math_utils.cpp  // definitions

// math_utils.h
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

int add(int a, int b);

#endif

// math_utils.cpp
#include "math_utils.h"

int add(int a, int b) {
    return a + b;
}

// main.cpp
#include <iostream>
#include "math_utils.h"

int main() {
    std::cout << "3 + 4 = " << add(3, 4) << std::endl;
    return 0;
}

Compile all files together:

g++ -Wall -std=c++17 -o program main.cpp math_utils.cpp

Tip: Use #pragma once as a modern alternative to include guards. It is supported by all major compilers and is more concise. However, #ifndef guards are standard C++ and guaranteed to work everywhere.

The main Function

The main function is special in C++. It is the only function whose declaration does not need to be visible before it is called (the compiler provides it implicitly). Valid signatures:

// Standard form
int main();

// With command-line arguments
int main(int argc, char* argv[]);

// Equivalent using C++ aliases
int main(int argc, char** argv);
  • argc — argument count (at least 1, since the program name is argv[0])
  • argv — array of C-style strings representing the arguments
#include <iostream>

int main(int argc, char* argv[]) {
    std::cout << "Program name: " << argv[0] << std::endl;
    std::cout << "Argument count: " << argc << std::endl;

    for (int i = 1; i < argc; ++i) {
        std::cout << "Arg " << i << ": " << argv[i] << std::endl;
    }

    return 0;
}
