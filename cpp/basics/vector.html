1.5.2 std::vector

std::vector is the most commonly used container in C++. It is a dynamic array that grows automatically when elements are added. It provides fast random access, cache-friendly contiguous storage, and amortized O(1) insertion at the end.

Creating Vectors

#include <iostream>
#include <vector>

int main() {
    // Empty vector
    std::vector<int> v1;

    // Vector with 5 elements, all initialized to 0
    std::vector<int> v2(5);

    // Vector with 5 elements, all initialized to 42
    std::vector<int> v3(5, 42);

    // Vector initialized with values
    std::vector<int> v4 = {1, 2, 3, 4, 5};

    // Copy constructor
    std::vector<int> v5 = v4;

    // Vector of strings
    std::vector<std::string> names = {"Alice", "Bob", "Charlie"};

    std::cout << "v3: ";
    for (int x : v3) std::cout << x << " ";
    std::cout << std::endl;
    // v3: 42 42 42 42 42

    return 0;
}

Adding Elements

#include <iostream>
#include <vector>
#include <string>

int main() {
    std::vector<int> v;

    // push_back: copy the value into the vector
    v.push_back(10);
    v.push_back(20);
    v.push_back(30);

    // emplace_back: construct in-place (more efficient for complex types)
    std::vector<std::string> words;
    words.emplace_back("hello");    // Constructs string in place
    words.emplace_back("world");

    std::cout << "v: ";
    for (int x : v) std::cout << x << " ";
    std::cout << std::endl;  // v: 10 20 30

    std::cout << "words: ";
    for (const auto& w : words) std::cout << w << " ";
    std::cout << std::endl;  // words: hello world

    return 0;
}

Tip: Prefer emplace_back over push_back for complex types. It constructs the element directly in the vector's storage, avoiding a temporary object. For primitive types like int, there is no practical difference.

Capacity Management

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;

    std::cout << "Initial: size=" << v.size()
              << ", capacity=" << v.capacity() << std::endl;
    // Initial: size=0, capacity=0

    // reserve: pre-allocate space without adding elements
    v.reserve(100);
    std::cout << "After reserve(100): size=" << v.size()
              << ", capacity=" << v.capacity() << std::endl;
    // After reserve(100): size=0, capacity=100

    // Add elements
    for (int i = 0; i < 10; ++i) {
        v.push_back(i);
    }
    std::cout << "After 10 pushes: size=" << v.size()
              << ", capacity=" << v.capacity() << std::endl;
    // After 10 pushes: size=10, capacity=100

    // resize: change the number of elements
    v.resize(5);  // Shrink to 5 elements
    std::cout << "After resize(5): size=" << v.size() << std::endl;
    // After resize(5): size=5

    v.resize(15, 99);  // Expand to 15, fill new elements with 99
    std::cout << "After resize(15, 99): size=" << v.size() << std::endl;
    // After resize(15, 99): size=15

    return 0;
}

Important: capacity() returns the amount of allocated storage, which may be larger than size(). When size() reaches capacity(), the vector reallocates (typically doubles capacity), copies all elements, and frees the old storage. Use reserve() to avoid repeated reallocations when you know the final size.

Accessing Elements

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {10, 20, 30, 40, 50};

    // operator[]: no bounds checking (fast, but unsafe)
    std::cout << "v[2] = " << v[2] << std::endl;        // 30

    // at(): bounds checking (throws std::out_of_range)
    std::cout << "v.at(0) = " << v.at(0) << std::endl;   // 10

    // front() and back()
    std::cout << "front = " << v.front() << std::endl;  // 10
    std::cout << "back = " << v.back() << std::endl;   // 50

    // data(): raw pointer to underlying array
    int* ptr = v.data();
    std::cout << "data()[3] = " << ptr[3] << std::endl;   // 40

    return 0;
}

Warning: operator[] does not perform bounds checking. Accessing v[100] on a 5-element vector is undefined behavior — it reads from memory beyond the vector's data. Use at() in debug builds or when correctness matters more than speed.

Iterating Over Vectors

#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // Method 1: Index-based (classic)
    for (size_t i = 0; i < v.size(); ++i) {
        std::cout << v[i] << " ";
    }
    std::cout << std::endl;

    // Method 2: Range-based for (C++11, preferred)
    for (int x : v) {
        std::cout << x << " ";
    }
    std::cout << std::endl;

    // Method 3: Range-based for with reference (to modify)
    for (int& x : v) {
        x *= 2;
    }
    for (const auto& x : v) std::cout << x << " ";
    std::cout << std::endl;
    // 2 4 6 8 10

    // Method 4: Iterator-based
    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << std::endl;

    return 0;
}

Guideline: Almost always prefer std::vector over C-style arrays. Vectors are safer, more convenient, and just as fast. Only use C arrays when interfacing with C APIs or in performance-critical embedded code where heap allocation is not acceptable.

Removing Elements

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5, 3, 7, 3};

    // Remove last element
    v.pop_back();
    // v: {1, 2, 3, 4, 5, 3, 7}

    // Remove element at index 1 (shifts everything after it)
    v.erase(v.begin() + 1);
    // v: {1, 3, 4, 5, 3, 7}

    // Remove all elements with value 3
    v.erase(std::remove(v.begin(), v.end(), 3), v.end());
    // v: {1, 4, 5, 7}

    for (int x : v) std::cout << x << " ";
    std::cout << std::endl;

    return 0;
}

Erase-remove idiom: std::remove does not actually remove elements — it shifts matching elements to the end and returns an iterator to the new logical end. erase then physically removes them. This is the standard pattern for removing elements by value from a vector.
