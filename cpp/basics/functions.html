1.1.5 Functions

Functions allow you to encapsulate reusable logic into named blocks. A well-designed function has a single responsibility, a clear interface, and is easy to test and debug independently.

Declaration and Definition

A function declaration (prototype) tells the compiler the function's name, return type, and parameters. A function definition provides the actual body. Declarations typically go in header files; definitions go in source files.

#include <iostream>

// Declaration (prototype)
int add(int a, int b);
double rectangle_area(double width, double height);
void greet(const std::string& name);

int main() {
    std::cout << "3 + 4 = " << add(3, 4) << std::endl;
    std::cout << "Area: " << rectangle_area(5.0, 3.0) << std::endl;
    greet("Alice");
    return 0;
}

// Definitions
int add(int a, int b) {
    return a + b;
}

double rectangle_area(double width, double height) {
    return width * height;
}

void greet(const std::string& name) {
    std::cout << "Hello, " << name << "!" << std::endl;
}

Tip: In C++, you can place the definition before main() and skip the separate declaration. In larger projects with multiple files, declarations go in .h files and definitions in .cpp files. See Hello World & Compilation for multi-file project setup.

Parameters and Return Types

Functions can take parameters by value or by reference (see References for the full discussion). They can also return values or be void.

#include <iostream>
#include <string>

// Pass by value: a copy is made (expensive for large objects)
void by_value(std::string s) {
    s += " (modified)";
    std::cout << "Inside by_value: " << s << std::endl;
}

// Pass by reference: no copy, original can be modified
void by_reference(std::string& s) {
    s += " (modified)";
    std::cout << "Inside by_reference: " << s << std::endl;
}

// Pass by const reference: no copy, original protected
void by_const_reference(const std::string& s) {
    std::cout << "Inside by_const_reference: " << s << std::endl;
    // s += " (modified)";  // ERROR: cannot modify const ref
}

int main() {
    std::string msg = "Hello";

    by_value(msg);
    std::cout << "After by_value: " << msg << std::endl;

    by_reference(msg);
    std::cout << "After by_reference: " << msg << std::endl;

    return 0;
}

Rule of thumb: Use const& for read-only parameters of any type beyond basic primitives. Use & only when the function must modify the argument. Use pass-by-value only for small types like int, double, bool.

Default Arguments

C++ allows parameters to have default values. Callers can omit those arguments and the defaults are used.

#include <iostream>

// Default arguments: must be rightmost parameters
void print_info(const std::string& name, int age = 0, const std::string& role = "user") {
    std::cout << "Name: " << name;
    std::cout << ", Age: " << age;
    std::cout << ", Role: " << role << std::endl;
}

int main() {
    print_info("Alice", 30, "admin");   // All arguments
    print_info("Bob", 25);              // Uses default role
    print_info("Charlie");              // Uses default age and role

    return 0;
}

Warning: Default arguments can only be specified in the declaration (typically in the header file), not in the definition (the .cpp file). If you specify them in both places, the compiler will error.

Function Overloading

C++ allows multiple functions with the same name but different parameter lists. The compiler selects the correct overload based on the arguments provided.

#include <iostream>

int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

int add(int a, int b, int c) {
    return a + b + c;
}

int main() {
    std::cout << add(3, 4) << std::endl;           // calls int version: 7
    std::cout << add(1.5, 2.5) << std::endl;       // calls double version: 4.0
    std::cout << add(1, 2, 3) << std::endl;         // calls 3-arg version: 6

    return 0;
}

Note: Return type alone is not sufficient to distinguish overloads. Two functions with the same name and same parameter types but different return types will cause a compile error.

Recursion

A recursive function calls itself. Every recursive function needs a base case (stopping condition) to avoid infinite recursion and stack overflow.

#include <iostream>

// Factorial: n! = n * (n-1)!
int factorial(int n) {
    // Base case
    if (n <= 1) {
        return 1;
    }
    // Recursive case
    return n * factorial(n - 1);
}

// Fibonacci (naive recursive)
int fibonacci(int n) {
    if (n <= 0) return 0;
    if (n == 1) return 1;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int main() {
    std::cout << "5! = " << factorial(5) << std::endl;       // 120
    std::cout << "10! = " << factorial(10) << std::endl;     // 3628800

    std::cout << "Fib(10) = " << fibonacci(10) << std::endl; // 55

    return 0;
}

Warning: The naive Fibonacci implementation above has exponential time complexity O(2^n) due to repeated computation. In practice, use iteration or memoization techniques. Recursion depth is also limited by stack size (typically around 1 MB, allowing ~10,000 nested calls).

Inline Functions

The inline keyword suggests to the compiler that the function body should be inserted at each call site, avoiding function call overhead.

inline int square(int x) {
    return x * x;
}

int main() {
    int result = square(5);  // Compiler may replace with: 5 * 5
    return 0;
}

Tip: Modern compilers are excellent at inlining automatically, even without the inline keyword. Use inline primarily for function definitions in header files to prevent multiple-definition linker errors. Avoid inline for large functions.

constexpr Functions (C++11)

constexpr functions can be evaluated at compile time if their arguments are constant expressions:

#include <iostream>

constexpr int square(int x) {
    return x * x;
}

constexpr int factorial(int n) {
    return (n <= 1) ? 1 : n * factorial(n - 1);
}

int main() {
    constexpr int val = square(5);  // Evaluated at compile time
    constexpr int fact10 = factorial(10);

    static_assert(val == 25, "square(5) should be 25");

    std::cout << "square(5) = " << val << std::endl;
    std::cout << "10! = " << fact10 << std::endl;

    return 0;
}
