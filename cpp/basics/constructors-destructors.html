1.2.2 Constructors & Destructors

Constructors and destructors are special member functions that manage the lifecycle of objects. A constructor initializes an object when it is created. A destructor cleans up resources when an object is destroyed. Together, they form the basis of the RAII (Resource Acquisition Is Initialization) pattern.

Default Constructor

A default constructor takes no arguments (or all arguments have defaults). It is called when an object is created without explicit initialization:

#include <iostream>

class Counter {
private:
    int count;

public:
    // Default constructor
    Counter() : count(0) {
        std::cout << "Default constructor called" << std::endl;
    }

    void increment() { ++count; }
    int get() const { return count; }
};

int main() {
    Counter c1;           // calls default constructor
    Counter c2{};         // also calls default constructor (C++11)
    Counter c3 = Counter{}; // calls default constructor

    std::cout << "c1.count = " << c1.get() << std::endl;  // 0

    return 0;
}

Note: If you define any constructor explicitly, the compiler no longer generates a default constructor automatically. If you need objects to be default-constructible, you must define the default constructor yourself or use = default.

Parameterized Constructor

A parameterized constructor accepts arguments to initialize the object with specific values:

#include <iostream>
#include <string>

class Student {
private:
    std::string name;
    double gpa;

public:
    // Parameterized constructor
    Student(const std::string& n, double g) : name(n), gpa(g) {
        std::cout << "Parameterized constructor: " << name << std::endl;
    }

    // Constructor with default arguments (serves as both default and parameterized)
    Student(const std::string& n = "Unknown", double g = 0.0)
        : name(n), gpa(g) {}

    void display() const {
        std::cout << name << " (GPA: " << gpa << ")" << std::endl;
    }
};

Warning: You cannot have both a default constructor and a constructor with all-default arguments at the same time — this would create ambiguity. Use one or the other.

Copy Constructor

The copy constructor creates a new object as a copy of an existing object. It is called when an object is passed by value, returned by value, or explicitly copied:

#include <iostream>

class Buffer {
private:
    int* data;
    int size;

public:
    // Parameterized constructor
    Buffer(int s) : size(s), data(new int[s]) {
        for (int i = 0; i < size; ++i) data[i] = 0;
        std::cout << "Constructor: allocated " << size << " ints" << std::endl;
    }

    // Copy constructor (deep copy)
    Buffer(const Buffer& other) : size(other.size), data(new int[other.size]) {
        for (int i = 0; i < size; ++i) data[i] = other.data[i];
        std::cout << "Copy constructor: deep copied " << size << " ints" << std::endl;
    }

    // Destructor
    ~Buffer() {
        delete[] data;
        std::cout << "Destructor: freed memory" << std::endl;
    }

    int& operator[](int i) { return data[i]; }
};

int main() {
    Buffer buf1{5};
    buf1[2] = 42;

    Buffer buf2 = buf1;  // calls copy constructor
    std::cout << "buf2[2] = " << buf2[2] << std::endl;  // 42

    return 0;
}

Tip: The compiler generates a default copy constructor that performs a shallow copy (member-wise copy). If your class manages resources (raw pointers, file handles, network sockets), you must write a custom copy constructor to perform a deep copy. See Copy Elision & RVO for how the compiler optimizes copies away.

Destructor

The destructor is called automatically when an object goes out of scope or is deleted. It has no parameters and no return type. Its name is prefixed with a tilde ~:

class Logger {
private:
    std::string filename;

public:
    Logger(const std::string& fname) : filename(fname) {
        std::cout << "Opening log file: " << filename << std::endl;
    }

    ~Logger() {
        std::cout << "Closing log file: " << filename << std::endl;
        // Clean up resources here (close files, release memory, etc.)
    }

    void log(const std::string& message) const {
        std::cout << "[" << filename << "] " << message << std::endl;
    }
};

void process() {
    Logger logger{"app.log"};
    logger.log("Processing started");
    // logger's destructor called automatically here
}

int main() {
    process();
    std::cout << "Back in main" << std::endl;
    return 0;
}
// Output:
// Opening log file: app.log
// [app.log] Processing started
// Closing log file: app.log
// Back in main

Constructor Initialization List

Member initialization lists are the preferred way to initialize members in a constructor. They are more efficient than assignment in the constructor body because members are initialized directly rather than being default-constructed and then assigned:

class Vector3D {
private:
    double x, y, z;

public:
    // Preferred: initialization list
    Vector3D(double x, double y, double z) : x(x), y(y), z(z) {}

    // Also works but less efficient for class-type members
    // Vector3D(double x, double y, double z) {
    //     this->x = x;  // members default-constructed first, then assigned
    //     this->y = y;
    //     this->z = z;
    // }

    void print() const {
        std::cout << "(" << x << ", " << y << ", " << z << ")" << std::endl;
    }
};

Required: const members and reference members must be initialized in the initialization list. They cannot be assigned in the constructor body because they must be initialized at the point of creation.

class Required {
private:
    const int id;          // must be in init list
    int& ref;              // must be in init list
    std::string name;

public:
    Required(int i, int& r, const std::string& n)
        : id(i), ref(r), name(n) {}  // initialization list is required
};

Order of Construction and Destruction

Members are initialized in the order they are declared in the class, not the order they appear in the initialization list. Destruction happens in the reverse order:

#include <iostream>

class A {
public:
    A()  { std::cout << "A constructed" << std::endl; }
    ~A() { std::cout << "A destroyed" << std::endl; }
};

class B {
public:
    B()  { std::cout << "B constructed" << std::endl; }
    ~B() { std::cout << "B destroyed" << std::endl; }
};

class Container {
private:
    A a;  // constructed first (declared first)
    B b;  // constructed second (declared second)

public:
    Container() {
        std::cout << "Container constructed" << std::endl;
    }
    ~Container() {
        std::cout << "Container destroyed" << std::endl;
    }
};

int main() {
    Container c;
    return 0;
}
// Output:
// A constructed
// B constructed
// Container constructed
// Container destroyed
// B destroyed       (reverse order)
// A destroyed

Warning: Always match your initialization list order to the declaration order to avoid confusion. Compilers will warn if the order differs (enable with -Wall).
