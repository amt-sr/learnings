1.5.3 std::map & std::unordered_map

Maps are associative containers that store key-value pairs. std::map keeps elements sorted by key (using a red-black tree), while std::unordered_map uses a hash table for average O(1) access. Choosing the right one depends on whether you need ordering, peak lookup speed, or predictable worst-case performance.

std::map (Ordered Map)

std::map is implemented as a balanced binary search tree (typically a red-black tree). Elements are always sorted by key, and lookup, insertion, and deletion are O(log n).

#include <iostream>
#include <map>
#include <string>

int main() {
    // Create an ordered map (string keys, int values)
    std::map<std::string, int> ages;

    // Insert elements
    ages["Alice"] = 30;
    ages["Bob"] = 25;
    ages["Charlie"] = 35;

    // Insert using emplace (more efficient)
    ages.emplace("Diana", 28);

    // Insert using insert (won't overwrite existing key)
    ages.insert({"Eve", 22});

    // Accessing values
    std::cout << "Alice's age: " << ages["Alice"] << std::endl;  // 30
    std::cout << "Bob's age:   " << ages.at("Bob") << std::endl;    // 25

    // Iterate -- elements are in alphabetical order by key
    std::cout << "\nAll entries (sorted by key):" << std::endl;
    for (const auto& [name, age] : ages) {
        std::cout << "  " << name << ": " << age << std::endl;
    }

    return 0;
}

std::unordered_map (Hash Map)

std::unordered_map is implemented as a hash table. It provides average O(1) lookup, insertion, and deletion, but elements have no defined order.

#include <iostream>
#include <unordered_map>
#include <string>

int main() {
    std::unordered_map<std::string, int> scores;

    scores["Team A"] = 95;
    scores["Team B"] = 82;
    scores["Team C"] = 91;
    scores["Team D"] = 78;

    // Average O(1) lookup
    std::cout << "Team A score: " << scores["Team A"] << std::endl;

    // Check if key exists
    std::string key = "Team E";
    if (scores.find(key) != scores.end()) {
        std::cout << key << " exists" << std::endl;
    } else {
        std::cout << key << " not found" << std::endl;
    }

    // Iterate -- order is undefined
    std::cout << "\nAll entries:" << std::endl;
    for (const auto& [team, score] : scores) {
        std::cout << "  " << team << ": " << score << std::endl;
    }

    // Size and erase
    std::cout << "Size: " << scores.size() << std::endl;  // 4
    scores.erase("Team B");
    std::cout << "After erase: " << scores.size() << std::endl;  // 3

    return 0;
}

Checking for Key Existence

#include <iostream>
#include <unordered_map>
#include <string>

int main() {
    std::unordered_map<std::string, int> dict = {
        {"hello", 1}, {"world", 2}
    };

    std::string word = "hello";

    // Method 1: find() -- idiomatic, safe
    auto it = dict.find(word);
    if (it != dict.end()) {
        std::cout << it->first << ": " << it->second << std::endl;
    }

    // Method 2: count() -- simple boolean check
    if (dict.count(word) > 0) {
        std::cout << "Found: " << dict[word] << std::endl;
    }

    // Method 3: contains() -- C++20, most readable
    if (dict.contains(word)) {
        std::cout << "Contains: " << dict[word] << std::endl;
    }

    // Method 4: [] -- DANGEROUS for checking! Creates entry if missing
    // if (dict["missing"] > 0) { ... }
    // This inserts "missing" with value 0 into the map!

    return 0;
}

Warning: Using m[key] to check for existence is a common bug. If the key does not exist, operator[] inserts it with a default value (0 for integers, empty string for strings). This silently grows your map. Use find(), count(), or contains() instead.

Practical Example: Word Frequency Counter

#include <iostream>
#include <unordered_map>
#include <string>
#include <sstream>

int main() {
    std::string text = "the cat sat on the mat the cat";

    std::unordered_map<std::string, int> freq;
    std::istringstream iss(text);
    std::string word;

    while (iss >> word) {
        freq[word]++;  // [] returns reference, increment in place
    }

    std::cout << "Word frequencies:" << std::endl;
    for (const auto& [w, count] : freq) {
        std::cout << "  " << w << ": " << count << std::endl;
    }

    return 0;
}
// Output (order may vary):
//   the: 3
//   cat: 2
//   sat: 1
//   on: 1
//   mat: 1

When to Use Which

Default choice: Use std::unordered_map unless you have a specific reason to use std::map. Unordered maps are faster for lookups and use less memory in most cases.

Use std::map when:

  • You need elements iterated in sorted key order.
  • You frequently need range queries (e.g., "all keys between A and M").
  • Your key type does not have a good hash function but supports operator<.
  • You need guaranteed O(log n) worst-case performance (no hash collisions).

Use std::unordered_map when:

  • You primarily need fast point lookups by key.
  • Iteration order does not matter.
  • Your key type has a good hash function (built-in types, std::string).
  • Performance is critical and average O(1) is sufficient.
