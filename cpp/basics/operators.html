1.1.3 Operators

Operators are symbols that tell the compiler to perform specific mathematical, relational, or logical operations. Understanding all operator categories is essential before moving into Control Flow where you will use them in conditions and loops.

Arithmetic Operators

OperatorDescriptionExample
+Additiona + b
-Subtractiona - b
*Multiplicationa * b
/Divisiona / b
%Modulo (remainder)a % b
++Incrementa++ or ++a
--Decrementa-- or --a
#include <iostream>

int main() {
    int a = 17, b = 5;

    std::cout << "a + b = " << a + b << std::endl;  // 22
    std::cout << "a - b = " << a - b << std::endl;  // 12
    std::cout << "a * b = " << a * b << std::endl;  // 85
    std::cout << "a / b = " << a / b << std::endl;  // 3 (integer division!)
    std::cout << "a % b = " << a % b << std::endl;  // 2

    // Floating-point division
    std::cout << "(double)a / b = " << (double)a / b << std::endl;  // 3.4

    return 0;
}

Warning: Integer division truncates toward zero. 17 / 5 yields 3, not 3.4. -17 / 5 yields -3, not -4. To get a floating-point result, at least one operand must be a floating-point type. Cast one operand or use a double literal.

Pre-increment vs Post-increment

#include <iostream>

int main() {
    int x = 5;

    // Post-increment: use current value, then increment
    int a = x++;  // a = 5, x becomes 6
    std::cout << "a = " << a << ", x = " << x << std::endl;

    // Pre-increment: increment first, then use new value
    int b = ++x;  // x becomes 7, b = 7
    std::cout << "b = " << b << ", x = " << x << std::endl;

    return 0;
}

Tip: In standalone statements like x++; or ++x;, there is no difference. Prefer ++x when it does not matter, because pre-increment avoids creating a temporary copy (relevant for iterator types in advanced STL usage).

Warning: Do not confuse = (assignment) with == (comparison). Writing if (x = 5) assigns 5 to x and evaluates to true (non-zero). Some compilers warn about this with -Wall, but it is a frequent source of bugs.

Logical Operators

OperatorDescriptionExample
&&Logical ANDtrue && false → false
||Logical ORtrue || false → true
!Logical NOT!true → false
#include <iostream>

int main() {
    int age = 25;
    bool has_id = true;

    // Short-circuit evaluation: && stops at first false
    // || stops at first true
    if (age >= 18 && has_id) {
        std::cout << "Access granted" << std::endl;
    }

    if (age < 13 || age > 65) {
        std::cout << "Discount eligible" << std::endl;
    }

    // Negation
    if (!has_id) {
        std::cout << "No ID provided" << std::endl;
    }

    return 0;
}

Tip: C++ uses short-circuit evaluation. In a && b, if a is false, b is never evaluated. In a || b, if a is true, b is never evaluated. This is useful for null checks: ptr && ptr->method() will not dereference a null pointer.

Bitwise Operators

OperatorDescriptionExample
&Bitwise AND5 & 3 → 1
|Bitwise OR5 | 3 → 7
^Bitwise XOR5 ^ 3 → 6
~Bitwise NOT (complement)~5
<<Left shift1 << 4 → 16
>>Right shift16 >> 2 → 4
#include <iostream>

int main() {
    unsigned int flags = 0;

    // Using bitwise ops to set/check flags
    const unsigned int READ    = 1 << 0;  // 1
    const unsigned int WRITE   = 1 << 1;  // 2
    const unsigned int EXECUTE = 1 << 2;  // 4

    // Set flags
    flags = READ | WRITE;         // flags = 3 (0b011)

    // Check a flag
    if (flags & READ) {
        std::cout << "Read permission set" << std::endl;
    }

    if (flags & EXECUTE) {
        std::cout << "Execute permission set" << std::endl;
    } else {
        std::cout << "Execute permission NOT set" << std::endl;
    }

    // Toggle a flag
    flags = flags ^ WRITE;        // Remove WRITE (flags = 1)
    std::cout << "After toggle: " << flags << std::endl;

    return 0;
}

Note: Bitwise operators operate on the binary representation of values. They are essential for systems programming, embedded development, and working with flags. The shift operators << and >> shift bits left or right. Left-shifting by n is equivalent to multiplying by 2^n; right-shifting is equivalent to dividing by 2^n (for unsigned types).

Ternary Operator

The ternary operator ?: is a compact way to write simple if-else expressions:

#include <iostream>

int main() {
    int score = 85;

    // Ternary: condition ? value_if_true : value_if_false
    std::string result = (score >= 60) ? "Pass" : "Fail";
    std::cout << result << std::endl;

    // Can be nested (but avoid deep nesting for readability)
    char grade = (score >= 90) ? 'A'
               : (score >= 80) ? 'B'
               : (score >= 70) ? 'C'
               : (score >= 60) ? 'D'
               : 'F';
    std::cout << "Grade: " << grade << std::endl;

    return 0;
}

Warning: Do not nest ternary operators deeply. For complex logic, use if/else statements which are more readable. See Control Flow for details.

Operator Precedence

When multiple operators appear in an expression, precedence determines the order of evaluation. Here is a simplified list (highest to lowest):

  1. Postfix: ++, --, (), []
  2. Prefix: ++, --, !, ~
  3. Multiplicative: *, /, %
  4. Additive: +, -
  5. Shift: <<, >>
  6. Relational: <, >, <=, >=
  7. Equality: ==, !=
  8. Bitwise AND: &
  9. Bitwise XOR: ^
  10. Bitwise OR: |
  11. Logical AND: &&
  12. Logical OR: ||
  13. Assignment: =, +=, -=, etc.

Tip: When in doubt, use parentheses to make the intent clear. (a + b) * c is unambiguous, whereas a + b * c relies on the reader knowing precedence. Readability always wins.
