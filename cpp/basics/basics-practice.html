Basics Practice: Build a Student Management System

Topics Covered: Variables & Data Types, Control Flow, Functions, Arrays & Strings, Pointers, References, Classes & Objects, Constructors & Destructors, Inheritance, Polymorphism, Encapsulation & Abstraction, Function Templates, Class Templates, std::vector, std::map / std::unordered_map, std::string, Iterators

Problem Statement

Build a StudentManager class hierarchy with CRUD operations, search, sorting, and a template-based report generator. This project exercises nearly every core C++ concept: classes, inheritance, polymorphism, templates, STL containers, algorithms, and control flow.

Requirements:

  • A Student base class with encapsulated fields and proper constructors.
  • A StudentManager that stores students in a std::vector and supports add, remove, update, and search operations.
  • Derived classes GraduateStudent and UndergradStudent with polymorphic display.
  • A template-based report generator that works with any container of student pointers.
  • A simple CLI menu in main() for user interaction.

Guidelines:

  • Use proper encapsulation: make data members private and expose via getters/setters.
  • Prefer std::string over C-style strings for names and titles.
  • Use std::vector as the primary storage container.
  • Use std::sort with custom comparators for sorting.
  • Apply the virtual keyword for polymorphic behavior.

Step 1: Define the Student Class

1. Define the Student class

Create a Student class with private members for name (string), id (int), age (int), grade (string), and gpa (double). Provide a parameterized constructor, a default constructor, getters, setters, and a virtual display() method for polymorphism.

Click to reveal Solution

#include <iostream>
#include <string>

class Student {
private:
    std::string name;
    int id;
    int age;
    std::string grade;
    double gpa;

public:
    // Default constructor
    Student()
        : name(""), id(0), age(0), grade(""), gpa(0.0) {}

    // Parameterized constructor
    Student(const std::string& name, int id, int age,
            const std::string& grade, double gpa)
        : name(name), id(id), age(age), grade(grade), gpa(gpa) {}

    // Getters
    std::string getName()   const { return name; }
    int         getId()     const { return id; }
    int         getAge()    const { return age; }
    std::string getGrade()  const { return grade; }
    double      getGpa()    const { return gpa; }

    // Setters
    void setName(const std::string& n)  { name = n; }
    void setId(int i)                     { id = i; }
    void setAge(int a)                   { age = a; }
    void setGrade(const std::string& g) { grade = g; }
    void setGpa(double g)                { gpa = g; }

    // Virtual display for polymorphism
    virtual void display() const {
        std::cout << "Student: " << name
                  << " | ID: " << id
                  << " | Age: " << age
                  << " | Grade: " << grade
                  << " | GPA: " << gpa << "\n";
    }

    // Virtual destructor for proper cleanup in inheritance
    virtual ~Student() = default;
};

Explanation:

  • All data members are private, enforcing encapsulation.
  • The constructor uses a member initializer list for efficient initialization.
  • Getters are marked const since they do not modify the object.
  • display() is virtual so derived classes can override it.
  • The virtual destructor ensures correct cleanup when deleting through a base pointer.

Complexity Analysis:

  • Time: O(1) for all getters/setters; O(L) for string operations where L is string length.
  • Space: O(L) per Student object for the stored strings.

Step 2: Implement the StudentManager with Vector Storage

2. Implement the StudentManager with vector storage

Use std::vector<Student*> to store student pointers. Implement addStudent(), removeStudent() (by ID), updateStudent() (by ID), and displayAll() methods.

Click to reveal Solution

#include <iostream>
#include <string>
#include <vector>
#include <algorithm>

class StudentManager {
private:
    std::vector<Student*> students;

    // Helper: find index of student by ID (-1 if not found)
    int findIndex(int id) const {
        for (size_t i = 0; i < students.size(); ++i) {
            if (students[i]->getId() == id) {
                return static_cast<int>(i);
            }
        }
        return -1;
    }

public:
    StudentManager() = default;

    ~StudentManager() {
        for (auto* s : students) {
            delete s;
        }
    }

    // Prevent copying (owns raw pointers)
    StudentManager(const StudentManager&) = delete;
    StudentManager& operator=(const StudentManager&) = delete;

    void addStudent(Student* student) {
        students.push_back(student);
        std::cout << "Added: ";
        student->display();
    }

    bool removeStudent(int id) {
        int idx = findIndex(id);
        if (idx == -1) {
            std::cout << "Student with ID " << id
                      << " not found.\n";
            return false;
        }
        delete students[idx];
        students.erase(students.begin() + idx);
        std::cout << "Removed student with ID " << id << ".\n";
        return true;
    }

    bool updateStudent(int id, const std::string& newName,
                       int newAge, const std::string& newGrade,
                       double newGpa) {
        int idx = findIndex(id);
        if (idx == -1) {
            std::cout << "Student with ID " << id
                      << " not found.\n";
            return false;
        }
        students[idx]->setName(newName);
        students[idx]->setAge(newAge);
        students[idx]->setGrade(newGrade);
        students[idx]->setGpa(newGpa);
        std::cout << "Updated: ";
        students[idx]->display();
        return true;
    }

    void displayAll() const {
        if (students.empty()) {
            std::cout << "No students in the system.\n";
            return;
        }
        std::cout << "=== All Students (" << students.size()
                  << ") ===\n";
        for (const auto* s : students) {
            s->display();
        }
    }

    Student* getStudent(int id) const {
        int idx = findIndex(id);
        return (idx != -1) ? students[idx] : nullptr;
    }
};

Explanation:

  • std::vector<Student*> stores pointers so polymorphism works correctly with derived classes.
  • The destructor deletes each student to prevent memory leaks.
  • Copy constructor and assignment operator are deleted because raw pointer ownership is not safely copyable.
  • findIndex() is a private helper used by multiple methods to locate a student by ID.
  • erase() takes an iterator obtained from begin() + idx.

Complexity Analysis:

  • Add: O(1) amortized (vector push_back).
  • Remove: O(n) — search is O(n), erase shifts elements.
  • Update / Search: O(n) — linear scan for the ID.
  • Display All: O(n) — visits each student once.

Step 3: Add Search Functionality

3. Add search functionality

Implement three search methods in StudentManager: search by name (partial/substring match), search by exact ID, and search by GPA range (min, max). Return matching results as a vector of pointers.

Click to reveal Solution

#include <vector>
#include <string>
#include <algorithm>
#include <iostream>

// Add these methods to StudentManager:

// Search by name (case-insensitive partial match)
std::vector<Student*> searchByName(const std::string& query) const {
    std::vector<Student*> results;
    std::string lowerQuery = query;
    std::transform(lowerQuery.begin(), lowerQuery.end(),
                   lowerQuery.begin(), ::tolower);

    for (auto* s : students) {
        std::string lowerName = s->getName();
        std::transform(lowerName.begin(), lowerName.end(),
                       lowerName.begin(), ::tolower);
        if (lowerName.find(lowerQuery) != std::string::npos) {
            results.push_back(s);
        }
    }
    return results;
}

// Search by exact ID
Student* searchById(int id) const {
    int idx = findIndex(id);
    if (idx != -1) {
        std::cout << "Found: ";
        students[idx]->display();
        return students[idx];
    }
    std::cout << "No student with ID " << id << ".\n";
    return nullptr;
}

// Search by GPA range
std::vector<Student*> searchByGpaRange(double minGpa, double maxGpa) const {
    std::vector<Student*> results;
    for (auto* s : students) {
        if (s->getGpa() >= minGpa && s->getGpa() <= maxGpa) {
            results.push_back(s);
        }
    }
    return results;
}

// Helper to display search results
void displayResults(const std::vector<Student*>& results) const {
    if (results.empty()) {
        std::cout << "No matching students found.\n";
        return;
    }
    std::cout << "Found " << results.size() << " student(s):\n";
    for (const auto* s : results) {
        s->display();
    }
}

Explanation:

  • searchByName converts both the query and the student name to lowercase for case-insensitive partial matching using std::string::find().
  • searchById reuses the findIndex() helper for O(n) lookup.
  • searchByGpaRange checks each student's GPA against the provided range.
  • Search methods return vectors of pointers (not copies) for efficiency and polymorphic access.

Complexity Analysis:

  • All search methods: O(n * L) where n is the number of students and L is string length for name searches. GPA and ID searches are O(n).
  • Space: O(k) for the results vector, where k is the number of matches.

Step 4: Implement Sorting

4. Implement sorting

Add sorting methods to StudentManager that sort students by GPA (descending), by name (alphabetical), or by ID (ascending). Use std::sort with lambda-based custom comparators.

Click to reveal Solution

#include <algorithm>

// Add these methods to StudentManager:

void sortByGpa() {
    std::sort(students.begin(), students.end(),
        [](const Student* a, const Student* b) {
            return a->getGpa() > b->getGpa();
        });
    std::cout << "Students sorted by GPA (descending).\n";
}

void sortByName() {
    std::sort(students.begin(), students.end(),
        [](const Student* a, const Student* b) {
            return a->getName() < b->getName();
        });
    std::cout << "Students sorted by name (ascending).\n";
}

void sortById() {
    std::sort(students.begin(), students.end(),
        [](const Student* a, const Student* b) {
            return a->getId() < b->getId();
        });
    std::cout << "Students sorted by ID (ascending).\n";
}

Explanation:

  • std::sort takes a pair of iterators and an optional comparator.
  • Lambda expressions [...](...){ ... } provide concise inline comparators.
  • Each comparator returns true when the first argument should come before the second.
  • GPA sort uses > for descending order; name and ID use < for ascending.
  • Sorting operates on the pointer vector, so the actual Student objects are not copied.

Complexity Analysis:

  • Time: O(n log n) — std::sort is typically introsort (quicksort + heapsort).
  • Space: O(log n) — recursion depth for quicksort partitioning.

Warning: Since we store pointers, sorting does not move the actual Student objects — it only rearranges the pointers in the vector. This is efficient but means the "order" is a property of the vector, not the objects themselves.

Step 5: Add Inheritance — GraduateStudent and UndergradStudent

5. Add inheritance: GraduateStudent and UndergradStudent

Derive GraduateStudent and UndergradStudent from Student. Add thesisTitle (string) for graduate students and major (string) for undergraduates. Override the display() method to show the extra field. Demonstrate polymorphism by calling display() through base-class pointers.

Click to reveal Solution

class GraduateStudent : public Student {
private:
    std::string thesisTitle;

public:
    GraduateStudent(const std::string& name, int id, int age,
                     const std::string& grade, double gpa,
                     const std::string& thesisTitle)
        : Student(name, id, age, grade, gpa),
          thesisTitle(thesisTitle) {}

    std::string getThesisTitle() const { return thesisTitle; }
    void setThesisTitle(const std::string& t) { thesisTitle = t; }

    void display() const override {
        std::cout << "[Grad] " << getName()
                  << " | ID: " << getId()
                  << " | Age: " << getAge()
                  << " | Grade: " << getGrade()
                  << " | GPA: " << getGpa()
                  << " | Thesis: " << thesisTitle << "\n";
    }
};

class UndergradStudent : public Student {
private:
    std::string major;

public:
    UndergradStudent(const std::string& name, int id, int age,
                     const std::string& grade, double gpa,
                     const std::string& major)
        : Student(name, id, age, grade, gpa),
          major(major) {}

    std::string getMajor() const { return major; }
    void setMajor(const std::string& m) { major = m; }

    void display() const override {
        std::cout << "[Ugrad] " << getName()
                   << " | ID: " << getId()
                   << " | Age: " << getAge()
                   << " | Grade: " << getGrade()
                   << " | GPA: " << getGpa()
                   << " | Major: " << major << "\n";
    }
};

// Demonstration of polymorphism
void demonstratePolymorphism() {
    StudentManager manager;

    manager.addStudent(new UndergradStudent("Alice", 101, 20, "A", 3.8, "Computer Science"));
    manager.addStudent(new GraduateStudent("Bob", 202, 25, "A-", 3.9, "Machine Learning"));
    manager.addStudent(new UndergradStudent("Charlie", 103, 22, "B+", 3.2, "Physics"));

    std::cout << "\n--- Polymorphic Display ---\n";
    manager.displayAll();
    // Each student's own display() is called, even through Student*
}

Explanation:

  • Both derived classes publicly inherit from Student, gaining all base-class functionality.
  • The derived constructors pass parameters to the base constructor via the initializer list.
  • display() is marked override to explicitly indicate it overrides a virtual function. The compiler will error if the signature does not match.
  • When StudentManager calls s->display() through a Student*, the correct derived version is invoked at runtime (dynamic dispatch).
  • Each derived class prefixes its output with [Grad] or [Ugrad] for easy identification.

Complexity Analysis:

  • Time: O(1) for construction. Polymorphic dispatch adds a small constant-time overhead (vtable lookup) per call.
  • Space: O(L) extra bytes per derived object for the additional string member.

Step 6: Template-Based Report Generator

6. Template-based report generator

Create a template function generateReport() that accepts any container of Student* pointers (vector, list, deque, etc.) and prints a formatted report. The template should also accept a title string and use a formatting lambda for custom output per container.

Click to reveal Solution

#include <iostream>
#include <string>
#include <vector>
#include <list>
#include <deque>
#include <iomanip>

// Template report generator: works with any container of Student*
template <typename Container>
void generateReport(const std::string& title, const Container& students) {
    std::cout << "\n========================================\n";
    std::cout << "  " << title << "\n";
    std::cout << "========================================\n";
    std::cout << std::left
              << std::setw(5)  << "ID"
              << std::setw(20) << "Name"
              << std::setw(8) << "Age"
              << std::setw(8) << "GPA"
              << "Grade\n";
    std::cout << "----------------------------------------\n";

    for (const auto* student : students) {
        std::cout << std::left
                  << std::setw(5)  << student->getId()
                  << std::setw(20) << student->getName()
                  << std::setw(8)  << student->getAge()
                  << std::setw(8)  << std::fixed << std::setprecision(2)
                                        << student->getGpa()
                  << student->getGrade() << "\n";
    }

    std::cout << "========================================\n";
    std::cout << "  Total students: " << students.size() << "\n";
    std::cout << "========================================\n";
}

// Overload that accepts a filter predicate
template <typename Container, typename Predicate>
void generateFilteredReport(const std::string& title,
                            const Container& students,
                            Predicate filter) {
    std::cout << "\n========================================\n";
    std::cout << "  " << title << " (filtered)\n";
    std::cout << "========================================\n";

    int count = 0;
    for (const auto* student : students) {
        if (filter(student)) {
            student->display();
            ++count;
        }
    }

    if (count == 0) {
        std::cout << "  No students match the filter.\n";
    }
    std::cout << "----------------------------------------\n";
    std::cout << "  Matched: " << count << " / "
              << students.size() << "\n";
    std::cout << "========================================\n";
}

// Usage example
void demonstrateReports() {
    std::vector<Student*> allStudents;
    allStudents.push_back(new UndergradStudent("Alice", 101, 20, "A", 3.8, "CS"));
    allStudents.push_back(new GraduateStudent("Bob", 202, 25, "A-", 3.9, "ML"));
    allStudents.push_back(new UndergradStudent("Charlie", 103, 22, "B+", 3.2, "Physics"));

    // Works with vector
    generateReport("All Students Report", allStudents);

    // Works with std::list (copy pointers for demo)
    std::list<Student*> studentList(allStudents.begin(), allStudents.end());
    generateReport("List-Based Report", studentList);

    // Filtered report: only GPA >= 3.5
    generateFilteredReport("High GPA Students", allStudents,
        [](const Student* s) { return s->getGpa() >= 3.5; });

    // Cleanup
    for (auto* s : allStudents) delete s;
}

Explanation:

  • The template function generateReport accepts any container type that supports range-based for iteration.
  • std::setw and std::left from <iomanip> align columns in the report output.
  • generateFilteredReport is a second template that accepts a predicate (lambda/function) to filter students.
  • Both templates work with std::vector, std::list, std::deque, or any custom container with begin()/end().

Complexity Analysis:

  • Time: O(n) to iterate and print all students. Filtered version is also O(n).
  • Space: O(1) extra space (output only). The template itself has zero runtime overhead due to compile-time instantiation.

Step 7: Add a Simple CLI Menu

7. Add a simple CLI menu

Create a main() function with a loop-based text menu that lets the user interact with the StudentManager. Support options for adding students (both undergrad and grad), displaying all, searching, sorting, and exiting.

Click to reveal Solution

#include <iostream>
#include <string>
#include <limits>

void printMenu() {
    std::cout << "\n========= Student Management System =========\n";
    std::cout << "  1. Add Undergraduate Student\n";
    std::cout << "  2. Add Graduate Student\n";
    std::cout << "  3. Display All Students\n";
    std::cout << "  4. Search by Name\n";
    std::cout << "  5. Search by ID\n";
    std::cout << "  6. Search by GPA Range\n";
    std::cout << "  7. Sort by GPA\n";
    std::cout << "  8. Sort by Name\n";
    std::cout << "  9. Sort by ID\n";
    std::cout << "  10. Remove Student\n";
    std::cout << "  0. Exit\n";
    std::cout << "============================================\n";
    std::cout << "Enter choice: ";
}

int main() {
    StudentManager manager;

    while (true) {
        printMenu();
        int choice;
        if (!(std::cin >> choice)) {
            std::cin.clear();
            std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
            std::cout << "Invalid input. Please enter a number.\n";
            continue;
        }

        if (choice == 0) {
            std::cout << "Goodbye!\n";
            break;
        }

        switch (choice) {
        case 1: {
            std::string name, grade, major;
            int id, age;
            double gpa;
            std::cout << "Name: "; std::cin.ignore();
            std::getline(std::cin, name);
            std::cout << "ID: "; std::cin >> id;
            std::cout << "Age: "; std::cin >> age;
            std::cout << "Grade: "; std::cin >> grade;
            std::cout << "GPA: "; std::cin >> gpa;
            std::cout << "Major: "; std::cin.ignore();
            std::getline(std::cin, major);
            manager.addStudent(
                new UndergradStudent(name, id, age, grade, gpa, major));
            break;
        }
        case 2: {
            std::string name, grade, thesis;
            int id, age;
            double gpa;
            std::cout << "Name: "; std::cin.ignore();
            std::getline(std::cin, name);
            std::cout << "ID: "; std::cin >> id;
            std::cout << "Age: "; std::cin >> age;
            std::cout << "Grade: "; std::cin >> grade;
            std::cout << "GPA: "; std::cin >> gpa;
            std::cout << "Thesis Title: "; std::cin.ignore();
            std::getline(std::cin, thesis);
            manager.addStudent(
                new GraduateStudent(name, id, age, grade, gpa, thesis));
            break;
        }
        case 3:
            manager.displayAll();
            break;
        case 4: {
            std::string query;
            std::cout << "Search name: "; std::cin.ignore();
            std::getline(std::cin, query);
            manager.displayResults(manager.searchByName(query));
            break;
        }
        case 5: {
            int id;
            std::cout << "Student ID: "; std::cin >> id;
            manager.searchById(id);
            break;
        }
        case 6: {
            double minGpa, maxGpa;
            std::cout << "Min GPA: "; std::cin >> minGpa;
            std::cout << "Max GPA: "; std::cin >> maxGpa;
            manager.displayResults(
                manager.searchByGpaRange(minGpa, maxGpa));
            break;
        }
        case 7:
            manager.sortByGpa();
            manager.displayAll();
            break;
        case 8:
            manager.sortByName();
            manager.displayAll();
            break;
        case 9:
            manager.sortById();
            manager.displayAll();
            break;
        case 10: {
            int id;
            std::cout << "Student ID to remove: "; std::cin >> id;
            manager.removeStudent(id);
            break;
        }
        default:
            std::cout << "Invalid choice. Try again.\n";
        }
    }

    return 0;
}

Explanation:

  • The while (true) loop keeps the program running until the user selects exit (0).
  • std::cin.ignore() before std::getline() clears the newline left in the buffer by >>.
  • Invalid non-numeric input is handled by clearing std::cin's error state and ignoring the rest of the line.
  • new UndergradStudent(...) and new GraduateStudent(...) create derived objects stored as Student* — polymorphism in action.
  • The switch statement dispatches each menu option to the corresponding StudentManager method.

Production Note: In a real application you would use std::unique_ptr<Student> or std::shared_ptr<Student> instead of raw pointers to guarantee memory safety. This exercise uses raw pointers to keep the focus on core OOP concepts. You can refactor to smart pointers as an advanced follow-up.

Follow-up Questions

Variations to Try

  1. Add persistence: Save and load the student database to/from a text file using <fstream>. Implement saveToFile() and loadFromFile() methods. How do you serialize the derived class type so you know whether to reconstruct a GraduateStudent or UndergradStudent?
  2. Replace raw pointers with smart pointers: Refactor StudentManager to use std::unique_ptr<Student> or std::shared_ptr<Student>. How does this simplify the destructor, copy semantics, and exception safety?
  3. Add an academic record history: Each student tracks a std::map<std::string, double> (course name to grade) and can compute the GPA dynamically. Change gpa from a stored field to a computed property.
  4. Build a full-featured comparator system: Allow the user to sort by any field in ascending or descending order by composing comparators at runtime using a SortField enum and a SortOrder enum.

Key Takeaways:

  • Encapsulation: Private data members with public getters/setters protect invariants and make the class safe to refactor. Always prefer private access unless there is a compelling reason otherwise.
  • Inheritance + Polymorphism: Deriving GraduateStudent and UndergradStudent from Student lets the manager store heterogeneous objects uniformly. The virtual keyword and override specifier ensure correct dynamic dispatch.
  • Templates: The report generator works with any container type, demonstrating generic programming. Templates are compiled once per type but have zero runtime overhead.
  • STL Algorithms: std::sort with lambda comparators is idiomatic C++. Avoid writing your own sort unless you need a specific algorithm.
  • RAII and Ownership: When storing raw pointers in a container, the owning class must delete them in its destructor. In modern C++, prefer smart pointers to automate this.
  • CLI Design: A simple loop-and-switch menu is the starting point for any console application. Robust input handling (clearing error state, ignoring buffer) prevents crashes from bad user input.
