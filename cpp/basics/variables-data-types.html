1.1.2 Variables & Data Types

Variables are named storage locations in memory. C++ is a statically typed language, meaning the type of every variable must be known at compile time. This allows the compiler to allocate the correct amount of memory and enforce type safety.

Declaring and Initializing Variables

#include <iostream>

int main() {
    // Declaration + initialization (preferred style)
    int age = 25;
    double temperature = 98.6;
    char grade = 'A';
    bool is_valid = true;

    // C++11 uniform initialization
    int count{0};
    double pi{3.14159265358979};
    bool found{false};

    // C++17 allows structured bindings and more
    // C++17 if-init (covered in intermediate topics)

    std::cout << "Age: " << age << std::endl;
    std::cout << "Temperature: " << temperature << std::endl;
    std::cout << "Grade: " << grade << std::endl;
    std::cout << "Valid: " << std::boolalpha << is_valid << std::endl;

    return 0;
}

Tip: Prefer brace initialization int x{0} over copy initialization int x = 0 in modern C++. Brace initialization prevents implicit narrowing conversions. For example, int x{3.14} will not compile (lossy double-to-int conversion), while int x = 3.14 silently truncates.

Integer Modifiers

Integer types support modifiers for signedness and size:

#include <iostream>

int main() {
    unsigned int population = 331000000;  // No negative values
    signed int balance = -500;            // Explicitly signed (default)
    long long big_number = 9223372036854775807LL;

    std::cout << "Unsigned: " << population << std::endl;
    std::cout << "Signed:   " << balance << std::endl;
    std::cout << "Long long: " << big_number << std::endl;

    return 0;
}

Warning: Be careful with unsigned arithmetic. Subtracting from an unsigned value can underflow, wrapping around to a very large number. For example, unsigned int x = 0; x - 1; yields 4294967295 (on a 32-bit system) rather than -1.

The auto Keyword

C++11 introduced auto for type deduction. The compiler deduces the type from the initializer:

#include <iostream>
#include <vector>

int main() {
    auto x = 42;           // int
    auto y = 3.14;         // double
    auto z = "hello";      // const char*
    auto flag = true;      // bool

    // auto with iterators (very common in STL code)
    std::vector<int> nums = {10, 20, 30};
    for (auto it = nums.begin(); it != nums.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << std::endl;

    return 0;
}

Note: auto strips const and & by default. Use const auto& to deduce a const reference, and auto& for a mutable reference. See References for more detail.

Type Casting

C++ offers several ways to convert between types. Prefer C++ style casts over C casts:

#include <iostream>

int main() {
    double pi = 3.14159;

    // C-style cast (avoid in C++)
    int c_style = (int)pi;                // 3

    // C++ static_cast (preferred)
    int cpp_style = static_cast<int>(pi);  // 3

    std::cout << "C-style: " << c_style << std::endl;
    std::cout << "C++ style: " << cpp_style << std::endl;

    // Implicit conversions happen with mixed types
    int a = 5;
    double b = 2.5;
    double result = a + b;  // 'a' is promoted to double

    std::cout << "5 + 2.5 = " << result << std::endl;

    return 0;
}
CastUse Case
static_cast<T>(expr)Well-defined conversions (int to double, enum to int)
dynamic_cast<T>(expr)Safe downcasting in inheritance hierarchies
const_cast<T>(expr)Adding or removing const
reinterpret_cast<T>(expr)Low-level reinterpreting of bit patterns (dangerous)

Warning: Avoid reinterpret_cast unless absolutely necessary. It reinterprets the raw bit representation and is inherently unsafe. Prefer static_cast for nearly all everyday conversions.

sizeof Operator

The sizeof operator returns the size of a type or variable in bytes at compile time:

#include <iostream>

int main() {
    std::cout << "bool:       " << sizeof(bool) << " bytes" << std::endl;
    std::cout << "char:       " << sizeof(char) << " bytes" << std::endl;
    std::cout << "short:      " << sizeof(short) << " bytes" << std::endl;
    std::cout << "int:        " << sizeof(int) << " bytes" << std::endl;
    std::cout << "long long:  " << sizeof(long long) << " bytes" << std::endl;
    std::cout << "float:      " << sizeof(float) << " bytes" << std::endl;
    std::cout << "double:     " << sizeof(double) << " bytes" << std::endl;

    // sizeof on an expression
    int arr[10];
    std::cout << "Array size: " << sizeof(arr) << " bytes" << std::endl;
    std::cout << "Elements:   " << sizeof(arr) / sizeof(arr[0]) << std::endl;

    return 0;
}
