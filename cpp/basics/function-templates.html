1.4.1 Function Templates

Function templates allow you to write a single function definition that works with multiple data types. Instead of writing separate max(int, int), max(double, double), and max(string, string), you write one max(T, T) template and the compiler generates the concrete versions for you.

Template Syntax

A function template begins with the template keyword followed by a type parameter list in angle brackets. The type parameter (commonly named T) acts as a placeholder for any type.

#include <iostream>
#include <string>

// Basic function template
template <typename T>
T max_val(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    // Compiler generates: max_val<int>(int, int)
    std::cout << max_val(3, 7) << std::endl;         // 7

    // Compiler generates: max_val<double>(double, double)
    std::cout << max_val(3.14, 2.71) << std::endl;   // 3.14

    // Compiler generates: max_val<std::string>(string, string)
    std::cout << max_val(std::string{"apple"},
                          std::string{"banana"}) << std::endl;  // banana

    return 0;
}

Tip: Both typename and class are interchangeable in template parameter declarations. template<typename T> and template<class T> mean the same thing. Modern C++ style prefers typename as it is more descriptive.

Multiple Type Parameters

Templates can have multiple type parameters. Each type is independent, and you can use them in any combination.

#include <iostream>

// Two independent type parameters
template <typename T, typename U>
void print_pair(T first, U second) {
    std::cout << "(" << first << ", " << second << ")" << std::endl;
}

// Template with a return type that differs from parameters
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

int main() {
    print_pair(42, 3.14);              // (42, 3.14)  int + double
    print_pair("hello", 100);          // (hello, 100) const char* + int
    print_pair(3.14, std::string{"pi"}); // (3.14, pi)  double + string

    // C++14: auto return type deduction (no trailing return needed)
    std::cout << add(3, 4.5) << std::endl;      // 7.5
    std::cout << add(1.5, 2) << std::endl;      // 3.5

    return 0;
}

Type Deduction

The compiler deduces template arguments from the types of the function arguments. In most cases you do not need to specify the type explicitly.

#include <iostream>

template <typename T>
T multiply(T a, T b) {
    return a * b;
}

int main() {
    // Implicit deduction (compiler figures out T from arguments)
    std::cout << multiply(3, 4) << std::endl;         // 12 (T = int)
    std::cout << multiply(2.5, 4.0) << std::endl;     // 10 (T = double)

    // Explicit specification (override deduction)
    std::cout << multiply<double>(3, 4) << std::endl; // 12.0 (T = double)

    // Error: type ambiguity (int vs double)
    // multiply(3, 4.5);  // Compiler cannot deduce single T

    // Fix: explicit specification
    std::cout << multiply<double>(3, 4.5) << std::endl; // 13.5

    return 0;
}

Note: Type deduction requires all uses of a type parameter to deduce to the same type. multiply(3, 4.5) fails because 3 deduces T=int but 4.5 deduces T=double. Use explicit specification or two separate type parameters to resolve this.

Practical Examples

Generic swap

#include <iostream>
#include <string>

template <typename T>
void my_swap(T& a, T& b) {
    T temp = a;
    a = b;
    b = temp;
}

int main() {
    int x = 10, y = 20;
    my_swap(x, y);
    std::cout << "x=" << x << ", y=" << y << std::endl;  // x=20, y=10

    double a = 1.1, b = 2.2;
    my_swap(a, b);
    std::cout << "a=" << a << ", b=" << b << std::endl;  // a=2.2, b=1.1

    std::string s1 = "hello", s2 = "world";
    my_swap(s1, s2);
    std::cout << "s1=" << s1 << ", s2=" << s2 << std::endl;

    return 0;
}

Generic print function

#include <iostream>
#include <vector>

// Print any single value
template <typename T>
void print(const T& value) {
    std::cout << value << std::endl;
}

// Print a vector of any type
template <typename T>
void print_vector(const std::vector<T>& vec) {
    std::cout << "[";
    for (size_t i = 0; i < vec.size(); ++i) {
        std::cout << vec[i];
        if (i + 1 < vec.size()) std::cout << ", ";
    }
    std::cout << "]" << std::endl;
}

int main() {
    print(42);
    print(3.14);
    print("hello");

    std::vector<int> nums = {1, 2, 3, 4, 5};
    print_vector(nums);  // [1, 2, 3, 4, 5]

    std::vector<std::string> words = {"foo", "bar", "baz"};
    print_vector(words);  // [foo, bar, baz]

    return 0;
}

Template Specialization (Overview)

Sometimes a template needs special behavior for a specific type. Template specialization lets you provide a custom implementation for particular types. This is covered in depth in Template Specialization.

#include <iostream>
#include <cstring>

// Generic template
template <typename T>
T max_val(T a, T b) {
    return (a > b) ? a : b;
}

// Full specialization for const char* (C-style strings)
template <>
const char* max_val<const char*>(const char* a, const char* b) {
    return (strcmp(a, b) > 0) ? a : b;
}

int main() {
    std::cout << max_val(10, 20) << std::endl;        // 20 (generic)
    std::cout << max_val("apple", "banana") << std::endl; // banana (specialized)
    return 0;
}

Deep dive: Template Specialization covers full and partial specialization, Variadic Templates covers templates with a variable number of type parameters, and Concepts (C++20) adds constraints to template parameters.

How Templates Work

Templates are not functions — they are recipes for generating functions. The compiler creates a concrete function (an instantiation) for each unique set of type arguments used in your code. This happens at compile time, so there is zero runtime overhead.

#include <iostream>

template <typename T>
T square(T x) {
    return x * x;
}

int main() {
    // Compiler generates: int square<int>(int x) { return x * x; }
    int a = square(5);

    // Compiler generates: double square<double>(double x) { return x * x; }
    double b = square(3.14);

    // Compiler reuses the int instantiation (same type)
    int c = square(10);

    std::cout << a << " " << b << " " << c << std::endl;
    return 0;
}

Tip: Templates are header-only — the full definition must be visible where the template is used (typically in a .h file). The compiler needs to see the definition to instantiate the template. Unlike regular functions, you cannot separate the declaration into a .h and definition into a .cpp.
