1.1.4 Control Flow

Control flow statements determine the order in which statements are executed. C++ provides conditional branching, looping, and jump statements that let you express complex logic.

if / else if / else

The if statement is the most fundamental control flow construct. It evaluates a condition and executes a block if the condition is true.

#include <iostream>

int main() {
    int score = 85;

    if (score >= 90) {
        std::cout << "Grade: A" << std::endl;
    } else if (score >= 80) {
        std::cout << "Grade: B" << std::endl;
    } else if (score >= 70) {
        std::cout << "Grade: C" << std::endl;
    } else if (score >= 60) {
        std::cout << "Grade: D" << std::endl;
    } else {
        std::cout << "Grade: F" << std::endl;
    }

    return 0;
}

Tip: Always use braces {} around the body of if/else statements, even for single-line bodies. This prevents the dangling else problem and makes the code easier to maintain. Many style guides (Google, LLVM, etc.) enforce this rule.

switch Statement

The switch statement is a cleaner alternative to long if/else if chains when comparing a single expression against constant values.

#include <iostream>

int main() {
    int day = 3;

    switch (day) {
        case 1:
            std::cout << "Monday" << std::endl;
            break;
        case 2:
            std::cout << "Tuesday" << std::endl;
            break;
        case 3:
            std::cout << "Wednesday" << std::endl;
            break;
        case 4:
            std::cout << "Thursday" << std::endl;
            break;
        case 5:
            std::cout << "Friday" << std::endl;
            break;
        default:
            std::cout << "Weekend" << std::endl;
            break;
    }

    return 0;
}

Warning: Forgetting break causes fall-through, where execution continues into the next case. This is almost always a bug. If intentional fall-through is desired, add a // fall through comment. C++17 added the [[fallthrough]] attribute to suppress compiler warnings explicitly.

for Loop

The for loop is the most common loop in C++. It combines initialization, condition, and increment in a single line.

#include <iostream>

int main() {
    // Classic for loop
    for (int i = 0; i < 5; ++i) {
        std::cout << "Iteration: " << i << std::endl;
    }

    // Range-based for loop (C++11)
    int nums[] = {10, 20, 30, 40, 50};

    std::cout << "\nRange-based for:" << std::endl;
    for (int num : nums) {
        std::cout << num << " ";
    }
    std::cout << std::endl;

    // Range-based for with reference (to modify elements)
    for (int& num : nums) {
        num *= 2;  // Double each element
    }

    for (const int& num : nums) {
        std::cout << num << " ";
    }
    std::cout << std::endl;

    return 0;
}

Tip: Use const auto& in range-based for loops when you only need to read elements: for (const auto& item : container). This avoids unnecessary copies and prevents accidental modification.

while Loop

The while loop evaluates its condition before each iteration. It is ideal when the number of iterations is unknown beforehand.

#include <iostream>

int main() {
    int count = 0;

    // while: check condition first
    while (count < 5) {
        std::cout << "Count: " << count << std::endl;
        ++count;
    }

    // Common pattern: processing input
    int sum = 0;
    int value;
    std::cout << "\nEnter numbers (0 to stop):" << std::endl;
    while (std::cin >> value && value != 0) {
        sum += value;
    }
    std::cout << "Sum: " << sum << std::endl;

    return 0;
}

do-while Loop

The do-while loop guarantees at least one iteration since the condition is checked after the loop body.

#include <iostream>

int main() {
    int choice;

    do {
        std::cout << "\nMenu:" << std::endl;
        std::cout << "1. Option A" << std::endl;
        std::cout << "2. Option B" << std::endl;
        std::cout << "3. Quit" << std::endl;
        std::cout << "Choice: ";
        std::cin >> choice;

        switch (choice) {
            case 1: std::cout << "You chose A" << std::endl; break;
            case 2: std::cout << "You chose B" << std::endl; break;
            case 3: std::cout << "Goodbye!" << std::endl; break;
            default: std::cout << "Invalid choice" << std::endl;
        }
    } while (choice != 3);

    return 0;
}

break and continue

break exits the innermost loop or switch immediately. continue skips the rest of the current iteration and moves to the next one.

#include <iostream>

int main() {
    // break: find first even number
    int nums[] = {1, 3, 7, 4, 9, 2};
    for (int i = 0; i < 6; ++i) {
        if (nums[i] % 2 == 0) {
            std::cout << "First even: " << nums[i]
                      << " at index " << i << std::endl;
            break;
        }
    }

    // continue: skip odd numbers
    std::cout << "\nEven numbers only:" << std::endl;
    for (int i = 0; i < 10; ++i) {
        if (i % 2 != 0) {
            continue;  // Skip odd
        }
        std::cout << i << " ";
    }
    std::cout << std::endl;

    return 0;
}

Warning: Overusing break and continue can make code harder to follow because the control flow is not linear. Use them when they genuinely simplify the logic (like the search pattern above), but avoid using them in complex nested loops where they reduce readability.

Nested Loops

Loops can be nested inside each other. A common use case is working with multi-dimensional data:

#include <iostream>

int main() {
    // Multiplication table
    for (int i = 1; i <= 5; ++i) {
        for (int j = 1; j <= 5; ++j) {
            std::cout << i * j << "\t";
        }
        std::cout << std::endl;
    }

    // Searching a 2D array
    int matrix[3][3] = {
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9}
    };

    int target = 5;
    bool found = false;
    for (int i = 0; i < 3 && !found; ++i) {
        for (int j = 0; j < 3; ++j) {
            if (matrix[i][j] == target) {
                std::cout << "Found " << target
                          << " at [" << i <<"][" << j << "]" << std::endl;
                found = true;
                break;
            }
        }
    }

    return 0;
}
