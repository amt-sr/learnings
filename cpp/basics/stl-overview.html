1.5.1 STL Overview

The Standard Template Library (STL) is a collection of generic, reusable data structures and algorithms that are part of the C++ standard library. It is one of the most powerful features of C++, providing production-ready components for common programming tasks.

Containers

Containers are objects that store collections of elements. The STL provides several categories of containers, each optimized for different use cases.

Sequence Containers

Elements are stored in a linear order, accessed by position.

Iterators

Iterators provide a uniform way to traverse any container. They generalize the concept of a pointer: you can dereference an iterator to access the element it points to, and increment it to move to the next element.

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9, 3};

    // Forward iteration with iterators
    std::cout << "Forward: ";
    for (std::vector<int>::iterator it = nums.begin();
         it != nums.end(); ++it) {
        std::cout << *it << " ";
    }
    std::cout << std::endl;

    // Reverse iteration
    std::cout << "Reverse: ";
    for (auto rit = nums.rbegin(); rit != nums.rend(); ++rit) {
        std::cout << *rit << " ";
    }
    std::cout << std::endl;

    return 0;
}

Algorithms

The <algorithm> header provides 100+ generic algorithms that operate on iterator ranges. These include sorting, searching, transforming, and numeric operations.

#include <iostream>
#include <vector>
#include <algorithm>
#include <numeric>

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9, 3};

    // Sort
    std::sort(nums.begin(), nums.end());

    // Search
    auto it = std::find(nums.begin(), nums.end(), 8);
    if (it != nums.end()) {
        std::cout << "Found 8 at position "
                  << (it - nums.begin()) << std::endl;
    }

    // Count
    int count = std::count(nums.begin(), nums.end(), 5);
    std::cout << "Count of 5: " << count << std::endl;

    // Accumulate
    int sum = std::accumulate(nums.begin(), nums.end(), 0);
    std::cout << "Sum: " << sum << std::endl;

    // Print sorted result
    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;

    return 0;
}

Functors and Lambdas

Functors (function objects) and lambda expressions allow you to customize algorithm behavior. A lambda is a concise way to create an inline function object.

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9, 3};

    // Sort descending using a lambda
    std::sort(nums.begin(), nums.end(), [](int a, int b) {
        return a > b;
    });

    // Count elements greater than 5
    int big = std::count_if(nums.begin(), nums.end(), [](int x) {
        return x > 5;
    });
    std::cout << "Elements > 5: " << big << std::endl;

    // Transform: multiply each element by 2
    std::for_each(nums.begin(), nums.end(), [](int& x) {
        x *= 2;
    });

    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;

    return 0;
}

Why STL Matters

Productivity: The STL saves you from reimplementing common data structures and algorithms. Need a sorted dictionary? Use std::map. Need to find an element? Use std::find. Each component is thoroughly tested, optimized, and standardized.

  • Type safety: Templates ensure containers and algorithms work with the correct types at compile time.
  • Performance: STL components are highly optimized. std::vector and std::sort often match or beat hand-written alternatives.
  • Correctness: Standardized behavior means portable, predictable results across compilers and platforms.
  • Uniformity: All containers use iterators, so algorithms work seamlessly with any container.
