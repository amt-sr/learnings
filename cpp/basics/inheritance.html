1.2.3 Inheritance

Inheritance is a mechanism where a new class (the derived class) acquires the properties and behaviors of an existing class (the base class). It models the is-a relationship and enables code reuse, making it a core pillar of object-oriented programming alongside classes, encapsulation, and polymorphism.

Single Inheritance Syntax

A derived class specifies its base class after a colon. By default, inheritance is private for class — you almost always want public inheritance:

#include <iostream>
#include <string>

class Shape {
protected:
    std::string color;

public:
    Shape(const std::string& c = "black") : color(c) {
        std::cout << "Shape constructor" << std::endl;
    }

    void set_color(const std::string& c) { color = c; }
    std::string get_color() const { return color; }

    ~Shape() {
        std::cout << "Shape destructor" << std::endl;
    }
};

// Single public inheritance
class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r, const std::string& c = "black")
        : Shape(c), radius(r) {      // base class constructor called first
        std::cout << "Circle constructor" << std::endl;
    }

    double area() const {
        return 3.14159265 * radius * radius;
    }

    void print() const {
        std::cout << "Circle: r=" << radius
                  << ", color=" << color
                  << ", area=" << area() << std::endl;
    }

    ~Circle() {
        std::cout << "Circle destructor" << std::endl;
    }
};

int main() {
    Circle c{5.0, "red"};
    c.print();  // Circle: r=5, color=red, area=78.5398

    // Inherited members are accessible
    c.set_color("blue");
    std::cout << "Color: " << c.get_color() << std::endl;

    return 0;
}

Types of Inheritance

The access specifier before the base class controls how base class members are accessible in the derived class:

Inheritance TypeBase public membersBase protected membersBase private members
publicpublic in derivedprotected in derivedInaccessible
protectedprotected in derivedprotected in derivedInaccessible
privateprivate in derivedprivate in derivedInaccessible

Rule of thumb: Use public inheritance for is-a relationships (a Dog is an Animal). Use private inheritance for has-a / implemented-in-terms-of relationships (a Stack implemented using a Vector). Avoid protected inheritance unless you have a specific reason.

Constructor Chaining with Base Class

When a derived class is constructed, the base class constructor is called first, then member variables, then the derived class constructor body. Destruction happens in reverse order:

#include <iostream>

class Base {
public:
    Base() { std::cout << "Base constructor" << std::endl; }
    Base(int x) { std::cout << "Base(int) constructor: " << x << std::endl; }
    ~Base() { std::cout << "Base destructor" << std::endl; }
};

class Derived : public Base {
public:
    // Calls Base() by default
    Derived() { std::cout << "Derived constructor" << std::endl; }

    // Calls Base(int) explicitly via init list
    Derived(int x) : Base(x) {
        std::cout << "Derived(int) constructor" << std::endl;
    }

    ~Derived() { std::cout << "Derived destructor" << std::endl; }
};

int main() {
    std::cout << "--- Default ---" << std::endl;
    Derived d1;
    std::cout << "--- Parameterized ---" << std::endl;
    Derived d2{42};
    return 0;
}
// Output:
// --- Default ---
// Base constructor
// Derived constructor
// --- Parameterized ---
// Base(int) constructor: 42
// Derived(int) constructor
// Derived destructor
// Base destructor
// Derived destructor
// Base destructor

Important: If the base class has no default constructor, the derived class must explicitly call a base class constructor in its initialization list. Otherwise, compilation fails because the compiler cannot implicitly call Base().

Is-a Relationship

Public inheritance models an is-a relationship. A derived class object can be used wherever a base class object is expected:

#include <iostream>

class Animal {
public:
    std::string name;

    Animal(const std::string& n = "Animal") : name(n) {}
    virtual void speak() const {
        std::cout << name << " makes a sound" << std::endl;
    }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    Dog(const std::string& n = "Dog") : Animal(n) {}

    void speak() const override {
        std::cout << name << " says: Woof!" << std::endl;
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& n = "Cat") : Animal(n) {}

    void speak() const override {
        std::cout << name << " says: Meow!" << std::endl;
    }
};

void make_speak(const Animal& animal) {
    animal.speak();  // calls the correct override
}

int main() {
    Dog dog{"Rex"};
    Cat cat{"Whiskers"};

    dog.speak();       // Rex says: Woof!
    cat.speak();       // Whiskers says: Meow!

    // Dog IS-A Animal, so it can be passed to a function expecting Animal
    make_speak(dog);   // Rex says: Woof!
    make_speak(cat);   // Whiskers says: Meow!

    return 0;
}

Method Overriding

A derived class can provide its own implementation of a base class method. Use the override keyword (C++11) to ensure the method actually overrides a base class virtual function:

class Base {
public:
    virtual void draw() const {
        std::cout << "Base::draw" << std::endl;
    }

    void describe() const {
        std::cout << "Base::describe" << std::endl;
    }
};

class Derived : public Base {
public:
    // Correct: overrides virtual draw()
    void draw() const override {
        std::cout << "Derived::draw" << std::endl;
    }

    // ERROR: describe() is not virtual, cannot override
    // void describe() const override { ... }

    // ERROR: signature mismatch, does not override
    // void draw() override { ... }  // missing const
};

Tip: Always use the override keyword when overriding virtual functions. It catches bugs at compile time — if the base class signature changes, the compiler will warn you instead of silently creating a new function.
