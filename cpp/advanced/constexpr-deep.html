Advanced constexpr

constexpr tells the compiler to evaluate expressions at compile time. Since C++14, constexpr functions can contain loops, local variables, and conditionals. C++20 added constexpr allocators, vector, string, and more. C++23 introduces consteval (immediate functions) that must execute at compile time. This topic follows Cache-Oriented Programming and leads into CRTP.

constexpr Evolution

Compile-Time Computation

// C++14: constexpr functions with full control flow
constexpr int fibonacci(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; ++i) {
        int c = a + b;
        a = b;
        b = c;
    }
    return b;
}

// Computed at compile time
constexpr int fib20 = fibonacci(20);  // 6765

// Compile-time string hashing (for switch/case on strings)
constexpr std::uint32_t djb2_hash(const char* str) {
    std::uint32_t hash = 5381;
    while (*str) {
        hash = hash * 33 + static_cast<std::uint32_t>(*str);
        ++str;
    }
    return hash;
}

// Use in switch statement (impossible with std::string)
void dispatch(const char* command) {
    switch (djb2_hash(command)) {
        case djb2_hash("start"):  handle_start();  break;
        case djb2_hash("stop"):   handle_stop();   break;
        case djb2_hash("reset"):  handle_reset();  break;
        default:                  handle_unknown();
    }
}

constexpr Containers (C++20)

#include <array>
#include <string_view>
#include <algorithm>

// Compile-time lookup table
template<typename K, typename V, std::size_t N>
struct ConstexprMap {
    std::array<std::pair<K, V>, N> data;

    constexpr ConstexprMap(std::array<std::pair<K, V>, N> init) : data(init) {}

    constexpr V at(const K& key) const {
        for (const auto& [k, v] : data) {
            if (k == key) return v;
        }
        throw "Key not found";  // compile-time error if not found
    }

    constexpr bool contains(const K& key) const {
        return std::find_if(data.begin(), data.end(),
            [&key](const auto& pair) { return pair.first == key; }) != data.end();
    }
};

// Protocol message type lookup — fully compile-time
constexpr auto message_types = ConstexprMap{
    std::array<std::pair<int, std::string_view>, 4>{{
        {1, "RRC Setup Request"},
        {2, "RRC Setup Response"},
        {3, "RRC Release"},
        {4, "RRC Reconfiguration"},
    }}
};

// This is evaluated at compile time
constexpr auto msg_name = message_types.at(2);  // "RRC Release"

// Compile-time sieve of Eratosthenes
constexpr std::size_t kMax = 1000;

constexpr auto generate_primes() {
    std::array<bool, kMax> sieve{};
    for (std::size_t i = 2; i < kMax; ++i) sieve[i] = true;

    for (std::size_t i = 2; i * i < kMax; ++i) {
        if (sieve[i]) {
            for (std::size_t j = i * i; j < kMax; j += i) {
                sieve[j] = false;
            }
        }
    }
    return sieve;
}

constexpr auto primes = generate_primes();  // computed at compile time

consteval — Immediate Functions (C++23)

consteval (also called "immediate functions") must be evaluated at compile time. If the compiler cannot evaluate them at compile time, it is a hard error (not a fallback to runtime like constexpr):

// C++23: consteval — MUST execute at compile time
consteval int square(int x) {
    return x * x;
}

constexpr int cube(int x) {
    return x * x * x;
}

int main() {
    constexpr int a = square(5);       // OK: compile-time context
    int runtime_val = 10;
    constexpr int b = square(runtime_val);  // ERROR: not a constant expression
    int c = square(runtime_val);              // ERROR: square is consteval, not const

    // But this works: square() result is a compile-time constant
    int d = square(10);              // OK: argument is a constant

    // constexpr CAN fall back to runtime:
    int e = cube(runtime_val);        // OK: evaluated at runtime
}

Compile-Time Assertions with consteval

// C++23 consteval: enforce compile-time checks
consteval void validate_config(int max_ue, int max_bearers) {
    if (max_ue <= 0 || max_ue > 65536) {
        throw "Invalid max_ue configuration";
    }
    if (max_bearers <= 0 || max_bearers > 8) {
        throw "Invalid bearer configuration";
    }
    if (max_bearers * max_ue > 524288) {
        throw "Resource budget exceeded";
    }
}

// This runs at compile time — errors become compilation errors
int main() {
    validate_config(1000, 4);  // OK: validated at compile time
    // validate_config(0, 4);  // ERROR: "Invalid max_ue configuration"
}

constexpr String Processing

#include <array>
#include <string_view>

// Compile-time string parsing
template<std::size_t N>
struct FixedString {
    char data[N]{};

    constexpr FixedString(const char (&str)[N]) {
        for (std::size_t i = 0; i < N; ++i) {
            data[i] = str[i];
        }
    }

    constexpr std::size_t size() const { return N - 1; }
    constexpr std::string_view view() const { return {data, N - 1}; }

    constexpr bool starts_with(std::string_view prefix) const {
        return view().starts_with(prefix);
    }

    constexpr bool contains(char c) const {
        for (std::size_t i = 0; i < size(); ++i) {
            if (data[i] == c) return true;
        }
        return false;
    }
};

// Compile-time format validation
template<FixedString fmt>
consteval bool valid_format_string() {
    int open_braces = 0;
    for (std::size_t i = 0; i < fmt.size(); ++i) {
        if (fmt.data[i] == '{') ++open_braces;
        if (fmt.data[i] == '}') --open_braces;
    }
    return open_braces == 0;
}

// Usage
static_assert(valid_format_string<"Hello {}">());
static_assert(!valid_format_string<"Hello {">());  // unbalanced

Telecom Relevance

  • Protocol tables: LTE/5G protocol message types, cause codes, and IE identifiers can be stored in constexpr maps — zero runtime initialization, no heap allocation.
  • Configuration validation: consteval validates system configuration (max UEs, bearer limits, timer values) at compile time, catching misconfiguration before deployment.
  • Lookup tables: CRC tables, bit-reversal tables, and encoding tables computed at compile time avoid startup overhead in real-time systems.
  • String hashing: Compile-time string hashing enables efficient message dispatch without string comparison at runtime.

Best practice: Mark lookup tables and configuration structs as constexpr whenever possible. Use consteval for validation functions that must never have runtime overhead. Remember that constexpr evaluation has a default step limit (typically 33554432 for GCC) to prevent compilation hangs.
