clang-tidy

clang-tidy is a clang-based static analysis tool that operates on the Abstract Syntax Tree (AST). It provides an extensible framework for diagnosing and auto-fixing a wide range of C++ issues -- from coding style violations to potential bugs and performance problems.

Installing and Running

# macOS (Homebrew)
brew install llvm

# Ubuntu/Debian
sudo apt install clang-tidy

# Run on a single file (requires compile_commands.json)
clang-tidy src/parser.cpp

# Run on all source files in parallel
run-clang-tidy -p build/ -j$(nproc) src/

# Run specific checks
clang-tidy -checks='modernize-*,bugprone-*' src/main.cpp

# Auto-fix with --fix (modifies files in place)
clang-tidy --fix -checks='modernize-use-auto' src/main.cpp

Prerequisite: clang-tidy needs a compile_commands.json file to understand compiler flags, includes, and definitions. Generate it with CMake: cmake -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON

.clang-tidy Configuration File

Place a .clang-tidy file in your project root to configure checks consistently across the team:

# .clang-tidy
Checks: >
  -*,
  bugprone-*,
  modernize-*,
  readability-*,
  performance-*,
  -modernize-use-trailing-return-type,
  -readability-magic-numbers,
  -readability-identifier-length
WarningsAsErrors: ''
HeaderFilterRegex: 'src/.*\.h$'
FormatStyle:     file
CheckOptions:
  - key: readability-identifier-naming.ClassCase
    value: CamelCase
  - key: readability-identifier-naming.FunctionCase
    value: camelBack
  - key: readability-identifier-naming.VariableCase
    value: camelBack
  - key: modernize-use-override.IgnoreDestructors
    value: false

Understanding the Checks Line

# Enable everything, then disable what you don't want
Checks: '*, -llvm-*, -google-*'

# Enable specific categories only
Checks: 'bugprone-*, modernize-*, performance-*'

# Enable individual checks
Checks: 'modernize-use-auto, modernize-use-nullptr,
         bugprone-use-after-move, performance-noexcept-destructor'

Check Categories

modernize -- Modernize to C++11/14/17/20

CheckWhat it catches
modernize-use-autoReplace explicit type with auto where the type is obvious
modernize-use-nullptrReplace NULL/0 with nullptr
modernize-use-overrideAdd override specifier to virtual function overrides
modernize-use-emplaceUse emplace_back() instead of push_back(T{})
modernize-use-usingReplace typedef with using (C++11)
modernize-use-noexceptMark non-throwing functions noexcept
modernize-make-uniqueReplace new T with std::make_unique
modernize-make-sharedReplace new T with std::make_shared
modernize-loop-convertConvert for loops to range-based for

bugprone -- Likely Bugs

CheckWhat it catches
bugprone-use-after-moveUsing an object after std::move
bugprone-argument-commentMismatched argument names in calls
bugprone-copy-constructor-initCopy constructor initializing self
bugprone-integer-divisionInteger division where floating-point was intended
bugprone-string-constructorWrong string constructor arguments
bugprone-sizeof-expressionsizeof on pointer instead of type

performance -- Performance Improvements

CheckWhat it catches
performance-noexcept-destructorDestructors should be noexcept
performance-faster-string-findUse string::find(char) instead of string::find(string)
performance-unnecessary-value-paramPass parameters by const reference instead of value
performance-for-range-copyRange for loop copies instead of using reference

Auto-fix with --fix

# Before auto-fix
std::vector<int> numbers = {1, 2, 3};
for (std::vector<int>::iterator it = numbers.begin();
     it != numbers.end(); ++it) {
    std::cout << *it << "\n";
}

# After: clang-tidy --fix -checks='modernize-use-auto,modernize-loop-convert'
std::vector<int> numbers = {1, 2, 3};
for (auto it = numbers.begin(); it != numbers.end(); ++it) {
    std::cout << *it << "\n";
}

# After also applying modernize-loop-convert:
std::vector<int> numbers = {1, 2, 3};
for (int number : numbers) {
    std::cout << number << "\n";
}

Caution: Always review auto-fix changes before committing. Run --fix on a clean git branch so you can easily revert if needed. Some fixes may subtly change behavior (e.g., modernize-use-noexcept adds noexcept which affects overload resolution).

CMake Integration

# CMakeLists.txt
find_program(CLANG_TIDY clang-tidy)
if(CLANG_TIDY)
    set_target_properties(my_target PROPERTIES
        CXX_CLANG_TIDY "${CLANG_TIDY};--config-file=${CMAKE_SOURCE_DIR}/.clang-tidy"
    )
endif()

Best practice: Start with a focused set of checks (bugprone-*, modernize-*), fix existing issues, then gradually enable more. Pair with cppcheck for memory-focused analysis and Clang Static Analyzer for deep path-sensitive analysis.
