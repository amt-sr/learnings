Memory-Mapped Files

Memory-mapped files (mmap) map a file's contents directly into the process's virtual address space. Instead of using read()/write() system calls, the program accesses file data through memory pointers. The OS handles paging — loading data from disk on demand and writing back changes. This approach is especially powerful for large file processing, shared memory IPC, and efficient random access.

Prerequisites: Shared Memory IPC, File Streams, Memory Model, Placement New & Memory Pools.

Mapping a File for Reading

#include <iostream>
#include <fstream>
#include <cstring>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>
#include <sys/stat.h>

class MappedFile {
    void* data_ = nullptr;
    std::size_t size_ = 0;
public:
    MappedFile(const char* path, bool readonly = true) {
        int fd = open(path, readonly ? O_RDONLY : O_RDWR);
        if (fd < 0) throw std::runtime_error("open failed");

        struct stat sb{};
        if (fstat(fd, &sb) < 0) { close(fd); throw std::runtime_error("fstat failed"); }
        size_ = static_cast<std::size_t>(sb.st_size);

        int prot = readonly ? PROT_READ : (PROT_READ | PROT_WRITE);
        data_ = mmap(nullptr, size_, prot, MAP_PRIVATE, fd, 0);
        close(fd);  // fd not needed after mmap

        if (data_ == MAP_FAILED) throw std::runtime_error("mmap failed");
    }

    ~MappedFile() {
        if (data_ && data_ != MAP_FAILED)
            munmap(data_, size_);
    }

    MappedFile(const MappedFile&) = delete;
    MappedFile& operator=(const MappedFile&) = delete;

    const char* data() const { return static_cast<const char*>(data_); }
    std::size_t size() const { return size_; }

    // Bounds-checked access
    char operator[](std::size_t i) const {
        if (i >= size_) throw std::out_of_range("MappedFile index");
        return data()[i];
    }
};

int main() {
    MappedFile file("data.bin");

    std::cout << "File size: " << file.size() << " bytes\n";

    // Access file contents through memory pointer
    std::size_t line_count = 0;
    for (std::size_t i = 0; i < file.size(); ++i) {
        if (file[i] == '\n') ++line_count;
    }
    std::cout << "Lines: " << line_count << "\n";
}

Memory-Mapped File for Writing (msync)

#include <iostream>
#include <cstring>
#include <fcntl.h>
#include <unistd.h>
#include <sys/mman.h>

int main() {
    // Open/create a file for read-write
    int fd = open("mmap_output.bin", O_RDWR | O_CREAT | O_TRUNC, 0666);
    if (fd < 0) return 1;

    // Set file size (must be done before mmap)
    constexpr std::size_t FILE_SIZE = 4096;
    if (ftruncate(fd, FILE_SIZE) < 0) { close(fd); return 1; }

    // MAP_SHARED: writes go to the file
    char* mapped = static_cast<char*>(
        mmap(nullptr, FILE_SIZE, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0)
    );
    close(fd);

    if (mapped == MAP_FAILED) return 1;

    // Write data through memory pointer
    std::strcpy(mapped, "Hello from mmap!");
    std::strcat(mapped + 18, "This is shared memory backed by a file.");

    // Flush to disk (MS_SYNC = synchronous, MS_ASYNC = asynchronous)
    if (msync(mapped, FILE_SIZE, MS_SYNC) != 0) {
        std::cerr << "msync failed\n";
    }

    std::cout << "Data written and synced to disk\n";

    // Verify the file
    // $ cat mmap_output.bin
    // Hello from mmap!This is shared memory backed by a file.

    munmap(mapped, FILE_SIZE);
    return 0;
}

MAP_SHARED vs MAP_PRIVATE

FlagBehaviorWritesVisibility
MAP_SHAREDChanges shared with other processesWritten to fileAll processes see changes
MAP_PRIVATECopy-on-write, private viewMemory onlyChanges invisible to others

Advantages of Memory-Mapped Files

  • Zero-copy I/O: No buffering between kernel and user space — direct memory access
  • Lazy loading: OS loads pages on demand (only the pages you actually access)
  • Random access: Jump to any offset in O(1) — no seeking through read()
  • Large files: Access files larger than physical RAM — OS manages paging
  • Shared access: MAP_SHARED enables IPC between processes (same as shared memory)
  • Efficient sequential scan: Prefetching works naturally with sequential memory access

madvise — Performance Hints

#include <sys/mman.h>

// Tell the kernel about access patterns
void* mapped = mmap(...);

// Will access sequentially (e.g., scanning a log file)
madvise(mapped, size, MADV_SEQUENTIAL);

// Will access randomly (e.g., a database index)
madvise(mapped, size, MADV_RANDOM);

// Will access this region soon (preload into page cache)
madvise(mapped, size, MADV_WILLNEED);

// Don't need this region anymore (free pages)
madvise(mapped + offset, len, MADV_DONTNEED);

Best practice: Use RAII wrappers for mmap (construct in constructor, unmap in destructor). Always check for MAP_FAILED. Use MAP_PRIVATE for read-only access unless you need shared writes. Call msync before munmap for MAP_SHARED mappings to ensure durability.

Alignment: mmap returns page-aligned memory (typically 4096-byte aligned). The offset parameter must be a multiple of the page size. Accessing past the mapped region is undefined behavior (SIGSEGV).
