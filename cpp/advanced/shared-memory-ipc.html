Shared Memory & IPC

Shared memory is the fastest form of inter-process communication (IPC). Multiple processes map the same physical memory region into their address spaces, enabling zero-copy data exchange. Unlike sockets or pipes, data is not copied through the kernel — processes read and write directly to shared memory, achieving maximum throughput.

Prerequisites: Socket Programming, Signal Handling, Memory Model & Atomics, Memory-Mapped Files.

POSIX Shared Memory API

FunctionPurpose
shm_open()Create or open a shared memory object
ftruncate()Set the size of the shared memory object
mmap()Map shared memory into process address space
munmap()Unmap shared memory from address space
shm_unlink()Remove shared memory object (like unlink for files)
close()Close the shared memory file descriptor

POSIX Shared Memory — Producer

#include <iostream>
#include <cstring>
#include <fcntl.h>
#include <sys/mman.h>
#include <unistd.h>
#include <semaphore.h>

constexpr const char* SHM_NAME = "/my_shm";
constexpr std::size_t SHM_SIZE = 4096;

struct SharedData {
    sem_t sem_producer;  // producer writes when available
    sem_t sem_consumer;  // consumer reads when data ready
    char buffer[SHM_SIZE - 2 * sizeof(sem_t)];
};

int main() {
    // 1. Create shared memory object
    int fd = shm_open(SHM_NAME, O_CREAT | O_RDWR, 0666);
    if (fd < 0) {
        std::cerr << "shm_open failed\n";
        return 1;
    }

    // 2. Set size
    ftruncate(fd, sizeof(SharedData));

    // 3. Map into address space
    auto* shared = static_cast<SharedData*>(
        mmap(nullptr, sizeof(SharedData),
             PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0)
    );
    close(fd);  // fd no longer needed after mmap

    if (shared == MAP_FAILED) {
        std::cerr << "mmap failed\n";
        return 1;
    }

    // 4. Initialize semaphores (1 = shared between processes, 0 = initial value)
    sem_init(&shared->sem_producer, 1, 1);  // producer starts with token
    sem_init(&shared->sem_consumer, 1, 0);  // consumer waits

    // 5. Produce data
    for (int i = 0; i < 5; ++i) {
        sem_wait(&shared->sem_producer);  // wait for slot

        std::string msg = "Message " + std::to_string(i);
        std::strncpy(shared->buffer, msg.c_str(), sizeof(shared->buffer) - 1);
        std::cout << "Produced: " << shared->buffer << "\n";

        sem_post(&shared->sem_consumer);  // signal data ready
        sleep(1);  // simulate work
    }

    // Cleanup
    sem_destroy(&shared->sem_producer);
    sem_destroy(&shared->sem_consumer);
    munmap(shared, sizeof(SharedData));
    shm_unlink(SHM_NAME);

    std::cout << "Producer done\n";
    return 0;
}

POSIX Shared Memory — Consumer

#include <iostream>
#include <cstring>
#include <fcntl.h>
#include <sys/mman.h>
#include <unistd.h>
#include <semaphore.h>

// Same SharedData struct as producer
struct SharedData {
    sem_t sem_producer;
    sem_t sem_consumer;
    char buffer[4096 - 2 * sizeof(sem_t)];
};

int main() {
    // Open existing shared memory (O_RDWR, no O_CREAT)
    int fd = shm_open("/my_shm", O_RDWR, 0666);
    if (fd < 0) {
        std::cerr << "shm_open failed — start producer first\n";
        return 1;
    }

    auto* shared = static_cast<SharedData*>(
        mmap(nullptr, sizeof(SharedData),
             PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0)
    );
    close(fd);

    if (shared == MAP_FAILED) {
        std::cerr << "mmap failed\n";
        return 1;
    }

    // Consume data
    for (int i = 0; i < 5; ++i) {
        sem_wait(&shared->sem_consumer);  // wait for data

        std::cout << "Consumed: " << shared->buffer << "\n";

        sem_post(&shared->sem_producer);  // signal slot available
    }

    munmap(shared, sizeof(SharedData));
    std::cout << "Consumer done\n";
    return 0;
}

Synchronization with Semaphores

Shared memory provides no built-in synchronization. Without explicit coordination, concurrent reads and writes cause data races. Two common approaches:

  • POSIX named semaphores (sem_open/sem_wait/sem_post) — shared between unrelated processes via name
  • POSIX unnamed semaphores (sem_init with pshared=1) — placed in shared memory itself
  • Mutexes in shared memory — pthread_mutexattr_setpshared for process-shared mutex
  • Atomics — std::atomic with MAP_SHARED (hardware-supported on modern systems)

Cache coherency: When multiple processes modify shared memory, cache line bouncing between CPU cores can degrade performance. Align data structures to cache lines and minimize false sharing — see Cache-Oriented Programming.

Best practice: Use RAII wrappers for shared memory and semaphores. Always handle the case where shm_open returns an existing object (after a crash, the shared memory may still exist). Clean up shared memory in signal handlers or destructors.
