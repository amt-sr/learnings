Advanced Practice: Build a Compile-Time Plugin System

Topics Covered: Concepts, Ranges, Coroutines, Memory Model & Atomics, Lock-Free Programming, Custom Allocators, CRTP, Type Erasure, Singleton Pattern, Observer Pattern, Factory Pattern, Socket Programming

Problem Statement

Build a production-quality plugin system that demonstrates mastery of advanced C++ techniques. The system must support:

  • Concept-constrained plugins — enforce a plugin interface at compile time using C++20 concepts.
  • Type-erased plugin storage — store heterogeneous plugin types in a single container without virtual inheritance overhead.
  • Lock-free task dispatch — a Michael-Scott style lock-free queue for submitting tasks to plugins concurrently.
  • CRTP-based lifecycle mixin — add logging, timing, and reference counting to every plugin at compile time with zero virtual overhead.
  • Factory-based plugin loading — abstract factory for dynamic loading from shared libraries or static registration.
  • Network notification — send UDP notifications to a monitoring endpoint when a plugin completes a task.
  • Signal-safe shutdown — graceful teardown via POSIX signal handling.

This exercise ties together concepts, type erasure, atomics, CRTP, design patterns, and socket programming into one cohesive system.

Prerequisites: This is a capstone exercise. You should be comfortable with C++20 concepts, std::atomic with memory ordering, CRTP, std::any / type erasure, the Factory and Observer patterns, and basic BSD socket programming before attempting.

1. Define the Plugin Concept

Click to reveal Solution

#include <concepts>
#include <string_view>

// C++20 concept that constrains what qualifies as a Plugin.
// Any type satisfying this concept can be registered with the PluginManager.
template<typename T>
concept Plugin = requires(T t, const T ct) {
    // Must have a name() returning a string-like value
    { t.name() } -> std::convertible_to<std::string_view>;
    // Must have a version() returning a string-like value
    { t.version() } -> std::convertible_to<std::string_view>;
    // Must have an execute() method accepting a string argument
    { t.execute(std::string{}) } -> std::convertible_to<std::string>;
    // Must have lifecycle hooks: on_load() and on_unload()
    { t.on_load() } -> std::same_as<void>;
    { t.on_unload() } -> std::same_as<void>;
};

// Example plugin satisfying the concept
struct GreeterPlugin {
    std::string name() { return "Greeter"; }
    std::string version() { return "1.0.0"; }

    std::string execute(const std::string& task) {
        return "Hello, " + task + "!";
    }

    void on_load() {
        std::cout << "[GreeterPlugin] loaded\n";
    }

    void on_unload() {
        std::cout << "[GreeterPlugin] unloaded\n";
    }
};

// Compile-time check
static_assert(Plugin<GreeterPlugin>, "GreeterPlugin must satisfy Plugin concept");

// This will NOT compile (demonstrates concept rejection):
// struct BadPlugin {};
// static_assert(Plugin<BadPlugin>, "");

Explanation:

  • The Plugin concept uses requires expressions to specify five required methods: name(), version(), execute(), on_load(), and on_unload().
  • Return type constraints use std::convertible_to for flexibility (accepts std::string, std::string_view, const char*) and std::same_as for exact matches (void for lifecycle hooks).
  • The static_assert verifies the plugin at compile time. If a type does not satisfy the concept, the compiler produces a clear error listing which requirements are missing.

2. Implement the PluginManager with Type Erasure

Click to reveal Solution

#include <iostream>
#include <string>
#include <unordered_map>
#include <memory>
#include <functional>
#include <mutex>
#include <stdexcept>
#include "concepts_check.h"   // assumes Plugin concept is defined here

// Type-erased wrapper: stores any Plugin-constrained type behind
// a uniform interface without virtual inheritance on the plugin itself.
class TypeErasedPlugin {
public:
    template<Plugin T>
    explicit TypeErasedPlugin(T plugin)
        : name_fn_([p = std::move(plugin)]() mutable { return p.name(); })
        , version_fn_([p = std::move(plugin)]() mutable { return p.version(); })
        , execute_fn_([p = std::move(plugin)](std::string task) mutable {
              return p.execute(std::move(task));
          })
        , on_load_fn_([p = std::move(plugin)]() mutable { p.on_load(); })
        , on_unload_fn_([p = std::move(plugin)]() mutable { p.on_unload(); })
    {
        // Note: in production, use a single captured object (e.g. shared_ptr)
        // to avoid issues with multiple moves above. Simplified here for clarity.
    }

    std::string name()              { return name_fn_(); }
    std::string version()           { return version_fn_(); }
    std::string execute(std::string t) { return execute_fn_(std::move(t)); }
    void on_load()                  { on_load_fn_(); }
    void on_unload()                { on_unload_fn_(); }

private:
    std::function<std::string()>               name_fn_;
    std::function<std::string()>               version_fn_;
    std::function<std::string(std::string)>      execute_fn_;
    std::function<void()>                     on_load_fn_;
    std::function<void()>                     on_unload_fn_;
};

// PluginManager: thread-safe registry of type-erased plugins
class PluginManager {
public:
    template<Plugin T>
    bool register_plugin(T plugin) {
        std::string plugin_name = plugin.name();
        std::lock_guard<std::mutex> lock(mutex_);
        if (plugins_.contains(plugin_name)) {
            return false;  // already registered
        }
        plugins_.emplace(plugin_name, std::make_unique<TypeErasedPlugin>(std::move(plugin)));
        plugins_[plugin_name]->on_load();
        return true;
    }

    bool unregister_plugin(const std::string& name) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = plugins_.find(name);
        if (it == plugins_.end()) return false;
        it->second->on_unload();
        plugins_.erase(it);
        return true;
    }

    TypeErasedPlugin* lookup(const std::string& name) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = plugins_.find(name);
        return (it != plugins_.end()) ? it->second.get() : nullptr;
    }

    // Execute a task on a named plugin
    std::string dispatch(const std::string& plugin_name, const std::string& task) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = plugins_.find(plugin_name);
        if (it == plugins_.end()) {
            throw std::runtime_error("Plugin not found: " + plugin_name);
        }
        return it->second->execute(task);
    }

    // List all registered plugin names
    std::vector<std::string> list_plugins() const {
        std::lock_guard<std::mutex> lock(mutex_);
        std::vector<std::string> names;
        names.reserve(plugins_.size());
        for (const auto& [name, _] : plugins_) {
            names.push_back(name);
        }
        return names;
    }

private:
    mutable std::mutex mutex_;
    std::unordered_map<std::string, std::unique_ptr<TypeErasedPlugin>> plugins_;
};

Explanation:

  • TypeErasedPlugin uses std::function to wrap each plugin method. This is one form of type erasure: the caller interacts with a uniform interface while the concrete type is captured inside lambdas.
  • The constructor is constrained with template<Plugin T>, so only types satisfying the Plugin concept can be stored.
  • PluginManager provides thread-safe registration, lookup, and dispatch via a std::mutex and std::unordered_map.
  • In a production system you would capture a single std::shared_ptr to the plugin object rather than moving it into each lambda separately.

3. Build a Lock-Free Task Queue

Click to reveal Solution

#include <atomic>
#include <memory>
#include <optional>
#include <string>
#include <functional>
#include <iostream>

// A task represents work to be dispatched to a plugin.
struct Task {
    std::string plugin_name;
    std::string payload;
    std::function<void(const std::string&)> callback;  // called with result

    Task() = default;
    Task(std::string pn, std::string pl,
         std::function<void(const std::string&)> cb)
        : plugin_name(std::move(pn))
        , payload(std::move(pl))
        , callback(std::move(cb)) {}
};

// Node for the Michael-Scott lock-free queue
template<typename T>
struct LockFreeNode {
    T data;
    std::atomic<LockFreeNode*> next{nullptr};

    template<typename... Args>
    explicit LockFreeNode(Args&&... args)
        : data(std::forward<Args>(args)...)
        , next{nullptr} {}
};

// Michael-Scott style lock-free queue (SPSC-safe, MPMC best-effort)
template<typename T>
class LockFreeQueue {
public:
    LockFreeQueue() {
        LockFreeNode<T>* sentinel = new LockFreeNode<T>();
        head_.store(sentinel, std::memory_order_relaxed);
        tail_.store(sentinel, std::memory_order_relaxed);
    }

    ~LockFreeQueue() {
        while (try_dequeue().has_value()) {}  // drain remaining
        delete head_.load(std::memory_order_relaxed);  // delete sentinel
    }

    // Enqueue an item (lock-free)
    void enqueue(T item) {
        LockFreeNode<T>* node = new LockFreeNode<T>(std::move(item));
        while (true) {
            LockFreeNode<T>* tail = tail_.load(std::memory_order_acquire);
            LockFreeNode<T>* next = tail->next.load(std::memory_order_acquire);
            if (tail == tail_.load(std::memory_order_acquire)) {
                if (next == nullptr) {
                    // tail->next is null, try to link our node
                    if (tail->next.compare_exchange_weak(
                            next, node,
                            std::memory_order_release,
                            std::memory_order_relaxed)) {
                        // Successfully linked; try to swing tail to our node
                        tail_.compare_exchange_weak(
                            tail, node,
                            std::memory_order_release,
                            std::memory_order_relaxed);
                        return;
                    }
                } else {
                    // Tail is falling behind; help advance it
                    tail_.compare_exchange_weak(
                        tail, next,
                        std::memory_order_release,
                        std::memory_order_relaxed);
                }
            }
        }
    }

    // Dequeue an item (lock-free). Returns nullopt if empty.
    std::optional<T> try_dequeue() {
        while (true) {
            LockFreeNode<T>* head = head_.load(std::memory_order_acquire);
            LockFreeNode<T>* tail = tail_.load(std::memory_order_acquire);
            LockFreeNode<T>* next = head->next.load(std::memory_order_acquire);

            if (head == head_.load(std::memory_order_acquire)) {
                if (head == tail) {
                    if (next == nullptr) {
                        return std::nullopt;  // Queue is empty
                    }
                    // Tail is behind; help advance it
                    tail_.compare_exchange_weak(
                        tail, next,
                        std::memory_order_release,
                        std::memory_order_relaxed);
                } else {
                    // Read value before CAS, otherwise another dequeue
                    // might free the node before we get the value
                    T value = std::move(next->data);
                    if (head_.compare_exchange_weak(
                            head, next,
                            std::memory_order_release,
                            std::memory_order_relaxed)) {
                        // Successfully dequeued; reclaim old sentinel
                        delete head;
                        return value;
                    }
                }
            }
        }
    }

    bool empty() const {
        LockFreeNode<T>* head = head_.load(std::memory_order_acquire);
        LockFreeNode<T>* next = head->next.load(std::memory_order_acquire);
        return next == nullptr;
    }

    // Non-copyable, non-movable
    LockFreeQueue(const LockFreeQueue&) = delete;
    LockFreeQueue& operator=(const LockFreeQueue&) = delete;

private:
    std::atomic<LockFreeNode<T>*> head_;
    std::atomic<LockFreeNode<T>*> tail_;
};

// --- Demo ---
int main() {
    LockFreeQueue<Task> taskQueue;

    // Enqueue tasks
    taskQueue.enqueue(Task("Greeter", "World", [](const std::string& result) {
        std::cout << "Callback: " << result << "\n";
    }));
    taskQueue.enqueue(Task("Greeter", "C++", [](const std::string& result) {
        std::cout << "Callback: " << result << "\n";
    }));

    // Dequeue tasks
    int count = 0;
    while (auto task = taskQueue.try_dequeue()) {
        std::cout << "Dequeued task for plugin: "
                  << task->plugin_name
                  << ", payload: " << task->payload << "\n";
        if (task->callback) {
            task->callback("Result of " + task->payload);
        }
        ++count;
    }
    std::cout << "Processed " << count << " tasks\n";

    return 0;
}

Explanation:

  • This is a Michael-Scott queue: a sentinel node at the head allows the head and tail pointers to be updated independently without ABA problems.
  • enqueue: appends to the tail. Uses compare_exchange_weak on tail->next to link the new node. If successful, tries to advance the tail (non-critical if it fails, another thread will help).
  • try_dequeue: reads head->next and moves the value before the CAS. This prevents another thread from freeing the node while we are reading it.
  • Memory ordering: acquire on loads and release on stores ensure that all writes to a node are visible before the node is published or consumed.
  • Each try_dequeue reclaims the old sentinel node (the previous head). This is safe for single-consumer; for multi-consumer you would need hazard pointers or epoch-based reclamation.

Important: The Michael-Scott queue as implemented above is safe for single-producer-single-consumer (SPSC) and work-conserving for MPMC. However, without a memory reclamation scheme (hazard pointers, epoch-based reclamation, or RCU), the multi-consumer case risks use-after-free. For production MPMC, add hazard pointers or use a reference-counted node approach.

4. Add CRTP-Based Plugin Lifecycle

Click to reveal Solution

#include <iostream>
#include <string>
#include <chrono>
#include <sstream>
#include <iomanip>
#include <atomic>

// CRTP base: provides logging, timing, and reference counting.
// Zero virtual overhead -- all calls are resolved at compile time.
template<typename Derived>
class PluginLifecycle {
public:
    void on_load() {
        auto start = std::chrono::steady_clock::now();
        derived().on_load_impl();
        auto end = std::chrono::steady_clock::now();
        auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
        log("Loaded in " + std::to_string(ms.count()) + "ms");
    }

    void on_unload() {
        auto start = std::chrono::steady_clock::now();
        derived().on_unload_impl();
        auto end = std::chrono::steady_clock::now();
        auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
        log("Unloaded in " + std::to_string(ms.count()) + "ms");
    }

    std::string execute(const std::string& task) {
        auto start = std::chrono::steady_clock::now();
        std::string result = derived().execute_impl(task);
        auto end = std::chrono::steady_clock::now();
        auto us = std::chrono::duration_cast<std::chrono::microseconds>(end - start);
        log("Executed '" + task + "' in " + std::to_string(us.count()) + "us");
        return result;
    }

    // Reference counting (thread-safe)
    void add_ref() { ref_count_.fetch_add(1, std::memory_order_relaxed); }
    void release() {
        if (ref_count_.fetch_sub(1, std::memory_order_acq_rel) == 1) {
            log("Reference count reached zero");
        }
    }
    long ref_count() const {
        return ref_count_.load(std::memory_order_relaxed);
    }

protected:
    // CRTP accessor
    Derived& derived() { return static_cast<Derived&>(*this); }
    const Derived& derived() const {
        return static_cast<const Derived&>(*this);
    }

    void log(const std::string& message) {
        auto now = std::chrono::system_clock::now();
        auto time_t = std::chrono::system_clock::to_time_t(now);
        std::tm tm_buf{};
        localtime_r(&time_t, &tm_buf);
        std::ostringstream oss;
        oss << std::put_time(&tm_buf, "%H:%M:%S")
            << " [" << derived().name() << "] " << message;
        std::cout << oss.str() << "\n";
    }

    std::atomic<long> ref_count_{0};
};

// Concrete plugin using CRTP mixin
class LoggingGreeter : public PluginLifecycle<LoggingGreeter> {
public:
    std::string name() const { return "LoggingGreeter"; }
    std::string version() const { return "2.0.0"; }

    // These are called by the CRTP base; the base wraps them with logging/timing
    void on_load_impl() {
        // Expensive initialization simulated
    }
    void on_unload_impl() {
        // Cleanup simulated
    }
    std::string execute_impl(const std::string& task) {
        return "Hello, " + task + "!";
    }
};

// CRTP-based plugin for data processing
class DataProcessor : public PluginLifecycle<DataProcessor> {
public:
    std::string name() const { return "DataProcessor"; }
    std::string version() const { return "1.3.0"; }

    void on_load_impl() {}
    void on_unload_impl() {}
    std::string execute_impl(const std::string& task) {
        // Reverse the input string as a "processing" example
        return std::string(task.rbegin(), task.rend());
    }
};

int main() {
    LoggingGreeter greeter;
    greeter.add_ref();
    greeter.add_ref();

    greeter.on_load();
    std::cout << "Result: " << greeter.execute("World") << "\n";
    std::cout << "Result: " << greeter.execute("Plugin System") << "\n";
    greeter.release();
    greeter.on_unload();
    greeter.release();

    std::cout << "\n--- DataProcessor ---\n";
    DataProcessor processor;
    processor.on_load();
    std::cout << "Result: " << processor.execute("abcdef") << "\n";
    processor.on_unload();

    return 0;
}

Explanation:

  • PluginLifecycle<Derived> uses the Curiously Recurring Template Pattern (CRTP) to inject logging, timing, and reference counting into any derived plugin class.
  • The base class calls derived().on_load_impl(), derived().execute_impl(), etc. Because the derived type is known at compile time, these calls are statically dispatched -- no vtable, no virtual overhead.
  • Each concrete plugin only implements the *_impl methods. The CRTP base wraps every call with timestamped logging.
  • Reference counting uses std::atomic<long> with relaxed ordering for the counter and acquire-release for the final check, ensuring the cleanup side-effect is visible to the thread that sees the count drop to zero.
  • Because there are no virtual functions, the compiler can inline the entire call chain in optimized builds.

5. Implement the PluginLoader with Factory Pattern

Click to reveal Solution

#include <iostream>
#include <string>
#include <unordered_map>
#include <memory>
#include <functional>
#include <dlfcn.h>

// Abstract factory interface for creating plugins
class IPluginFactory {
public:
    virtual ~IPluginFactory() = default;

    // The returned pointer must be a type satisfying the Plugin concept.
    // We return void* here to avoid forcing a common base class on plugins.
    virtual void* create() = 0;
    virtual std::string plugin_name() const = 0;
};

// Concrete factory for a statically known plugin type
template<typename T>
class PluginFactory : public IPluginFactory {
public:
    void* create() override {
        return static_cast<void*>(new T());
    }
    std::string plugin_name() const override {
        T instance;
        return instance.name();
    }
};

// Macro for static plugin registration (used in each plugin .cpp file)
#define REGISTER_PLUGIN(PluginClass) \
    static bool _registered_##PluginClass = []() { \
        PluginLoader::instance().register_factory( \
            std::string(PluginClass{}.name()), \
            std::make_unique<PluginFactory<PluginClass>>()); \
        return true; \
    }()

// PluginLoader: singleton that owns factories and can load plugins
// either statically (via REGISTER_PLUGIN) or dynamically (via dlopen)
class PluginLoader {
public:
    static PluginLoader& instance() {
        static PluginLoader loader;
        return loader;
    }

    void register_factory(const std::string& name,
                           std::unique_ptr<IPluginFactory> factory) {
        std::lock_guard<std::mutex> lock(mutex_);
        factories_[name] = std::move(factory);
        std::cout << "[PluginLoader] Registered factory: " << name << "\n";
    }

    // Load a plugin by name (must have a registered factory)
    void* load(const std::string& name) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = factories_.find(name);
        if (it == factories_.end()) {
            // Try dynamic loading as a fallback
            return load_dynamic(name);
        }
        return it->second->create();
    }

    // Dynamic loading: open a shared library (.so / .dylib)
    void* load_dynamic(const std::string& lib_path) {
        // POSIX: dlopen opens the shared library
        void* handle = dlopen(lib_path.c_str(), RTLD_NOW);
        if (!handle) {
            std::cerr << "[PluginLoader] dlopen failed: " << dlerror() << "\n";
            return nullptr;
        }

        // Convention: each plugin .so exports a create_plugin() function
        using CreateFn = void* (*)();
        CreateFn create_fn = reinterpret_cast<CreateFn>(dlsym(handle, "create_plugin"));
        if (!create_fn) {
            std::cerr << "[PluginLoader] dlsym failed: " << dlerror() << "\n";
            dlclose(handle);
            return nullptr;
        }

        void* plugin = create_fn();

        // Store handle for cleanup
        handles_[lib_path] = handle;
        std::cout << "[PluginLoader] Dynamically loaded: " << lib_path << "\n";
        return plugin;
    }

    // Unload a dynamic library
    void unload_dynamic(const std::string& lib_path) {
        std::lock_guard<std::mutex> lock(mutex_);
        auto it = handles_.find(lib_path);
        if (it != handles_.end()) {
            dlclose(it->second);
            handles_.erase(it);
            std::cout << "[PluginLoader] Unloaded: " << lib_path << "\n";
        }
    }

    // List all statically registered factory names
    std::vector<std::string> available_factories() const {
        std::lock_guard<std::mutex> lock(mutex_);
        std::vector<std::string> names;
        names.reserve(factories_.size());
        for (const auto& [name, _] : factories_) {
            names.push_back(name);
        }
        return names;
    }

private:
    PluginLoader() = default;

    std::mutex mutex_;
    std::unordered_map<std::string, std::unique_ptr<IPluginFactory>> factories_;
    std::unordered_map<std::string, void*> handles_;  // dlopen handles
};

// --- Demo: static registration ---
struct EchoPlugin {
    std::string name() { return "Echo"; }
    std::string version() { return "1.0.0"; }
    std::string execute(const std::string& task) { return "Echo: " + task; }
    void on_load() { std::cout << "[Echo] on_load\n"; }
    void on_unload() { std::cout << "[Echo] on_unload\n"; }
};

REGISTER_PLUGIN(EchoPlugin);

int main() {
    // The REGISTER_PLUGIN macro has already registered EchoPlugin
    auto& loader = PluginLoader::instance();

    std::cout << "Available factories: ";
    for (const auto& name : loader.available_factories()) {
        std::cout << name << " ";
    }
    std::cout << "\n";

    // Load plugin via factory
    void* raw = loader.load("Echo");
    if (raw) {
        auto* echo = static_cast<EchoPlugin*>(raw);
        echo->on_load();
        std::cout << echo->execute("Hello Factory") << "\n";
        echo->on_unload();
        delete echo;
    }

    // Dynamic loading example (would work with a real .so file):
    // void* dyn = loader.load_dynamic("./plugins/libmath_plugin.so");

    return 0;
}

Explanation:

  • Abstract Factory: IPluginFactory defines a uniform interface for creating plugins. PluginFactory<T> is the concrete factory templated on the plugin type.
  • Static registration: The REGISTER_PLUGIN macro leverages a static variable initializer to self-register at program startup, before main() runs.
  • Dynamic loading: Uses POSIX dlopen/dlsym to load shared libraries at runtime. The convention is that each plugin .so exports a create_plugin() C function.
  • The PluginLoader is a Meyer's Singleton (thread-safe by C++11 guarantee).
  • Handles from dlopen are stored for proper cleanup via dlclose.

Tip: For cross-platform dynamic loading, wrap dlopen/dlsym/dlclose behind an interface. On Windows, use LoadLibrary/GetProcAddress/FreeLibrary instead. Consider using std::unique_ptr with a custom deleter for automatic handle cleanup.

6. Add Network Notification via Sockets

Click to reveal Solution

#include <iostream>
#include <string>
#include <cstring>
#include <stdexcept>
#include <sstream>
#include <iomanip>
#include <chrono>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <unistd.h>

// RAII wrapper for a UDP socket
class UdpNotifier {
public:
    UdpNotifier(const std::string& host, uint16_t port)
        : host_(host), port_(port)
    {
        sock_fd_ = socket(AF_INET, SOCK_DGRAM, 0);
        if (sock_fd_ < 0) {
            throw std::runtime_error("Failed to create socket: " +
                                     std::string(strerror(errno)));
        }

        // Resolve destination address
        std::memset(&dest_addr_, 0, sizeof(dest_addr_));
        dest_addr_.sin_family = AF_INET;
        dest_addr_.sin_port = htons(port_);
        if (inet_pton(AF_INET, host_.c_str(), &dest_addr_.sin_addr) <= 0) {
            close(sock_fd_);
            throw std::runtime_error("Invalid address: " + host_);
        }

        std::cout << "[UdpNotifier] Configured: " << host_
                  << ":" << port << "\n";
    }

    ~UdpNotifier() {
        if (sock_fd_ >= 0) {
            close(sock_fd_);
            std::cout << "[UdpNotifier] Socket closed\n";
        }
    }

    // Send a notification message to the monitoring endpoint
    void notify(const std::string& plugin_name,
                const std::string& task,
                const std::string& result) {
        std::ostringstream oss;
        auto now = std::chrono::system_clock::now();
        auto time_t = std::chrono::system_clock::to_time_t(now);
        std::tm tm_buf{};
        localtime_r(&time_t, &tm_buf);

        oss << std::put_time(&tm_buf, "%Y-%m-%dT%H:%M:%S")
            << " plugin=\"" << plugin_name << "\""
            << " task=\"" << task << "\""
            << " result=\"" << result << "\""
            << " status=\"complete\"";

        std::string message = oss.str();

        ssize_t sent = sendto(sock_fd_,
                              message.c_str(),
                              message.size(),
                              0,
                              reinterpret_cast<const sockaddr*>(&dest_addr_),
                              sizeof(dest_addr_));

        if (sent < 0) {
            std::cerr << "[UdpNotifier] sendto failed: "
                      << strerror(errno) << "\n";
        } else {
            std::cout << "[UdpNotifier] Sent " << sent
                      << " bytes to " << host_
                      << ":" << port_ << "\n";
        }
    }

    // Non-copyable, non-movable
    UdpNotifier(const UdpNotifier&) = delete;
    UdpNotifier& operator=(const UdpNotifier&) = delete;
    UdpNotifier(UdpNotifier&&) = delete;
    UdpNotifier& operator=(UdpNotifier&&) = delete;

private:
    std::string host_;
    uint16_t port_;
    int sock_fd_ = -1;
    struct sockaddr_in dest_addr_{};
};

// --- Demo ---
int main() {
    try {
        // Send notifications to localhost:9999 (run a UDP listener there)
        UdpNotifier notifier("127.0.0.1", 9999);

        // Simulate plugin completions
        notifier.notify("Greeter", "World", "Hello, World!");
        notifier.notify("DataProcessor", "abc123", "321cba");
        notifier.notify("Echo", "test", "Echo: test");

        std::cout << "\nTip: run `nc -u -l 9999` to receive the UDP "
                  << "messages in another terminal.\n";
    } catch (const std::exception& e) {
        std::cerr << "Error: " << e.what() << "\n";
        return 1;
    }

    return 0;
}

Explanation:

  • UdpNotifier is an RAII wrapper around a POSIX UDP socket. The socket is created in the constructor and closed in the destructor, ensuring no resource leaks even on exceptions.
  • socket(AF_INET, SOCK_DGRAM, 0) creates a UDP (datagram) socket. Unlike TCP, UDP does not require a connection -- each sendto is independent.
  • inet_pton converts the IP address string to binary form. htons converts the port to network byte order.
  • The notify method formats a structured message with timestamp, plugin name, task, and result -- suitable for parsing by a monitoring service.
  • UDP is chosen over TCP because notifications are fire-and-forget: the plugin system should not block waiting for a monitor acknowledgement.

Production Notes:

  • Add retry logic with exponential backoff if the monitoring endpoint is temporarily unreachable.
  • Consider using a local buffer or a lock-free queue for notifications to decouple the plugin execution from network I/O.
  • Set SO_SNDBUF socket option if sending high-volume notifications to avoid kernel buffer drops.

// Step 1: Plugin concept
template<typename T>
concept Plugin = requires(T t, const T ct) {
    { t.name() } -> std::convertible_to<std::string_view>;
    { t.version() } -> std::convertible_to<std::string_view>;
    { t.execute(std::string{}) } -> std::convertible_to<std::string>;
    { t.on_load() } -> std::same_as<void>;
    { t.on_unload() } -> std::same_as<void>;
};

// Step 2: Type-erased wrapper (simplified)
class TypeErasedPlugin {
public:
    template<Plugin T>
    explicit TypeErasedPlugin(T plugin)
        : name_fn_([p = std::move(plugin)]() mutable { return p.name(); })
        , execute_fn_([p = std::move(plugin)](std::string t) mutable {
              return p.execute(std::move(t));
          })
        , on_load_fn_([p = std::move(plugin)]() mutable { p.on_load(); })
        , on_unload_fn_([p = std::move(plugin)]() mutable { p.on_unload(); })
    {}

    std::string name()              { return name_fn_(); }
    std::string execute(std::string t) { return execute_fn_(std::move(t)); }
    void on_load()                  { on_load_fn_(); }
    void on_unload()                { on_unload_fn_(); }

private:
    std::function<std::string()>               name_fn_;
    std::function<std::string(std::string)>      execute_fn_;
    std::function<void()>                     on_load_fn_;
    std::function<void()>                     on_unload_fn_;
};

// Step 3: Lock-free queue (simplified -- uses mutex for this demo)
struct Task {

    std::string payload;

// Step 6: UdpNotifier (simplified inline)
class UdpNotifier {
public:
    UdpNotifier(const std::string& host, uint16_t port) {
        sock_fd_ = socket(AF_INET, SOCK_DGRAM, 0);
        if (sock_fd_ < 0) throw std::runtime_error("socket() failed");
        std::memset(&dest_addr_, 0, sizeof(dest_addr_));
        dest_addr_.sin_family = AF_INET;
        dest_addr_.sin_port = htons(port);
        inet_pton(AF_INET, host.c_str(), &dest_addr_.sin_addr);
    }
    ~UdpNotifier() { if (sock_fd_ >= 0) close(sock_fd_); }

    void notify(const std::string& plugin_name,
                const std::string& task,
                const std::string& result) {
        std::string msg = "plugin=" + plugin_name +
                          " task=" + task +
                          " result=" + result;
        sendto(sock_fd_, msg.c_str(), msg.size(), 0,
               reinterpret_cast<const sockaddr*>(&dest_addr_),
               sizeof(dest_addr_));
    }
private:
    int sock_fd_ = -1;
    struct sockaddr_in dest_addr_{};
};

// ---- Global state for signal handling ----
std::atomic<bool> g_running{true};

void signal_handler(int signum) {
    std::cout << "\n[Signal] Received signal " << signum
              << ", shutting down...\n";
    g_running.store(false, std::memory_order_release);
}

// ---- PluginManager ----
class PluginManager {
public:
    template<Plugin T>
    void register_plugin(T plugin) {
        std::string n = plugin.name();
        plugins_[n] = std::make_unique<TypeErasedPlugin>(std::move(plugin));
        plugins_[n]->on_load();
    }

    void unregister_all() {
        for (auto& [name, p] : plugins_) p->on_unload();
        plugins_.clear();
    }

    std::string dispatch(const std::string& name, const std::string& task) {
        auto it = plugins_.find(name);
        if (it == plugins_.end()) return "ERROR: plugin '" + name + "' not found";
        return it->second->execute(task);
    }

private:
    std::unordered_map<std::string, std::unique_ptr<TypeErasedPlugin>> plugins_;
};

// ---- Worker thread: drains the lock-free queue and dispatches tasks ----
void worker_thread(PluginManager& manager, UdpNotifier& notifier) {
    while (g_running.load(std::memory_order_acquire)) {
        auto task = taskQueue.try_dequeue();
        if (!task) {
            std::this_thread::sleep_for(std::chrono::milliseconds(10));
            continue;
        }

        std::cout << "[Worker] Dispatching to " << task->plugin_name
                  << ": " << task->payload << "\n";

        std::string result = manager.dispatch(task->plugin_name, task->payload);
        std::cout << "[Worker] Result: " << result << "\n";

        // Send UDP notification
        notifier.notify(task->plugin_name, task->payload, result);

        // Invoke callback
        if (task->callback) {
            task->callback(result);
        }
    }
    std::cout << "[Worker] Exiting\n";
}

// ---- main() ----
int main() {
    // Register signal handlers for graceful shutdown
    std::signal(SIGINT, signal_handler);
    std::signal(SIGTERM, signal_handler);

    std::cout << "=== Compile-Time Plugin System ===\n\n";

    // 1. Set up UDP notification (to localhost:9999)
    std::unique_ptr<UdpNotifier> notifier;
    try {
        notifier = std::make_unique<UdpNotifier>("127.0.0.1", 9999);
    } catch (const std::exception& e) {
        std::cout << "[Warning] UDP notifier unavailable: "
                  << e.what() << "\n";
        std::cout << "Continuing without notifications...\n\n";
    }

    // 2. Register plugins (concept-constrained)
    PluginManager manager;
    manager.register_plugin(GreeterPlugin{});
    manager.register_plugin(UpperPlugin{});
    manager.register_plugin(ReversePlugin{});
    std::cout << "\n";

    // 3. Start worker threads
    const int num_workers = 2;
    std::vector<std::thread> workers;
    for (int i = 0; i < num_workers; ++i) {
        workers.emplace_back([&manager, &notifier]() {
            if (notifier) {
                worker_thread(manager, *notifier);
            } else {
                // Dummy notifier fallback
                UdpNotifier dummy("127.0.0.1", 1);
                worker_thread(manager, dummy);
            }
        });
    }

    // 4. Submit tasks
    auto submit = [](std::string plugin, std::string payload,
                      std::function<void(const std::string&)> cb = {}) {
        taskQueue.enqueue(Task{
            std::move(plugin),
            std::move(payload),
            std::move(cb)
        });
    };

    submit("Greeter", "World", [](const std::string& r) {
        std::cout << "[Callback] " << r << "\n";
    });
    submit("UpperCase", "hello plugin system");
    submit("Reverse", "abcdef");
    submit("Greeter", "C++ Advanced");
    submit("UpperCase", "lock-free queue");
    submit("Reverse", "Compile-Time Plugin System");

    std::cout << "[Main] Submitted 6 tasks\n\n";

    // 5. Wait for tasks to drain or shutdown signal
    int drain_count = 0;
    while (g_running.load(std::memory_order_acquire) && drain_count < 100) {
        if (taskQueue.empty()) {
            // Give workers time to process the last items
            std::this_thread::sleep_for(std::chrono::milliseconds(200));
            if (taskQueue.empty()) break;
        }
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
        ++drain_count;
    }

    // 6. Graceful shutdown
    std::cout << "\n=== Shutting down ===\n";
    g_running.store(false, std::memory_order_release);
    for (auto& w : workers) w.join();

    manager.unregister_all();
    std::cout << "\n=== Shutdown complete ===\n";

    return 0;
}

Explanation:

  • Signal handling: std::signal registers handlers for SIGINT (Ctrl+C) and SIGTERM. The handler sets a global std::atomic<bool> flag. All threads check this flag and exit cooperatively.
  • Plugin registration: register_plugin is constrained by the Plugin concept, so only valid plugins compile through.
  • Worker threads: Two worker threads drain the lock-free queue, dispatch tasks to the PluginManager, send UDP notifications, and invoke callbacks.
  • Graceful shutdown: After submitting tasks, main waits for the queue to drain (with a timeout). Then it sets g_running = false and joins all worker threads. Finally, unregister_all() calls on_unload() on each plugin.
  • The UdpNotifier is created with a try/catch so the system continues to function even if the notification endpoint is unavailable.

Follow-up Questions

Variations to Try

  1. Hot-reloadable plugins: Extend the PluginLoader to support reloading a plugin from an updated .so without restarting the process. How do you handle in-flight tasks that reference the old plugin instance?
  2. Coroutine-based task dispatch: Replace the lock-free queue with a C++20 coroutine task system where each execute() call is a co_await-able coroutine, allowing plugins to suspend and resume mid-execution.
  3. Custom allocator for plugin nodes: Use a pool allocator for the lock-free queue nodes to reduce allocation contention. Benchmark against the default new allocator under high-throughput conditions.
  4. Observer pattern for plugin events: Add an observer/event system so that other components can subscribe to plugin load, unload, and task-completion events without the PluginManager needing to know about them.

Key Takeaways:

  • C++20 concepts provide compile-time interface enforcement that is more expressive and readable than SFINAE or static_assert alone.
  • Type erasure (via std::function, std::any, or custom wrappers) enables heterogeneous storage without forcing a common base class, preserving the ability to use CRTP and value semantics in concrete plugins.
  • Lock-free data structures eliminate mutex contention on hot paths but require careful attention to memory ordering and memory reclamation. The Michael-Scott queue is a foundational pattern.
  • CRTP lets you inject cross-cutting concerns (logging, timing, reference counting) at compile time with zero virtual overhead, making it ideal for performance-critical plugin systems.
  • Abstract factory decouples plugin creation from usage, enabling both static registration (for known plugins) and dynamic loading (for third-party or hot-swappable plugins).
  • Socket programming adds observability to the plugin system. UDP is the right choice for fire-and-forget notifications where reliability is not critical.
  • Signal-safe shutdown with std::atomic flags ensures that the system can be stopped gracefully without leaving resources leaked or threads orphaned.
