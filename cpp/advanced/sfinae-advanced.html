SFINAE & enable_if Advanced

SFINAE (Substitution Failure Is Not An Error) is the C++ mechanism that allows the compiler to discard invalid template overloads during overload resolution rather than producing a hard error. Advanced SFINAE patterns form the foundation of type traits and generic library design in pre-C++20 code. In C++20, concepts largely replace SFINAE, but understanding these patterns is essential for reading and maintaining legacy code.

Prerequisites: SFINAE Basics, Type Traits, Templates, Variadic Templates.

The void_t Idiom

void_t is a metafunction that maps any list of types to void. Combined with SFINAE, it detects whether a type has a particular member or nested type:

#include <type_traits>
#include <iostream>

// C++17: std::void_t is in <type_traits>
// Detect if T has a .size() method
template<typename T, typename = void>
struct has_size : std::false_type {};

template<typename T>
struct has_size<T, std::void_t<decltype(std::declval<T>().size())>>
    : std::true_type {};

// Detect if T has a nested type::iterator
template<typename T, typename = void>
struct has_iterator_type : std::false_type {};

template<typename T>
struct has_iterator_type<T, std::void_t<typename T::iterator>>
    : std::true_type {};

// Detect multiple members at once
template<typename T, typename = void>
struct is_container : std::false_type {};

template<typename T>
struct is_container<T, std::void_t<
    typename T::value_type,
    typename T::iterator,
    decltype(std::declval<T>().begin()),
    decltype(std::declval<T>().end()),
    decltype(std::declval<T>().size())
>> : std::true_type {};

int main() {
    std::cout << std::boolalpha;
    std::cout << has_size<std::vector<int>>::value << "\n";  // true
    std::cout << has_size<int>::value << "\n";                 // false
    std::cout << is_container<std::vector<int>>::value << "\n";  // true
    std::cout << is_container<int>::value << "\n";                // false
}

// SFINAE: different overloads based on type properties
template<typename T>
std::enable_if_t<std::is_integral_v<T>, std::string>
to_string(T val) {
    return std::to_string(val);
}

template<typename T>
std::enable_if_t<std::is_floating_point_v<T>, std::string>
to_string(T val) {
    char buf[64];
    std::snprintf(buf, sizeof(buf), "%.6f", val);
    return buf;
}

// if constexpr: single function, internal branching
template<typename T>
std::string to_string_modern(T val) {
    if constexpr (std::is_integral_v<T>) {
        return std::to_string(val);
    } else if constexpr (std::is_floating_point_v<T>) {
        char buf[64];
        std::snprintf(buf, sizeof(buf), "%.6f", val);
        return buf;
    } else {
        return std::string(val);
    }
}

Detection Idiom (C++17)

The detection idiom (from Library Fundamentals TS v2) is a generalization of void_t that provides a cleaner interface for checking type properties:

#include <type_traits>

// Simplified detection idiom implementation
namespace detail {
    template<typename Default, typename AlwaysVoid,
             template<typename...> class Op, typename... Args>
    struct detector {
        using value_t = std::false_type;
        using type = Default;
    };

    template<typename Default, template<typename...> class Op, typename... Args>
    struct detector<Default, std::void_t<Op<Args...>>, Op, Args...> {
        using value_t = std::true_type;
        using type = Op<Args...>;
    };
}

// Convenient aliases
template<template<typename...> class Op, typename... Args>
using is_detected = typename detail::detector<void, void, Op, Args...>::value_t;

template<template<typename...> class Op, typename... Args>
using detected_t = typename detail::detector<void, void, Op, Args...>::type;

template<typename Default, template<typename...> class Op, typename... Args>
using detected_or = typename detail::detector<Default, void, Op, Args...>::type;

// Define detectors for specific properties
template<typename T>
using has_size_t = decltype(std::declval<T>().size());

template<typename T>
using has_reserve_t = decltype(std::declval<T>().reserve(std::size_t{}));

template<typename T>
using has_begin_end_t = decltype(
    std::declval<T>().begin(),
    std::declval<T>().end()
);

// Usage
static_assert(is_detected<has_size_t, std::vector<int>>::value);
static_assert(!is_detected<has_size_t, int>::value);
static_assert(is_detected<has_reserve_t, std::vector<int>>::value);
static_assert(!is_detected<has_reserve_t, std::list<int>>::value);

// Fallback type: detected_or provides a default if detection fails
using default_size_type = detected_or<std::size_t, has_size_t, int>;

Real-World SFINAE Patterns

Conditional Member Function Enablement

class Serializer {
public:
    // Only available if T is arithmetic
    template<typename T>
    std::enable_if_t<std::is_arithmetic_v<T>>
    write(T value) {
        // Direct binary write for arithmetic types
    }

    // Only available if T has a serialize() method
    template<typename T>
    std::enable_if_t<!std::is_arithmetic_v<T> &&
        is_detected<decltype(std::declval<T>().serialize()), T>::value>
    write(const T& obj) {
        obj.serialize(buffer_);
    }
};

// Modern equivalent with concepts (C++20):
class SerializerModern {
public:
    template<std::integral T>
    void write(T value) { /* ... */ }

    template<typename T> requires requires(const T& t) { t.serialize(); }
    void write(const T& obj) { obj.serialize(buffer_); }
};

SFINAE with Return Type

// Return type SFINAE — cannot be replaced with if constexpr
template<typename Container>
auto front(const Container& c)
    -> decltype(c.front())  // SFINAE: removes overload if no .front()
{
    return c.front();
}

template<typename Container>
auto front(const Container& c)
    -> decltype(*c.begin())  // SFINAE: removes overload if no .begin()
{
    return *c.begin();
}

Migration path: In new code, prefer C++20 Concepts over SFINAE. Concepts are more readable, produce better error messages, and integrate with overload resolution. Use requires clauses and if constexpr instead of enable_if and void_t.

Common mistake: Putting SFINAE in the wrong place. enable_if in a default template parameter affects overload resolution. enable_if in a function parameter creates an implicit conversion. Always use the default template parameter form unless you specifically need parameter-based SFINAE.
