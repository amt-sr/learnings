Compile-Time Computation

Compile-time computation moves work from runtime to build time, producing zero-overhead results baked into the binary. C++ supports two primary mechanisms: constexpr/consteval functions and template metaprogramming (TMP). Both achieve the same goal but with different trade-offs in readability, expressiveness, and compilation speed.

Prerequisites: Advanced constexpr, CRTP, Type Erasure, Templates.

constexpr Functions vs Template Metaprogramming

Aspectconstexpr FunctionsTemplate Metaprogramming
SyntaxNormal function syntax with constexprStruct templates with specializations
ReadabilityHigh — looks like regular codeLow — recursive instantiation

Compilation speedFaster (evaluated once)Slower (template instantiation)

Compile-Time Fibonacci

constexpr Approach (Recommended)

#include <iostream>

// C++14 constexpr — full control flow, loops
constexpr long long fibonacci(int n) {
    if (n <= 1) return n;
    long long a = 0, b = 1;
    for (int i = 2; i <= n; ++i) {
        long long c = a + b;
        a = b;
        b = c;
    }
    return b;
}

// Values computed at compile time
constexpr long long fib10 = fibonacci(10);   // 55
constexpr long long fib20 = fibonacci(20);   // 6765
constexpr long long fib50 = fibonacci(50);  // 12586269025

int main() {
    // These are compile-time constants — no runtime cost
    std::cout << "fib(10) = " << fib10 << "\n";
    std::cout << "fib(20) = " << fib20 << "\n";
    std::cout << "fib(50) = " << fib50 << "\n";

    // fibonacci() can also be called at runtime
    int n;
    std::cin >> n;
    std::cout << "fib(" << n << ") = " << fibonacci(n) << "\n";
}

Template Metaprogramming Approach (Legacy)

// Classic TMP: recursive template instantiation
template<int N>
struct Fibonacci {
    // Compile-time recursion
    static constexpr long long value =
        Fibonacci<N - 1>::value + Fibonacci<N - 2>::value;
};

// Base cases via template specialization
template<> struct Fibonacci<0> { static constexpr long long value = 0; };
template<> struct Fibonacci<1> { static constexpr long long value = 1; };

// Usage
constexpr long long fib10 = Fibonacci<10>::value;  // 55

// Note: O(N^2) instantiations — slow compile for large N

Compilation cost: Template Fibonacci instantiates O(2^N) templates (with memoization it becomes O(N), but the compiler cache is not guaranteed). The constexpr approach is O(N) with minimal compile overhead. Prefer constexpr for value computation.

Compile-Time Factorial

#include <iostream>
#include <type_traits>

// constexpr factorial
constexpr unsigned long long factorial(unsigned int n) {
    unsigned long long result = 1;
    for (unsigned int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

// Template factorial (type-safe, can be used as template parameter)
template<unsigned int N>
struct Factorial {
    static constexpr unsigned long long value = N * Factorial<N - 1>::value;
};
template<> struct Factorial<0> { static constexpr unsigned long long value = 1; };

// Generate compile-time array of factorials
template<std::size_t... Is>
constexpr auto make_factorial_table(std::index_sequence<Is...>) {
    return std::array<unsigned long long, sizeof...(Is)>{
        {factorial(Is)...}
    };
}

constexpr auto factorials = make_factorial_table(
    std::make_index_sequence<21>{}  // 0! through 20!
);

int main() {
    for (std::size_t i = 0; i < factorials.size(); ++i) {
        std::cout << i << "! = " << factorials[i] << "\n";
    }
}

Compile-Time String Processing

#include <array>
#include <string_view>
#include <algorithm>

// Fixed-size compile-time string
template<std::size_t N>
struct CompileTimeString {
    char data[N]{};

    constexpr CompileTimeString(const char (&str)[N]) {
        for (std::size_t i = 0; i < N; ++i)
            data[i] = str[i];
    }

    constexpr std::size_t size() const { return N - 1; }
    constexpr std::string_view view() const { return {data, N - 1}; }
};

// Compile-time string concatenation
template<std::size_t N1, std::size_t N2>
constexpr auto ct_concat(const char (&a)[N1], const char (&b)[N2]) {
    std::array<char, N1 + N2 - 1> result{};
    for (std::size_t i = 0; i < N1 - 1; ++i) result[i] = a[i];
    for (std::size_t i = 0; i < N2 - 1; ++i) result[N1 - 1 + i] = b[i];
    return result;
}

// Compile-time CRC32 table generation
constexpr std::uint32_t crc32_table(std::size_t index) {
    std::uint32_t crc = static_cast<std::uint32_t>(index);
    for (int i = 0; i < 8; ++i) {
        if (crc & 1)
            crc = (crc >> 1) ^ 0xEDB88320u;
        else
            crc >>= 1;
    }
    return crc;
}

template<std::size_t N = 256>
struct CRC32Table {
    std::array<std::uint32_t, N> table{};

    constexpr CRC32Table() {
        for (std::size_t i = 0; i < N; ++i)
            table[i] = crc32_table(i);
    }

    constexpr std::uint32_t operator()(std::string_view data) const {
        std::uint32_t crc = 0xFFFFFFFF;
        for (char c : data) {
            crc = (crc >> 8) ^ table[(crc ^ static_cast<std::uint32_t>(c)) & 0xFF];
        }
        return crc ^ 0xFFFFFFFF;
    }
};

// Entire CRC table computed at compile time
constexpr CRC32Table crc32{};

// Compile-time string hashing
constexpr std::uint64_t fnv1a(const char* str, std::uint64_t hash = 14695981039346656037ull) {
    return *str ? fnv1a(str + 1, (hash ^ static_cast<std::uint64_t>(*str)) * 1099511628211ull) : hash;
}

// Use as template parameter for type dispatch
template<auto Hash>
struct MessageType {};

void process(MessageType<fnv1a("RRC_SETUP")>) { /* ... */ }
void process(MessageType<fnv1a("RRC_RELEASE")>) { /* ... */ }

Compile-Time Type Lists

// Type list — a fundamental TMP building block
template<typename... Ts>
struct TypeList {};

// Count types in a list
template<typename TList>
struct TypeCount;

template<typename... Ts>
struct TypeCount<TypeList<Ts...>> {
    static constexpr std::size_t value = sizeof...(Ts);
};

// Find index of a type in a type list
template<typename T, typename TList>
struct TypeIndex;

template<typename T, typename... Ts>
struct TypeIndex<T, TypeList<T, Ts...>> {
    static constexpr std::size_t value = 0;
};

template<typename T, typename Head, typename... Tail>
struct TypeIndex<T, TypeList<Head, Tail...>> {
    static constexpr std::size_t value = 1 + TypeIndex<T, TypeList<Tail...>>::value;
};

// Usage
using MyTypes = TypeList<int, double, char, std::string>;
static_assert(TypeCount<MyTypes>::value == 4);
static_assert(TypeIndex<double, MyTypes>::value == 1);
static_assert(TypeIndex<std::string, MyTypes>::value == 3);

When to Use TMP Over constexpr

  • Use constexpr for computing values (Fibonacci, CRC tables, hash functions, mathematical constants)
  • Use TMP for type manipulation (selecting types, building type lists, conditional type selection)
  • Use concepts (C++20) for type constraints instead of SFINAE-based TMP
  • Use if constexpr for compile-time branching within constexpr functions

Best practice: Modern C++ (C++14+) makes constexpr the default choice for compile-time value computation. Reserve template metaprogramming for operations on types themselves. The C++20 consteval keyword enforces compile-time-only evaluation when you need a hard guarantee.

Watch out: Large constexpr computations can exceed the compiler's default step limit. GCC defaults to 33554432 steps. Increase with -fconstexpr-ops-limit=N or -fconstexpr-loop-limit=N if needed. excessively deep template recursion will produce extremely long error messages.
