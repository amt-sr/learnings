Clang Static Analyzer

The Clang Static Analyzer is a path-sensitive analysis tool built into the Clang compiler infrastructure. Unlike clang-tidy (which performs AST-based pattern matching), the analyzer performs symbolic execution -- it explores execution paths through the program, tracking symbolic values to find bugs that only manifest under specific conditions.

What Makes It Different

Path-sensitive analysis means the analyzer tracks the state of the program along each possible execution path. It can determine that a bug is reachable only if a specific branch is taken:

void process(int* ptr) {
    if (!ptr) {
        // Path 1: ptr is null
        return;  // returns early -- safe
    }
    // Path 2: ptr is non-null -- analyzer knows ptr != nullptr here
    int value = *ptr;  // no warning on this path

    if (value < 0) {
        // Path 2a: value is negative
        log_error(value);  // Path-specific reasoning
    }
    // Path 2b: value is non-negative -- analyzer tracks this
}

Running the Analyzer with scan-build

# Basic analysis (replaces the compiler invocation)
scan-build cmake --build build

# Run with specific checks enabled
scan-build -enable-checker core,unix,osx cmake --build build

# Run with specific checks disabled
scan-build -disable-checker deadcode.DeadStores cmake --build build

# Keep analyzer results even if build succeeds
scan-build -keep-cc cmake --build build

# View results in browser
scan-view /tmp/scan-build-2024-01-15-123456-1234-1/

Checker Categories

CategoryChecksWhat It Finds
coreCallAndMessage, DivZero, DynamicTypePropagation, NullDereference, StackAddrEscape, UndefResult, UninitializedObjectNull dereferences, division by zero, undefined results, stack address leaks

Enabling Specific Checkers

# Run only core null dereference and memory leak checks
scan-build \
  -enable-checker core.NullDereference \
  -enable-checker cplusplus.NewDeleteLeaks \
  cmake --build build

# Run all checkers in a category
scan-build -enable-checker core \
  -enable-checker security \
  cmake --build build

Integration with CMake

# CMakeLists.txt -- configure CMAKE_CXX_COMPILER_LAUNCHER
# Method 1: Using scan-build wrapper
# Run: scan-build cmake -B build && scan-build cmake --build build

# Method 2: CMake option to enable analyzer
option(ENABLE_ANALYZER "Run Clang Static Analyzer" OFF)

if(ENABLE_ANALYZER)
    find_program(SCAN_BUILD scan-build)
    if(SCAN_BUILD)
        set(CMAKE_C_COMPILER "${SCAN_BUILD}" "${CMAKE_C_COMPILER}")
        set(CMAKE_CXX_COMPILER "${SCAN_BUILD}" "${CMAKE_CXX_COMPILER}")
    endif()
endif()

Performance note: The analyzer is significantly slower than cppcheck or clang-tidy because it performs deep path-sensitive analysis. Expect analysis times of 2-10x the normal build time. Run it on CI, not on every local build.

Strengths: The Clang Static Analyzer excels at finding bugs that require tracking state across function calls -- use-after-return, double-free, null pointer propagation through conditionals, and inter-procedural memory leaks. These are exactly the kinds of bugs that undefined behavior detection tools complement.
