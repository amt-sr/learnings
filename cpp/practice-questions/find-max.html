Find Maximum Element in an Array

Topics Covered: Arrays, STL Algorithms, Iterators

Problem Statement

Given an array (or any container) of integers, find the maximum element. Provide both a manual loop implementation and a solution using std::max_element from the STL.

Examples:

  • {3, 7, 1, 9, 4} → 9
  • {-5, -2, -9} → -2
  • {42} → 42

Edge Cases to Consider:

  • Empty container — how do you signal an error?
  • All identical elements.
  • Negative numbers only.
  • Single-element container.

Manual Loop Approach

Click to reveal Solution

#include <iostream>
#include <vector>
#include <optional>
#include <stdexcept>

// Using std::optional to handle empty container safely (C++17)
std::optional<int> findMaxManual(const std::vector<int>& arr) {
    if (arr.empty()) {
        return std::nullopt;
    }

    int maxVal = arr[0];
    for (std::size_t i = 1; i < arr.size(); ++i) {
        if (arr[i] > maxVal) {
            maxVal = arr[i];
        }
    }
    return maxVal;
}

// Alternative: using iterators
std::optional<int> findMaxIterators(const std::vector<int>& arr) {
    if (arr.empty()) return std::nullopt;

    auto maxIt = arr.begin();
    for (auto it = arr.begin() + 1; it != arr.end(); ++it) {
        if (*it > *maxIt) {
            maxIt = it;
        }
    }
    return *maxIt;
}

// Template version for any container
template<typename T>
std::optional<T> findMaxTemplate(const std::vector<T>& arr) {
    if (arr.empty()) return std::nullopt;

    T maxVal = arr[0];
    for (const auto& elem : arr) {
        if (elem > maxVal) {
            maxVal = elem;
        }
    }
    return maxVal;
}

int main() {
    std::vector<int> nums = {3, 7, 1, 9, 4};

    auto result = findMaxManual(nums);
    if (result) {
        std::cout << "Max (manual): " << *result << "\n";  // 9
    }

    auto result2 = findMaxTemplate(nums);
    if (result2) {
        std::cout << "Max (template): " << *result2 << "\n";  // 9
    }

    // Empty case
    std::vector<int> empty;
    auto emptyResult = findMaxManual(empty);
    if (!emptyResult) {
        std::cout << "Empty container: no max\n";
    }

    return 0;
}

Explanation:

  • We initialize maxVal with the first element, then scan the rest.
  • std::optional<int> is used to cleanly handle the empty container case without exceptions or sentinel values.
  • The template version works with any comparable type (int, double, float, etc.).
  • The range-based for loop version is the most idiomatic modern C++.

Using std::max_element

Click to reveal Solution

#include <iostream>
#include <vector>
#include <algorithm>   // std::max_element
#include <climits>

int main() {
    std::vector<int> nums = {3, 7, 1, 9, 4};

    // std::max_element returns an iterator to the max element
    auto maxIt = std::max_element(nums.begin(), nums.end());
    if (maxIt != nums.end()) {
        std::cout << "Max element: " << *maxIt << "\n";        // 9
        std::cout << "At index: " << (maxIt - nums.begin()) << "\n";  // 3
    }

    // Works with C-arrays too
    int arr[] = {5, 2, 8, 1, 7};
    int* maxPtr = std::max_element(std::begin(arr), std::end(arr));
    std::cout << "Max in C-array: " << *maxPtr << "\n";  // 8

    // With a custom comparator (find max by absolute value)
    std::vector<int> vals = {-10, 3, -7, 5};
    auto absMax = std::max_element(vals.begin(), vals.end(),
        [](int a, int b) { return std::abs(a) < std::abs(b); });
    std::cout << "Max by abs value: " << *absMax << "\n";  // -10

    return 0;
}

Explanation:

  • std::max_element returns an iterator to the maximum element, not the value itself. Dereference with *.
  • It works with any forward-iterable container: std::vector, std::deque, std::list, C-arrays, etc.
  • If multiple elements share the maximum value, it returns an iterator to the first such element.
  • A custom comparator allows you to define "maximum" in any way (e.g., by absolute value).

Complexity Analysis:

  • Time: O(n) — every element is examined exactly once.
  • Space: O(1) — only one variable for tracking the maximum.
  • This is optimal: you must examine every element at least once to guarantee the maximum is found.

Follow-up Questions

Variations to Try

  1. Find min and max together: Write a function that finds both minimum and maximum in fewer than 2n comparisons (compare pairs of elements, then compare the larger with current max and smaller with current min — 3n/2 comparisons).
  2. Find second maximum: Find the second largest element in a single pass.
  3. Find top K elements: Given an array and an integer K, return the K largest elements (consider using std::partial_sort or std::nth_element).
  4. Find max in a sorted rotated array: Given an array that was sorted then rotated, find the maximum element in O(log n).

Key Takeaways:

  • Prefer std::max_element over a manual loop in production code. It is generic, well-tested, and communicates intent clearly.
  • Remember that std::max_element returns an iterator, not a value. This is more flexible (you get position and value).
  • Use std::optional (C++17) to represent "no result" cleanly, avoiding sentinel values like INT_MIN.
  • The pattern of initializing with the first element and scanning the rest is fundamental and appears in many algorithms.
