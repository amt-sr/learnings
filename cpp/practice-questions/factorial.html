Calculate Factorial

Topics Covered: Functions, Recursion, Templates

Problem Statement

Write functions to calculate the factorial of a non-negative integer n. Provide both iterative and recursive implementations.

Definition: n! = n * (n-1) * (n-2) * ... * 1, with 0! = 1.

Examples:

  • 5! = 120
  • 0! = 1
  • 10! = 3,628,800
  • 20! = 2,432,902,008,176,640,000

Important Constraints:

  • Factorial grows extremely fast. 20! is the largest factorial that fits in a 64-bit unsigned integer.
  • Handle invalid input: negative numbers.
  • Discuss the trade-offs between iterative and recursive approaches.

Iterative Solution

Click to reveal Solution

#include <iostream>
#include <stdexcept>
#include <string>
#include <cstdint>

uint64_t factorialIterative(int n) {
    if (n < 0) {
        throw std::invalid_argument("Factorial is not defined for negative numbers");
    }

    uint64_t result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= static_cast<uint64_t>(i);
    }
    return result;
}

int main() {
    std::cout << "0!  = " << factorialIterative(0) << "\n";   // 1
    std::cout << "1!  = " << factorialIterative(1) << "\n";   // 1
    std::cout << "5!  = " << factorialIterative(5) << "\n";   // 120
    std::cout << "10! = " << factorialIterative(10) << "\n";  // 3628800
    std::cout << "20! = " << factorialIterative(20) << "\n";  // 2432902008176640000

    try {
        factorialIterative(-1);
    } catch (const std::invalid_argument& e) {
        std::cout << "Error: " << e.what() << "\n";
    }

    return 0;
}

Explanation:

  • We start with result = 1 (since 0! = 1) and multiply from 2 to n.
  • We use uint64_t to maximize the range before overflow (supports up to 20!).
  • Negative input throws std::invalid_argument.
  • The loop starts at 2, not 1, because multiplying by 1 is a no-op (minor optimization).

Recursive Solution

Click to reveal Solution

#include <iostream>
#include <stdexcept>
#include <cstdint>

uint64_t factorialRecursive(int n) {
    if (n < 0) {
        throw std::invalid_argument("Factorial is not defined for negative numbers");
    }
    if (n <= 1) {
        return 1;  // Base case: 0! = 1! = 1
    }
    return static_cast<uint64_t>(n) * factorialRecursive(n - 1);
}

// Tail-recursive version (compiler may optimize to avoid stack growth)
uint64_t factorialTailRecursive(int n, uint64_t accumulator = 1) {
    if (n < 0) {
        throw std::invalid_argument("Factorial is not defined for negative numbers");
    }
    if (n <= 1) {
        return accumulator;  // Return accumulated result
    }
    return factorialTailRecursive(n - 1, n * accumulator);
}

int main() {
    std::cout << "5! (recursive) = " << factorialRecursive(5) << "\n";  // 120
    std::cout << "5! (tail-rec)  = " << factorialTailRecursive(5) << "\n";  // 120

    return 0;
}

Explanation:

  • The recursive version directly mirrors the mathematical definition: n! = n * (n-1)!.
  • Base case: n <= 1 returns 1.
  • The tail-recursive version carries an accumulator so that the recursive call is the very last operation. Compilers (like GCC/Clang with -O2) can optimize tail recursion into a loop, eliminating stack growth.

Compile-Time Factorial (Bonus)

Click to reveal Solution

#include <iostream>
#include <cstdint>

// Compile-time using constexpr (C++11)
constexpr uint64_t factorialConstexpr(int n) {
    return (n <= 1) ? 1 : static_cast<uint64_t>(n) * factorialConstexpr(n - 1);
}

// Compile-time using template metaprogramming (C++98 compatible)
template<int N>
struct Factorial {
    static constexpr uint64_t value = static_cast<uint64_t>(N) * Factorial<N - 1>::value;
};

template<>
struct Factorial<0> {
    static constexpr uint64_t value = 1;
};

int main() {
    // Evaluated at compile time
    constexpr auto f5 = factorialConstexpr(5);
    constexpr auto f10 = Factorial<10>::value;

    static_assert(factorialConstexpr(5) == 120, "5! should be 120");
    static_assert(Factorial<7>::value == 5040, "7! should be 5040");

    std::cout << "5!  (constexpr) = " << f5 << "\n";   // 120
    std::cout << "10! (template)  = " << f10 << "\n";   // 3628800

    return 0;
}

Explanation:

  • constexpr functions can be evaluated at compile time if all arguments are compile-time constants.
  • Template metaprogramming uses recursive template instantiation. The specialization Factorial<0> provides the base case.
  • static_assert verifies the result at compile time — if the factorial is wrong, the program will not compile.
  • In C++14 and later, constexpr functions can have loops, making them as expressive as runtime code.

Complexity Analysis:

Follow-up Questions

Variations to Try

  1. Big integer factorial: Use std::vector<uint32_t> or a string-based representation to compute factorials beyond 20! (e.g., 100!).
  2. Number of trailing zeros: Count the number of trailing zeros in n! without computing the factorial itself (hint: count factors of 5).
  3. Approximate factorial: Use Stirling's approximation: n! ~ sqrt(2*pi*n) * (n/e)^n.
  4. Factorial using std::accumulate: Show how to compute factorial using std::accumulate with a custom multiplier functor.

Key Takeaways:

  • Factorial is the simplest recursive problem — it demonstrates the pattern of a base case and recursive case.
  • The iterative version is preferred for runtime use: no stack overflow risk, lower overhead.
  • Compile-time computation with constexpr or template metaprogramming has zero runtime cost and catches errors at compile time.
  • Be aware of integer overflow limits: uint64_t handles up to 20!. For larger values, you need a big integer library.
