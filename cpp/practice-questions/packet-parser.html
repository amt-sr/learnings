Parse a 5G MAC PDU Header

Topics Covered: Bit Operations, Buffers, Protocol Handling

Problem Statement

Implement a parser for a 5G NR MAC PDU (Protocol Data Unit) header. The MAC PDU consists of one or more MAC sub-PDUs, each with a subheader followed by payload. The subheader format varies based on the Logical Channel ID (LCID).

MAC Subheader Format (simplified):

BitsFieldDescription
7-5 (3 bits)R / F / LCID (high bits)R=reserved, F=format flag
4-0 (5 bits)LCID (low bits)Logical Channel ID
15-8 (8 bits)L (length)Present only for non-fixed-size LCIDs

Notes:

  • LCID 0-32: normal data channels (variable-length, L field present).
  • LCID 33-62: fixed-size logical channels (no L field).
  • LCID 63: padding (end of PDU).

#include <iostream>
#include <vector>
#include <cstdint>
#include <string>
#include <sstream>
#include <iomanip>
#include <stdexcept>
#include <bitset>

// ---- Bit-level helper functions ----

// Extract `numBits` bits starting at `bitOffset` from the byte buffer
uint32_t extractBits(const uint8_t* data, std::size_t bitOffset, int numBits) {
    uint32_t result = 0;
    for (int i = 0; i < numBits; ++i) {
        std::size_t byteIdx = (bitOffset + i) / 8;
        int bitIdx = 7 - static_cast<int>((bitOffset + i) % 8);
        bool bitSet = (data[byteIdx] >> bitIdx) & 1;
        result = (result << 1) | static_cast<uint32_t>(bitSet);
    }
    return result;
}

// ---- Parsed Sub-PDU structures ----

struct MacSubPdu {
    uint8_t lcid = 0;        // Logical Channel ID (6 bits)
    uint8_t formatFlag = 0;   // F bit
    uint16_t length = 0;     // L field (present only for variable-length)
    bool hasLength = false;  // Whether L field is present
    std::size_t payloadOffset = 0;  // Byte offset to payload
    std::size_t payloadLength = 0;   // Length of payload in bytes
};

struct MacPduResult {
    std::vector<MacSubPdu> subPdus;
    std::size_t totalBytesParsed = 0;
    bool hasPadding = false;
};

// ---- Parser ----

class MacPduParser {
public:
    // LCID ranges
    static constexpr uint8_t LCID_PADDING = 63;
    static constexpr uint8_t LCID_FIXED_START = 33;
    static constexpr uint8_t LCID_FIXED_END = 62;
    static constexpr uint16_t MAX_L_VALUE = 32767;

    // Fixed-size payload lengths for specific LCIDs (example values)
    static constexpr int FIXED_SIZE = 4;

    MacPduResult parse(const uint8_t* data, std::size_t length) {
        MacPduResult result;
        std::size_t offset = 0;  // byte offset

        while (offset < length) {
            MacSubPdu pdu;

            if (offset + 2 > length) {
                throw std::runtime_error("Incomplete MAC subheader at offset " +
                    std::to_string(offset));
            }

            // Parse first byte: bits [7:5] = R,F,LCID_high ; bits [4:0] = LCID_low
            uint8_t byte0 = data[offset];
            uint8_t lcidHigh = (byte0 >> 5) & 0x07;  // bits 7-5
            pdu.formatFlag = (lcidHigh >> 1) & 0x01;   // F is bit 6
            uint8_t lcidLow = byte0 & 0x1F;             // bits 4-0
            pdu.lcid = (lcidHigh & 0x01) << 5 | lcidLow;  // combine

            offset += 1;

            // Check for padding
            if (pdu.lcid == LCID_PADDING) {
                result.hasPadding = true;
                break;
            }

            // Determine if L field is present
            bool isFixedSize = (pdu.lcid >= LCID_FIXED_START &&
                               pdu.lcid <= LCID_FIXED_END);

            if (!isFixedSize) {
                // Variable-length: read 8-bit L field
                if (offset + 1 > length) {
                    throw std::runtime_error("Incomplete L field at offset " +
                        std::to_string(offset));
                }
                pdu.length = data[offset];
                pdu.hasLength = true;
                offset += 1;
                pdu.payloadLength = pdu.length;
            } else {
                // Fixed-size: no L field, known payload size
                pdu.payloadLength = FIXED_SIZE;
            }

            pdu.payloadOffset = offset;

            // Verify enough data for payload
            if (offset + pdu.payloadLength > length) {
                throw std::runtime_error("Incomplete payload for LCID " +
                    std::to_string(pdu.lcid) + ": need " +
                    std::to_string(pdu.payloadLength) + " bytes at offset " +
                    std::to_string(offset) + " but only " +
                    std::to_string(length - offset) + " available");
            }

            result.subPdus.push_back(pdu);
            offset += pdu.payloadLength;
        }

        result.totalBytesParsed = offset;
        return result;
    }

    // Pretty-print the parsed result
    std::string formatResult(const MacPduResult& result) const {
        std::ostringstream oss;
        oss << "=== MAC PDU Parse Result ===\n";
        oss << "Sub-PDUs: " << result.subPdus.size() << "\n";
        oss << "Total parsed: " << result.totalBytesParsed << " bytes\n";
        if (result.hasPadding) oss << "Has padding: yes\n";
        oss << "\n";

        for (std::size_t i = 0; i < result.subPdus.size(); ++i) {
            const auto& pdu = result.subPdus[i];
            oss << "Sub-PDU #" << (i + 1) << ":\n";
            oss << "  LCID: " << static_cast<int>(pdu.lcid) << "\n";
            oss << "  F:    " << static_cast<int>(pdu.formatFlag) << "\n";
            if (pdu.hasLength) {
                oss << "  L:    " << pdu.length << " bytes\n";
            }
            oss << "  Payload: offset=" << pdu.payloadOffset
                << ", length=" << pdu.payloadLength << "\n";
        }
        return oss.str();
    }
};

// ---- Test ----

int main() {
    // Construct a test MAC PDU byte buffer
    // Sub-PDU 1: LCID=5, L=10 (variable-length)
    //   Byte 0: F=0, LCID=5  => 000 00 00101 = 0x05
    //   Byte 1: L=10         => 0x0A
    //   Bytes 2-11: payload (10 bytes)
    // Sub-PDU 2: LCID=40 (fixed-size)
    //   Byte 12: F=0, LCID=40 => 000 00 101000 (6-bit LCID=40, padded to byte)
    //   Bits: byte = 0b000101000... but we only use 1 byte for subheader
    //   Byte 12: high bits of LCID (bit 7-5 = 000), F=0 (bit 6), LCID low 5 bits = 01000 = 0x08
    //   Wait: LCID=40 = 0b101000. In 6 bits: bit 5 = 1 (MSB), bits 4-0 = 01000
    //   Byte: R(0) F(0) LCID5(1) LCID4-0(01000) = 0b00010100 = 0x14
    //   Bytes 13-16: payload (4 bytes fixed)
    // Sub-PDU 3: Padding (LCID=63)
    //   Byte 17: 0b01111111 = 0x7F

    std::vector<uint8_t> pdu;

    // Sub-PDU 1: LCID=5, Length=10
    // LCID=5, F=0: bits [7:5]=000, bit6=0(F), bits[4:0]=00101
    pdu.push_back(0x05);  // R=0, F=0, LCID=5
    pdu.push_back(10);    // L=10
    for (int i = 0; i < 10; ++i) {
        pdu.push_back(static_cast<uint8_t>(i));  // dummy payload
    }

    // Sub-PDU 2: LCID=40 (fixed-size, no L field)
    // LCID=40 = 0b101000. LCID_high = 1, LCID_low = 01000 = 0x08
    // Byte: R(0) F(0) LCID_high(1) LCID_low(01000) = 0b000101000... wait
    // Actually: bits 7=R, 6=F, 5=LCID[5], 4-0=LCID[4:0]
    // LCID=40 = 101000: LCID[5]=1, LCID[4:0]=01000
    // Byte = 0b0_0_1_01000 = 0b00101000 = 0x28
    pdu.push_back(0x28);  // R=0, F=0, LCID=40
    for (int i = 0; i < 4; ++i) {
        pdu.push_back(0xFF);  // dummy payload
    }

    // Padding
    // LCID=63 = 0b111111
    // Byte: R=0, F=1, LCID[5]=1, LCID[4:0]=11111 => 0b0_1_1_11111 = 0x7F
    pdu.push_back(0x7F);  // padding

    // Print raw bytes
    std::cout << "Raw PDU (" << pdu.size() << " bytes): ";
    for (auto b : pdu) {
        std::cout << std::hex << std::setw(2) << std::setfill('0')
                  << static_cast<int>(b) << " ";
    }
    std::cout << "\n\n";

    // Parse
    MacPduParser parser;
    try {
        auto result = parser.parse(pdu.data(), pdu.size());
        std::cout << parser.formatResult(result);
    } catch (const std::exception& e) {
        std::cerr << "Parse error: " << e.what() << "\n";
    }

    return 0;
}

Explanation:

  • extractBits: Reads individual bits from a byte buffer. Given a bit offset, it extracts numBits bits, constructing the value MSB-first. This is the fundamental building block for protocol parsing.
  • Subheader parsing: Each subheader starts with 1 byte containing the R, F, and LCID fields. The LCID spans 6 bits: 1 bit in the high nibble and 5 bits in the low nibble.
  • Variable vs Fixed: LCIDs 0-32 have a length field (L); LCIDs 33-62 have fixed-size payloads; LCID 63 is padding and terminates parsing.
  • Bounds checking: Every field access is verified against the buffer length to prevent out-of-bounds reads.
  • Exception handling: Parse errors throw std::runtime_error with a descriptive message including the offset and context.

Complexity Analysis:

  • Time: O(n) — single pass through the PDU buffer.
  • Space: O(k) where k is the number of sub-PDUs (for the result vector).

Follow-up Questions

Variations to Try

  1. Extended length (2-byte L): Add support for L values > 255 using 2-byte encoding when the format flag F=1.
  2. MAC CE parsing: For LCID values corresponding to MAC Control Elements, parse the CE payload (e.g., Timing Advance Command, BSR, PHR).
  3. Zero-copy parsing: Store views (pointers + lengths) into the original buffer instead of copying payload data.
  4. Byte-level parsing (no bit extraction): Rewrite the parser using byte-level access only, with shift and mask operations, for maximum performance.
  5. MAC PDU builder: Implement a serializer that constructs a valid MAC PDU from a list of sub-PDUs (reverse of parsing).

Real-World Context:

  • 5G MAC PDU parsing is a fundamental operation in every 5G base station (gNB) and user equipment (UE). It runs on every received transport block.
  • In real implementations, parsers must handle multi-slot PDU aggregation, segmented payloads, and various MAC CE types.
  • Performance matters: parsers typically handle millions of PDUs per second and must be cache-friendly and branch-minimal.
  • Zero-copy design is essential in high-throughput telecom systems to avoid memory bandwidth bottlenecks.
