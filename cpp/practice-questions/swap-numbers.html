Swap Two Numbers

Topics Covered: Pointers, References, Functions

Problem Statement

Write functions to swap two numbers using three different approaches:

  1. Using pointers (pass by pointer).
  2. Using references (pass by reference).
  3. Using XOR swap (no temporary variable).

Explain the differences, advantages, and disadvantages of each approach.

Approach 1: Swap Using Pointers

Click to reveal Solution

#include <iostream>

void swapByPointer(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 10, y = 20;
    std::cout << "Before: x = " << x << ", y = " << y << "\n";

    swapByPointer(&x, &y);
    std::cout << "After:  x = " << x << ", y = " << y << "\n";
    // Output: After:  x = 20, y = 10

    return 0;
}

Explanation:

  • We pass the addresses of the variables using the address-of operator &.
  • Inside the function, we dereference the pointers with * to access and modify the original values.
  • This is the C-style approach. It works but requires explicit & and * syntax at the call site and inside the function.

Approach 2: Swap Using References

Click to reveal Solution

#include <iostream>

void swapByReference(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}

// Template version that works with any type
template<typename T>
void swapGeneric(T& a, T& b) {
    T temp = std::move(a);  // move instead of copy for efficiency
    a = std::move(b);
    b = std::move(temp);
}

int main() {
    int x = 10, y = 20;
    std::cout << "Before: x = " << x << ", y = " << y << "\n";

    swapByReference(x, y);
    std::cout << "After:  x = " << x << ", y = " << y << "\n";
    // Output: After:  x = 20, y = 10

    // With strings
    std::string s1 = "hello", s2 = "world";
    swapGeneric(s1, s2);
    std::cout << "s1 = " << s1 << ", s2 = " << s2 << "\n";
    // Output: s1 = world, s2 = hello

    return 0;
}

Explanation:

  • References (int&) are aliases to the original variables. No explicit & or * needed at the call site.
  • The syntax is cleaner and less error-prone than pointers.
  • The template version uses std::move to avoid unnecessary copies for large types (like std::string).
  • This is the idiomatic C++ approach. In practice, use std::swap which already does this.

Approach 3: XOR Swap (No Temporary Variable)

Click to reveal Solution

#include <iostream>

void swapXOR(int& a, int& b) {
    // Only works for integer types
    if (&a == &b) return;  // Guard: same variable would zero it out

    a ^= b;
    b ^= a;  // b = b ^ (a ^ b) = a
    a ^= b;  // a = (a ^ b) ^ a = b
}

int main() {
    int x = 10, y = 20;
    std::cout << "Before: x = " << x << ", y = " << y << "\n";

    swapXOR(x, y);
    std::cout << "After:  x = " << x << ", y = " << y << "\n";
    // Output: After:  x = 20, y = 10

    return 0;
}

Explanation:

  • XOR swap uses three XOR operations to swap values without a temporary variable.
  • a ^= b stores the XOR of both in a.
  • b ^= a recovers the original a into b (since a ^ b ^ b = a).
  • a ^= b recovers the original b into a.
  • Warning: If a and b are the same variable, this zeros out the value. The guard if (&a == &b) return; prevents this.

Why XOR Swap is Usually a Bad Idea:

  • Modern compilers optimize temporary-variable swaps into register moves — often faster than XOR.
  • XOR swap only works on integer types (not floats, strings, or objects).
  • It prevents the compiler from doing certain optimizations (read-after-write dependency chain).
  • It is a classic interview trick but should never be used in real code.

Follow-up Questions

Variations to Try

  1. Why can't you swap with pass-by-value? Write a version that takes int a, int b by value and observe that the original variables are unchanged. Explain why.
  2. Swap three variables: Rotate three variables: a → b, b → c, c → a.
  3. Swap bits: Swap the nth and mth bits of an integer using bit manipulation.
  4. Implement your own std::swap: Create a generic my_swap function template that handles all types efficiently using std::move.

Key Takeaways:

  • Pointers vs references: References provide the same ability to modify caller variables but with cleaner syntax. Prefer references in C++.
  • The difference between pass-by-value, pass-by-pointer, and pass-by-reference is one of the most fundamental concepts in C++.
  • In production code, always use std::swap from <utility> — it is optimized, generic, and handles move semantics correctly.
