Thread Pool for 5G RAN Processing

Topics Covered: Multithreading, Atomics, Scheduling, Smart Pointers

Problem Statement

Implement a thread pool suitable for 5G Radio Access Network (RAN) processing. The pool must support:

  • Priority-based task scheduling: Tasks have priorities (HIGH, NORMAL, LOW). Higher-priority tasks are processed first.
  • Work stealing: Idle threads can steal tasks from other threads' local queues to improve load balancing.
  • Graceful shutdown: Stop accepting new tasks, finish pending work, then exit cleanly.
  • Task statistics: Track completed task count, queue depth, and active worker count.

In a real 5G RAN, task priorities map to processing deadlines: slot-level processing (TTI-based) has the highest priority, followed by control-plane processing, then background management tasks.

Design Considerations:

  • Each worker thread has a local deque of tasks. The worker pushes/pops from its own deque (lock-free for the owner).
  • Stealing occurs from the bottom (oldest tasks) of another thread's deque, while the owner pushes/pops from the top (newest tasks).
  • A global priority queue handles initially submitted tasks and distributes them to workers.
  • Work stealing reduces contention on the global queue and improves cache locality.

enum class Priority { HIGH = 0, NORMAL = 1, LOW = 2, COUNT = 3 };

// ---- Task with priority ----

struct PrioritizedTask {
    std::function<void()> func;
    Priority priority;
    int id;  // For tracking

    // Higher priority = lower value, so we reverse the comparison
    // for min-heap behavior (HIGH comes first)
    bool operator>(const PrioritizedTask& other) const {
        return static_cast<int>(priority) > static_cast<int>(other.priority);
    }
};

// ---- Per-thread local work queue ----

class LocalQueue {
public:
    void push(PrioritizedTask task) {
        std::lock_guard<std::mutex> lock(mutex_);
        tasks_.push_back(std::move(task));
    }

    bool pop(PrioritizedTask& task) {
        std::lock_guard<std::mutex> lock(mutex_);
        if (tasks_.empty()) return false;
        task = std::move(tasks_.back());  // Pop from top (LIFO)
        tasks_.pop_back();
        return true;
    }

    bool steal(PrioritizedTask& task) {
        std::lock_guard<std::mutex> lock(mutex_);
        if (tasks_.empty()) return false;
        task = std::move(tasks_.front());  // Steal from bottom (FIFO)
        tasks_.pop_front();
        return true;
    }

    std::size_t size() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return tasks_.size();
    }

private:
    mutable std::mutex mutex_;
    std::deque<PrioritizedTask> tasks_;
};

// ---- Thread Pool ----

class RanThreadPool {
public:
    RanThreadPool(std::size_t numThreads)
        : stopped_(false), nextTaskId_(0) {
        for (std::size_t i = 0; i < numThreads; ++i) {
            workers_.emplace_back(&RanThreadPool::workerLoop, this, i);
        }
        std::cout << "ThreadPool: " << numThreads << " workers started\n";
    }

    ~RanThreadPool() {
        shutdown();
    }

    // Submit a task with priority, returns a future for the result
    template<typename F, typename... Args>
    auto submit(Priority priority, F&& f, Args&&... args)
        -> std::future<typename std::invoke_result_t<F, Args...>>
    {
        using ReturnType = typename std::invoke_result_t<F, Args...>;

        auto task = std::make_shared<std::packaged_task<ReturnType()>>(
            std::bind(std::forward<F>(f), std::forward<Args>(args)...)
        );

        std::future<ReturnType> result = task->get_future();
        int taskId = nextTaskId_++;

        {
            std::lock_guard<std::mutex> lock(globalMutex_);
            globalQueue_.push({[task]() { (*task)(); }, priority, taskId});
        }
        globalCond_.notify_one();

        return result;
    }

    // Submit with default NORMAL priority
    template<typename F, typename... Args>
    auto submit(F&& f, Args&&... args)
        -> std::future<typename std::invoke_result_t<F, Args...>>
    {
        return submit(Priority::NORMAL, std::forward<F>(f),
                      std::forward<Args>(args)...);
    }

    void shutdown() {
        if (stopped_) return;
        {
            std::lock_guard<std::mutex> lock(globalMutex_);
            stopped_ = true;
        }
        globalCond_.notify_all();

        for (auto& w : workers_) {
            if (w.joinable()) w.join();
        }
        std::cout << "ThreadPool: all workers stopped\n";
    }

    // Statistics
    std::size_t totalCompleted() const { return completedCount_.load(); }
    std::size_t globalQueueSize() const {
        std::lock_guard<std::mutex> lock(globalMutex_);
        return globalQueue_.size();
    }

    std::string stats() const {
        std::ostringstream oss;
        oss << "completed=" << completedCount_.load()
            << ", globalQ=" << globalQueueSize()
            << ", workers=" << workers_.size();
        return oss.str();
    }

private:
    void workerLoop(std::size_t workerId) {
        LocalQueue localQueue;
        std::mt19937 rng(workerId);

        while (true) {
            PrioritizedTask task;

            // 1. Try local queue first (fast path, no contention)
            if (localQueue.pop(task)) {
                executeTask(task);
                continue;
            }

            // 2. Try global queue
            {
                std::unique_lock<std::mutex> lock(globalMutex_);
                if (stopped_ && globalQueue_.empty()) {
                    return;  // Shutdown and no more work
                }
                if (!globalQueue_.empty()) {
                    task = std::move(globalQueue_.top());
                    globalQueue_.pop();
                    lock.unlock();
                    executeTask(task);
                    continue;
                }
            }

            // 3. Try work stealing from random other workers
            bool stolen = false;
            if (localQueues_.size() > 1) {
                for (int attempt = 0; attempt < 2; ++attempt) {
                    std::size_t victim = (workerId + 1 + attempt) % localQueues_.size();
                    if (victim != workerId) {
                        if (localQueues_[victim]->steal(task)) {
                            stolen = true;
                            break;
                        }
                    }
                }
            }

            if (stolen) {
                executeTask(task);
                continue;
            }

            // 4. Wait for new work
            {
                std::unique_lock<std::mutex> lock(globalMutex_);
                globalCond_.wait(lock, [this]() {
                    return !globalQueue_.empty() || stopped_;
                });
            }
        }
    }

    void executeTask(PrioritizedTask& task) {
        try {
            task.func();
        } catch (const std::exception& e) {
            std::cerr << "Task " << task.id << " exception: "
                      << e.what() << "\n";
        }
        completedCount_.fetch_add(1, std::memory_order_relaxed);
    }

    // Global priority queue
    mutable std::mutex globalMutex_;
    std::condition_variable globalCond_;
    std::priority_queue<PrioritizedTask,
                          std::vector<PrioritizedTask>,
                          std::greater<PrioritizedTask>> globalQueue_;
    bool stopped_;
    int nextTaskId_;
    std::atomic<std::size_t> completedCount_{0};

    // Workers and their local queues
    std::vector<std::thread> workers_;
    std::vector<LocalQueue*> localQueues_;  // Populated by workers
};

// ---- Demo: Simulate 5G RAN processing ----

void simulateSlotProcessing(int slotNum) {
    std::this_thread::sleep_for(std::chrono::milliseconds(10));
    // Simulated: decode PDCCH, PDSCH, process HARQ-ACK
}

void simulateControlProcessing(int ueId) {
    std::this_thread::sleep_for(std::chrono::milliseconds(20));
    // Simulated: RRC connection handling, measurement reports
}

void simulateBackgroundTask(int taskId) {
    std::this_thread::sleep_for(std::chrono::milliseconds(30));
    // Simulated: statistics collection, logging, neighbor cell discovery
}

int main() {
    RanThreadPool pool(4);

    // Submit slot-level processing (HIGH priority - time-critical)
    std::vector<std::future<void>> slotFutures;
    for (int i = 0; i < 8; ++i) {
        slotFutures.push_back(pool.submit(
            Priority::HIGH,
            [i]() { simulateSlotProcessing(i); }
        ));
    }

    // Submit control-plane tasks (NORMAL priority)
    std::vector<std::future<void>> ctrlFutures;
    for (int i = 0; i < 4; ++i) {
        ctrlFutures.push_back(pool.submit(
            Priority::NORMAL,
            [i]() { simulateControlProcessing(i); }
        ));
    }

    // Submit background tasks (LOW priority)
    std::vector<std::future<void>> bgFutures;
    for (int i = 0; i < 3; ++i) {
        bgFutures.push_back(pool.submit(
            Priority::LOW,
            [i]() { simulateBackgroundTask(i); }
        ));
    }

    // Wait for all high-priority tasks first (most important)
    std::cout << "Waiting for slot processing...\n";
    for (auto& f : slotFutures) f.get();
    std::cout << "All slot processing complete. "
              << "Stats: " << pool.stats() << "\n";

    // Wait for remaining tasks
    for (auto& f : ctrlFutures) f.get();
    for (auto& f : bgFutures) f.get();

    std::cout << "All tasks complete. Final stats: " << pool.stats() << "\n";

    pool.shutdown();

    return 0;
}

Explanation:

  • Priority queue: Tasks are stored in a std::priority_queue with std::greater comparator so that HIGH priority (value 0) is at the top.
  • Work stealing: Each worker thread has a LocalQueue. When the local queue is empty and the global queue is empty, workers try to steal from other workers' queues. Stealing takes from the front (oldest tasks), while the owner pops from the back (newest tasks).
  • Future-based return: submit() returns a std::future using std::packaged_task. The caller can wait for results or check completion.
  • Graceful shutdown: Sets stopped_ flag and wakes all threads. Workers finish their current task, then drain the global queue before exiting.
  • Exception safety: Exceptions in tasks are caught in executeTask and logged, preventing one task from crashing the pool.
  • 5G RAN context: HIGH priority maps to slot-level processing (TTI = 0.5ms or 1ms deadlines), NORMAL to control-plane (RRC, measurements), LOW to background tasks.

Follow-up Questions

Variations to Try

  1. Lock-free local deque: Replace LocalQueue's mutex with the Chase-Lev work-stealing deque algorithm (lock-free for single producer, single consumer).
  2. Affinity-aware scheduling: Pin worker threads to specific CPU cores using pthread_setaffinity_np or SetThreadAffinityMask to minimize cache migration in NUMA systems.
  3. Deadline-aware scheduling: Instead of fixed priorities, each task has an absolute deadline. The scheduler picks the task with the earliest deadline (Earliest Deadline First algorithm).
  4. Dynamic worker count: Add/remove worker threads based on load (e.g., spin up extra workers during high traffic, remove during low traffic).
  5. Task chaining: Support dependent tasks where one task's output feeds another. Implement a DAG scheduler.
  6. Per-UE work queues: In a real RAN, each UE has its own processing context. Implement per-UE serialization so tasks for the same UE run in order while different UEs run in parallel.

Real-World 5G RAN Requirements:

  • TTI (Transmission Time Interval): In 5G NR, the minimum TTI is 0.125ms (mini-slots). All slot-level processing must complete within this window.
  • Latency budget: URLLC (Ultra-Reliable Low-Latency Communication) requires end-to-end latency < 1ms. The thread pool must schedule and complete tasks within microseconds.
  • Determinism: Telecom systems require bounded worst-case execution time. Work stealing helps but does not guarantee determinism. Real RAN systems often use partitioned architectures where each core handles a fixed subset of UEs.
  • NUMA awareness: High-end base stations use multi-socket systems. Memory locality and CPU pinning are critical for performance.

Advanced Topics to Explore:

  • The Chase-Lev deque is the standard lock-free work-stealing algorithm. It uses a circular array with atomic indices and careful memory ordering to allow the owner to push/pop without locks while stealers can take from the other end.
  • Fork-join parallelism (like Intel TBB's parallel_for) is an alternative to thread pools for data-parallel workloads.
  • Coroutine-based scheduling (C++20) can replace thread-based scheduling for fine-grained tasks, reducing context switch overhead.
