Lock-Free Ring Buffer

Topics Covered: Circular Buffers, Atomics, Templates

Problem Statement

Implement a lock-free Single Producer Single Consumer (SPSC) ring buffer template class. The buffer must:

  • Support push (write) from a single producer thread and pop (read) from a single consumer thread.
  • Be completely lock-free: use only std::atomic operations, no mutexes.
  • Use power-of-two capacity for efficient modulo via bitmask.
  • Handle buffer full and buffer empty conditions gracefully.
  • Be generic over the element type T.

Important: This is an SPSC (single producer, single consumer) ring buffer. A multi-producer/multi-consumer (MPMC) lock-free ring buffer is significantly more complex and requires CAS loops on both head and tail.

#include <iostream>
#include <atomic>
#include <vector>
#include <cstddef>
#include <optional>
#include <thread>
#include <chrono>

template<typename T, std::size_t Capacity>
class SPSCRingBuffer {
    static_assert((Capacity & (Capacity - 1)) == 0,
        "Capacity must be a power of 2");

    static constexpr std::size_t Mask = Capacity - 1;

public:
    SPSCRingBuffer()
        : buffer_(Capacity), head_(0), tail_(0) {}

    // Non-copyable, non-movable
    SPSCRingBuffer(const SPSCRingBuffer&) = delete;
    SPSCRingBuffer& operator=(const SPSCRingBuffer&) = delete;

    // Push: called by single producer thread
    bool push(const T& item) {
        const auto currentHead = head_.load(std::memory_order_relaxed);
        const auto nextHead = currentHead + 1;

        // Check if buffer is full
        // Buffer is full when (head - tail) == Capacity
        if (nextHead - tail_.load(std::memory_order_acquire) > Capacity) {
            return false;  // Buffer full
        }

        buffer_[currentHead & Mask] = item;
        head_.store(nextHead, std::memory_order_release);
        return true;
    }

    bool push(T&& item) {
        const auto currentHead = head_.load(std::memory_order_relaxed);
        const auto nextHead = currentHead + 1;

        if (nextHead - tail_.load(std::memory_order_acquire) > Capacity) {
            return false;
        }

        buffer_[currentHead & Mask] = std::move(item);
        head_.store(nextHead, std::memory_order_release);
        return true;
    }

    // Pop: called by single consumer thread
    std::optional<T> pop() {
        const auto currentTail = tail_.load(std::memory_order_relaxed);

        // Check if buffer is empty
        if (currentTail == head_.load(std::memory_order_acquire)) {
            return std::nullopt;  // Buffer empty
        }

        T item = std::move(buffer_[currentTail & Mask]);
        tail_.store(currentTail + 1, std::memory_order_release);
        return item;
    }

    // Query methods
    bool empty() const {
        return tail_.load(std::memory_order_acquire) ==
               head_.load(std::memory_order_acquire);
    }

    bool full() const {
        return (head_.load(std::memory_order_acquire) -
                tail_.load(std::memory_order_acquire)) == Capacity;
    }

    std::size_t size() const {
        return head_.load(std::memory_order_acquire) -
               tail_.load(std::memory_order_acquire);
    }

    std::size_t capacity() const { return Capacity; }

private:
    std::vector<T> buffer_;
    // head is written by producer, read by consumer
    std::atomic<std::size_t> head_;
    // tail is written by consumer, read by producer
    std::atomic<std::size_t> tail_;
};

// --- Demo ---

int main() {
    constexpr std::size_t BUF_SIZE = 16;  // Must be power of 2
    SPSCRingBuffer<int, BUF_SIZE> buffer;

    std::cout << "Capacity: " << buffer.capacity() << "\n";
    std::cout << "Initial empty: " << buffer.empty() << "\n\n";

    // Producer thread
    std::thread producer([&]() {
        for (int i = 1; i <= 25; ++i) {
            while (!buffer.push(i)) {
                // Buffer full, spin-wait (or could yield/sleep)
                std::this_thread::yield();
            }
            std::cout << "  Produced: " << i
                      << " (size=" << buffer.size() << ")\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(50));
        }
        std::cout << "Producer done\n";
    });

    // Consumer thread
    std::thread consumer([&]() {
        int received = 0;
        int expected = 1;
        while (received < 25) {
            auto item = buffer.pop();
            if (item) {
                ++received;
                if (*item != expected++) {
                    std::cerr << "MISMATCH! Expected " << expected - 1
                              << " got " << *item << "\n";
                }
                std::cout << "  Consumed: " << *item
                          << " (size=" << buffer.size() << ")\n";
            } else {
                std::this_thread::yield();
            }
        }
        std::cout << "Consumer done (received " << received << " items)\n";
    });

    producer.join();
    consumer.join();

    std::cout << "\nFinal state: empty=" << buffer.empty()
              << ", size=" << buffer.size() << "\n";

    return 0;
}

Explanation:

  • Power-of-two capacity: Allows index & (Capacity - 1) instead of index % Capacity. Bitmask is a single CPU instruction vs. division which is expensive.
  • Monotonic counters: head_ and tail_ never wrap around. They are monotonically increasing size_t values. The actual buffer index is obtained via & Mask.
  • Memory ordering:
    • Producer writes data, then does release store on head_. Consumer does acquire load on head_ before reading data. This ensures the producer's writes are visible to the consumer.
    • Similarly for tail_: consumer does release, producer does acquire.
    • Within the same thread, relaxed ordering is sufficient for the initial load of the local counter.
  • SPSC guarantee: Only one thread writes head_ (producer) and only one thread writes tail_ (consumer). No CAS loop is needed for either, making it truly lock-free.
  • Full/Empty detection: Full when (head - tail) == Capacity. Empty when head == tail. The monotonic counters make this comparison correct regardless of overflow.

Follow-up Questions

Variations to Try

  1. MPMC ring buffer: Extend to support multiple producers and multiple consumers using CAS (compare-and-swap) loops on both head and tail. This is significantly more complex.
  2. Bulk push/pop: Add methods to push/pop multiple items at once, reducing atomic operations per item.
  3. Overwrite mode: When full, overwrite the oldest item instead of rejecting. Useful for telemetry/logging buffers.
  4. Cache-line padding: Pad the head and tail atomics to separate cache lines (using alignas(64)) to avoid false sharing between producer and consumer threads.
  5. Fixed-size byte buffer: Implement a ring buffer for raw bytes (not typed elements) with variable-length push/pop, suitable for network packet buffers.

Common Pitfalls:

  • False sharing: If head_ and tail_ are on the same cache line, the producer and consumer constantly invalidate each other's cache. Use alignas(64) to separate them.
  • Memory ordering bugs: Using relaxed ordering everywhere can lead to data races. The release/acquire pattern is essential for cross-thread visibility.
  • Non-power-of-two capacity: The bitmask optimization only works for powers of two. For arbitrary sizes, use % Capacity.
  • SPSC assumption violation: If multiple threads call push or pop, the SPSC buffer produces data races. Use a mutex or MPMC implementation instead.

Real-World Applications:

  • 5G base stations use ring buffers for inter-thread communication between the PHY layer and MAC layer.
  • Audio processing pipelines use lock-free ring buffers between the audio callback thread and the processing thread.
  • The Disruptor pattern (LMAX) is a high-performance ring buffer used in financial trading systems.
