Thread-Safe Queue

Topics Covered: Multithreading, Mutex, Condition Variables, Templates

Problem Statement

Implement a thread-safe queue that can be safely used by multiple producer and consumer threads simultaneously. The queue must support:

  • push: Add an element to the queue (blocking or non-blocking).
  • pop: Remove and return an element (blocking with optional timeout).
  • try_pop: Non-blocking pop that returns immediately if empty.
  • empty / size: Query the queue state.
  • stop: Gracefully shutdown, waking all waiting threads.

Use std::mutex, std::condition_variable, and std::queue as the underlying container.

#include <iostream>
#include <queue>
#include <mutex>
#include <condition_variable>
#include <thread>
#include <chrono>
#include <optional>
#include <atomic>
#include <vector>
#include <random>

template<typename T>
class ThreadSafeQueue {
public:
    ThreadSafeQueue() = default;

    // Disable copy and move (mutex is not copyable)
    ThreadSafeQueue(const ThreadSafeQueue&) = delete;
    ThreadSafeQueue& operator=(const ThreadSafeQueue&) = delete;
    ThreadSafeQueue(ThreadSafeQueue&&) = delete;
    ThreadSafeQueue& operator=(ThreadSafeQueue&&) = delete;

    // Push an element into the queue
    void push(const T& value) {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (stopped_) return;  // Reject new items after stop
            queue_.push(value);
        }
        cond_.notify_one();  // Notify one waiting consumer
    }

    void push(T&& value) {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            if (stopped_) return;
            queue_.push(std::move(value));
        }
        cond_.notify_one();
    }

    // Pop with blocking (waits until an element is available or stopped)
    std::optional<T> pop() {
        std::unique_lock<std::mutex> lock(mutex_);
        cond_.wait(lock, [this] {
            return !queue_.empty() || stopped_;
        });

        if (queue_.empty()) return std::nullopt;  // Stopped and empty

        T value = std::move(queue_.front());
        queue_.pop();
        return value;
    }

    // Pop with timeout
    std::optional<T> pop_for(const std::chrono::milliseconds& timeout) {
        std::unique_lock<std::mutex> lock(mutex_);
        if (!cond_.wait_for(lock, timeout, [this] {
            return !queue_.empty() || stopped_;
        })) {
            return std::nullopt;  // Timeout
        }

        if (queue_.empty()) return std::nullopt;  // Stopped and empty

        T value = std::move(queue_.front());
        queue_.pop();
        return value;
    }

    // Non-blocking pop
    std::optional<T> try_pop() {
        std::lock_guard<std::mutex> lock(mutex_);
        if (queue_.empty()) return std::nullopt;

        T value = std::move(queue_.front());
        queue_.pop();
        return value;
    }

    bool empty() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return queue_.empty();
    }

    std::size_t size() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return queue_.size();
    }

    // Stop the queue and wake all waiting threads
    void stop() {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            stopped_ = true;
        }
        cond_.notify_all();  // Wake all waiting consumers
    }

private:
    mutable std::mutex mutex_;
    std::condition_variable cond_;
    std::queue<T> queue_;
    bool stopped_ = false;
};

// --- Demo: Producer-Consumer ---

int main() {
    ThreadSafeQueue<int> queue;
    std::atomic<int> produced{0};
    std::atomic<int> consumed{0};

    // Producer thread
    std::thread producer([&]() {
        for (int i = 1; i <= 20; ++i) {
            queue.push(i);
            produced.fetch_add(1);
            std::cout << "Produced: " << i << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(50));
        }
        queue.stop();  // Signal consumers to stop
    });

    // Consumer threads
    auto consumerFunc = [&](int id) {
        while (true) {
            auto value = queue.pop();
            if (!value) {
                std::cout << "Consumer " << id << " exiting\n";
                break;  // Queue stopped and empty
            }
            consumed.fetch_add(1);
            std::cout << "Consumer " << id << " got: " << *value << "\n";
            std::this_thread::sleep_for(std::chrono::milliseconds(100));
        }
    };

    std::thread consumer1(consumerFunc, 1);
    std::thread consumer2(consumerFunc, 2);

    producer.join();
    consumer1.join();
    consumer2.join();

    std::cout << "\nProduced: " << produced.load() << "\n";
    std::cout << "Consumed: " << consumed.load() << "\n";

    return 0;
}

Explanation:

  • mutex_ protects all access to the internal std::queue. Every public method locks it.
  • condition_variable allows consumer threads to sleep efficiently when the queue is empty, instead of busy-waiting.
  • notify_one (in push) wakes a single waiting consumer. notify_all (in stop) wakes all consumers to let them exit.
  • stopped_ flag enables graceful shutdown. When set, pop() returns nullopt even if the queue is empty, allowing consumers to exit cleanly.
  • We use std::optional<T> for the return type to distinguish "got a value" from "queue empty/stopped".
  • Move semantics: we std::move values out of the queue to avoid copies for large types.
  • The class is non-copyable and non-movable because std::mutex is not copyable.

Follow-up Questions

Variations to Try

  1. Bounded queue: Add a maximum capacity. push should block (or return false) when the queue is full. Use two condition variables: one for "not full" and one for "not empty".
  2. Priority queue: Replace std::queue with std::priority_queue for a thread-safe priority queue.
  3. Batch pop: Add a popAll() method that drains the entire queue in one lock acquisition.
  4. Fine-grained locking: Implement the queue using a linked list with separate head and tail mutexes to allow push and pop to proceed concurrently.
  5. Lock-free queue: Use std::atomic compare-and-swap operations to implement a lock-free MPMC queue (significantly more complex).

Common Pitfalls:

  • Spurious wakeups: condition_variable::wait can wake up without being notified. Always use the predicate overload: wait(lock, predicate) to guard against spurious wakeups.
  • Missed notifications: If notify is called before wait, the notification is lost. The predicate pattern handles this correctly.
  • Deadlock: Never lock the mutex and then call notify inside the lock (it works, but is suboptimal). Unlock first, then notify. In our implementation, notify_one is called after the lock_guard goes out of scope.
  • Exception safety: If the element type's copy/move constructor throws, the queue must remain in a valid state.

Key Takeaways:

  • A condition variable + mutex is the canonical pattern for blocking queues in C++.
  • Always use the predicate form of wait() to handle spurious wakeups.
  • Use std::optional to signal "no result" cleanly, avoiding sentinel values or exceptions for expected conditions.
  • The stop() pattern is essential for graceful shutdown in multi-threaded applications.
