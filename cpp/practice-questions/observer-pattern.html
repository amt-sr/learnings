Observer Pattern with std::function

Topics Covered: Design Patterns, std::function, Lambdas, Templates

Problem Statement

Implement an Observer (Event/Signal-Slot) pattern using std::function and lambdas instead of classical inheritance-based observers. This approach is more flexible: any callable (free functions, lambdas, member functions, functors) can subscribe to events.

The implementation should support:

  • Subscribe with any callable (lambda, free function, member function).
  • Unsubscribe by token/ID.
  • Thread-safe notification (optional, discuss trade-offs).
  • Type-safe events with different payload types.

#include <iostream>
#include <functional>
#include <string>
#include <vector>
#include <unordered_map>
#include <memory>
#include <mutex>

// Type alias for observer callback
template<typename... Args>
using Callback = std::function<void(Args...)>;

// Event/Signal class that holds subscribers and notifies them
template<typename... Args>
class Event {
public:
    // Subscribe a callback, returns a token for unsubscription
    int subscribe(Callback<Args...> callback) {
        int token = nextToken_++;
        callbacks_[token] = std::move(callback);
        return token;
    }

    // Unsubscribe by token
    bool unsubscribe(int token) {
        return callbacks_.erase(token) > 0;
    }

    // Notify all subscribers
    void emit(Args... args) const {
        for (const auto& [token, callback] : callbacks_) {
            callback(args...);
        }
    }

    // Clear all subscribers
    void clear() { callbacks_.clear(); }

    std::size_t subscriberCount() const { return callbacks_.size(); }

private:
    std::unordered_map<int, Callback<Args...>> callbacks_;
    int nextToken_ = 0;
};

// Thread-safe version
template<typename... Args>
class ThreadSafeEvent {
public:
    int subscribe(Callback<Args...> callback) {
        std::lock_guard<std::mutex> lock(mutex_);
        int token = nextToken_++;
        callbacks_[token] = std::move(callback);
        return token;
    }

    bool unsubscribe(int token) {
        std::lock_guard<std::mutex> lock(mutex_);
        return callbacks_.erase(token) > 0;
    }

    void emit(Args... args) const {
        std::lock_guard<std::mutex> lock(mutex_);
        for (const auto& [token, callback] : callbacks_) {
            callback(args...);
        }
    }

private:
    mutable std::mutex mutex_;
    std::unordered_map<int, Callback<Args...>> callbacks_;
    int nextToken_ = 0;
};

// --- Demo ---

class TemperatureSensor {
public:
    // Events with different payload types
    Event<double> onTemperatureChange;
    Event<int> onCriticalAlert;

    void updateTemperature(double temp) {
        currentTemp_ = temp;
        onTemperatureChange.emit(temp);

        if (temp > 100.0) {
            onCriticalAlert.emit(static_cast<int>(temp));
        }
    }

private:
    double currentTemp_ = 0.0;
};

class Display {
public:
    void show(double temp) {
        std::cout << "  [Display] Temperature: " << temp << " C\n";
    }
};

// Free function observer
void logTemperature(double temp) {
    std::cout << "  [Logger] Logged: " << temp << " C\n";
}

int main() {
    TemperatureSensor sensor;
    Display display;

    // Subscribe with a lambda
    auto token1 = sensor.onTemperatureChange.subscribe(
        [&display](double temp) {
            display.show(temp);
        }
    );

    // Subscribe with a free function
    auto token2 = sensor.onTemperatureChange.subscribe(logTemperature);

    // Subscribe lambda to alert event
    auto token3 = sensor.onCriticalAlert.subscribe(
        [](int temp) {
            std::cout << "  [ALERT] CRITICAL: " << temp << " C!\n";
        }
    );

    // Subscribe another lambda
    auto token4 = sensor.onTemperatureChange.subscribe(
        [](double temp) {
            if (temp > 50.0) {
                std::cout << "  [Warning] High temperature: " << temp << "\n";
            }
        }
    );

    std::cout << "Subscribers: " << sensor.onTemperatureChange.subscriberCount() << "\n\n";

    // Emit events
    std::cout << "--- Update 25C ---\n";
    sensor.updateTemperature(25.0);

    std::cout << "\n--- Update 75C ---\n";
    sensor.updateTemperature(75.0);

    std::cout << "\n--- Update 105C ---\n";
    sensor.updateTemperature(105.0);

    // Unsubscribe the display
    sensor.onTemperatureChange.unsubscribe(token1);
    std::cout << "\n--- Update 30C (after display unsubscribed) ---\n";
    sensor.updateTemperature(30.0);

    return 0;
}

Explanation:

  • Event<Args...> is a variadic template that can hold callbacks with any signature.
  • std::function<void(Args...)> type-erases any callable: lambdas (with or without captures), free functions, member functions (with std::bind), and functors.
  • Each subscription returns a token (integer ID) that can be used to unsubscribe later. This avoids the complexity of tracking callable identity.
  • The ThreadSafeEvent variant adds mutex protection for use in multi-threaded contexts.
  • TemperatureSensor demonstrates domain-specific events: onTemperatureChange fires on every reading, onCriticalAlert fires only when temperature exceeds a threshold.

Advantages of std::function-based Observer:

Classical (Inheritance)std::function (This Approach)
Requires abstract base classAny callable works
Observers must inherit from interfaceNo inheritance needed
Can only observe events from known typesLambdas can capture anything
Harder to unsubscribe specific callbacksToken-based unsubscription
Compile-time type safetyType safety via template parameters

Follow-up Questions

Variations to Try

  1. Filter by event type: Extend the Event class to support filtering, so subscribers only get notified when a predicate is satisfied (e.g., "notify me only when temp > 50").
  2. Event Bus: Implement a central EventBus that routes named events to subscribers. Useful for decoupling in large systems.
  3. Thread-safe deferred emission: In the ThreadSafeEvent, collect callbacks in the lock, then invoke them after releasing the lock. Why is this important? (Hint: callback might try to subscribe/unsubscribe.)
  4. Member function observer: Use std::bind or a lambda to subscribe a member function as a callback. Example: event.subscribe(std::bind(&Foo::onEvent, &foo, std::placeholders::_1)).

Important Caveat:

If a callback is invoked while the emitter object is being destroyed, you get undefined behavior. Always disconnect all observers in the destructor, or use weak references (as shown in the Smart Pointer Patterns question) to check if observers still exist before calling them.

Key Takeaways:

  • std::function + lambdas provides a more flexible observer pattern than classical inheritance-based approaches.
  • Variadic templates allow type-safe events with any payload.
  • Token-based subscription is simpler and more robust than trying to compare callables for equality.
  • In production, always consider thread safety and lifetime management when implementing observer patterns.
