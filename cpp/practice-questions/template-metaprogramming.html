Template Metaprogramming

Topics Covered: Templates, Constexpr, Variadic Templates, Type Traits

Problem Statement

Implement the following at compile time using template metaprogramming and constexpr:

  1. Compile-time factorial using template specialization and using constexpr.
  2. Compile-time Fibonacci using template specialization and using constexpr.
  3. Type list operations: a type list that supports Append, Contains, Count, and Filter.

All results must be verifiable with static_assert at compile time.

1. Compile-Time Factorial

Click to reveal Solution

#include <iostream>
#include <cstdint>

// ---- Approach A: Template Metaprogramming (recursive templates) ----
template<int N>
struct FactorialTMP {
    static constexpr uint64_t value = N * FactorialTMP<N - 1>::value;
};

template<>
struct FactorialTMP<0> {
    static constexpr uint64_t value = 1;
};

// ---- Approach B: constexpr function (C++11+) ----
constexpr uint64_t factorialConstexpr(int n) {
    return (n <= 1) ? 1 : static_cast<uint64_t>(n) * factorialConstexpr(n - 1);
}

// ---- Approach C: constexpr with loop (C++14+) ----
constexpr uint64_t factorialLoop(int n) {
    uint64_t result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= static_cast<uint64_t>(i);
    }
    return result;
}

// Compile-time verification
static_assert(FactorialTMP<0>::value == 1);
static_assert(FactorialTMP<5>::value == 120);
static_assert(FactorialTMP<10>::value == 3628800);

static_assert(factorialConstexpr(0) == 1);
static_assert(factorialConstexpr(5) == 120);

static_assert(factorialLoop(0) == 1);
static_assert(factorialLoop(20) == 2432902008176640000ULL);

int main() {
    std::cout << "FactorialTMP<5>::value = " << FactorialTMP<5>::value << "\n";
    std::cout << "factorialConstexpr(5) = " << factorialConstexpr(5) << "\n";
    std::cout << "factorialLoop(5) = " << factorialLoop(5) << "\n";
    return 0;
}

Explanation:

  • Template approach: Recursive template instantiation generates a chain of types. The specialization FactorialTMP<0> provides the base case. This works in C++98 and later.
  • constexpr approach: A function annotated constexpr can be evaluated at compile time if called with constant arguments. C++11 allows only single return statement + recursion; C++14+ allows loops.
  • constexpr loop: The most readable and efficient approach (C++14+). No recursion depth limit concerns.
  • Template instantiation depth is limited (typically 1024), so FactorialTMP<1025> would fail. constexpr functions do not have this limit.

2. Compile-Time Fibonacci

Click to reveal Solution

#include <iostream>
#include <cstdint>

// ---- Template Metaprogramming ----
template<int N>
struct FibTMP {
    static constexpr uint64_t value = FibTMP<N - 1>::value + FibTMP<N - 2>::value;
};

template<> struct FibTMP<0> { static constexpr uint64_t value = 0; };
template<> struct FibTMP<1> { static constexpr uint64_t value = 1; };

// ---- Constexpr ----
constexpr uint64_t fibConstexpr(int n) {
    if (n <= 0) return 0;
    if (n == 1) return 1;
    uint64_t a = 0, b = 1;
    for (int i = 2; i <= n; ++i) {
        uint64_t c = a + b;
        a = b;
        b = c;
    }
    return b;
}

// Verification
static_assert(FibTMP<0>::value == 0);
static_assert(FibTMP<1>::value == 1);
static_assert(FibTMP<10>::value == 55);
static_assert(FibTMP<20>::value == 6765);

static_assert(fibConstexpr(0) == 0);
static_assert(fibConstexpr(1) == 1);
static_assert(fibConstexpr(10) == 55);
static_assert(fibConstexpr(20) == 6765);

int main() {
    std::cout << "FibTMP<10> = " << FibTMP<10>::value << "\n";  // 55
    std::cout << "fibConstexpr(20) = " << fibConstexpr(20) << "\n";     // 6765
    return 0;
}

Explanation:

  • The template version uses the mathematical definition directly. Note that naive template Fibonacci is exponential in instantiation depth, but the compiler typically memoizes template specializations (each FibTMP<N> is instantiated only once).
  • The constexpr version uses an iterative approach for O(n) compile-time computation.

3. Type List Operations

Click to reveal Solution

#include <iostream>
#include <type_traits>

// ---- Type List Definition ----
template<typename... Ts>
struct TypeList {};

// ---- Length: count the number of types ----
template<typename TList>
struct Length;

template<typename... Ts>
struct Length<TypeList<Ts...>> {
    static constexpr std::size_t value = sizeof...(Ts);
};

// ---- Append: add a type to the end of the list ----
template<typename TList, typename T>
struct Append;

template<typename... Ts, typename T>
struct Append<TypeList<Ts...>, T> {
    using type = TypeList<Ts..., T>;
};

// ---- Contains: check if a type is in the list ----
template<typename TList, typename T>
struct Contains;

template<typename T>
struct Contains<TypeList<>, T> : std::false_type {};

template<typename T, typename... Rest>
struct Contains<TypeList<T, Rest...>, T> : std::true_type {};

template<typename Head, typename... Rest, typename T>
struct Contains<TypeList<Head, Rest...>, T> : Contains<TypeList<Rest...>, T> {};

// ---- Count: how many times a type appears ----
template<typename TList, typename T>
struct Count;

template<typename T>
struct Count<TypeList<>, T> {
    static constexpr std::size_t value = 0;
};

template<typename T, typename... Rest>
struct Count<TypeList<T, Rest...>, T> {
    static constexpr std::size_t value = 1 + Count<TypeList<Rest...>, T>::value;
};

template<typename Head, typename... Rest, typename T>
struct Count<TypeList<Head, Rest...>, T> : Count<TypeList<Rest...>, T> {};

// ---- Filter: keep only types satisfying a predicate ----
template<typename TList, template<typename> class Predicate>
struct Filter;

template<template<typename> class Predicate>
struct Filter<TypeList<>, Predicate> {
    using type = TypeList<>;
};

template<typename Head, typename... Rest, template<typename> class Predicate>
struct Filter<TypeList<Head, Rest...>, Predicate> {
    using rest = typename Filter<TypeList<Rest...>, Predicate>::type;
    using type = std::conditional_t<
        Predicate<Head>::value,
        typename Append<rest, Head>::type,  // Head passes predicate
        rest                                  // Skip Head
    >;
};

// ---- Predicate: is integral type ----
template<typename T>
struct IsIntegralType : std::is_integral<T> {};

// ---- Print TypeList (helper for debugging) ----
template<typename TList>
struct PrintTypeList;

template<>
struct PrintTypeList<TypeList<>> {
    static std::string value() { return "{}"; }
};

template<typename Head, typename... Rest>
struct PrintTypeList<TypeList<Head, Rest...>> {
    static std::string value() {
        return "{ " + PrintTypeList<TypeList<Rest...>>::value() + " }";
    }
};

// ---- Compile-time verification ----
using MyTypes = TypeList<int, double, char, int, float>;
using MyTypes2 = typename Append<MyTypes, std::string>::type;

static_assert(Length<MyTypes>::value == 5);
static_assert(Contains<MyTypes, int>::value == true);
static_assert(Contains<MyTypes, std::string>::value == false);
static_assert(Count<MyTypes, int>::value == 2);
static_assert(Count<MyTypes, double>::value == 1);
static_assert(Length<MyTypes2>::value == 6);
static_assert(Contains<MyTypes2, std::string>::value == true);

// Filter: keep only integral types
using IntegralTypes = typename Filter<MyTypes, IsIntegralType>::type;
static_assert(Length<IntegralTypes>::value == 3);  // int, char, int
static_assert(Contains<IntegralTypes, double>::value == false);
static_assert(Contains<IntegralTypes, float>::value == false);

int main() {
    std::cout << "Length of MyTypes: " << Length<MyTypes>::value << "\n";
    std::cout << "Contains int: " << Contains<MyTypes, int>::value << "\n";
    std::cout << "Count of int: " << Count<MyTypes, int>::value << "\n";
    std::cout << "Filtered (integral) length: " << Length<IntegralTypes>::value << "\n";
    return 0;
}

Explanation:

  • TypeList is a variadic template that holds a list of types. It has no runtime data — it exists purely at compile time.
  • Length uses sizeof...(Ts) to count the pack expansion directly.
  • Append uses pack expansion to create a new TypeList with the additional type.
  • Contains uses recursive template matching: check the head, then recurse on the rest.
  • Count is similar to Contains but adds 1 each time it finds a match.
  • Filter uses std::conditional_t to either include or skip each type based on a predicate template.
  • All operations are verified with static_assert, meaning errors are caught at compile time.

Variations to Try

  1. Type map: Implement a compile-time associative map from types to values. Example: TypeMap<Pair<int, 1>, Pair<double, 2.0>> with a Get<T> operation.
  2. Compile-time string: Implement a fixed-size compile-time string class using const char* template parameters (C++20) and support concatenation.
  3. Conditional types (if constexpr): Use C++17 if constexpr to write compile-time Fibonacci without template recursion.
  4. Compile-time CRC32: Implement a CRC32 calculator using a constexpr lookup table generated at compile time.

Key Takeaways:

  • Template metaprogramming operates on types, not just values. This is its unique power — you cannot do type manipulation with constexpr alone.
  • static_assert is your friend: it catches errors at compile time with meaningful messages.
  • C++14/17/20 have progressively made metaprogramming easier: constexpr functions, if constexpr, auto template parameters, and concepts.
  • In real-world telecom code, compile-time computation is used for lookup tables (CRC, bit manipulation), type-safe message handling, and configuration validation.
