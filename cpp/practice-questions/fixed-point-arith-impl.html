Q15.16 Fixed-Point Arithmetic Class

Topics Covered: Fixed-Point Arithmetic, Operator Overloading, Templates

Problem Statement

Implement a Q15.16 fixed-point arithmetic class. In the Q notation, Q15.16 means 15 bits for the integer part and 16 bits for the fractional part, stored in a 32-bit signed integer.

The class must support:

  • Addition (+) and subtraction (-)
  • Multiplication (*) with proper shift
  • Division (/) with proper shift
  • Conversion to/from float, double, and int
  • Comparison operators

Q15.16 Format Reference:

ValueRaw (hex)Explanation
1.00x000100001 << 16
0.50x000080001 << 15
0.250x000040001 << 14
-1.50xFFFE8000Two's complement
3.141590x0003243F3 + 0.14159 * 65536

Key formulas:

  • To convert float to Q15.16: raw = (int32_t)(value * 65536.0f)
  • To convert Q15.16 to float: value = (float)raw / 65536.0f
  • Multiplication: (a * b) >> 16 (shift right by fractional bits)
  • Division: (a << 16) / b (shift left before divide to preserve precision)

#include <iostream>
#include <iostream>
#include <cstdint>
#include <cmath>
#include <stdexcept>
#include <string>
#include <sstream>
#include <iomanip>

class FixedQ15_16 {
public:
    static constexpr int FRACTIONAL_BITS = 16;
    static constexpr int64_t SCALE = 1LL << FRACTIONAL_BITS;  // 65536

    // ---- Constructors ----

    constexpr FixedQ15_16() : raw_(0) {}

    // From raw value (advanced use)
    explicit constexpr FixedQ15_16(int32_t raw) : raw_(raw) {}

    // From integer
    constexpr FixedQ15_16(int value) : raw_(static_cast<int32_t>(value << FRACTIONAL_BITS)) {}

    // From float (implicit)
    constexpr FixedQ15_16(float value)
        : raw_(static_cast<int32_t>(static_cast<double>(value) * SCALE)) {}

    // From double (implicit)
    constexpr FixedQ15_16(double value)
        : raw_(static_cast<int32_t>(value * static_cast<double>(SCALE))) {}

    // ---- Conversions ----

    constexpr float toFloat() const {
        return static_cast<float>(raw_) / static_cast<float>(SCALE);
    }

    constexpr double toDouble() const {
        return static_cast<double>(raw_) / static_cast<double>(SCALE);
    }

    constexpr int toInt() const {
        return raw_ >> FRACTIONAL_BITS;  // Truncate toward zero
    }

    constexpr int32_t raw() const { return raw_; }

    // ---- Arithmetic operators ----

    constexpr FixedQ15_16 operator+(const FixedQ15_16& rhs) const {
        return FixedQ15_16(raw_ + rhs.raw_);
    }

    constexpr FixedQ15_16 operator-(const FixedQ15_16& rhs) const {
        return FixedQ15_16(raw_ - rhs.raw_);
    }

    FixedQ15_16 operator*(const FixedQ15_16& rhs) const {
        // Use 64-bit intermediate to prevent overflow
        int64_t result = static_cast<int64_t>(raw_) *
                         static_cast<int64_t>(rhs.raw_);
        return FixedQ15_16(static_cast<int32_t>(result >> FRACTIONAL_BITS));
    }

    FixedQ15_16 operator/(const FixedQ15_16& rhs) const {
        if (rhs.raw_ == 0) {
            throw std::runtime_error("Fixed-point division by zero");
        }
        // Shift left BEFORE divide to preserve precision
        int64_t temp = (static_cast<int64_t>(raw_) << FRACTIONAL_BITS);
        return FixedQ15_16(static_cast<int32_t>(temp / rhs.raw_));
    }

    // ---- Compound assignment ----

    FixedQ15_16& operator+=(const FixedQ15_16& rhs) {
        raw_ += rhs.raw_;
        return *this;
    }

    FixedQ15_16& operator-=(const FixedQ15_16& rhs) {
        raw_ -= rhs.raw_;
        return *this;
    }

    FixedQ15_16& operator*=(const FixedQ15_16& rhs) {
        *this = *this * rhs;
        return *this;
    }

    FixedQ15_16& operator/=(const FixedQ15_16& rhs) {
        *this = *this / rhs;
        return *this;
    }

    // ---- Unary operators ----

    constexpr FixedQ15_16 operator-() const {
        return FixedQ15_16(-raw_);
    }

    constexpr FixedQ15_16 operator+() const {
        return *this;
    }

    // ---- Comparison operators ----

    constexpr bool operator==(const FixedQ15_16& rhs) const { return raw_ == rhs.raw_; }
    constexpr bool operator!=(const FixedQ15_16& rhs) const { return raw_ != rhs.raw_; }
    constexpr bool operator< (const FixedQ15_16& rhs) const { return raw_ <  rhs.raw_; }
    constexpr bool operator<=(const FixedQ15_16& rhs) const { return raw_ <= rhs.raw_; }
    constexpr bool operator> (const FixedQ15_16& rhs) const { return raw_ >  rhs.raw_; }
    constexpr bool operator>=(const FixedQ15_16& rhs) const { return raw_ >= rhs.raw_; }

    // ---- Utility ----

    std::string toString() const {
        std::ostringstream oss;
        oss << std::fixed << std::setprecision(6) << toDouble();
        return oss.str();
    }

    // Absolute value
    FixedQ15_16 abs() const {
        return FixedQ15_16(raw_ >= 0 ? raw_ : -raw_);
    }

private:
    int32_t raw_;
};

// ---- Stream output ----

std::ostream& operator<<(std::ostream& os, const FixedQ15_16& val) {
    return os << val.toString();
}

// ---- Demo ----

int main() {
    FixedQ15_16 a = 3.5;
    FixedQ15_16 b = 2.25;
    FixedQ15_16 c = -1.5;

    std::cout << "=== FixedQ15_16 Arithmetic Demo ===\n\n";

    std::cout << "a = " << a << " (raw: 0x" << std::hex << a.raw() << std::dec << ")\n";
    std::cout << "b = " << b << " (raw: 0x" << std::hex << b.raw() << std::dec << ")\n";
    std::cout << "c = " << c << "\n\n";

    // Addition and subtraction
    std::cout << "a + b = " << (a + b) << "\n";   // 5.750000
    std::cout << "a - b = " << (a - b) << "\n";   // 1.250000
    std::cout << "a + c = " << (a + c) << "\n";   // 2.000000

    // Multiplication
    std::cout << "\na * b = " << (a * b) << "\n";   // 7.875000
    std::cout << "a * c = " << (a * c) << "\n";   // -5.250000

    // Division
    std::cout << "\na / b = " << (a / b) << "\n";   // 1.555555...
    std::cout << "b / a = " << (b / a) << "\n";   // 0.642857...

    // Comparisons
    std::cout << "\na > b: " << (a > b) << "\n";    // true (1)
    std::cout << "a == c: " << (a == c) << "\n";  // false (0)
    std::cout << "a >= b: " << (a >= b) << "\n";  // true (1)

    // Integer conversion
    std::cout << "\na.toInt() = " << a.toInt() << "\n";  // 3

    // Chained operations
    FixedQ15_16 result = (a + b) * c / FixedQ15_16(2);
    std::cout << "\n(a + b) * c / 2 = " << result << "\n";

    // Accuracy comparison with float
    float fa = 3.5f, fb = 2.25f;
    std::cout << "\n--- Accuracy check ---\n";
    std::cout << "Float:     a/b = " << (fa / fb) << "\n";
    std::cout << "Fixed:     a/b = " << (a / b).toDouble() << "\n";
    std::cout << "Difference: " << std::abs((a / b).toDouble() - fa / fb) << "\n";

    return 0;
}

Explanation:

  • The value is stored as a plain int32_t using an implicit scale factor of 65536 (216).
  • Add/Subtract: No shift needed. Both values have the same scale, so adding the raw integers is equivalent to adding the real values.
  • Multiply: Two Q15.16 values multiplied produce a Q30.32 result (32 integer bits, 32 fractional bits). We shift right by 16 to get back to Q15.16. The 64-bit intermediate prevents overflow of the multiplication.
  • Divide: We shift the numerator left by 16 first to compensate for the scale. Without this, the result would lose all fractional precision (integer division). Example: 1.0 / 2.0 — raw 65536 / 131072 = 0. Wrong. With shift: (65536 << 16) / 131072 = 32768 = 0.5. Correct.
  • Conversions: Float-to-fixed multiplies by 65536; fixed-to-float divides by 65536.

On processors without FPU (common in low-cost microcontrollers), fixed-point is dramatically faster than software-emulated floating point.

Follow-up Questions

Variations to Try

  1. Template-based Q format: Generalize to FixedPoint<int32_t, IntBits, FracBits> with compile-time validation.
  2. Square root: Implement a fixed-point square root using the Newton-Raphson method.
  3. Sin/Cos lookup table: Implement sin/cos using a precomputed 256-entry lookup table with linear interpolation.
  4. Saturation arithmetic: Modify the class so that overflow clamps to the maximum/minimum value instead of wrapping (common in DSP).
  5. Q31 format: Implement Q0.31 (32 fractional bits) where the range is [-1, 1). This is used extensively in audio DSP.

Real-World Applications:

  • Fixed-point arithmetic is used in 5G baseband processing for channel estimation, equalization, and FFT on platforms without hardware FPU.
  • DSP chips (like TI C55x) use fixed-point natively for audio and video codecs.
  • Embedded systems (Cortex-M0/M0+ without FPU) rely on fixed-point for sensor processing and motor control.
  • The CMSIS-DSP library from ARM provides optimized fixed-point math functions.
