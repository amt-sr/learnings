Port

Difficulty: ⭐☆☆☆☆ Interview Frequency: ★★★★☆

Goal

Understand:

  • What a port is and how it works with IP addresses
  • Well-known, registered, and dynamic port ranges
  • The socket concept (IP + Port = unique endpoint)
  • Common ports every engineer should know

Definition

A port is a numbered endpoint that identifies a specific process or service on a machine. Combined with an IP address, it forms a socket — the actual communication endpoint.

Socket = IP Address + Port Number Example: 192.168.1.10:8080 identifies process on port 8080 on that specific machine
Analogy: If an IP address is a building's street address, a port is the apartment number. Multiple services (tenants) can run on the same machine (building), each reachable by its port number.

Port Ranges

RangeNamePortsUsage
System PortsWell-known0 – 1023Standard services (need root/admin)
User PortsRegistered1024 – 49151Registered applications
Dynamic PortsEphemeral49152 – 65535Client-side, temporary connections
Key point: Well-known ports are assigned by IANA. Registered ports are for known applications but don't require root. Ephemeral ports are assigned by the OS for outgoing connections.

How Connections Work

Client (browser) Server ───────────── ────── ephemeral port → connects to → port 443 (HTTPS) Example connection: 10.0.1.5:52341 ──────→ 142.250.80.46:443 The client gets a random ephemeral port (52341). The server listens on a well-known port (443). A single server can handle thousands of connections: 142.250.80.46:443 ← 10.0.1.5:52341 142.250.80.46:443 ← 10.0.1.6:52342 142.250.80.46:443 ← 10.0.1.7:52343 All three connections share port 443 on the server, but are distinguished by the client IP:port tuple.

Tuple Uniqueness

A TCP connection is uniquely identified by a 5-tuple:

(Source IP, Source Port, Destination IP, Destination Port, Protocol) Two connections are the same ONLY if all 5 elements match.
System design implication: A single server listening on port 443 can theoretically handle ~65,535 concurrent connections from the same client IP (limited by ephemeral port range). From different client IPs, it can handle millions. This is why TCP connection reuse (keep-alive) matters — it frees up ephemeral ports.

Port Exhaustion

When a machine makes too many outbound connections, it can run out of ephemeral ports (typically 49152-65535 = ~16,383 ports).

Symptoms: "Address already in use" errors, connection timeouts

Solutions:
  1. Enable TCP keep-alive to reuse connections
  2. Use connection pooling
  3. Tune ephemeral port range (sysctl net.ipv4.ip_local_port_range)
  4. Enable SO_REUSEADDR socket option
  5. Use multiple source IPs on the client

Common Interview Questions

  • "What's the difference between IP address and port?"
    IP identifies the machine. Port identifies the process on that machine. Together (socket) they uniquely identify a communication endpoint.
  • "What happens when you run out of ephemeral ports?"
    New outbound connections fail. Solutions: connection pooling, keep-alive, tuning OS port range, using multiple source IPs.
  • "Why can two services both use port 8080 on different machines?"
    Because ports are per-machine. The socket includes the IP address, so 10.0.1.1:8080 and 10.0.1.2:8080 are different endpoints.
  • "Can two processes on the same machine use the same port?"
    No — unless they bind to different IPs (e.g., 127.0.0.1:8080 and 0.0.0.0:8080). Same IP + same port = conflict.
