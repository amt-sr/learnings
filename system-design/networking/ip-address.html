IP Address

Difficulty: ⭐⭐☆☆☆ Interview Frequency: ★★★★☆

Goal

Understand:

  • What an IP address is and the difference between IPv4 and IPv6
  • Public vs private IPs and NAT
  • Subnetting, CIDR, and why it matters for system design
  • Loopback, link-local, and special addresses

Definition

An IP address is a numerical label assigned to each device on a network, used for routing traffic between source and destination.

Analogy: An IP address is like a postal address for your computer. Without it, no data can find its way to the right machine — just like a letter can't be delivered without a street address.

IPv4 vs IPv6

IPv4IPv6
Address length32-bit (4 bytes)128-bit (16 bytes)
FormatDotted decimal: 192.168.1.1Colon hex: 2001:0db8::1
Total addresses~4.3 billion~3.4 × 10^38
Header size20-60 bytes40 bytes (fixed)
FragmentationRouters and sendersSenders only
BroadcastYesNo (multicast only)
NATRequired (address exhaustion)Not needed
AdoptionStill dominantGrowing (~45% Google traffic)
Why IPv6 matters for system design: With billions of IoT devices and containers (Kubernetes pods), IPv4 exhaustion is real. Large-scale systems use IPv6 internally or dual-stack. Cloud providers (AWS, GCP) default to IPv6 for new VPCs.

Private vs Public IP

NAT (Network Address Translation): Private IPs are not routable on the internet. NAT translates private → public at the router level, allowing many internal devices to share one public IP. This is why your home network works with a single ISP-assigned IP.

CIDR Notation

CIDR (Classless Inter-Domain Routing) expresses the network portion of an IP address.

IP/CIDR = Network prefix + Host portion /24 means first 24 bits are network, last 8 bits are host
/32 = single host, /0 = all addresses 
Examples:
  10.0.0.0/8    → 16,777,216 addresses  (entire Class A private)
  192.168.1.0/24 → 256 addresses        (common home network)
  172.16.0.0/12  → 1,048,576 addresses  (entire Class B private)
  10.0.1.0/24    → 256 addresses        (single subnet)
  10.0.1.5/32    → 1 address           (single host)

Subnetting in System Design

Why subnetting matters for large systems: VPC (AWS): ┌─────────────────────────────────────────┐ │ 10.0.0.0/16 (65,536 IPs total) │ │ ├── 10.0.1.0/24 Public subnets (LBs) │ │ ├── 10.0.2.0/24 App servers │ │ ├── 10.0.3.0/24 DB primaries │ │ ├── 10.0.4.0/24 DB replicas │ │ ├── 10.0.5.0/24 Cache (Redis) │ │ └── 10.0.6.0/24 Worker nodes │ └─────────────────────────────────────────┘ Each subnet = isolated failure domain + security group boundary
Interview tip: When designing a system, mention subnetting: "We'd put load balancers in public subnets, app servers and databases in private subnets, and use security groups to restrict traffic between tiers." This shows infrastructure awareness.

DNS and IP

Humans use domain names (google.com). Computers use IP addresses. DNS resolves one to the other.

User types:  google.com
DNS resolves: → 142.250.80.46 (IPv4)
              → 2607:f8b0:4005:808::200e (IPv6)

Every HTTP request starts with DNS → IP resolution.
Cached DNS = 0ms. Uncached = 1-100ms.

Common Interview Questions

  • "What's the difference between public and private IP?"
    Public: routable on the internet, globally unique. Private: non-routable, used within internal networks (10.x, 172.16-31.x, 192.168.x). NAT bridges them.
  • "Why do we still use IPv4 if IPv6 is better?"
    NAT works well enough, IPv4 infrastructure is entrenched, migration cost is high. IPv6 adoption is growing driven by cloud, mobile, and IoT.
  • "What's 127.0.0.1?"
    Loopback address. Traffic to 127.0.0.1 never leaves the machine — used for local development and inter-process communication on the same host.
  • "What's CIDR /24?"
    First 24 bits define the network, last 8 bits for hosts = 256 addresses (254 usable, minus network + broadcast).
  • "How does NAT work?"
    Router maintains a translation table mapping (private IP:port) → (public IP:port). Outbound traffic gets NATed. Inbound responses are reverse-NATed back to the original private IP.
