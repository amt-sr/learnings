HTTP/3

Difficulty: ⭐⭐⭐☆☆ Interview Frequency: ★★★☆☆

Goal

Understand:

  • Why HTTP/3 exists (TCP-level HOL blocking in HTTP/2)
  • QUIC protocol and how it replaces TCP + TLS
  • 0-RTT connection setup
  • Connection migration

Definition

HTTP/3 (2022, RFC 9114) is the latest HTTP version. It uses QUIC (a transport protocol built on UDP) instead of TCP, eliminating TCP-level head-of-line blocking and reducing connection setup time.

The Problem HTTP/3 Solves

HTTP/2 over TCP: TCP packet loss blocks ALL streams ┌─────────────────────────────────────────────┐ │ TCP Connection │ │ [Stream 1 ████████████████] (blocked) │ │ [Stream 2 ████████░░░░░░░░] (blocked too!) │ │ [Stream 3 ██████░░░░░░░░░░] (blocked too!) │ │ │ │ One lost packet → everything waits │ └─────────────────────────────────────────────┘ HTTP/3 over QUIC: Each stream is independent ┌─────────────────────────────────────────────┐ │ QUIC Connection (UDP) │ │ [Stream 1 █████████████████] (blocked) │ │ [Stream 2 ████████████ ✓] (fine!) │ │ [Stream 3 ██████████ ✓] (fine!) │ │ │ │ Lost packet only blocks its own stream │ └─────────────────────────────────────────────┘

QUIC: TCP + TLS Combined

HTTP/1.1 + TLS 1.2: TCP handshake: 1 RTT TLS handshake: 2 RTTs First data: Total 3 RTTs (7 round trips for full setup) HTTP/2 + TLS 1.3: TCP + TLS combined: 2 RTTs First data: 2 RTTs (3 round trips for full setup) HTTP/3 (QUIC + TLS 1.3): QUIC combines transport + crypto in one handshake: 1 RTT First data: 1 RTT 0-RTT (repeat connection): 0 RTTs HTTP/3 is ~2x faster to first byte on new connections, and ~3x faster than HTTP/1.1 with TLS 1.2.

Connection Migration

QUIC uses connection IDs instead of the (IP, port) tuple to identify connections. This allows seamless switching between networks.

Scenario: User walks from WiFi to cellular

TCP: Connection breaks (IP changed)
  → New TCP handshake needed (2-3 RTTs)
  → In-flight data lost
  → Visible interruption

QUIC: Connection ID stays the same
  → Packets continue on new network
  → No interruption
  → In-flight data retransmitted on new path

Common Interview Questions

  • "Why does HTTP/3 use UDP instead of TCP?"
    Because TCP's reliable ordered delivery causes head-of-line blocking across all streams. When one TCP packet is lost, ALL HTTP/2 streams on that connection are blocked. QUIC's independent streams over UDP avoid this.
  • "What's 0-RTT?"
    On repeat connections to the same server, QUIC can send data immediately (0 round trips) using cached cryptographic parameters. The server validates the cached parameters and processes the data. Similar to TLS 1.3 session resumption but built into the transport layer.
  • "What's connection migration?"
    QUIC identifies connections by a connection ID (not IP:port). When the client's IP changes (WiFi to cellular, VPN toggle), the connection survives. TCP would break because the IP:port tuple changed.
