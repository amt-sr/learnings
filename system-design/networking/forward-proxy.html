Forward Proxy

Difficulty: ⭐⭐☆☆☆ Interview Frequency: ★★★☆☆

Goal

Understand:

  • What a forward proxy is and how it sits between clients and the internet
  • How it differs from a reverse proxy
  • Common use cases: corporate proxy, VPN, content filtering

Definition

A forward proxy is a server that sits between clients and the internet, forwarding client requests to external servers on their behalf. The destination server only sees the proxy's IP address, not the client's. The client explicitly configures the proxy (or is routed through it via network policy).

Forward Proxy Request Flow: ┌────────┐ ┌──────────────┐ ┌─────────────┐ │ Client │──────▶│Forward Proxy │──────▶│ Internet │ │(Browser)│ │ (Squid/Corp) │◀──────│ Server │ └────────┘ └──────────────┘ └─────────────┘ │ Server sees the proxy's IP, not the client's real IP. Compare with Reverse Proxy: ┌────────┐ ┌──────────────┐ ┌─────────────┐ │ Client │──────▶│Reverse Proxy │──────▶│ Backend Svc │ └────────┘ │ (Nginx/ALB) │ │ (Internal) │ └──────────────┘ └─────────────┘ Client sees proxy, not the backend.

How Forward Proxies Work

  1. Client sends a request to the forward proxy (e.g., GET http://example.com)
  2. The proxy evaluates the request against its rules (allow/deny lists, authentication)
  3. If allowed, the proxy opens its own connection to the destination server
  4. The proxy forwards the client's request, optionally modifying headers
  5. The proxy relays the server's response back to the client
Key distinction: With a forward proxy, the client knows it's using a proxy and configures it explicitly. With a reverse proxy, the client has no idea a proxy exists — it thinks it's talking directly to the server.

Common Use Cases

Corporate Proxy

Enterprises deploy forward proxies to control all outbound internet traffic from employee devices. This enables:

  • Content filtering — Block access to malicious or non-work-related sites
  • Logging and monitoring — Audit all outbound HTTP requests
  • Data loss prevention (DLP) — Prevent sensitive data from leaving the network
  • Bandwidth optimization — Cache frequently accessed resources

VPN as Forward Proxy

A VPN tunnels all client traffic through a remote server, acting as a forward proxy at the network level. Unlike an HTTP proxy (which handles HTTP/HTTPS only), a VPN routes all traffic (DNS, TCP, UDP) through the proxy server.

Anonymization

Services like Tor route traffic through multiple forward proxies (nodes) to hide the client's identity. Each node only knows the previous and next hop, not the full path.

Caching Proxy

A forward proxy can cache responses to reduce bandwidth usage. This is common in ISP-level proxies and content delivery networks.

Configuration Example

# Client-side proxy configuration
# Browser: Settings → Proxy → Manual
#   HTTP Proxy:  proxy.corp.com:8080
#   HTTPS Proxy: proxy.corp.com:8080

# Linux environment variable
export http_proxy=http://proxy.corp.com:8080
export https_proxy=http://proxy.corp.com:8080

# Squid proxy configuration (simplified)
# /etc/squid/squid.conf
acl corp_network src 10.0.0.0/8
acl blocked_sites dstdomain .facebook.com .twitter.com

http_access deny blocked_sites
http_access allow corp_network
http_access deny all

Common Interview Questions

  • "What is the difference between a forward proxy and a reverse proxy?"
    A forward proxy sits in front of clients and makes requests on their behalf (client-side). A reverse proxy sits in front of servers and receives requests on their behalf (server-side). Forward proxy hides the client; reverse proxy hides the server. See: Reverse Proxy
  • "Why would a company use a forward proxy?"
    To monitor and control outbound traffic, enforce content filtering, log employee internet usage for compliance, prevent data exfiltration, and cache responses to reduce bandwidth costs.
  • "Is a VPN a forward proxy?"
    Conceptually similar — both sit between client and internet. But a VPN operates at the network layer (routes all traffic), while a forward proxy typically handles HTTP/HTTPS at the application layer. A VPN is more like a tunnel; a proxy is more like a relay for specific protocols.
