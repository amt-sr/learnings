L7 Load Balancer

Difficulty: ⭐⭐⭐☆☆ Interview Frequency: ★★★★★

Goal

Understand:

  • What Layer 7 load balancing means and how it differs from L4
  • HTTP-aware routing: path-based, host-based, header-based
  • TLS termination, rate limiting, and request transformation
  • Real-world examples: AWS ALB, Nginx, Envoy, HAProxy

Definition

A Layer 7 load balancer operates at the application layer of the OSI model. It inspects HTTP requests — including URL path, host header, query parameters, cookies, and custom headers — to make intelligent routing decisions. Unlike L4, it understands the content of each request and can route differently based on it.

L7 Load Balancing — Path-Based Routing: Client Request: GET https://api.example.com/api/users/123 │ │ │ │ │ └── Path: /api/users/123 │ └── Host: api.example.com └── Method: GET L7 LB parses HTTP and routes: ┌──────────────────────────────────────────────────────────┐ │ L7 Load Balancer │ │ │ │ /api/users/* ──▶ User Service (10.0.1.10:8080) │ │ /api/orders/* ──▶ Order Service (10.0.1.11:8080) │ │ /api/payments/* ──▶ Payment Svc (10.0.1.12:8080) │ │ /* ──▶ Web Frontend (10.0.1.20:3000) │ │ │ │ Host: blog.example.com ──▶ Blog Service │ │ Host: api.example.com ──▶ API Gateway │ └──────────────────────────────────────────────────────────┘

Key Features

TLS Termination

The L7 LB terminates TLS connections, decrypts HTTPS traffic, and forwards plain HTTP to backends. This centralizes certificate management and enables the LB to inspect and modify HTTP content.

SSL Offloading

TLS encryption/decryption is CPU-intensive (especially RSA with large key sizes). Offloading this to the LB frees backend servers to handle application logic.

# Nginx L7 load balancing with TLS termination
upstream api_backend {
    least_conn;
    server 10.0.1.10:8080;
    server 10.0.1.11:8080;
    server 10.0.1.12:8080;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate     /etc/ssl/api.pem;
    ssl_certificate_key /etc/ssl/api.key;
    ssl_protocols       TLSv1.2 TLSv1.3;

    location /api/users/ {
        proxy_pass http://api_backend;
    }

    location /api/orders/ {
        proxy_pass http://order_backend;
    }
}

Rate Limiting

L7 LB can enforce per-client, per-IP, or per-API-key rate limits before traffic hits backends.

# L7 rate limiting strategies:
# 1. Per IP:     100 requests/minute per IP address
# 2. Per API key: 1000 requests/minute per key
# 3. Per user:   500 requests/minute (from JWT sub claim)
# 4. Global:     10,000 requests/second total

# AWS ALB supports:
# - Fixed-window rate limiting
# - Rate-based rules in AWS WAF (5 requests per 5 min per IP, etc.)

Request/Response Transformation

L7 LB can add, remove, or modify headers, rewrite URLs, and transform request/response bodies.

# Common header manipulations:
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
proxy_set_header X-Real-IP         $remote_addr;
proxy_set_header X-Request-ID      $request_id;

# URL rewriting:
rewrite ^/api/v1/(.*) /$1 break;

Real-World Examples

AWS Application Load Balancer (ALB)

  • Layer 7 only — HTTP/HTTPS with path-based routing
  • Supports up to 100 rules per listener
  • Integrated with AWS WAF for DDoS protection
  • Supports sticky sessions, health checks, and slow start mode
  • Can route to Lambda functions directly (no EC2 needed)

Envoy Proxy

  • Open-source L7 proxy designed for microservices and service mesh
  • xDS API for dynamic configuration
  • Built-in circuit breaking, retries, rate limiting, observability
  • Used as the data plane in Istio service mesh

HAProxy (HTTP Mode)

# HAProxy L7 configuration
frontend http_front
    bind *:443 ssl crt /etc/ssl/api.pem

    # Path-based routing
    acl is_users path_beg /api/users
    acl is_orders path_beg /api/orders

    use_backend user_backend if is_users
    use_backend order_backend if is_orders
    default_backend default_backend

backend user_backend
    balance roundrobin
    server user1 10.0.1.10:8080 check
    server user2 10.0.1.11:8080 check

Performance Considerations

FactorImpact
HTTP parsing overheadEach request requires full HTTP parsing — more CPU than L4
TLS terminationCPU-intensive; use hardware acceleration or dedicated certs
Rule evaluationMore routing rules = more evaluation time per request
Connection reuseHTTP/2 multiplexing reduces connection overhead significantly
BufferingL7 LB buffers full request/response — higher memory usage
Typical throughput: A well-provisioned L7 LB handles 10,000-50,000 requests/second per instance. L4 LB handles 1-10 million connections/second. If you need raw throughput for non-HTTP protocols, use L4. If you need intelligent HTTP routing, use L7.

Common Interview Questions

  • "What is a Layer 7 load balancer and how is it different from L4?"
    L7 operates at the application layer and inspects HTTP content (path, headers, cookies) to make routing decisions. L4 only sees IP and port. L7 enables path-based routing, TLS termination, and rate limiting, but adds more latency and CPU overhead. See: L4 Load Balancer
  • "How would you design routing for a microservices architecture?"
    Use an L7 load balancer (ALB or Envoy) with path-based routing: /api/users/* → user-service, /api/orders/* → order-service. For internal services, use a service mesh (Envoy sidecars) with L7 routing based on headers and gRPC methods. The L7 LB also handles TLS termination and rate limiting at the edge.
  • "What is TLS termination and why do it at the L7 LB?"
    The LB decrypts HTTPS and forwards plain HTTP to backends. Centralizes cert management (one cert on the LB, not every backend). Frees backend CPU for application work. Enables the LB to inspect HTTP content for routing and WAF rules.
  • "Explain how an API Gateway differs from an L7 load balancer."
    An API Gateway is a specialized L7 LB with additional features: authentication/authorization, request transformation, API versioning, developer portal, and usage analytics. An L7 LB focuses on routing and distribution. In practice, many API Gateways (Kong, AWS API Gateway) are built on top of L7 proxies like Nginx or Envoy.
