Caching Headers

Difficulty: ⭐⭐⭐☆☆ Interview Frequency: ★★★★☆

Goal

Understand:

  • Cache-Control directives and how they control browser/CDN caching
  • ETag and Last-Modified for validation
  • Fresh vs stale vs revalidate
  • How to set caching headers for different content types

Definition

Caching headers are HTTP headers that control how and for how long responses can be cached by browsers, CDNs, and intermediaries. They're the primary mechanism for reducing redundant requests and improving performance.

Cache-Control

The most important caching header. Controls caching behavior at all levels (browser, CDN, proxy).

DirectiveMeaningUse When
max-age=3600Cache for 1 hourStatic assets (images, CSS, JS)
max-age=0Always revalidateDynamic content that changes frequently
no-cacheMust revalidate before using cached copyAPI responses, user-specific data
no-storeNever cacheSecret data, payment info
publicAny cache (including CDN) can storePublic static assets
privateOnly browser cache can storeUser-specific data
s-maxage=600CDN caches for 10 min (overrides max-age for shared caches)API responses with different browser vs CDN TTL
must-revalidateStale cache must be revalidated (not used without checking)Content that can go stale but shouldn't be stale long
Common confusion: no-cache does NOT mean "don't cache." It means "you can cache this, but you must check with the server before using it." no-store means "absolutely never cache this."

Validation Headers

ETag

A unique identifier for a specific version of a resource. If the content changes, the ETag changes.

Server response:
  ETag: "abc123"
  Cache-Control: max-age=3600

Client on subsequent request:
  If-None-Match: "abc123"

Server checks: Has the resource changed since ETag "abc123"?
  No change → 304 Not Modified (no body, saves bandwidth)
  Changed  → 200 OK + new ETag + full body

Last-Modified

Timestamp of when the resource was last modified.

Server response:
  Last-Modified: Wed, 21 Oct 2024 07:28:00 GMT

Client on subsequent request:
  If-Modified-Since: Wed, 21 Oct 2024 07:28:00 GMT

Same logic as ETag — 304 if unchanged, 200 if changed.
ETag is preferred (more precise, handles sub-second changes).

Caching Strategies by Content Type

Static assets (images, fonts, CSS, JS): Cache-Control: public, max-age=31536000, immutable (Cache for 1 year, never revalidate) → Use content hashing in filename: app.a1b2c3.js → When file changes, URL changes → cache automatically invalidated API responses (JSON): Cache-Control: private, max-age=60 (Browser only, 60 seconds) → Or: no-cache for real-time data User-specific data: Cache-Control: private, no-cache (Must revalidate, browser only) Highly sensitive data: Cache-Control: no-store, private (Never cache, anywhere) CDN-friendly API: Cache-Control: public, s-maxage=300, max-age=60 (CDN caches 5 min, browser caches 1 min)
The immutable trick: Set Cache-Control: max-age=31536000, immutable on static assets. Use content hashes in filenames (app.a1b2c3.js). When the file changes, the hash changes, creating a new URL. The old URL stays cached forever (no revalidation needed). This eliminates all 304 requests for static assets.

Common Interview Questions

  • "What's the difference between no-cache and no-store?"
    no-cache: cache it, but always validate with the server before using (304 check). no-store: never cache it at all. Use no-store for sensitive data (payment info, secrets). Use no-cache for dynamic data that changes frequently.
  • "What's an ETag?"
    A unique fingerprint for a resource version. Server sends it with the response. Client sends it back as If-None-Match. If unchanged, server returns 304 (no body). If changed, 200 with new content and ETag.
  • "How would you cache API responses?"
    For public data: Cache-Control: public, s-maxage=300 (CDN caches 5 min). For user-specific: private, max-age=60 (browser 1 min). For real-time: no-cache (always validate). See: Cache-Aside Pattern

What to Skip

  • Expires header (superseded by Cache-Control)
  • Pragma: no-cache (HTTP/1.0 legacy)
  • Vary header (content negotiation — advanced topic)
