Solid State Drives (SSD)

Difficulty: ⭐⭐⭐☆☆ Interview Frequency: ★★★★☆

Goal

Understand:

  • How NAND flash memory stores data — no moving parts
  • Why random reads are fast but writes have unique costs
  • Write amplification, garbage collection, and the TRIM command
  • TBW (Terabytes Written) and SSD endurance
  • How SSD characteristics affect database design decisions

Definition

A Solid State Drive (SSD) stores data in NAND flash memory cells, using electronic addressing instead of mechanical seeking. With no moving parts, SSDs deliver orders-of-magnitude better random I/O performance than HDDs, but they introduce unique write characteristics that database engines must account for.

EraseBlock (256–512 KB)~1–3 msSlow, and you can only erase entire blocks
The fundamental asymmetry: You can read and write individual pages, but you can only erase entire blocks. This mismatch is the root cause of write amplification and the reason SSDs have complex internal controllers.

Write Amplification

When updating existing data, the SSD cannot overwrite in place (NAND cells must be erased before reprogramming). The controller must:

  1. Read the entire block containing the old page
  2. Erase the block
  3. Write the old pages back with the updated page

This means a single 4KB logical write can cause an entire 256KB block to be rewritten — a 64x amplification in the worst case.

Interview insight: QLC SSDs have worse random write performance than HDDs for sustained workloads. When an interviewer asks "are SSDs always faster than HDDs?", the answer is nuanced — QLC SSDs can be slower for sustained random writes due to garbage collection overhead and limited SLC caching.

TBW (Terabytes Written) and Endurance

Every NAND cell has a finite number of Program/Erase (P/E) cycles. TBW represents the total amount of data that can be written to the SSD before the manufacturer's endurance warranty expires.

Drive ClassTBW (1TB drive)Typical Workload
Consumer TLC300–600 TBDesktop, light server
Enterprise TLC1,000–3,000 TBDatabase, analytics
Enterprise MLC5,000–10,000 TBWrite-heavy OLTP

Factors That Affect Endurance

  • Write amplification factor — higher WAF means more flash writes per logical write
  • Wear leveling — the FTL distributes writes across all blocks to prevent early failure
  • Over-provisioning — extra capacity reserved for garbage collection (enterprise drives have 28% OP vs 7% consumer)
  • Temperature — higher temperatures accelerate NAND degradation

TRIM and Garbage Collection

  • TRIM — OS tells the SSD which logical blocks are no longer in use, allowing the SSD's garbage collector to proactively erase those blocks
  • Garbage Collection — background process that consolidates blocks containing valid and invalid pages, freeing whole blocks for new writes
  • Without TRIM, the SSD does not know which pages are stale, leading to performance degradation over time

Common Interview Questions

  • "Why are SSDs faster than HDDs for random reads?"
    No mechanical seeking or rotational latency. The SSD controller addresses flash pages electronically in microseconds. Multiple dies can be read in parallel. An SSD can achieve 10,000–1,000,000 random read IOPS vs 100–200 for HDDs. See: HDD
  • "What is write amplification on an SSD?"
    Because NAND flash cannot overwrite in place, updating a page requires reading the entire block, erasing it, and rewriting all valid pages plus the updated one. The ratio of physical writes to logical writes is the Write Amplification Factor. A WAF of 3x means every 1 GB of logical writes causes 3 GB of physical writes to flash.
  • "How does SSD wear leveling work?"
    The Flash Translation Layer (FTL) maintains a mapping from logical block addresses (LBAs) to physical flash pages. When writing, it selects the physical page with the lowest erase count (dynamic wear leveling) or ensures all blocks are erased roughly equally (static wear leveling). This prevents any single block from wearing out prematurely.
  • "Should you use SSDs or HDDs for a database?"
    SSDs for OLTP (random read/write dominated) and any latency-sensitive workload. HDDs for cold storage, backups, and analytical workloads dominated by sequential scans. Many production systems use a hybrid: SSDs for hot data, HDDs for warm/cold data. See: Buffer Pool
