Services

Difficulty: вӯҗвӯҗвӯҗ Interview Frequency: вҳ…вҳ…вҳ…вҳ…

Goal

Understand the Kubernetes Service abstraction as a stable networking layer over ephemeral pods, the three service types (ClusterIP, NodePort, LoadBalancer), and how service discovery works within a cluster.

Definition

Kubernetes Service is an abstraction that provides a stable network endpoint (IP address and DNS name) for a set of pods. Since pods are ephemeral and their IPs change on recreation, Services provide a fixed address that load-balances traffic across healthy, ready pods.

Without Services, pods would need to know each other's IP addresses, which change on every restart. Services decouple consumers from pod locations and provide built-in load balancing.

Service Discovery

Kubernetes provides two service discovery mechanisms:

1. Environment Variables

# When a pod starts, K8s injects environment variables for every service:
MY_SERVICE_SERVICE_HOST=10.96.0.1
MY_SERVICE_SERVICE_PORT=8080
MY_SERVICE_PORT=tcp://10.96.0.1:8080

# Downside: services must exist BEFORE the consuming pod is created

2. DNS (Primary Method)

# CoreDNS runs in the cluster and resolves service names
# Within same namespace:
my-service вҶ’ 10.96.0.1

# Across namespaces:
my-service.my-namespace.svc.cluster.local вҶ’ 10.96.0.1

# Headless services (ClusterIP: None): returns individual pod IPs
my-service-headless.my-namespace.svc.cluster.local вҶ’ 10.244.1.5, 10.244.2.8, 10.244.3.1
Best practice: Always use DNS names (not IPs) to reference services. This makes deployments portable and eliminates hardcoding.

Endpoint objects: Kubernetes maintains an Endpoints object per Service that lists the IPs of all currently ready pods matching the selector. kube-proxy watches this object and updates iptables/IPVS rules accordingly. A pod is included only if its readiness probe passes.

Common Interview Questions

  1. What is a Kubernetes Service and why is it needed?
  2. Explain the three service types: ClusterIP, NodePort, LoadBalancer.
  3. How does service discovery work inside a Kubernetes cluster?
  4. How does a Service know which pods to send traffic to?
  5. What happens to a Service when pods are added or removed?
  6. What is a headless Service and when would you use it?
  7. How does kube-proxy work?
  8. Compare Kubernetes Services with an external load balancer like AWS ALB.

What to Skip

Avoid diving into: iptables vs IPVS implementation details, kube-proxy mode configuration, CoreDNS configuration, or EndpointSlice objects. Focus on the Service abstraction, its types, label selectors, and DNS resolution.
