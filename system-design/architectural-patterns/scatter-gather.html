Scatter-Gather

Difficulty: ⭐⭐⭐ Interview Frequency: ★★★

Goal

Understand the scatter-gather pattern for querying multiple services in parallel and aggregating results, handling timeouts, partial failures, and deduplication in a distributed search context.

Definition

Scatter-gather is a pattern where a request is scattered (dispatched in parallel) to multiple services or partitions, each processing a subset of the query, and the results are gathered (collected and merged) into a single response. It is the distributed computing version of map-reduce at the request level.

Scatter-gather is used when no single service can answer a query completely. The query must be sent to multiple services, each contributing a partial result, and then combined. The classic example: a search engine that queries multiple index shards in parallel.

Handling Failures

Critical design decision: What happens when one scatter target is unavailable? The pattern must define degradation behavior.
  • Return partial results: Gather results from available shards, indicate "X of Y shards responded." Client sees fewer results but the service does not fail.
  • Retry with backoff: Retry failed scatter requests with exponential backoff. Only if retries also fail, return partial results.
  • Fail fast: If a critical shard fails, fail the entire request. Used when incomplete results are misleading (e.g., financial data).
  • Fallback to cached: Serve cached/stale data from the failed shard while retrying in the background.

Scatter-Gather Variants

Real-World Examples

  • Google Search: Scatters query to thousands of index servers, gathers and ranks results in milliseconds.
  • Amazon Product Search: Queries multiple category-specific indexes in parallel.
  • Elasticsearch: A query is scattered to all shards of an index, each returns its top N results, the coordinating node gathers, merges, and returns final results.
  • Database sharding: Scatter a query to all relevant shards, gather partial results.
Interview tip: "For a product search across multiple categories, I would use scatter-gather. The query is sent to category-specific index shards in parallel, each returning its top results. A gather service deduplicates, ranks by relevance, and applies pagination. Each shard request has a 200ms timeout; if a shard is slow, I return results from the other shards to avoid blocking the user."
